# Инциденты безопасности

---

## Что это и зачем

Инцидент — это любое нештатное событие на строительной площадке: травма работника, падение предмета, обрушение конструкции, пожар, утечка химикатов, удар электрическим током. Каждый инцидент фиксируется в системе, расследуется и закрывается с корректирующими действиями.

По законодательству РФ **каждый несчастный случай на производстве** обязательно расследуется. Тяжёлые случаи (госпитализация, инвалидность, смерть) — уведомление Ростехнадзора и Государственной инспекции труда **в течение суток**. Без учёта инцидентов — штрафы и уголовная ответственность для руководителя.

## Виды инцидентов

| Тип | Описание | Пример |
|-----|---------|--------|
| **Травма** | Физическое повреждение работника | Перелом руки при падении с лесов |
| **Предпосылка (near miss)** | Опасная ситуация без последствий | Упала труба рядом с рабочим, но не задела |
| **Повреждение имущества** | Поломка оборудования или конструкций | Экскаватор повредил подземный кабель |
| **Экологический** | Загрязнение окружающей среды | Разлив ГСМ на площадке |
| **Пожар** | Возгорание на объекте | Загорелась проводка во время сварки |
| **Падение** | Падение человека или предмета с высоты | Рабочий упал с крыши 3-го этажа |
| **Электрический** | Удар электрическим током | Прикосновение к оголённому проводу |
| **Прочее** | Любой другой инцидент | Тепловой удар, отравление |

## Тяжесть инцидента

| Уровень | Цвет | Что означает | Пример |
|---------|------|-------------|--------|
| **Незначительный** | Серый | Мелкий инцидент, без последствий | Поцарапал руку, заноза |
| **Умеренный** | Синий | Незначительная травма, без потери рабочих дней | Ушиб, растяжение |
| **Серьёзный** | Жёлтый | Травма с потерей рабочих дней | Перелом пальца, порез с наложением швов |
| **Критический** | Оранжевый | Тяжёлая травма, госпитализация | Перелом позвоночника, ожог 2 степени |
| **Смертельный** | Красный | Гибель работника | Падение с высоты со смертельным исходом |

> **Важно:** При критической и смертельной тяжести система выделяет инцидент красным цветом. Руководитель обязан немедленно начать расследование.

## Как зафиксировать инцидент

### Быстрая регистрация (модальное окно)

На странице списка инцидентов нажмите **«+ Новый инцидент»** — откроется модальное окно для быстрой регистрации.

### Полная форма

Перейдите **Безопасность → Инциденты → Новый инцидент** для заполнения полной формы.

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Дата и время** | Да | Когда произошёл инцидент |
| **Проект** | Да | На каком объекте |
| **Место** | Нет | Где именно на площадке (секция, этаж, зона) |
| **Тяжесть** | Да | Незначительный / Умеренный / Серьёзный / Критический / Смертельный |
| **Тип инцидента** | Да | Травма / Предпосылка / Повреждение имущества / и т.д. |
| **Описание** | Да | Подробное описание того, что произошло |
| **Кто сообщил** | Нет | Кто первым зафиксировал инцидент |
| **Пострадавший** | Нет | ФИО пострадавшего (если есть) |
| **Свидетели** | Нет | Имена свидетелей |
| **Потерянные рабочие дни** | Нет | Сколько дней работник не выходил на работу |
| **Медпомощь** | Нет | Оказана ли медицинская помощь (галочка) |
| **Госпитализация** | Нет | Был ли пострадавший госпитализирован (галочка) |
| **Примечание** | Нет | Дополнительная информация |

## Статусы инцидента

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Зарегистрирован** | Синий | Инцидент только что создан, ждёт реакции |
| **Расследуется** | Жёлтый | Назначен следователь, идёт расследование |
| **Корректирующие действия** | Оранжевый | Причина установлена, назначены меры по устранению |
| **Решён** | Зелёный | Все меры выполнены, инцидент разрешён |
| **Закрыт** | Серый | Финальный статус. Изменения невозможны |

```
Зарегистрирован → Расследуется → Корректирующие действия → Решён → Закрыт
      ↓                                                       ↑
      └──────────────────────────────────────────────────────────┘
                        (можно закрыть напрямую)
```

### Что делать на каждом этапе

**1. Зарегистрирован → Начать расследование**
- На детальной странице нажмите **«Начать расследование»**
- Назначьте следователя (инженера по ОТ или руководителя)
- Статус меняется на «Расследуется»

**2. Расследуется → Назначить корректирующие меры**
- Укажите **корневую причину** — почему это произошло
- Назначьте **корректирующие действия** — что нужно сделать, чтобы не повторилось
- Статус меняется на «Корректирующие действия»

**3. Корректирующие действия → Отметить решённым**
- Когда все действия выполнены, нажмите **«Отметить решённым»**
- Статус меняется на «Решён»

**4. Решён → Закрыть**
- Финальная проверка и закрытие инцидента
- Нажмите **«Закрыть»**

## Детальная страница инцидента

При нажатии на инцидент в списке открывается полная карточка.

### Основная информация (2/3 ширины)
- **Описание** инцидента
- **3 блока-показателя:** пострадавшие, потерянные рабочие дни, тип инцидента
- **Корневая причина** (оранжевый блок, если заполнена)
- **Корректирующие действия** (список мер)
- **Хронология** — история изменений статуса

### Боковая панель (1/3 ширины)
- Проект, местоположение
- Кто сообщил, дата инцидента
- Следователь (если назначен)
- **Кнопки действий** (зависят от текущего статуса)

## Пострадавшие

Для каждого инцидента можно зафиксировать подробную информацию о пострадавших:

| Поле | Описание |
|------|---------|
| ФИО | Полное имя пострадавшего |
| Должность | Кем работал на момент инцидента |
| Отдел / Бригада | Подразделение |
| Стаж работы | Опыт (в годах) |
| **Тип травмы** | Перелом / Порез / Ушиб / Ожог / Электротравма / Отравление / Падение с высоты / Удар предметом / Тепловой удар / Прочее |
| **Часть тела** | Голова / Глаза / Шея / Грудь / Спина / Живот / Рука (лев/прав) / Кисть / Нога / Стопа / Множественные |
| Медпомощь | Оказана ли |
| Госпитализация | Больница (название) |
| Потерянные дни | Сколько дней не работал |
| Вернулся ли на работу | Да / Нет, дата возвращения |
| **Исход** | Выздоровление / Стойкая нетрудоспособность / Смертельный исход |

## Корректирующие действия

После расследования назначаются конкретные меры по устранению причин:

| Поле | Описание |
|------|---------|
| **Тип действия** | Немедленное / Корректирующее / Предупредительное |
| **Описание** | Что конкретно нужно сделать |
| **Ответственный** | Кто выполняет |
| **Срок** | До какой даты |
| **Статус** | Запланировано → В работе → Выполнено |
| **Дата проверки** | Когда проверили выполнение |
| **Эффективность** | Помогло ли (да / нет) |

**Типы действий:**

| Тип | Описание | Пример |
|-----|---------|--------|
| **Немедленное** | Срочно, прямо сейчас | Огородить опасную зону, остановить работы |
| **Корректирующее** | Устранить причину | Заменить неисправные леса, починить ограждение |
| **Предупредительное** | Не допустить в будущем | Провести дополнительный инструктаж, изменить технологию |

> **Просрочено:** Если срок прошёл, а действие не выполнено — статус автоматически меняется на «Просрочено» (красный).

## Акт Н-1 (расследование несчастного случая)

**Путь:** Безопасность → **Акты Н-1**

Акт по форме **Н-1** — это официальный документ расследования несчастного случая на производстве. Составляется комиссией, утверждается руководителем, направляется в Государственную инспекцию труда (ГИТ).

### Когда нужен акт Н-1

- **Любой несчастный случай** на производстве с потерей трудоспособности хотя бы на 1 день
- **Перевод на другую работу** по медицинским показаниям вследствие травмы
- **Смерть работника** на рабочем месте

### Как создать акт Н-1

1. Перейдите **Безопасность → Акты Н-1**
2. Нажмите **«+ Новый акт»**
3. Заполните форму:

| Раздел | Поля |
|--------|------|
| **Связь с инцидентом** | Номер инцидента (если уже зарегистрирован) |
| **Сведения о пострадавшем** | ФИО (обязательно), должность, дата рождения, пол, стаж, дата последнего инструктажа, вид инструктажа |
| **Обстоятельства** | Подробное описание: что произошло |
| **Причины** | Почему это произошло (корневые причины) |
| **Свидетели** | Кто видел |
| **Виновные лица** | Кто допустил нарушение |
| **Корректирующие меры** | Что сделать, чтобы не повторилось |

4. Нажмите **«Создать»** → акт в статусе «Черновик»

### Статусы акта Н-1

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Черновик** | Серый | Создан, можно редактировать |
| **Расследование** | Жёлтый | Комиссия работает |
| **На рассмотрении** | Синий | Акт готов, на подписи у руководства |
| **Утверждён** | Зелёный | Подписан руководителем |
| **Направлен в ГИТ** | Фиолетовый | Отправлен в Государственную инспекцию труда |
| **Закрыт** | Серый | Расследование завершено |

```
Черновик → Расследование → На рассмотрении → Утверждён → Направлен в ГИТ → Закрыт
                                  ↓
                            Расследование (вернуть на доработку)
```

### Содержание акта Н-1

Акт содержит все данные, требуемые формой Н-1 (утверждена Постановлением Минтруда):

1. **Сведения о пострадавшем** — ФИО, должность, стаж, дата рождения
2. **Сведения об инструктаже** — когда и какой инструктаж проходил пострадавший
3. **Состав комиссии** — председатель и члены комиссии по расследованию
4. **Даты расследования** — начало и окончание
5. **Обстоятельства** — подробное описание события
6. **Причины** — корневые причины несчастного случая
7. **Виновные лица** — кто нарушил требования безопасности
8. **Мероприятия** — что нужно сделать для предотвращения
9. **Травма** — описание повреждений, тяжесть, госпитализация, смертельный исход
10. **Потерянные дни** — количество дней нетрудоспособности

## Дашборд безопасности

**Путь:** Безопасность → **Дашборд**

На главном дашборде безопасности видна сводная статистика:

| Показатель | Что означает |
|-----------|-------------|
| **Инциденты за период** | Сколько инцидентов произошло за выбранный период |
| **Активные инциденты** | Сколько инцидентов не закрыты |
| **LTIFR** | Lost Time Injury Frequency Rate — количество травм с потерей рабочего времени на 1 млн человеко-часов. Хорошо: < 1.0. Средне: 1–5. Плохо: > 5 |
| **Потерянные дни** | Сумма рабочих дней, потерянных из-за инцидентов |
| **Нарушения** | Количество активных нарушений |
| **Просроченные** | Нарушения с истёкшим сроком устранения |

## Типичные сценарии

### Сценарий 1: Травма на площадке

```
1. Рабочий упал с лесов, сломал руку
2. Бригадир немедленно регистрирует инцидент:
   → Тип: Падение | Тяжесть: Серьёзный
   → Пострадавший: Иванов И.И., каменщик
   → Место: Секция А, 3 этаж, строительные леса
   → Статус: Зарегистрирован

3. Инженер по ОТ начинает расследование:
   → Назначен следователем
   → Статус: Расследуется
   → Опрашивает свидетелей, осматривает площадку

4. Результат расследования:
   → Корневая причина: Неисправные перила на лесах
   → Корректирующие действия:
     • Немедленное: Остановить работы на лесах, огородить
     • Корректирующее: Заменить перила на всех секциях
     • Предупредительное: Проверять леса каждое утро
   → Статус: Корректирующие действия

5. Действия выполнены:
   → Леса починены, проведён внеплановый инструктаж
   → Статус: Решён → Закрыт

6. Создан Акт Н-1 (т.к. потеря трудоспособности):
   → Заполнены все разделы формы
   → Утверждён → Направлен в ГИТ
```

### Сценарий 2: Предпосылка (near miss)

```
1. Кирпич упал с 5-го этажа, но никого не задел
2. Регистрация:
   → Тип: Предпосылка | Тяжесть: Умеренный
   → Пострадавших: 0
   → Статус: Зарегистрирован

3. Расследование:
   → Причина: Не закреплены материалы на краю перекрытия
   → Действие: Установить защитные козырьки, инструктаж по складированию
   → Статус: Решён → Закрыт

(Акт Н-1 НЕ требуется — нет пострадавших)
```

## Частые вопросы

**Кто может регистрировать инциденты?**
Любой пользователь с ролью ADMIN, MANAGER или ENGINEER. На практике — бригадир или инженер по ОТ.

**Обязательно ли создавать Акт Н-1?**
Акт Н-1 обязателен при любом несчастном случае с потерей трудоспособности. Для предпосылок (near miss) и незначительных инцидентов без пострадавших — не требуется.

**В какой срок нужно зарегистрировать инцидент?**
Немедленно. Чем быстрее зафиксирован инцидент — тем точнее информация для расследования.

**Что такое LTIFR?**
Lost Time Injury Frequency Rate — международный показатель. Формула: (количество травм с потерей рабочего времени × 1 000 000) / общее количество отработанных человеко-часов. Ниже — лучше.

**Можно ли удалить инцидент?**
Технически да (мягкое удаление), но не рекомендуется. Инциденты — часть аудиторского следа.

**Кто расследует инцидент?**
Назначается руководителем. Обычно это инженер по ОТ или руководитель проекта. При тяжёлых случаях — комиссия из нескольких человек.

**Куда направляется Акт Н-1?**
В Государственную инспекцию труда (ГИТ). Система фиксирует дату отправки в поле «Направлен в ГИТ».
