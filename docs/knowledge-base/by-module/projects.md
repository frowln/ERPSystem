# Проекты

---

## Что это и зачем

Проект — это ваш строительный объект. Всё, что связано с объектом — сметы, финансы, задачи, документы, бригады, безопасность — привязывается к проекту. Без проекта система пуста: это как папка, в которую складывается вся информация по стройке.

Один проект = один объект. Если у вас три стройки — создайте три проекта.

---

## Где найти

**Путь:** Главное меню → **Проекты**

---

## Список проектов

Когда вы открываете раздел «Проекты», вы видите таблицу со всеми вашими объектами.

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Шифр | Короткий код проекта (например, «ЖК-15») |
| Название | Полное название + имя заказчика под ним |
| Статус | Цветная метка: черновик, планирование, в работе, на паузе, завершён, отменён |
| Тип объекта | Жилое, коммерческое, промышленное, инфраструктура, реконструкция |
| Менеджер | Ответственный за проект |
| Плановый срок | Дата завершения |
| Бюджет | Плановая сумма проекта |
| Прогресс | Полоска выполнения в процентах |

**Вкладки-фильтры** (вверху таблицы):
- **Все** — все проекты в системе
- **В работе** — только активные стройки
- **Планирование** — черновики и объекты на стадии подготовки
- **Завершённые** — сданные объекты

**Карточки-показатели** (над таблицей):
- Всего проектов (и сколько из них активных)
- Общий бюджет (и сколько уже потрачено)
- Средний прогресс по всем проектам
- Размер команды (сколько людей задействовано)

**Фильтры и поиск:**
- Строка поиска — ищет по названию, шифру или имени заказчика
- Фильтр по статусу — выпадающий список

**Кнопки:**
- **Создать проект** — откроется форма создания
- **Экспорт в CSV** — скачать таблицу в Excel

## Как создать проект

1. Нажмите кнопку **«Создать проект»** в правом верхнем углу
2. Заполните форму:

**Основная информация:**
- **Шифр** (обязательно) — короткий код, до 30 символов. Например: «ЖК-15», «ЦЕХ-03»
- **Название** (обязательно) — полное название объекта. Например: «Жилой комплекс "Солнечный", корпус 3»
- **Вид строительства** (обязательно) — выберите из списка:
  - Новое строительство
  - Реконструкция
  - Капитальный ремонт
  - Демонтаж
  - Техническое переоснащение
- **Тип объекта** — жилое, коммерческое, промышленное, инфраструктура, реконструкция (можно добавить свой тип)
- **Описание** — свободный текст, до 3000 символов

**Адрес:**
- **Область** — регион
- **Город** — населённый пункт
- **Адрес** — точный адрес объекта

**Участники:**
- **Заказчик** (обязательно) — начните вводить название, система подскажет из базы контрагентов. Если заказчика нет — можно создать нового прямо из этого поля. Подтянется ИНН автоматически

**Сроки:**
- **Плановая дата начала** — когда планируете начать
- **Плановая дата окончания** — когда планируете сдать

**Проектная документация:**
- Перетащите файлы (PDF, DWG, Excel, Word, фото, архивы) — система определит категорию автоматически (чертёж, смета, разрешение, договор и т.д.)

3. Нажмите **«Создать проект»**

> **Что произойдёт автоматически:** После создания система автоматически создаст финансовую модель (ФМ) и коммерческое предложение (КП) для этого проекта. Статус проекта установится в «Планирование» — предстроительная подготовка.

## Страница проекта

Нажмите на название проекта в списке — откроется детальная страница.

**Вверху страницы:**
- Название проекта и шифр
- Текущий статус (цветная метка)
- Кнопка **«Изменить статус»** — переключить стадию
- Кнопка **«Редактировать»** — изменить данные проекта

**Вкладки:**

| Вкладка | Что внутри |
|---------|-----------|
| **Обзор** | Сводная информация: адрес, заказчик, сроки, финансовые показатели. Ссылки на реестр рисков, предстроительное совещание |
| **Команда** | Кто работает на этом проекте: список участников с ролями и датой подключения |
| **Документы** | Все документы проекта по категориям: чертежи, сметы, договоры, фото. Загрузка и скачивание файлов |
| **Финансы (ФМ)** | Финансовая модель проекта: плановый бюджет, сметная стоимость, себестоимость, цена для заказчика, маржа и рентабельность в процентах |
| **Почта** | Внутренняя переписка по проекту — сообщения между участниками |
| **Предстроительная подготовка** | Территория и разрешения, проектирование и экспертиза, мобилизация, длинные поставки, чек-лист готовности к стройке, трекер вех |

## Стадии проекта

Проект проходит через несколько стадий. Каждая стадия имеет свой цвет:

| Стадия | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** | Серый | Проект создан, но ещё не начали работать. Можно спокойно заполнять данные |
| **Планирование** | Синий | Идёт предстроительная подготовка: разрешения, проектирование, сметы, торги |
| **В работе** | Зелёный | Стройка идёт. Основной рабочий статус |
| **На паузе** | Жёлтый | Работы временно остановлены (ожидание финансирования, погода, согласования) |
| **Завершён** | Фиолетовый | Объект сдан, работы окончены |
| **Отменён** | Красный | Проект отменён (заказчик отказался, не получено разрешение и т.д.) |

**Как переключить стадию:**
1. Откройте проект
2. Нажмите кнопку **«Изменить статус»** в правом верхнем углу
3. Выберите новый статус из списка
4. Статус изменится мгновенно

> **Подсказка:** Новый проект при создании автоматически получает статус «Планирование». Когда выйдете на площадку — переключите на «В работе».

---

## Как редактировать проект

1. Откройте проект (нажмите на его название в списке)
2. Нажмите кнопку **«Редактировать»** в правом верхнем углу
3. Измените нужные поля — все те же поля, что были при создании
4. Нажмите **«Сохранить изменения»**

> **Подсказка:** Менять статус можно отдельно, не заходя в режим редактирования — через кнопку «Изменить статус».

---

## Как удалить / архивировать проект

**Удаление:**
- Удалить проект может только администратор
- При удалении будут удалены все привязанные данные: задачи, документы, записи бюджета, сметы
- Это действие нельзя отменить

**Архивирование:**
- Вместо удаления лучше переведите проект в статус **«Завершён»** или **«Отменён»**
- Завершённые проекты остаются в системе со всей историей, но не мешают в рабочем списке (отфильтруйте по вкладке «В работе»)

> **Внимание:** Перед удалением убедитесь, что все нужные документы и отчёты сохранены отдельно. Удаление проекта затронет сметы, бюджеты, задачи, КС-2, КС-3 и все другие связанные записи.

## Связь с другими модулями

| Модуль | Как связан с проектом |
|--------|----------------------|
| **Сметы** | Смета привязывается к проекту. Из сметы рассчитываются объёмы и цены |
| **Бюджет / ФМ** | Финансовая модель создаётся автоматически при создании проекта. Показывает план/факт по деньгам |
| **КП** | Коммерческое предложение формируется из финансовой модели проекта |
| **Задачи** | Задачи создаются внутри проекта — конкретные работы с ответственными и сроками |
| **Бригады** | Бригады назначаются на проект — кто работает на объекте |
| **Документы** | Все документы (чертежи, разрешения, акты) привязываются к проекту |
| **КС-2 / КС-3** | Акты выполненных работ формируются по проекту для сдачи заказчику |
| **Журнал работ** | Ежедневные записи о работе на объекте |
| **Склад** | Расход материалов списывается на проект |
| **Безопасность** | Инциденты, инспекции и инструктажи привязаны к проекту |
| **Качество** | Дефекты, чек-листы и проверки привязаны к проекту |
| **Календарь** | События и совещания привязываются к проекту |
| **Планирование** | Диаграмма Ганта и WBS строятся по задачам проекта |
| **Спецификации** | Ведомости материалов привязываются к проекту |

## Печатные формы

Из проекта можно скачать или распечатать:
- **Карточка проекта** — сводная информация в PDF
- **Реестр проектов** — экспорт списка в CSV (кнопка «Экспорт» на странице списка)
- **Отчёт передачи (Handoff)** — при передаче проекта другому менеджеру

## Частые вопросы

**Как передать проект другому менеджеру?**
Откройте проект → Редактировать → измените поле «Менеджер проекта». Старый менеджер потеряет доступ к управлению (если у него нет другой роли), новый получит. Также можно использовать «Отчёт передачи» на вкладке проекта.

**Можно ли видеть все проекты на одном экране?**
Да. Страница «Проекты» показывает все объекты в одной таблице. Карточки-показатели вверху показывают сводку. Для более глубокого анализа откройте **Портфель → Здоровье портфеля** — там RAG-матрица (красный/жёлтый/зелёный) по 7 метрикам.

**Что означают цвета статусов?**
Серый = черновик, синий = планирование, зелёный = в работе, жёлтый = на паузе, фиолетовый = завершён, красный = отменён. Подробнее — в разделе «Стадии проекта» выше.

**Можно ли добавить свой тип объекта?**
Да. При создании проекта в поле «Тип объекта» можно добавить собственный тип — он сохранится и будет доступен в будущем.

**Что такое «Предстроительная подготовка» на странице проекта?**
Это вкладка, которая помогает подготовиться к началу стройки: оценка площадки, получение ГПЗУ, разрешений, технических условий; управление проектированием, экспертизой; мобилизация техники и бригад; контроль длинных поставок. Есть чек-лист готовности и трекер вех.

**Зачем нужен шифр проекта?**
Шифр — это уникальный короткий код (до 30 символов). Он используется в документах, отчётах и переписке для быстрой идентификации. Удобно, когда у вас много проектов: «ЖК-15» проще написать, чем «Жилой комплекс "Солнечный", корпус 3».
