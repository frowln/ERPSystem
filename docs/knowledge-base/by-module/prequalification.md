# Предквалификация подрядчиков

---

## Что это и зачем

**Предквалификация** — процесс проверки подрядчиков ДО того, как их допустят к тендерам и работам. Цель — убедиться, что подрядчик надёжный, имеет опыт, лицензии, технику и финансовую устойчивость.

Зачем это нужно:
- **Снижение рисков** — отсеиваем ненадёжных на входе, а не когда они уже сорвали сроки
- **Экономия времени** — в тендерах участвуют только проверенные компании
- **Юридическая защита** — документальное подтверждение должной осмотрительности (для ФНС)
- **Качество работ** — проверенные подрядчики = меньше переделок

## Часть 1: Критерии оценки

Каждый подрядчик оценивается по нескольким категориям:

| Категория | Что проверяем | Вес |
|-----------|--------------|-----|
| **Опыт** | Сколько лет на рынке, какие объекты строили, объёмы | 25% |
| **Лицензии** | СРО, допуски, лицензии (действующие!) | 20% |
| **Финансы** | Выручка, долги, судебные дела, налоговые задолженности | 20% |
| **Техника** | Собственная техника, парк, вместимость | 15% |
| **Кадры** | Численность, квалификация ИТР, аттестации | 10% |
| **Безопасность** | Статистика несчастных случаев, обучение ОТ | 10% |

### Система оценок

| Оценка | Значение | Допуск к тендерам |
|--------|----------|-------------------|
| **A** | Отлично (80-100 баллов) | Да, без ограничений |
| **B** | Хорошо (60-79 баллов) | Да |
| **C** | Удовлетворительно (40-59 баллов) | С ограничениями (мелкие работы) |
| **D** | Неудовлетворительно (0-39 баллов) | Нет |

---

## Часть 2: Анкета подрядчика

Подрядчик заполняет анкету с информацией:

| Раздел | Документы |
|--------|-----------|
| **Общие сведения** | Устав, ОГРН, ИНН, выписка ЕГРЮЛ |
| **СРО** | Свидетельство о допуске, выписка из реестра СРО |
| **Финансы** | Бухгалтерский баланс, справка об отсутствии долгов |
| **Опыт** | Портфолио объектов, рекомендательные письма, контракты |
| **Техника** | Перечень техники (собственная и арендованная) |
| **Кадры** | Штатное расписание, удостоверения ИТР |
| **Охрана труда** | Статистика за 3 года, протоколы обучения |
| **Страхование** | Полисы страхования ответственности |

---

## Часть 3: Процесс проверки

```
1. Подрядчик подаёт заявку на предквалификацию
2. Заполняет анкету и прикладывает документы
3. Специалист проверяет документы:
   - СРО — через реестр Ростехнадзора
   - ИНН/ОГРН — через ФНС
   - Судебные дела — через arbitr.ru
   - Финансы — через ФНС и ФССП
4. Выставляет баллы по каждой категории
5. Система рассчитывает итоговую оценку (A/B/C/D)
6. Решение: допущен / не допущен / допущен с ограничениями
7. Подрядчик получает уведомление
8. Через 12 месяцев — повторная проверка
```

## Часть 4: Реестр подрядчиков

В реестре видно:

| Колонка | Описание |
|---------|----------|
| **Компания** | Название подрядчика |
| **ИНН** | Для идентификации |
| **Специализация** | СМР, электрика, сантехника и т.д. |
| **Оценка** | A / B / C / D |
| **Статус** | Действующий / Истёк / На проверке |
| **Дата проверки** | Когда последний раз проверяли |
| **Допуск** | К каким работам допущен |

Фильтры: по специализации, оценке, статусу.

## Типичный сценарий

```
Ситуация: Нужен субподрядчик на электромонтажные работы

1. Снабженец находит 5 компаний
2. Отправляет им ссылку на анкету предквалификации
3. 4 из 5 заполняют (один отказался — уже подозрительно)
4. Проверка:
   - ООО «ЭлектроПро» — оценка A (опыт 12 лет, СРО, нет долгов)
   - ООО «ТокСтрой» — оценка B (5 лет, СРО, небольшие долги)
   - ИП Иванов — оценка D (нет СРО, 2 судебных дела)
   - ООО «Вольт» — оценка C (опыт есть, но техники мало)
5. К тендеру допущены: ЭлектроПро, ТокСтрой, Вольт (с ограничением)
6. ИП Иванов — отказ с объяснением
```

## Частые вопросы

**Обязательна ли предквалификация?**
Юридически — нет (для частных компаний). Практически — да, если вы работаете с госзаказами (44-ФЗ, 223-ФЗ) или крупными заказчиками. Даже без обязательства — это защита от рисков.

**Как часто нужно перепроверять?**
Рекомендуется раз в 12 месяцев. Система напомнит, когда срок проверки истекает.

**Подрядчик не прошёл — можно ли всё равно с ним работать?**
Технически система не заблокирует, но зафиксирует, что работа ведётся с неквалифицированным подрядчиком. Это риск для компании и потенциальные вопросы от заказчика и надзорных органов.
