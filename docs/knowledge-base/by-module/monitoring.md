# IoT мониторинг

---

## Что это и зачем

**IoT мониторинг** — это система сбора данных с датчиков на строительном объекте. IoT (Internet of Things) — «интернет вещей» — это когда физические устройства (датчики температуры, влажности, вибрации) передают данные в систему автоматически.

Без IoT мониторинга:
- Температуру бетона замеряют вручную термометром — 3 раза в сутки, в том числе ночью
- Уровень грунтовых вод проверяют раз в неделю — а он поднялся за ночь и затопил котлован
- Шум и вибрация от стройки — жалобы жителей, а доказать, что нормы не превышены, нечем
- Кто-то проник на объект ночью — узнали утром, когда украли инструмент

Модуль «IoT мониторинг» в системе ПРИВОД:
- Подключение датчиков (температура, влажность, вибрация, шум и др.)
- Автоматический сбор показаний в реальном времени
- Правила оповещения — если значение выходит за допустимый диапазон
- Геозоны — виртуальные границы на карте (кто зашёл, кто вышел)
- Обнаружение аномалий — система сама находит отклонения

## Часть 1: Устройства (датчики)

### Типы датчиков

| Тип датчика | Что измеряет | Где используется |
|-------------|-------------|-----------------|
| **Температура** | °C | Контроль температуры бетона, помещений |
| **Влажность** | % | Условия хранения материалов, сушка |
| **Давление** | Па, бар | Системы водоснабжения, отопления |
| **Вибрация** | мм/с | Контроль при забивке свай, влияние на здания |
| **Шум** | дБ | Контроль уровня шума (жалобы жителей) |
| **Газ** | ppm | Обнаружение утечки газа, CO, CO₂ |
| **Движение** | да/нет | Охрана объекта, контроль доступа |
| **Уровень** | мм, см | Уровень воды в котловане, ёмкостях |

### Статусы устройства

| Статус | Цвет | Описание |
|--------|------|----------|
| **Активно** (ACTIVE) | Зелёный | Датчик работает, данные поступают |
| **Неактивно** (INACTIVE) | Серый | Датчик выключен |
| **Ошибка** (ERROR) | Красный | Датчик не передаёт данные (поломка, разряд) |
| **Обслуживание** (MAINTENANCE) | Жёлтый | Калибровка, замена батареи |

### Карточка устройства

| Поле | Описание |
|------|----------|
| **Название** | Например: «Термодатчик бетон — блок А, секция 1» |
| **Тип** | Температура / Влажность / Вибрация... |
| **Серийный номер** | Уникальный идентификатор устройства |
| **Расположение** | Где установлен (координаты или описание) |
| **Проект** | К какому объекту привязан |
| **Интервал передачи** | Как часто отправляет данные (каждые 5 мин, 15 мин, 1 час) |
| **Батарея** | Уровень заряда (%) |
| **Последний сигнал** | Когда последний раз передал данные |

## Часть 2: Показания датчиков

**Путь:** IoT мониторинг → **Показания**

На этой странице видны **графики показаний** в реальном времени.

### Что видно

- **Текущее значение** — последнее показание датчика
- **График за период** — как менялось значение (час, день, неделя, месяц)
- **Допустимый диапазон** — зелёная зона на графике
- **Отклонения** — красные точки, когда значение вышло за допустимые пределы
- **Обнаруженные аномалии** — система автоматически выявляет необычные паттерны

### Обнаружение аномалий

Система анализирует данные и находит **необычные отклонения**:
- Резкий скачок температуры
- Постепенное повышение вибрации (возможная деформация конструкции)
- Периодическое повышение уровня воды (приток грунтовых вод)

Аномалии помечаются специальным значком на графике и отправляются в виде оповещений.

## Часть 3: Правила оповещения

**Путь:** IoT мониторинг → **Оповещения**

### Типы условий

| Условие | Описание | Пример |
|---------|----------|--------|
| **Больше чем** | Значение превысило порог | Температура > 35°C |
| **Меньше чем** | Значение упало ниже порога | Температура < −10°C |
| **Вне диапазона** | Значение вышло за пределы | Влажность не в 40–60% |
| **Нет данных** | Датчик перестал передавать | Нет сигнала > 30 минут |

### Серьёзность оповещений

| Уровень | Цвет | Действие |
|---------|------|----------|
| **Информация** | Синий | Просто записать в журнал |
| **Предупреждение** | Жёлтый | Уведомить ответственного |
| **Тревога** | Оранжевый | Срочно уведомить + SMS |
| **Критическое** | Красный | Немедленное оповещение всех + звонок |

### Настройка правила

| Поле | Пример |
|------|--------|
| **Название** | «Высокая температура бетона» |
| **Датчик** | Термодатчик блок А |
| **Условие** | Больше чем 70°C |
| **Серьёзность** | Тревога |
| **Кого уведомить** | Прораб, главный инженер |
| **Как уведомить** | Система + Telegram + SMS |

## Часть 4: Геозоны

**Геозона** — это виртуальная граница на карте. Система фиксирует, когда человек или техника входит в зону или покидает её.

### Типы геозон

| Тип | Описание | Пример |
|-----|----------|--------|
| **Объект** (SITE) | Территория строительной площадки | Периметр стройки |
| **Опасная зона** (DANGER) | Зона с повышенной опасностью | Зона работы крана |
| **Ограниченный доступ** (RESTRICTED) | Только для допущенных лиц | Склад ГСМ |
| **Парковка** (PARKING) | Зона парковки техники | Площадка стоянки |

### Типы событий геозоны

| Событие | Описание |
|---------|----------|
| **Вход** (ENTER) | Кто-то вошёл в зону |
| **Выход** (EXIT) | Кто-то покинул зону |
| **Нарушение** (VIOLATION) | Вход в запрещённую зону без допуска |

### Зачем нужны геозоны

- **Охрана**: фиксация входа/выхода в нерабочее время
- **Безопасность**: предупреждение при входе в опасную зону
- **Учёт рабочего времени**: фактическое время пребывания на объекте
- **Контроль техники**: техника не должна покидать объект без путевого листа

## Типичные сценарии

### Сценарий 1: Контроль температуры бетона зимой

```
1. Ноябрь, бетонирование фундамента ЖК «Южный парк»
   При зимнем бетонировании критически важна температура

2. Установлены 6 термодатчиков в теле бетона:
   → Датчики передают данные каждые 15 минут
   → Допустимый диапазон: +5°C ... +40°C

3. Правила оповещения:
   → Если < +5°C → «Тревога» → прораб + главный инженер
   → Если < 0°C → «Критическое» → все + SMS + звонок

4. Ночью в 3:00:
   → Датчик №3 показал +4°C
   → Сработала тревога → SMS прорабу
   → Прораб звонит на объект: «Включить дополнительный обогрев!»

5. К утру:
   → Температура вернулась к +12°C ✅
   → Бетон не замёрз, качество не пострадало
   → Без датчика — узнали бы утром, когда уже поздно
```

### Сценарий 2: Вибромониторинг при забивке свай

```
1. Рядом со стройкой — жилой дом (30 метров)
   Жители жалуются на вибрацию

2. Устанавливают вибродатчики:
   → На фундаменте жилого дома — 2 штуки
   → На стройплощадке — 1 штука

3. Норма по СН 2.2.4/2.1.8.566-96:
   → Допустимая вибрация для жилых зданий: 0.2 мм/с
   → Правило: если > 0.2 мм/с → «Тревога»

4. Во время забивки свай:
   → Вибрация на доме: 0.15 мм/с → в норме ✅
   → Данные записываются — есть документальное подтверждение

5. Жители пишут жалобу:
   → Компания предоставляет данные мониторинга
   → Все показания в пределах нормы — жалоба необоснована ✅
```

### Сценарий 3: Геозона для охраны

```
1. Объект «Склад ГСМ» — ограниченный доступ
   Геозона: радиус 50 метров, тип «Ограниченный доступ»

2. Правило:
   → Вход в рабочее время (7:00–19:00) → «Информация» (лог)
   → Вход в нерабочее время → «Тревога» → охрана + руководитель

3. Суббота, 23:00:
   → GPS-трекер фиксирует движение в геозоне
   → Сработала тревога → охрана получает уведомление
   → Охрана проверяет — оказался сторож, делает обход ✅

4. Лог событий сохраняется — всегда видно, кто и когда
   заходил в зону
```

## Частые вопросы

**Какие датчики можно подключить?**
Любые датчики, которые передают данные по протоколам MQTT, HTTP или LoRaWAN. Конкретные модели — уточняйте у службы поддержки.

**Нужен ли интернет на объекте?**
Да, для передачи данных в систему. Некоторые датчики работают через мобильную сеть (SIM-карта), другие через Wi-Fi.

**Как часто датчики передают данные?**
Настраивается: от раз в секунду до раз в час. Чем чаще — тем точнее, но быстрее расходуется батарея.

**Сколько стоит установка датчиков?**
Зависит от количества и типа. Один датчик температуры — от 3 000 руб. Вибродатчик — от 15 000 руб. Установка и настройка — отдельно.

**Что если датчик сломался?**
Система обнаружит отсутствие сигнала и отправит оповещение «Нет данных». Нужно заменить или починить датчик.

**Хранятся ли данные с датчиков?**
Да, все показания хранятся. Можно посмотреть историю за любой период. Данные используются для отчётов и экспертиз.
