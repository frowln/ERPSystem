# Коммерческие предложения (КП)

---

## Что это и зачем

Коммерческое предложение — это документ с ценами, который вы отправляете заказчику. В нём указано, сколько стоит каждая позиция проекта (материалы и работы) и итоговая сумма. КП формируется на основе финансовой модели (ФМ), где уже сведены себестоимость из конкурентного листа и сметные цены из ЛСР.

В КП две колонки цен:
- **Себестоимость** — сколько стоит нам (внутренняя информация, заказчик не видит)
- **Цена заказчику** — сколько платит клиент

Разница — это маржа (прибыль).

**Цепочка:** [Спецификация](specifications.md) → [КЛ](competitive-list.md) → [Финансовая модель](financial-model.md) → **Коммерческое предложение**

---

## Где найти

**Путь:** Главное меню → **Финансы** → **Коммерческие предложения**

## Список коммерческих предложений

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Название | Имя КП (например, «КП — Жилой комплекс, корпус 1») |
| Статус | На каком этапе: черновик, на проверке, утверждено, активно |
| Общая себестоимость | Сумма всех позиций по себестоимости |
| Дата создания | Когда создано КП |

**Вкладки-фильтры:**
- **Все** — все КП
- **Черновик** — в разработке
- **На проверке** — отправлено на согласование
- **Утверждено** — согласовано руководством
- **Активно** — действующее КП

**Поиск:** по названию или объекту.

## Как создать КП

### Из финансовой модели (рекомендуется)

1. Откройте финансовую модель проекта
2. Нажмите кнопку **«Создать КП»** (фиолетовая, в шапке)
3. Укажите название (по умолчанию: «КП — {название бюджета}»)
4. Нажмите **«Создать КП»**

Система автоматически:
- Создаст КП в статусе «Черновик»
- Скопирует все позиции из ФМ (названия, количества, единицы измерения, себестоимость)
- Разделит позиции на **материалы** и **работы**
- Перенаправит вас на страницу нового КП

### Вручную

1. Перейдите в **Финансы → Коммерческие предложения**
2. Нажмите **«Новое КП»**
3. Заполните форму:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Финансовая модель** | Да | Выберите ФМ, на основе которой создаётся КП |
| **Название КП** | Да | Например: «КП по материалам и работам» |
| **Примечания** | Нет | Дополнительные комментарии |

4. Нажмите **«Создать КП»**

## Страница КП (детальный вид)

### Навигация по цепочке

В верхней части страницы — **навигационная цепочка**, показывающая путь данных:

```
Спецификация → Конкурентный лист → Коммерческое предложение → Финансовая модель
```

Каждый элемент — ссылка для быстрого перехода.

### Показатели (карточки вверху)

| Показатель | Что значит |
|-----------|-----------|
| **Себестоимость** | Общая сумма всех позиций по закупочным ценам |
| **Цена заказчику** | Общая сумма для клиента |
| **Маржа** | Цена заказчику − себестоимость (в рублях) |
| **Маржа %** | То же в процентах |
| **Материалы** | Количество позиций материалов |
| **Работы** | Количество позиций работ |

### Статусная линейка

Вверху КП — визуальная линейка из 4 шагов. Текущий шаг выделен, пройденные — с галочкой:

```
[1. Черновик] → [2. На проверке] → [3. Утверждено] → [4. Активно]
```

### Две вкладки

КП разделено на две вкладки:
- **Материалы** — все позиции типа «материал» и «оборудование»
- **Работы** — все позиции типа «работа»

## Вкладка «Материалы»

Позиции сгруппированы по **дисциплинам** (марки разделов: «ЭМ», «ОВ», «ВК» и т.д.). Если дисциплина не указана — группа «Прочее».

**Колонки таблицы:**

| Колонка | Что содержит |
|---------|-------------|
| **Наименование** | Название позиции из ФМ |
| **Кол-во** | Количество единиц |
| **Ед.** | Единица измерения |
| **Себестоимость** | Закупочная цена за единицу (жирным, ₽) |
| **Цена заказчику** | Цена для клиента (редактируемая, кликните для ввода) |
| **Источник цены** | Откуда взята себестоимость (цветная метка) |
| **Статус** | Состояние обработки позиции |
| **Счёт** | Привязан ли счёт поставщика |
| **Действия** | Кнопки согласования/отклонения |

**Метки источника цены:**
- **Из сметы** (синяя) — цена из локальной сметы
- **Из конкурентного листа** (фиолетовая) — цена от победителя КЛ
- **Из счёта** (зелёная) — цена из привязанного счёта поставщика
- **Вручную** (серая) — введена оператором

**Работа с материалами:**

1. **Привяжите счёт** — нажмите «Выбрать счёт» в колонке «Счёт». Система покажет подходящие строки из входящих счетов. Выберите нужный — себестоимость обновится автоматически
2. **Установите цену заказчику** — кликните на ячейку и введите цену. Под ценой автоматически рассчитается маржа в процентах (зелёная — прибыль, красная — убыток)
3. **Согласуйте позицию** — нажмите галочку (кнопка «Согласовать»)

> **Важно:** Для согласования материала необходимо сначала привязать счёт поставщика. Без счёта кнопка согласования будет неактивна.

## Вкладка «Работы»

Структура аналогична материалам, но с дополнительными возможностями.

**Дополнительные колонки:**

| Колонка | Что содержит |
|---------|-------------|
| **Сметная цена** | Расчётная цена из сметы (до применения торгового коэффициента) |
| **Торговый коэффициент** | Множитель: > 1 = наценка (зелёный), < 1 = скидка (красный), = 1 = без изменений |

**Работа с позициями работ:**

1. **Себестоимость** заполняется одним из способов:
   - Из привязанной сметы (автоматически)
   - Из результатов тендера подрядчиков (кнопка «Тендер подрядчиков»)
   - Вручную (ссылка «Задать вручную»)
2. **Торговый коэффициент** — если привязана смета, можно изменить коэффициент. Себестоимость = сметная цена × коэффициент
3. **Цена заказчику** — аналогично материалам

> **Для согласования работы** необходимо: привязанная смета ИЛИ введённая вручную себестоимость > 0.

## Тендер подрядчиков

Для работ доступна функция **«Тендер подрядчиков»** — массовое назначение себестоимости из результатов тендера.

В шапке каждой дисциплины (раздела) есть кнопка **«Тендер подрядчиков»** (янтарная, с иконкой кубка).

**Как пользоваться:**

1. Нажмите кнопку — откроется модальное окно
2. Выберите тендер из списка (показываются доступные тендеры проекта)
3. Если у тендера есть победитель — появится кнопка **«Применить победителя тендера»** (зелёная)
4. Укажите себестоимость за единицу (или система возьмёт цену победителя)
5. Нажмите **«Применить к N позициям»**

Себестоимость обновится сразу у **всех** позиций дисциплины.

---

## Торговые коэффициенты / скидки

Торговый коэффициент — это множитель, который связывает сметную цену с себестоимостью для работ.

| Коэффициент | Значение | Цвет | Пример |
|------------|---------|------|--------|
| **> 1** | Наценка | Зелёный | 1.15 = сметная + 15% |
| **= 1** | Без изменений | Серый | Себестоимость = сметная |
| **< 1** | Скидка (торг) | Красный | 0.85 = скидка 15% от сметной |

**Формула:**
```
Себестоимость = Сметная цена × Торговый коэффициент
```

**Пример:** Сметная цена монтажа = 100 000 руб. Подрядчик дал скидку 10%. Торговый коэффициент = 0.90. Себестоимость = 100 000 × 0.90 = 90 000 руб.

## Статусы КП

| Статус | Цвет | Что означает | Что можно делать |
|--------|------|-------------|-----------------|
| **Черновик** | Серый | В разработке | Свободное редактирование, привязка счетов, установка цен |
| **На проверке** | Жёлтый | Отправлено на согласование | Согласование/отклонение позиций. Кнопка «Подтвердить все» |
| **Утверждено** | Зелёный | Согласовано руководством | Передача в ФМ. Создание новой версии |
| **Активно** | Синий | Действующее КП | Только просмотр. Можно создать новую версию |

**Переходы:**
```
Черновик → На проверке → Утверждено → Активно
                ↓
          Черновик (возврат на доработку)
```

## Статусы позиций КП

Каждая позиция КП проходит свой путь обработки:

| Статус | Что означает |
|--------|-------------|
| **Не обработана** | Начальное состояние. Нужно привязать счёт (для материала) или смету (для работы) |
| **Цена выбрана** | Себестоимость определена (из счёта, сметы, КЛ или вручную) |
| **Согласовано** | Позиция одобрена ответственным |
| **Подтверждено** | Финальное подтверждение перед передачей в ФМ |
| **В финансовой модели** | Данные переданы в ФМ |

## Передача в ФМ

Когда все позиции согласованы и подтверждены, данные из КП можно передать обратно в финансовую модель.

1. Убедитесь, что КП в статусе **«Утверждено»** или **«Активно»**
2. Нажмите кнопку **«Передать в ФМ»**
3. Откроется окно подтверждения:
   - Количество передаваемых позиций (материалы + работы)
   - Общая стоимость
   - Список позиций с источниками цен
4. Нажмите **«Передать»**

Система обновит в ФМ:
- Себестоимость (из КП) → costPrice в бюджете
- Цену заказчику (из КП) → customerPrice в бюджете
- Статус позиций КП → «В финансовой модели»

### Чек-лист готовности

Перед передачей система проверяет:
- Материалы без привязанного счёта — покажет количество
- Работы без привязанной сметы — покажет количество
- Несогласованные позиции — покажет количество

Все три показателя должны быть = 0 для полной готовности.

## Версионирование

Можно создавать новые версии КП:

1. Нажмите **«Новая версия»** на странице КП
2. Система создаст копию текущего КП с увеличенным номером версии
3. Новая версия станет текущей
4. Предыдущая версия сохраняется для истории

Это удобно, если заказчик просит пересмотреть цены — вы создаёте новую версию и корректируете, а оригинал остаётся.

## Как отправить заказчику

### Через PDF

1. Нажмите **«Экспорт PDF»** на странице КП
2. Скачается PDF-файл с печатной формой
3. Отправьте файл заказчику по электронной почте или через мессенджер

### Через портал заказчика

Если заказчик подключён к порталу ПРИВОД:
1. КП автоматически появляется в его портале после утверждения
2. Заказчик может просмотреть, согласовать или запросить изменения
3. Согласование фиксируется в системе

Подробнее о портале — в разделе «Портал заказчика».

## Панель конкурентного листа

На странице КП для каждой позиции доступна **панель КЛ** — список предложений поставщиков из конкурентного листа.

- Зелёная точка — самая низкая цена
- Красная точка — самая высокая цена
- Галочка — выбранный поставщик

Можно выбрать другого поставщика — его цена станет новой себестоимостью позиции.

## Полная цепочка: Спецификация → КЛ → ФМ → КП

```
1. СПЕЦИФИКАЦИЯ (specifications.md)
   ЧТО нужно: арматура 15 т, кабель 5000 м, монтаж...
   Цен НЕТ — только технические данные из проектной документации
        ↓
2. КОНКУРЕНТНЫЙ ЛИСТ (competitive-list.md)
   ПОЧЁМ: собираем цены от 3+ поставщиков
   Выбираем победителя → себестоимость определена
        ↓
3. ФИНАНСОВАЯ МОДЕЛЬ (financial-model.md)
   СВОДНАЯ ТАБЛИЦА:
   ├── Себестоимость  ← из КЛ (45 000 руб./т)
   ├── Сметная        ← из ЛСР (50 000 руб./т)
   └── Цена заказчику ← наценка (60 000 руб./т)
   Маржа = 60 000 − 45 000 = 15 000 руб. (25%)
        ↓
4. КОММЕРЧЕСКОЕ ПРЕДЛОЖЕНИЕ (эта страница)
   ДОКУМЕНТ ДЛЯ ЗАКАЗЧИКА:
   ├── Материалы: позиции + себестоимость + цена заказчику
   ├── Работы: позиции + торговый коэффициент + цена заказчику
   └── Печатная форма → PDF → заказчику
```

## Частые вопросы

**Чем КП отличается от ФМ?**
ФМ — это внутренний инструмент для расчётов (три колонки цен, маржа, НДС). КП — это документ для заказчика. Из ФМ создаётся КП, но заказчик видит только цены «для него», а не вашу себестоимость.

**Можно ли создать КП без ФМ?**
Нет. КП всегда создаётся на основе финансовой модели. Сначала создайте проект (ФМ создастся автоматически), затем создавайте КП.

**Как изменить цену заказчику?**
На вкладке «Материалы» или «Работы» кликните на ячейку в колонке «Цена заказчику» — она станет редактируемой. Введите число и кликните вне ячейки.

**Что такое торговый коэффициент?**
Это множитель для пересчёта сметной цены в себестоимость работ. Если подрядчик даёт скидку 10%, коэффициент = 0.90. Если наценка 15%, коэффициент = 1.15.

**Как привязать счёт к материалу?**
На вкладке «Материалы» нажмите «Выбрать счёт» в строке нужного материала. Система покажет подходящие строки из входящих счетов. Выберите нужный — себестоимость обновится автоматически.

**Можно ли отправить КП заказчику прямо из системы?**
Да, через портал заказчика. Если заказчик подключён — он увидит КП в своём портале после утверждения. Также можно скачать PDF и отправить по почте.

**Как создать новую версию КП?**
Нажмите **«Новая версия»** на странице КП. Создастся копия с увеличенным номером версии. Оригинал сохранится для истории.

**Что означает статус «В финансовой модели»?**
Это значит, что данные этой позиции (себестоимость и цена заказчику) уже переданы в ФМ проекта. Передача делается кнопкой «Передать в ФМ» после утверждения КП.
