# Заказы на закупку (Purchase Orders)

---

## Что это и зачем

Заказ на закупку (PO — Purchase Order) — это официальный документ для поставщика: «Отгрузите нам 10 тонн арматуры по цене 80 000 руб./тонна к 20 марта на адрес...». Это юридическое основание для поставки — без PO поставщик не обязан ничего отгружать.

Полный цикл закупки:
```
Потребность → Заявка → Согласование → Заказ поставщику (PO) → Отгрузка → Приёмка → Счёт → Оплата
```

В этой статье описано всё: от заявки до закрытия заказа.

## Часть 1: Заявка на закупку (Purchase Request)

Заявка — это внутренний запрос: «Нам нужно купить 10 тонн арматуры для проекта X». Создаёт прораб или снабженец, одобряет руководитель.

### Как создать заявку

1. Перейдите **Закупки → Заявки**
2. Нажмите **«+ Новая заявка»**

3. Заполните форму:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Дата** | Да | Дата заявки |
| **Проект** | Да | Для какого объекта |
| **Договор** | Нет | Привязка к договору |
| **Спецификация** | Нет | Ссылка на техдокументацию |
| **Приоритет** | Да | Низкий / Средний / Высокий / Критический |
| **Примечание** | Нет | Пояснения, особые требования |

4. Добавьте позиции — что нужно купить:

| Поле позиции | Описание |
|-------------|----------|
| Наименование | Что закупаем (например, «Арматура А500С d12») |
| Количество | Сколько единиц |
| Ед. изм. | Штуки, метры, тонны и т.д. |
| Ориентировочная цена | Примерная стоимость за единицу |

5. Нажмите **«Создать»** — заявка сохраняется в статусе «Черновик».

### Статусы заявки

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Черновик** | Серый | Создана, можно редактировать позиции |
| **Отправлена** | Синий | Направлена на согласование |
| **На согласовании** | Жёлтый | Руководитель проверяет |
| **Согласована** | Зелёный | Одобрена — можно создавать заказ поставщику |
| **Назначена** | Фиолетовый | Передана снабженцу для работы |
| **Заказано** | Голубой | PO создан, ждём поставку |
| **Доставлено** | Тёмно-зелёный | Материалы на складе |
| **Закрыта** | Серый | Заявка выполнена полностью |
| **Отклонена** | Красный | Не одобрена (с указанием причины) |
| **Отменена** | Красный | Больше не нужна |

```
Черновик → Отправлена → На согласовании → Согласована → Назначена → Заказано → Доставлено → Закрыта
                              ↓
                          Отклонена → Черновик (доработка)
```

### Приоритеты

| Приоритет | Когда использовать |
|-----------|-------------------|
| **Низкий** | Можно подождать, плановая закупка |
| **Средний** | Стандартные сроки |
| **Высокий** | Нужно скоро, влияет на график |
| **Критический** | Работа стоит! Срочная закупка |

### Канбан-доска заявок

На доске заявки разложены по колонкам-статусам. Удобно видеть, сколько заявок на каждом этапе и где затор.

```
| Черновик (3) | На согласовании (5) | Согласована (2) | Заказано (8) | Доставлено (4) |
|   [ЗК-101]  |     [ЗК-095]        |    [ЗК-088]     |   [ЗК-076]   |    [ЗК-062]    |
|   [ЗК-102]  |     [ЗК-096]        |    [ЗК-090]     |   [ЗК-079]   |    [ЗК-065]    |
```

## Часть 2: Заказ поставщику (Purchase Order)

### Как создать заказ

**Способ 1: Из заявки (рекомендуется)**

1. Откройте **согласованную** заявку
2. Нажмите **«Создать заказ поставщику»**
3. Позиции и количества подтянутся автоматически
4. Выберите поставщика и укажите цены
5. Нажмите **«Создать»**

**Способ 2: Вручную**

1. Перейдите **Закупки → Заказы**
2. Нажмите **«+ Новый заказ»**

3. Заполните форму:

### Основные данные

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Номер** | Авто | Уникальный номер PO (генерируется системой) |
| **Дата заказа** | Да | Дата оформления |
| **Поставщик** | Да | Кому заказываем (из справочника контрагентов) |
| **Проект** | Нет | Для какого объекта |
| **Заявка** | Нет | Ссылка на исходную заявку (если есть) |
| **Договор** | Нет | По какому договору |

### Условия

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Ожидаемая дата доставки** | Да | Когда поставщик должен привезти |
| **Адрес доставки** | Нет | Куда доставлять |
| **Условия оплаты** | Нет | Например: «50% предоплата, 50% по факту» |
| **Валюта** | Нет | По умолчанию — рубли (₽) |
| **Примечание** | Нет | Особые условия для поставщика |

### Позиции заказа

Для каждой позиции:

| Поле | Описание |
|------|---------|
| **Материал** | Что заказываем (из справочника или текстом) |
| **Количество** | Сколько единиц |
| **Ед. изм.** | Единица измерения |
| **Цена за единицу** | Согласованная цена |
| **Ставка НДС** | По умолчанию 20% |
| **Сумма** | Авто: количество × цена |
| **НДС** | Авто: сумма × ставка НДС |

**Итоги заказа:**
```
Позиция 1: Арматура d12, 10 т × 80 000 ₽ = 800 000 ₽
Позиция 2: Цемент М400, 50 т × 5 000 ₽  = 250 000 ₽
─────────────────────────────────────────────────────
Подитого:                                  1 050 000 ₽
НДС 20%:                                    210 000 ₽
ИТОГО:                                     1 260 000 ₽
```

4. Нажмите **«Создать»** — заказ сохраняется в статусе «Черновик».

---

## Статусы заказа (PO)

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Черновик** | Серый | Заказ формируется, можно редактировать позиции |
| **Отправлен** | Синий | Заказ отправлен поставщику |
| **Подтверждён** | Зелёный | Поставщик подтвердил приём заказа |
| **Частично доставлен** | Жёлтый | Часть позиций привезена, ждём остальное |
| **Доставлен** | Тёмно-зелёный | Всё доставлено полностью |
| **Оплачен** | Фиолетовый | Счёт получен и сверен |
| **Закрыт** | Серый | Заказ завершён, все расчёты выполнены |
| **Отменён** | Красный | Заказ аннулирован |

```
Черновик → Отправлен → Подтверждён → Частично доставлен → Доставлен → Оплачен → Закрыт
                                              ↑                ↑
                                              └────────────────┘
                                           (несколько поставок)

Отменён ← (из любого статуса до «Доставлен»)
```

## Жизненный цикл заказа

### Шаг 1. Создание (Черновик)

Снабженец создаёт заказ, добавляет позиции, проверяет цены.

**Можно:** редактировать, добавлять/удалять позиции, удалить заказ.
**Нельзя:** ничего — это рабочая стадия.

### Шаг 2. Отправка поставщику (Отправлен)

Нажмите **«Отправить поставщику»**. Заказ уходит поставщику (по email или через портал).

**Можно:** ждать подтверждения, отменить.
**Нельзя:** менять позиции — заказ зафиксирован.

### Шаг 3. Подтверждение (Подтверждён)

Поставщик подтвердил: «Да, привезём к указанной дате».

**Можно:** ждать доставки, отменить (если передумали).
**Нельзя:** менять условия.

### Шаг 4. Доставка

Поставщик привозит материалы. Кладовщик фиксирует, что именно и сколько получено.

**Как зарегистрировать доставку:**

1. Откройте карточку PO
2. Нажмите **«Зарегистрировать доставку»**
3. Для каждой позиции укажите **фактически полученное количество**

| Позиция | Заказано | Доставлено ранее | Доставлено сейчас |
|---------|--------:|------------------:|------------------:|
| Арматура d12 | 10 т | 0 т | 7 т |
| Цемент М400 | 50 т | 0 т | 50 т |

4. Нажмите **«Подтвердить приёмку»**

**Что происходит:**
- Если **всё** доставлено → статус «Доставлен»
- Если **не всё** → статус «Частично доставлен» (ждём остальное)
- На [складе](warehouse.md) автоматически создаётся **приход** (движение типа RECEIPT)
- Остатки обновляются

### Шаг 5. Счёт и оплата (Оплачен)

После полной доставки — получаем счёт от поставщика и сверяем:

**Трёхстороннее сопоставление (3-way matching):**

| Документ | Что проверяем |
|----------|-------------|
| **Заказ (PO)** | Что заказывали: позиции, количества, цены |
| **Приёмка** | Что фактически получили |
| **Счёт** | Что поставщик выставляет к оплате |

Если всё совпадает — счёт подтверждается. Если нет — расхождения расследуются.

**Типичные расхождения:**
- Заказали 100 шт, получили 95 → недопоставка
- Цена в счёте выше, чем в заказе → несогласованное повышение
- В счёте позиции, которых нет в заказе → ошибка

Нажмите **«Привязать счёт»** → система свяжет PO со [счётом](invoices.md).

### Шаг 6. Закрытие (Закрыт)

Нажмите **«Закрыть заказ»** — финальный статус. Все данные зафиксированы для отчётности.

## Частичная доставка

Поставщик не всегда привозит всё сразу. Система поддерживает несколько поставок:

**Пример:**

```
Заказ: 100 дверей + 200 петель

Доставка 1 (5 марта):
  Двери: 75 шт (из 100)
  Петли: 200 шт (из 200) ✓
  → Статус: «Частично доставлен»

Доставка 2 (12 марта):
  Двери: 25 шт (из оставшихся 25) ✓
  → Статус: «Доставлен» (всё получено)
```

При каждой доставке:
- Количество «Доставлено» увеличивается
- На складе создаётся приход
- Когда всё доставлено — статус автоматически меняется на «Доставлен»

## Полный пример цикла

```
1. Прораб создаёт заявку:
   «Нужно 10 т арматуры d12 для проекта "Дом на Ленина"»
   Приоритет: Высокий
   → Статус заявки: ЧЕРНОВИК

2. Прораб отправляет на согласование
   → Статус заявки: ОТПРАВЛЕНА → НА СОГЛАСОВАНИИ

3. Руководитель проекта одобряет
   → Статус заявки: СОГЛАСОВАНА

4. Назначает снабженцу
   → Статус заявки: НАЗНАЧЕНА

5. Снабженец создаёт PO из заявки:
   Поставщик: ООО «МеталлТрейд»
   Цена: 80 000 ₽/т → Итого: 960 000 ₽ (с НДС)
   Доставка: 20 марта
   → Статус PO: ЧЕРНОВИК
   → Статус заявки: ЗАКАЗАНО

6. Снабженец отправляет заказ поставщику
   → Статус PO: ОТПРАВЛЕН

7. Поставщик подтверждает заказ
   → Статус PO: ПОДТВЕРЖДЁН

8. 20 марта — поставщик привозит 7 т (не всё)
   Кладовщик регистрирует доставку: 7 т
   → Статус PO: ЧАСТИЧНО ДОСТАВЛЕН
   → На складе: +7 т арматуры

9. 25 марта — привозят оставшиеся 3 т
   Кладовщик регистрирует: 3 т
   → Статус PO: ДОСТАВЛЕН
   → На складе: +3 т арматуры
   → Статус заявки: ДОСТАВЛЕНО

10. Бухгалтерия получает счёт от поставщика
    Сверяют: заказ (10 т) = доставка (10 т) = счёт (10 т × 80 000 ₽) ✓
    → Статус PO: ОПЛАЧЕН

11. Оплата проведена → заказ закрыт
    → Статус PO: ЗАКРЫТ
    → Статус заявки: ЗАКРЫТА
```

| Роль | Что может |
|------|----------|
| **Прораб / Инженер** | Создать заявку, добавить позиции, отправить на согласование |

| **Администратор** | Полный доступ ко всем операциям |
| **Наблюдатель** | Только просмотр, без действий |

## Частые вопросы

**Можно ли создать PO без заявки?**
Да. Заказ можно создать напрямую, без привязки к заявке. Но рекомендуется начинать с заявки — это даёт контроль и согласование.

**Можно ли изменить заказ после отправки поставщику?**
Нет. После отправки позиции заблокированы. Если нужны изменения — отмените заказ и создайте новый.

**Что если поставщик привёз меньше, чем заказали?**
Зарегистрируйте фактическое количество. Система поставит статус «Частично доставлен». Можно: ждать допоставки, отменить остаток, или создать отдельный заказ на недостающее.

**Что такое трёхстороннее сопоставление?**
Это проверка: заказ = доставка = счёт. Если заказали 100, получили 100 и счёт на 100 — всё сходится. Если нет — нужно разбираться в расхождениях.

**Может ли одна заявка породить несколько PO?**
Связь 1:1 — одна заявка порождает один PO. Если нужно заказать у разных поставщиков — создайте отдельные заявки.

**Кто видит мои заявки?**
Заявки видны руководителю проекта (для согласования), снабженцу (для исполнения) и администратору. Другие пользователи видят только свои заявки.

**Как отследить, что заказ доставлен?**
В списке заказов фильтруйте по статусу «Частично доставлен» или «Подтверждён» (ожидает доставки). В карточке PO виден прогресс доставки в процентах.

**Где увидеть историю поставок?**
В карточке заказа — таблица позиций с колонкой «Доставлено». Полная история приёмок — в модуле [Движение материалов](stock-movements.md).
