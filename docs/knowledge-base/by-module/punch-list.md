# Punch List (Перечень замечаний)

---

## Что это и зачем

**Punch list** (перечень замечаний) — это список всех недоделок и замечаний, которые нужно устранить перед сдачей объекта (или его части) заказчику. Название пришло из американской строительной практики: раньше замечания «пробивали» (punch) дыроколом в бумажном списке.

Пока punch list не закрыт — **объект не принимается**. Это финальный барьер перед подписанием акта приёмки.

В системе ПРИВОД punch list состоит из:
- **Перечень** (Punch List) — группа замечаний, привязанная к проекту или зоне
- **Замечание** (Punch Item) — конкретная недоделка с ответственным и сроком

## Как это устроено

### Перечень (Punch List)

Перечень — это «папка», которая объединяет замечания по проекту, зданию, этажу или зоне.

| Поле | Описание |
|------|----------|
| **Название** | Например: «Punch list — Здание А, 3 этаж» |
| **Проект** | К какому проекту |
| **Описание** | Дополнительные пояснения |
| **Статус** | Открыт / В работе / Завершён |
| **Процент закрытия** | Рассчитывается автоматически |

**Процент закрытия** = (закрытые замечания ÷ все замечания) × 100%

Например: 15 из 20 замечаний закрыты = **75%**

### Статусы перечня

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Открыт** (OPEN) | Красный | Создан, замечания ещё есть |
| **В работе** (IN_PROGRESS) | Жёлтый | Идёт устранение замечаний |
| **Завершён** (COMPLETED) | Зелёный | Все замечания закрыты (100%) |

## Замечания (Punch Item)

Каждое замечание — это конкретная недоделка, которую нужно устранить.

### Как создать замечание

1. Перейдите в перечень или на страницу **Punch List → Список замечаний**
2. Нажмите **«+ Новое замечание»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Название** | Да | Краткое описание (например: «Скол плитки у двери 305») |
| **Перечень** | Да | К какому punch list относится |
| **Описание** | Нет | Подробности: что, где, как выглядит |
| **Приоритет** | Да | Низкий / Средний / Высокий / Критический |
| **Категория** | Да | Конструктивные / Архитектурные / Электрика / Сантехника / Отделка / Пожарная безопасность / Прочее |
| **Ответственный** | Нет | Кто должен устранить |
| **Срок** | Нет | До когда |
| **Фото** | Рекомендуется | Фотофиксация |

4. Нажмите **«Создать»** — замечание в статусе «Открыто»

### Приоритеты

| Приоритет | Цвет | Описание | Пример |
|-----------|------|----------|--------|
| **Низкий** (LOW) | Зелёный | Мелкая косметика | Потёртость на подоконнике |
| **Средний** (MEDIUM) | Жёлтый | Заметный, но не критичный | Кривая розетка, царапина на двери |
| **Высокий** (HIGH) | Оранжевый | Влияет на функцию | Не закрывается дверь, течёт кран |
| **Критический** (CRITICAL) | Красный | Нельзя использовать | Не работает отопление, нет электричества |

### Категории

| Категория | Что входит |
|-----------|-----------|
| **Конструктивные** | Стены, перекрытия, фундамент, каркас |
| **Архитектурные** | Фасад, окна, двери, перегородки |
| **Электрика** | Проводка, розетки, выключатели, освещение |
| **Сантехника** | Трубы, краны, сантехприборы, канализация |
| **Отделка** | Штукатурка, покраска, плитка, полы |
| **Пожарная безопасность** | Системы оповещения, огнезащита, пути эвакуации |
| **Прочее** | Всё остальное |

### Статусы замечания

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Открыто** (OPEN) | Красный | Замечание зафиксировано |
| **В работе** (IN_PROGRESS) | Жёлтый | Подрядчик устраняет |
| **Исправлено** (FIXED) | Синий | Подрядчик сообщил: «Готово» |
| **Проверено** (VERIFIED) | Зелёный | Инспектор подтвердил: действительно исправлено |
| **Закрыто** (CLOSED) | Серый | Окончательно закрыто |

```
Открыто → В работе → Исправлено → Проверено → Закрыто
                    ↘               ↓
                     ← (не исправлено, возврат)
```

### Комментарии

К каждому замечанию можно добавлять комментарии — это история обсуждения:
- Инженер: «Не закрывается дверь, петля провисла»
- Подрядчик: «Регулировка выполнена 05.03»
- Инженер: «Проверил, дверь работает нормально. Закрываю.»

## Канбан-доска

**Путь:** Punch List → **Доска**

Замечания разложены по колонкам-статусам:

```
| Открыто (8) | В работе (5) | Исправлено (3) | Проверено (4) | Закрыто (20) |
|  [Скол]     |  [Дверь]     |  [Розетка]     |  [Окно]       |  [Плитка]    |
|  [Трещина]  |  [Плинтус]   |  [Кран]        |  [Свет]       |  [Штукатурка]|
```

Каждая карточка показывает:
- Название замечания
- Приоритет (цветная метка)
- Категория
- Ответственный
- Срок

Можно перетаскивать карточки между колонками для быстрого обновления статуса.

---

## Дашборд

**Путь:** Punch List → **Дашборд**

Показывает общую картину:

- **Всего замечаний** — общее количество
- **Открытых** — сколько ещё не устранено
- **Процент закрытия** — прогресс-бар (например, 75%)
- **По категориям** — сколько замечаний в каждой категории
- **По приоритетам** — сколько критических, высоких и т.д.
- **Просроченных** — сколько вышло за срок

## Типичные сценарии

### Сценарий 1: Предпусковая приёмка этажа

```
1. Здание А, 3 этаж — работы завершены, подрядчик просит принять

2. Инженер создаёт punch list: «Приёмка — Здание А, 3 этаж»

3. Обходит этаж с планшетом и фиксирует замечания:
   → Комната 301: скол плитки у двери (Отделка, Средний)
   → Комната 302: не работает розетка (Электрика, Высокий)
   → Коридор: трещина в штукатурке у окна (Отделка, Средний)
   → Комната 305: не закрывается окно (Архитектурные, Высокий)
   → Санузел 303: течёт стык трубы (Сантехника, Критический)

4. Итого: 5 замечаний, процент закрытия: 0%

5. Подрядчик получает список и начинает устранять:
   → День 1: течь трубы устранена (Критический) → Исправлено
   → День 2: розетка, окно, плитка → Исправлено
   → День 3: штукатурка → Исправлено

6. Инженер проводит повторный обход:
   → Все 5 пунктов исправлены ✅
   → Процент закрытия: 100%
   → Статус перечня: Завершён

7. Этаж принят, акт подписан
```

### Сценарий 2: Финальная сдача объекта

```
1. Заказчик приходит на объект для приёмки

2. Совместный обход с представителем заказчика:
   → Создаётся punch list «Приёмка заказчиком — Здание А»
   → Заказчик отмечает 12 замечаний
   → 3 критических (нет горячей воды, не работает лифт, нет пожарной сигнализации)
   → 9 средних и низких (косметика)

3. Подписывается протокол с перечнем замечаний

4. Подрядчик устраняет:
   → Неделя 1: критические (3) ✅
   → Неделя 2: средние (6) ✅
   → Неделя 3: низкие (3) ✅

5. Повторная приёмка: 12/12 = 100% ✅
   → Заказчик подписывает акт приёмки
```

## Частые вопросы

**Чем punch list отличается от реестра дефектов?**
Дефекты фиксируются на протяжении всего строительства. Punch list — это финальный перечень замечаний перед сдачей. Часть дефектов может перейти в punch list, если не была устранена ранее.

**Кто составляет punch list?**
Обычно инженер по качеству или руководитель проекта при совместном обходе с заказчиком (представителем технического надзора).

**Можно ли добавлять замечания после создания перечня?**
Да. Punch list — живой документ. Новые замечания добавляются по мере обнаружения.

**Что если подрядчик не согласен с замечанием?**
Обсуждается в комментариях к замечанию. Если замечание признано необоснованным — можно закрыть с пометкой. Но обычно решение принимает заказчик.

**Какой процент закрытия считается достаточным?**
100%. Для подписания акта приёмки все замечания должны быть устранены. В отдельных случаях заказчик может принять объект с незначительными замечаниями (и гарантийным письмом).

**Как посмотреть историю замечания?**
Откройте карточку замечания — там видна вся история: кто создал, кто менял статус, комментарии, фото до/после.
