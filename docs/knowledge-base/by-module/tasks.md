# Задачи

---

## Что это и зачем

Задача — это конкретная работа, которую нужно сделать. «Залить фундамент блока А», «Согласовать смету с заказчиком», «Заказать арматуру» — всё это задачи. У каждой задачи есть ответственный, срок и приоритет.

Задачи помогают ничего не забыть и видеть, кто что делает прямо сейчас. Руководитель ставит задачу — исполнитель её видит и берёт в работу. Всё прозрачно.

---

## Где найти

**Путь:** Главное меню → **Задачи**

---

## Канбан-доска

Когда вы открываете «Задачи», вы видите **канбан-доску** — визуальную доску с колонками. Каждая колонка — это этап работы. Задачи расположены карточками в соответствующих колонках.

**Стандартные колонки (можно настроить):**

| Колонка | Цвет | Что означает |
|---------|------|-------------|
| **Бэклог** | Серый | Запланировано, но пока не в работе. Идеи и задачи «на потом» |
| **К выполнению** | Синий | Нужно сделать в ближайшее время |
| **В работе** | Жёлтый | Сейчас выполняется. Лимит: 5 задач одновременно |
| **На проверке** | Фиолетовый | Работа сделана, нужно проверить результат. Лимит: 5 задач |
| **Готово** | Зелёный | Выполнено и проверено |

> **Подсказка:** Чтобы переместить задачу на следующий этап — просто перетащите карточку мышкой в нужную колонку.

**Что видно на карточке задачи:**
- Код задачи (например, «TASK-42»)
- Название задачи
- Приоритет (цветная метка)
- Ответственный (аватар)
- Проект (мелким шрифтом)
- Срок (красным, если просрочена)
- Прогресс (полоска 0–100%)
- Звёздочка — добавить в избранное

**Фильтры** (над доской):
- Поиск по названию, коду или исполнителю
- Фильтр по проекту
- Фильтр по исполнителю
- Фильтр по приоритету
- Переключатель «Только мои» — показать только ваши задачи
- Кнопка сброса фильтров

**Виды отображения** (переключатель вверху):
- **Доска** — канбан с колонками (по умолчанию)
- **Список** — таблица со всеми задачами
- **Ганта** — ленточный график по срокам
- **Календарь** — задачи на календарной сетке
- **Мои задачи** — персональный дашборд

## Как создать задачу

**Способ 1: Кнопка «+» в колонке**
Нажмите «+» внизу любой колонки — задача создастся сразу в этом статусе.

**Способ 2: Кнопка «Создать задачу»**
Нажмите кнопку в правом верхнем углу — откроется форма.

**Поля формы:**

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Название** | Да | Что нужно сделать. Например: «Завершить кладку 3-го этажа» |
| **Описание** | Нет | Подробности: что именно, какие условия, ссылки на документы |
| **Проект** | Нет | К какому объекту относится задача |
| **Исполнитель** | Нет | Кто будет выполнять. Можно назначить позже |
| **Приоритет** | Да | Низкий, обычный, высокий, срочный или критический |
| **Статус** | Да | В какую колонку попадёт задача (по умолчанию — «К выполнению») |
| **Дата начала** | Нет | Когда планируется начать |
| **Дата завершения** | Нет | Срок выполнения (дедлайн) |
| **Видимость** | Да | Кто видит задачу: только участники, весь проект или вся организация |
| **Участники** | Нет | Дополнительные люди: соисполнители (могут выполнять) и наблюдатели (только видят) |

> **Подсказка:** Не обязательно заполнять всё сразу. Создайте задачу с названием и приоритетом — остальное добавите позже.

---

## Приоритеты задач

| Приоритет | Цвет | Когда использовать |
|-----------|------|-------------------|
| **Низкий** | Серый | Можно сделать когда будет время. Не горит |
| **Обычный** | Синий | Стандартная рабочая задача |
| **Высокий** | Оранжевый | Важно, нужно обратить внимание |
| **Срочный** | Красный | Горит. Нужно сделать сегодня-завтра |
| **Критический** | Тёмно-красный | Блокирует другие работы. Всё бросить и делать |

## Карточка задачи (детальный вид)

Нажмите на задачу — справа откроется панель с полной информацией.

**Основные данные:**
- Код и название задачи
- Статус и приоритет (цветные метки)
- Проект
- Исполнитель и автор задачи
- Прогресс (ползунок 0–100%)
- WBS-код (если задача привязана к структуре работ)
- Метки/теги

**Вкладки в карточке:**

| Вкладка | Что внутри |
|---------|-----------|
| **Комментарии** | Обсуждение задачи. Кто что написал и когда. Можно добавить комментарий |
| **Подзадачи** | Дочерние задачи — если основную задачу нужно разбить на части |
| **Файлы** | Прикреплённые документы, фото, чертежи. Можно перетащить файл для загрузки |
| **Активность** | Лог всех изменений: кто когда менял статус, назначал исполнителя, добавлял комментарии |
| **Зависимости** | Связи с другими задачами. Например: «эту задачу нельзя начать, пока не закончена другая» |

**Сроки:**
- Плановая дата начала и окончания
- Фактическая дата начала и окончания
- Количество дней просрочки (красным, если есть)

**Трудозатраты:**
- Плановые часы
- Фактические часы
- Визуальное сравнение

**Действия:**
- Добавить в избранное (звёздочка)
- Поделиться (изменить видимость)
- Делегировать (передать другому исполнителю)
- Удалить

## Стадии задач (настройка колонок)

Стандартные 5 колонок можно изменить под ваш бизнес-процесс.

**Как настроить:**
1. На канбан-доске нажмите кнопку **«Настроить стадии»** (иконка шестерёнки)
2. Справа откроется панель управления стадиями
3. Можно:
   - **Добавить** новую стадию (например, «На согласовании»)
   - **Переименовать** существующую
   - **Удалить** ненужную
   - **Изменить порядок** — перетаскиванием

> **Подсказка:** Настройте колонки под свой процесс. Например, для строительной компании: «Подготовка» → «Материалы заказаны» → «Выполнение» → «Приёмка» → «Сдано».

## Зависимости задач

Зависимость — это когда одну задачу нельзя начать (или закончить), пока не завершена другая.

**Типы зависимостей:**
- **Окончание → Начало** — сначала закончить первую, потом начать вторую (самый частый тип). Пример: сначала залить фундамент, потом класть кирпич
- **Начало → Начало** — обе задачи начинаются одновременно
- **Окончание → Окончание** — обе задачи заканчиваются одновременно
- **Начало → Окончание** — используется редко

**Как добавить зависимость:**
1. Откройте задачу (нажмите на карточку)
2. Перейдите на вкладку **«Зависимости»**
3. Нажмите **«Добавить зависимость»**
4. Выберите задачу, от которой зависит текущая

> **Подсказка:** Зависимости видны на диаграмме Ганта в виде стрелок между задачами. Если задача-предшественник задержится — система покажет, что зависимые задачи тоже сдвигаются.

## Учёт времени

Для каждой задачи можно вести учёт потраченного времени.

**Как это работает:**
- Откройте задачу → в разделе «Трудозатраты»
- **Таймер** — нажмите «Старт», когда начинаете работу. Нажмите «Стоп» когда закончили
- **Ручной ввод** — добавьте запись вручную: дата, количество часов
- Система покажет общее время по задаче

## Назначение участников

К задаче можно подключить несколько людей с разными ролями:

| Роль | Что может делать |
|------|-----------------|
| **Исполнитель** | Основной ответственный за задачу |
| **Соисполнитель** | Помогает выполнять задачу. Видит всё, может менять статус |
| **Наблюдатель** | Только видит задачу и получает уведомления. Не может менять |

## Связь с другими модулями

| Модуль | Как связан |
|--------|-----------|
| **Проект** | Задача привязывается к проекту. Можно фильтровать задачи по проектам |
| **Календарь** | Дедлайны задач отображаются на календаре |
| **Планирование (Гант)** | Задачи показываются на диаграмме Ганта. Зависимости — стрелками |
| **Журнал работ** | Ежедневные записи могут ссылаться на задачи |
| **Наряды** | Наряд-задание формируется на основе задач |
| **Уведомления** | При назначении, изменении статуса или приближении срока — приходит уведомление |

## Частые вопросы

**Как увидеть только мои задачи?**
На канбан-доске нажмите переключатель **«Только мои»** в панели фильтров. Или перейдите на вкладку **«Мои задачи»** — там персональный дашборд со всеми вашими задачами по всем проектам.

**Задача просрочена — что делать?**
Просроченные задачи отмечены красным цветом даты на карточке. Откройте задачу, оцените ситуацию: либо перенесите срок (если причина объективная), либо повысьте приоритет и назначьте дополнительных исполнителей.

**Можно ли разбить задачу на подзадачи?**
Да. Откройте задачу → вкладка **«Подзадачи»** → **«Добавить подзадачу»**. Каждая подзадача — это полноценная задача со своим статусом и исполнителем.

**Как перенести задачу в другой проект?**
Откройте задачу → нажмите «Редактировать» → измените поле «Проект».

**Что такое лимит на колонку (WIP)?**
WIP (Work In Progress) — ограничение на количество задач в одной колонке. Например, если лимит «В работе» = 5, то система не даст взять шестую задачу, пока не закончите одну из текущих. Это помогает не распылять силы.

**Можно ли делегировать задачу?**
Да. Откройте задачу → кнопка **«Делегировать»** → выберите нового исполнителя. Задача передаётся ему, вы получите уведомление о выполнении.

**Как посмотреть все просроченные задачи по проекту?**
Система имеет API для получения просроченных задач по проекту. Они отмечены красной датой на доске и в списке. Также можно использовать AI-помощник — спросите «Покажи просроченные задачи по проекту [название]».
