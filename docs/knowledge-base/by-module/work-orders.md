# Наряд-заказы (Work Orders)

---

## Что это и зачем

**Наряд-заказ** — это задание на выполнение конкретной работы на объекте: что сделать, кому, к какому сроку, какие материалы нужны. Это ежедневный инструмент прораба.

В отличие от «Задач» (которые могут быть любыми — встреча, звонок, документ), наряд-заказ — это **производственное задание** с привязкой к объёмам работ, материалам и технике.

## Часть 1: Карточка наряд-заказа

| Поле | Описание |
|------|----------|
| **Номер** | Автоматический (WO-001, WO-002...) |
| **Название** | Краткое описание работы |
| **Проект** | К какому объекту |
| **Тип работы** | Строительно-монтажная / Отделочная / Электромонтажная / Земляная и др. |
| **Бригада** | Кто выполняет |
| **Дата начала / окончания** | Плановые сроки |
| **Объём** | Количество + единица измерения (м², м³, шт., т) |
| **Материалы** | Какие материалы нужны (из складских остатков) |
| **Техника** | Какая техника требуется |
| **Приоритет** | Обычный / Срочный / Критический |

### Статусы

| Статус | Цвет | Описание |
|--------|------|----------|
| **Черновик** | Серый | Создан, но не выдан |
| **Назначен** | Синий | Выдан бригаде |
| **В работе** | Жёлтый | Бригада приступила |
| **На проверке** | Фиолетовый | Работа завершена, ждёт приёмки |
| **Выполнен** | Зелёный | Принят прорабом / заказчиком |
| **Отменён** | Красный | Отменён (с причиной) |

## Часть 2: Дашборд операций

| Метрика | Что показывает |
|---------|---------------|
| **Всего нарядов** | За выбранный период |
| **В работе** | Сколько сейчас выполняется |
| **Просроченных** | Срок прошёл, а работа не сдана |
| **Выполнено сегодня** | Сколько закрыли за день |
| **По бригадам** | Загрузка каждой бригады |

---

## Часть 3: Диспетчерский календарь

Визуальное представление: дни × бригады. Каждый наряд-заказ — цветная полоска на календаре. Видно:
- Какая бригада чем занята
- Где есть окна (свободные дни)
- Где наложения (конфликты)

## Типичный сценарий

```
1. Прораб смотрит план на неделю:
   → Нужно забетонировать перекрытие 3 этажа

2. Создаёт наряд-заказ:
   → Бригада: Бетонщики (Иванов)
   → Объём: 45 м³ бетона М300
   → Материалы: бетон (проверяет склад — есть)
   → Техника: автобетононасос (бронирует)
   → Срок: пн-ср

3. Бригадир получает уведомление → начинает работу

4. Среда: бригадир отмечает «Выполнено»
   → Прораб проверяет качество → «Принято» ✅

5. Данные попадают в журнал КС-6а → основа для КС-2
```

## Частые вопросы

**Чем наряд-заказ отличается от задачи?**
Задача — это любое поручение (позвонить, согласовать документ, проверить). Наряд-заказ — это производственное задание с объёмами, материалами и техникой.

**Обязательно ли использовать наряд-заказы?**
Для небольших проектов достаточно задач. Наряд-заказы полезны, когда важно контролировать объёмы работ, расход материалов и загрузку бригад.
