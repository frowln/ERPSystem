# RFI (Запросы информации)

---

## Что это и зачем

**RFI** (Request for Information / Запрос информации) — это официальный вопрос от одного участника строительства к другому. Чаще всего подрядчик задаёт вопрос проектировщику: «В чертежах непонятно — поясните».

Зачем нужен RFI:
- **Уточнение проекта** — чертежи не всегда однозначны, бывают противоречия
- **Фиксация решений** — устный ответ забудется, а RFI = документ
- **Защита от претензий** — «мы строили согласно ответу на RFI №23»
- **Отслеживание сроков** — если проектировщик не отвечает неделю, стройка стоит

Примеры RFI:
- «На чертеже КР-12 стена 200 мм, а на КР-15 — 250 мм. Какой размер правильный?»
- «В проекте не указана марка бетона для подпорной стены. Какую использовать?»
- «На плане отсутствует закладная для трубопровода ø110. Нужна ли она?»

## Как создать RFI

1. Перейдите **RFI → Список**
2. Нажмите **«+ Новый RFI»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Тема** | Да | Краткое описание вопроса |
| **Проект** | Да | К какому проекту |
| **Вопрос** | Да | Подробный текст вопроса |
| **Приоритет** | Да | Низкий / Средний / Высокий / Срочный |
| **Кому** | Нет | Кому адресован (проектировщик, заказчик, субподрядчик) |
| **Влияние на стоимость** | Нет | Да/Нет — может ли ответ повлиять на стоимость |
| **Влияние на сроки** | Нет | Да/Нет — может ли ответ повлиять на график |
| **Срок ответа** | Нет | До когда нужен ответ |
| **Вложения** | Нет | Фрагменты чертежей, фото, схемы |

4. Нажмите **«Создать»** — RFI в статусе «Черновик»
5. Нажмите **«Отправить»** — RFI уходит адресату

---

## Статусы RFI

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** (DRAFT) | Серый | Создан, но не отправлен |
| **Открыт** (OPEN) | Синий | Отправлен, ожидает назначения ответственного |
| **Назначен** (ASSIGNED) | Жёлтый | Назначен конкретный специалист для ответа |
| **Отвечен** (ANSWERED) | Зелёный | Ответ получен |
| **Закрыт** (CLOSED) | Серый | Вопрос решён, RFI закрыт |
| **Аннулирован** (VOID) | Серый | Вопрос потерял актуальность или был ошибочным |

```
Черновик → Открыт → Назначен → Отвечен → Закрыт
                                           Аннулирован ← (из любого статуса)
```

---

## Приоритеты

| Приоритет | Цвет | Когда использовать | Типовой срок ответа |
|-----------|------|-------------------|:-------------------:|
| **Низкий** (LOW) | Зелёный | Вопрос для уточнения, работа не стоит | 14 дней |
| **Средний** (MEDIUM) | Жёлтый | Нужно для планирования ближайших работ | 7 дней |
| **Высокий** (HIGH) | Оранжевый | Работа задерживается без ответа | 3 дня |
| **Срочный** (URGENT) | Красный | Стройка стоит, нужен ответ немедленно | 1 день |

---

## Ответы на RFI

К одному RFI может быть **несколько ответов** — например, предварительный и окончательный, или ответы от разных специалистов.

Каждый ответ содержит:
| Поле | Описание |
|------|----------|
| **Автор** | Кто ответил |
| **Дата** | Когда |
| **Текст ответа** | Собственно ответ |
| **Вложения** | Файлы (исправленные чертежи, разъяснения) |
| **Официальный** | Да/Нет — является ли это официальным ответом (для юридической силы) |

**Важно:** Только ответ с отметкой **«Официальный»** считается основанием для выполнения работ. Предварительные ответы — для обсуждения.

---

## Влияние на стоимость и сроки

При создании RFI можно отметить:
- **Влияние на стоимость** — если ответ может привести к дополнительным затратам
- **Влияние на сроки** — если без ответа нельзя продолжать работу

Это важно для управления проектом: если 5 RFI отмечены как «влияющие на сроки» и ни один не отвечен — руководитель видит риск задержки.

---

## Отслеживание просрочки

Система автоматически отслеживает сроки ответа. Если срок ответа истёк, а ответа нет — RFI помечается как **просроченный** (красный индикатор).

На дашборде видно:
- Сколько RFI открыто
- Сколько просрочено
- Среднее время ответа
- Кто задерживает ответы чаще всего

---

## Канбан-доска

**Путь:** RFI → **Доска**

RFI разложены по колонкам-статусам:

```
| Черновик (2) | Открыт (5) | Назначен (3) | Отвечен (8) | Закрыт (25) |
|  [Вопрос1]   | [Вопрос3]  | [Вопрос6]    | [Вопрос9]   | [Вопрос12]  |
|  [Вопрос2]   | [Вопрос4]  | [Вопрос7]    | [Вопрос10]  | [Вопрос13]  |
```

Каждая карточка показывает:
- Тема RFI
- Приоритет (цветная метка)
- Кому назначен
- Срок ответа (красный, если просрочен)
- Есть ли влияние на стоимость/сроки (иконки)

## Типичные сценарии

### Сценарий 1: Противоречие в чертежах

```
1. Прораб обнаружил: на чертеже КР-12 стена 200 мм,
   а на КР-15 (в другом разрезе) — 250 мм

2. Создаёт RFI:
   → Тема: «Противоречие: толщина стены по ос. В/3 — лист КР-12 vs КР-15»
   → Приоритет: Высокий
   → Влияние на сроки: Да (кладка стоит до ответа)
   → Прикладывает фрагменты обоих чертежей
   → Адресат: проектировщик (раздел КР)

3. Проектировщик отвечает (через 2 дня):
   → «Правильно 250 мм (по расчёту). Лист КР-12 — опечатка.
      Выпускается корректировочный лист КР-12 rev.2»
   → Отметка: Официальный ответ ✅
   → Прикладывает исправленный лист

4. Прораб закрывает RFI:
   → Кладка продолжается со стеной 250 мм
   → В документации — ссылка на RFI №45
```

### Сценарий 2: Отсутствующая информация в проекте

```
1. Инженер-электрик не находит в проекте тип кабеля
   для питания вентустановки ВУ-3

2. Создаёт RFI:
   → Тема: «Тип кабеля для ВУ-3 не указан в однолинейной схеме»
   → Приоритет: Средний
   → Влияние на стоимость: Да (разные кабели — разная цена)
   → Адресат: проектировщик (раздел ЭО)

3. Проектировщик:
   → Предварительный ответ: «Ориентировочно ВВГнг 5×10»
     (НЕ официальный — для информации)
   → Через 3 дня — официальный ответ:
     «ВВГнг(А)-LS 5×10, кабельная линия L=45 м.
      Основание: расчёт нагрузок, лист ЭО-08»
   → Прикладывает дополнение к проекту

4. Инженер закрывает RFI
   → Заказывает кабель ВВГнг(А)-LS 5×10
```

### Сценарий 3: Массовые RFI тормозят стройку

```
1. По проекту открыто 15 RFI, из них 8 просрочены

2. Руководитель проекта смотрит статистику:
   → 12 из 15 RFI адресованы одному проектировщику
   → Средний срок ответа: 12 дней (при норме 7)
   → 5 RFI помечены «Влияет на сроки»

3. Руководитель организует совещание:
   → Показывает список просроченных RFI
   → Проектировщик обязуется ответить на все в течение 3 дней
   → Договариваются о еженедельных сверках по RFI

4. Через неделю: все 15 RFI отвечены и закрыты ✅
```

## Частые вопросы

**Чем RFI отличается от submittal?**
RFI — это вопрос («поясните, уточните»). Submittal — это документ на согласование («вот чертёж, одобрите»). RFI = запрос информации. Submittal = предоставление документа.

**Кто может создать RFI?**
Любой участник проекта: подрядчик, субподрядчик, инженер. Чаще всего — подрядчик проектировщику.

**Что делать, если на RFI не отвечают?**
Система показывает просроченные RFI. Эскалация: от инженера → к руководителю проекта → к заказчику. Задержка ответа — основание для продления сроков строительства.

**Имеет ли ответ на RFI юридическую силу?**
Да, если отмечен как «Официальный». Это равносильно письменному указанию проектировщика. Поэтому все важные решения должны оформляться через RFI, а не устно.

**Сколько RFI — это нормально?**
Зависит от сложности проекта. На крупном объекте — 50–200 RFI за проект. Если больше — возможно, качество проектной документации низкое.

**Может ли ответ на RFI изменить стоимость работ?**
Да. Если ответ требует дополнительных работ или более дорогих материалов — это основание для дополнительного соглашения (change order).

**Где посмотреть все RFI по проекту?**
На странице RFI используйте фильтр по проекту. На доске видно все RFI по статусам. Просроченные помечены красным.
