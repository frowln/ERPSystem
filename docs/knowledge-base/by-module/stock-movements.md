# Движение материалов

---

## Что это и зачем

Движение материалов — это любая операция, которая меняет количество на складе: пришло, ушло, переместилось, списалось. Каждое движение — это документ с номером, датой, ответственным и подробным перечнем материалов.

Без движений остатки не меняются. Если поставщик привёз 10 тонн цемента, а кладовщик не оформил приход — система не знает об этом цементе.

**Правило:** каждое физическое перемещение материала должно быть отражено в системе.

---

## Где найти

**Путь:** Главное меню → **Склад** → **Перемещения**

---

## Типы движений

В системе 6 типов — каждый для своей ситуации:

| Тип | Русское название | Что происходит | Пример |
|-----|-----------------|----------------|--------|
| **RECEIPT** | Приход | Материал поступает на склад | Поставщик привёз арматуру |
| **ISSUE** | Расход | Материал уходит со склада | Бригада забрала кирпич на объект |
| **TRANSFER** | Перемещение | Материал перевозится между складами | Из центрального склада на площадку |
| **RETURN** | Возврат | Неиспользованный материал возвращают | Бригада вернула лишний цемент |
| **ADJUSTMENT** | Корректировка | Исправление данных (после инвентаризации) | Посчитали — оказалось на 200 кирпичей меньше |
| **WRITE_OFF** | Списание | Безвозвратная потеря | Битая плитка, просроченный клей |

### Приход (RECEIPT)

Материал поступает на склад. Остаток **увеличивается**.

**Когда используется:**
- Поставщик доставил заказ
- Получение из другого склада (межпроектное перемещение)
- Оприходование найденных излишков (после инвентаризации)

**Что заполнять:**
| Поле | Описание |
|------|---------|
| Склад назначения | Куда приходуем |
| Поставщик | От кого получено |
| Номер накладной | Ссылка на товарную накладную |
| Позиции | Материал, количество, цена за единицу |

> **Совет:** Если приход связан с заказом поставщику (PO), можно создать приход автоматически: система подтянет все позиции из заказа.

### Расход (ISSUE)

Материал уходит со склада. Остаток **уменьшается**.

**Когда используется:**
- Выдача бригаде на объект
- Передача субподрядчику
- Расход на ремонт оборудования

**Что заполнять:**
| Поле | Описание |
|------|---------|
| Склад-источник | Откуда выдаём |
| Проект | На какой объект |
| Получатель | Кто получает (бригадир, прораб) |
| Позиции | Материал, количество |

> **Важно:** Система проверит доступный остаток. Если запрошено больше, чем есть — ошибка. Подробнее в разделе [Проверка остатка](#что-если-не-хватает).

### Перемещение (TRANSFER)

Материал перевозится с одного склада на другой. Остаток на складе-источнике **уменьшается**, на складе назначения **увеличивается**.

**Когда используется:**
- Перевозка с центрального склада на площадку
- Перераспределение между объектами
- Консолидация остатков в одном месте

**Что заполнять:**
| Поле | Описание |
|------|---------|
| Склад-источник | Откуда перемещаем |
| Склад назначения | Куда перемещаем |
| Позиции | Материал, количество |

### Возврат (RETURN)

Неиспользованный материал возвращается на склад. Остаток **увеличивается**.

**Когда используется:**
- Бригада завершила работу и осталось лишнее
- Материал не подошёл (другая марка)
- Объект закрыт досрочно

### Корректировка (ADJUSTMENT)

Исправление остатка по результатам пересчёта. Остаток **увеличивается или уменьшается**.

**Когда используется:**
- После [инвентаризации](warehouse.md#инвентаризация) обнаружены расхождения
- Исправление ошибки ввода
- Пересортица (один материал был учтён как другой)

> Корректировки обычно создаются **автоматически** при завершении инвентаризации.

### Списание (WRITE_OFF)

Безвозвратная потеря материала. Остаток **уменьшается**, материал никуда не передаётся.

**Когда используется:**
- Порча (разбили, залило, сгнило)
- Истёк срок годности
- Кража или утрата
- Брак, обнаруженный при использовании

## Статусы движения

Каждое движение проходит через этапы:

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Черновик** | Серый | Создано, можно редактировать. Остатки НЕ затронуты |
| **Подтверждено** | Синий | Проверено и заблокировано. Готово к выполнению |
| **Выполнено** | Зелёный | Остатки обновлены. Операция завершена |
| **Отменено** | Красный | Операция не состоялась. Если было «Выполнено» — откат |

```
Черновик → Подтверждено → Выполнено
    ↓           ↓              ↓
Отменено   Отменено       Отменено (с откатом остатков)
```

### Что происходит при подтверждении

Система проверяет:
- Все обязательные поля заполнены
- Для расхода/перемещения/списания: **достаточно ли материала на складе**
- Количества положительные
- Склад-источник и склад назначения указаны (по ситуации)

Если проверка не пройдена — движение остаётся в «Черновике» с сообщением об ошибке.

### Что происходит при выполнении

- Остатки **обновляются** в реальном времени
- Цена материала обновляется (при приходе с новой ценой)
- Данные попадают в [Cash Flow](cash-flow.md) (если есть стоимость)
- Лимитно-заборная ведомость обновляется (если есть привязка)
- Оповещения проверяются (не упал ли остаток ниже минимума)

### Что происходит при отмене

- Если движение было «Выполнено» — все изменения **откатываются** (остатки возвращаются назад)
- Если «Черновик» или «Подтверждено» — просто помечается как отменённое

## Как создать движение

1. Перейдите **Склад → Перемещения**
2. Нажмите **«Новое перемещение»**
3. Заполните:

### Шапка

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Тип** | Да | Приход / Расход / Перемещение / Возврат / Корректировка / Списание |
| **Дата** | Да | Дата операции |
| **Проект** | Нет | К какому проекту относится |
| **Склад-источник** | Для расхода, перемещения, списания | Откуда берём |
| **Склад назначения** | Для прихода, перемещения, возврата | Куда кладём |
| **Ответственный** | Нет | Кто выполняет операцию |
| **Примечание** | Нет | Комментарий (номер накладной, причина и т.д.) |

### Позиции (строки)

Для каждой позиции:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Материал** | Да | Выбрать из справочника |
| **Количество** | Да | Сколько единиц (должно быть > 0) |
| **Цена за единицу** | Для прихода | Стоимость по накладной |
| **Номер партии** | Нет | Для отслеживания по партиям |
| **Сертификат** | Нет | Номер сертификата качества |

4. Нажмите **«Сохранить»** — создаётся черновик
5. Проверьте данные → нажмите **«Подтвердить»**
6. Нажмите **«Выполнить»** — остатки обновятся

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Номер | Уникальный номер (SM-000245) |
| Дата | Дата операции |
| Тип | Цветной бейдж: Приход (зелёный), Расход (красный), Перемещение (синий) и т.д. |
| Откуда → Куда | Склад-источник и склад назначения |
| Статус | Черновик / Подтверждено / Выполнено / Отменено |
| Кол-во позиций | Сколько строк в документе |

**Фильтры:**
- По типу движения
- По статусу
- По складу
- По дате (период)
- По проекту

### Доска перемещений (канбан)

Также доступна **канбан-доска** — те же движения, но распределённые по колонкам-статусам:

```
| Черновик (5) | Подтверждено (3) | Выполнено (12) | Отменено (1) |
|  [SM-301]    |   [SM-298]       |   [SM-295]     |  [SM-280]    |
|  [SM-302]    |   [SM-299]       |   [SM-296]     |              |
|  ...         |   [SM-300]       |   ...          |              |
```

## Карточка движения

При нажатии на строку открывается детальная страница.

**Заголовок:**
- Номер документа и тип (бейдж)
- Статус (бейдж)
- Дата операции

**Информация:**
- Проект (если указан)
- Откуда → Куда
- Ответственный
- Примечание

**Таблица позиций:**

| Материал | Ед. изм. | Количество | Цена | Сумма |
|----------|---------|----------:|-----:|------:|
| Арматура d12 | м | 500 | 55 ₽ | 27 500 ₽ |
| Кирпич М-150 | шт | 5 000 | 12 ₽ | 60 000 ₽ |
| **Итого** | | | | **87 500 ₽** |

**Кнопки действий** — зависят от статуса:
- **Черновик:** «Добавить позицию», «Подтвердить», «Отменить»
- **Подтверждено:** «Выполнить», «Отменить»
- **Выполнено:** «Отменить» (откатит остатки)

## Что если не хватает?

При попытке **подтвердить** расход, перемещение или списание система проверяет доступный остаток:

```
Запрос: Выдать 300 кг цемента со Склада А
Проверка:
  Всего на Складе А: 350 кг
  Зарезервировано: 100 кг
  Доступно: 250 кг
Результат: ОТКАЗ

Сообщение: «Недостаточно материала "Цемент М400".
            Запрошено: 300 кг. Доступно: 250 кг»
```

**Что делать:**
1. **Уменьшить количество** до доступного (250 кг)
2. **Дождаться прихода** — когда поставщик привезёт ещё
3. **Снять резервирование** — если зарезервированный материал больше не нужен для другого проекта
4. **Выдать частями** — сначала 250 кг, остальное позже

> Проверка происходит только при **подтверждении**, не при создании черновика. В черновике можно указать любое количество — ошибка появится при попытке подтвердить.

## Приход из заказа на закупку

Если материалы заказаны через модуль **Снабжение** (Purchase Order), приход можно создать автоматически:

1. В списке заказов выберите PO со статусом «В доставке»
2. Нажмите **«Оформить приход»**
3. Система автоматически создаст движение типа «Приход» со всеми позициями из заказа
4. Проверьте количества (может прийти не всё) → скорректируйте при необходимости
5. Подтвердите и выполните

## История движений

**Путь:** Склад → Перемещения → **История**

Полная хронология всех движений с фильтрацией по:
- Складу
- Материалу
- Периоду (от–до)
- Типу движения
- Статусу

Используется для:
- Отслеживания «где был материал» (цепочка движений)
- Подготовки отчёта [М-29](warehouse.md#отчёт-м-29)
- Анализа оборачиваемости запасов
- Аудиторских проверок

## Пример: полный цикл материала

Рассмотрим жизненный цикл арматуры d12:

```
1. ПРИХОД (от поставщика)
   Дата: 1 марта
   Склад: Центральный
   Количество: 10 000 м
   Цена: 55 ₽/м
   Остаток: 0 → 10 000 м

2. РАСХОД (на объект)
   Дата: 5 марта
   Откуда: Центральный → Площадка «Дом на Ленина»
   Количество: 3 000 м
   Остаток на центральном: 10 000 → 7 000 м

3. ПЕРЕМЕЩЕНИЕ (между складами)
   Дата: 10 марта
   Откуда: Центральный → Площадка «Офис на Мира»
   Количество: 2 000 м
   Остаток на центральном: 7 000 → 5 000 м
   Остаток на «Офис на Мира»: 0 → 2 000 м

4. ВОЗВРАТ (лишнее)
   Дата: 15 марта
   Откуда: Площадка «Дом на Ленина» → Центральный
   Количество: 500 м
   Остаток на центральном: 5 000 → 5 500 м

5. СПИСАНИЕ (брак)
   Дата: 20 марта
   Склад: Площадка «Дом на Ленина»
   Количество: 100 м (ржавая, негодная)
   Остаток на площадке: 2 500 → 2 400 м

6. ИНВЕНТАРИЗАЦИЯ → КОРРЕКТИРОВКА
   Дата: 31 марта
   Склад: Центральный
   По системе: 5 500 м
   Факт: 5 480 м
   Корректировка: −20 м
   Остаток: 5 500 → 5 480 м
```

## Частые вопросы

**Можно ли изменить выполненное движение?**
Нет. Выполненное движение нельзя редактировать. Если ошибка — отмените его (остатки откатятся) и создайте новое правильное.

**Кто может создавать движения?**
Администратор, Руководитель проекта, Кладовщик. Роль «Наблюдатель» видит движения, но не может создавать.

**Как отследить, куда ушёл материал?**
В карточке материала → секция «Последние движения». Или: Склад → Перемещения → фильтр по материалу → видна вся цепочка.

**Чем отличается «перемещение» от «межпроектного перемещения»?**
Обычное перемещение — между складами (может быть в рамках одного проекта). Межпроектное — передача материала с одного проекта на другой (требует согласования руководителя).

**Обязательно ли указывать цену в приходе?**
Цена рекомендуется (для учёта стоимости запасов), но не обязательна. Без цены система не сможет рассчитать стоимость остатков.

**Как быстро оформить приход?**
Используйте **Быстрый приход** (Склад → Быстрый приход) — отсканировал штрих-код, ввёл количество, подтвердил. Два шага вместо полного оформления.

**Что такое «партия» и «сертификат» в позициях?**
Необязательные поля для прослеживаемости: номер партии помогает отследить, из какой поставки конкретный материал, а сертификат — подтверждение качества от производителя.

**Как узнать, сколько материала ушло за месяц?**
Склад → Перемещения → фильтр: тип = «Расход», период = нужный месяц. Или: Склад → Отчёт М-29 (автоматически собирает все расходы за месяц).

**Движение и ордер — это одно и то же?**
Почти. Складской ордер (М-4, М-11) — это первичный документ по российским стандартам. Движение — это запись в системе. Каждый проведённый ордер создаёт движение. Но движение может быть создано и без ордера (например, корректировка после инвентаризации).
