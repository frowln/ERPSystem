# Обследование объектов

---

## Что это и зачем

**Обследование объектов** — модуль для фиксации результатов осмотра строительных площадок до начала работ, в процессе строительства или при приёмке. Обследование помогает оценить состояние объекта, выявить риски и задокументировать текущее положение дел.

Когда проводят обследование:
- **До начала работ** — оценка площадки, существующих коммуникаций, соседних зданий
- **При входе на субподряд** — фиксация состояния «как было» (чтобы потом не отвечать за чужие дефекты)
- **Плановые обходы** — еженедельный/ежемесячный осмотр хода работ
- **При приёмке** — проверка готовности перед сдачей заказчику

## Часть 1: Создание обследования

| Поле | Описание |
|------|----------|
| **Проект** | К какому проекту относится |
| **Тип** | Первичное / Плановое / Приёмочное / Внеплановое |
| **Дата** | Когда проведено |
| **Инспектор** | Кто проводил осмотр |
| **Участок/Зона** | Какая часть объекта обследована |
| **Результат** | Удовлетворительно / Замечания / Не допускается |

### Чек-лист обследования

При создании обследования система предлагает чек-лист проверки:

| Категория | Что проверяют |
|-----------|--------------|
| **Ограждение** | Забор, ворота, проходная |
| **Дороги** | Подъездные пути, временные дороги |
| **Коммуникации** | Вода, электричество, канализация |
| **Безопасность** | Освещение, знаки, ограждения опасных зон |
| **Складирование** | Площадки для материалов, контейнеры |
| **Техника** | Площадки для крана, бетононасоса |
| **Экология** | Пыль, шум, мусор, водоотведение |

## Часть 2: Фотофиксация

Каждое обследование сопровождается фотографиями:

1. Фотографируйте с привязкой к чек-листу (каждый пункт — фото)
2. Система сохраняет дату и время снимка
3. Можно добавить подпись/комментарий к каждому фото
4. Фото с телефона загружаются прямо из камеры

> **Совет:** Фотографируйте общий план + крупный план проблемного места. Это поможет при разборе ситуации.

---

## Часть 3: Замечания

Если при обследовании обнаружены проблемы:

1. Добавьте замечание с описанием
2. Укажите категорию (безопасность, качество, организация)
3. Назначьте ответственного за устранение
4. Установите срок
5. После устранения — повторный осмотр и закрытие замечания

> Замечания по безопасности автоматически создают записи в модуле **[Охрана труда](safety-inspections.md)**.

---

## Типичный сценарий

```
Ситуация: Перед началом монтажных работ нужно обследовать площадку

1. Инженер ПТО создаёт обследование (тип: Первичное)
2. Выезжает на объект с телефоном
3. Проходит по чек-листу, фотографирует каждый пункт
4. Обнаруживает: нет временного ограждения котлована
5. Создаёт замечание → назначает подрядчику → срок 2 дня
6. Результат обследования: «Замечания»
7. Через 2 дня — повторный осмотр: ограждение установлено ✓
8. Замечание закрыто → обследование: «Удовлетворительно»
```

## Частые вопросы

**Чем обследование отличается от инспекции по ОТ?**
Обследование — общий осмотр объекта (состояние, готовность, ход работ). Инспекция ОТ — целевая проверка именно по охране труда. При обследовании можно найти нарушение ОТ — тогда оно автоматически попадёт в модуль охраны труда.

**Обязательно ли фотографировать?**
Юридически — нет. Практически — да. Фото — единственное доказательство состояния объекта на конкретную дату. Без фото любой спор решится не в вашу пользу.

**Кто должен проводить обследование?**
Обычно инженер ПТО или прораб. Для приёмочного обследования — комиссия (представители заказчика, подрядчика, авторского надзора).
