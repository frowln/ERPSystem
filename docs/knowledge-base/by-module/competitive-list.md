# Конкурентный лист (КЛ)

---

## Что это и зачем

Конкурентный лист — это таблица сравнения цен от разных поставщиков на позиции спецификации. По каждой позиции (труба, кабель, арматура...) вы собираете предложения от 3 и более поставщиков, сравниваете их по цене, срокам поставки, гарантии и условиям оплаты, и выбираете лучшего.

Выбранная цена — это **себестоимость**. Она автоматически попадает в финансовую модель (ФМ), а оттуда — в коммерческое предложение (КП) для заказчика.

**Цепочка:** [Спецификация](specifications.md) → **Конкурентный лист** → [Финансовая модель](financial-model.md) → [КП](commercial-proposals.md)

---

## Где найти

**Реестр всех КЛ:** Главное меню → **Спецификации** → кнопка **«Реестр КЛ»** (или напрямую `/competitive-lists`)

**КЛ конкретной спецификации:** откройте спецификацию → кнопка **«Создать КЛ»** (или перейдите по ссылке, если КЛ уже создан)

---

## Реестр конкурентных листов

В реестре собраны все КЛ по всем проектам компании.

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Название | Имя конкурентного листа (например, «КЛ — Материалы фундамента») |
| Проект | К какому объекту относится |
| Статус | На каком этапе: черновик, сбор предложений, оценка, решение, утверждён |
| Позиции | Сколько позиций спецификации включено |
| Предложения | Сколько всего предложений от поставщиков собрано |
| Поставщики | Сколько уникальных поставщиков участвуют |
| Лучшая цена | Самая низкая цена среди победителей |
| Лучший поставщик | Кто предложил лучшую цену |
| Дата создания | Когда создан КЛ |

**Вкладки-фильтры по статусу:**
- **Все** — все конкурентные листы
- **Черновик** — только что созданные
- **Сбор предложений** — активный сбор цен от поставщиков
- **Оценка** — предложения собраны, идёт анализ
- **Решение принято** — победители выбраны
- **Утверждён** — финальная версия, изменения запрещены

**Поиск:** по названию, проекту или поставщику.

## Как создать конкурентный лист

### Автоматически (рекомендуется)

При создании спецификации система может автоматически создать КЛ — для этого просто включите соответствующий флажок в форме спецификации.

### Вручную

1. Откройте нужную спецификацию
2. Нажмите кнопку **«Создать КЛ»**
3. Система создаст КЛ в статусе «Черновик» и скопирует все позиции спецификации

После создания вы увидите **страницу конкурентного листа** с двумя панелями:
- **Слева** — список позиций спецификации (наименование, количество, ед. измерения)
- **Справа** — предложения поставщиков по выбранной позиции

---

## Как добавить предложение поставщика

1. Выберите позицию из списка слева (нажмите на строку)
2. Справа нажмите **«Добавить предложение»**
3. Заполните данные:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Поставщик** | Да | Название компании-поставщика |
| **Цена за единицу** | Да | Цена за единицу измерения (руб.) |
| **Количество** | Нет | Подтягивается из спецификации автоматически |
| **Срок поставки (дни)** | Нет | За сколько дней поставщик доставит |
| **Условия оплаты** | Нет | Например: «50% предоплата, 50% по факту» |
| **Предоплата, %** | Нет | Процент предоплаты (0–100%) |
| **Отсрочка (дни)** | Нет | Сколько дней отсрочки после поставки |
| **Гарантия (мес)** | Нет | Гарантийный срок в месяцах |
| **Примечание** | Нет | Свободный текст |

4. Нажмите **«Сохранить»**

> **Подсказка:** Для каждой позиции рекомендуется собрать минимум 3 предложения. Для государственных строек это требование закона (конкурентная процедура). Для коммерческих — хорошая практика.

Повторите для всех поставщиков и всех позиций.

## Авторанжирование — как работает

После того как предложения собраны, система может автоматически оценить и ранжировать их.

Нажмите кнопку **«Авторанжирование»** в шапке страницы.

**Как рассчитывается балл (0–100 очков):**

| Критерий | Вес | Что оценивается | Лучший результат |
|----------|-----|-----------------|-----------------|
| **Цена** | 50% | Чем дешевле — тем лучше | Самая низкая цена = 50 очков |
| **Срок поставки** | 25% | Чем быстрее — тем лучше | Самая быстрая поставка = 25 очков |
| **Гарантия** | 15% | Чем дольше — тем лучше | Самая длинная гарантия = 15 очков |
| **Условия оплаты** | 10% | Чем меньше предоплата — тем лучше | 0% предоплата = 10 очков |

**Пример:**
- Поставщик А: цена 1000 руб., поставка 10 дней, гарантия 12 мес., предоплата 0% → Балл: 100
- Поставщик Б: цена 1200 руб., поставка 5 дней, гарантия 24 мес., предоплата 50% → Балл: ~72
- Поставщик В: цена 1500 руб., поставка 30 дней, гарантия 6 мес., предоплата 100% → Балл: ~15

После ранжирования предложения выстраиваются по убыванию балла. Место 1 — лучшее предложение, место 2 — второе и т.д.

## Выбор победителя

### Вручную

1. Нажмите иконку **кубка** (Award) на строке лучшего предложения
2. Откроется окно подтверждения — при желании укажите причину выбора
3. Нажмите **«Подтвердить»**

Предыдущий победитель по этой позиции (если был) автоматически сбрасывается.

### Автовыбор лучших

Нажмите кнопку **«Автовыбор лучших»**. Система автоматически:
- По каждой позиции выберет предложение с **самой низкой ценой**
- Пометит его как победителя
- Запишет причину: «Автовыбор: лучшая цена»

> **Важно:** При выборе победителя (любым способом) его цена **автоматически передаётся в финансовую модель** как себестоимость позиции.

---

## Как выбранные цены попадают в ФМ

При каждом выборе победителя система автоматически:

1. Находит соответствующую позицию в бюджете (ФМ) по связке «позиция спецификации → позиция бюджета»
2. Записывает цену победителя в поле **«Себестоимость»** (costPrice) позиции бюджета
3. Указывает источник цены: «Из конкурентного листа»
4. Пересчитывает маржу по позиции

Это происходит **мгновенно** — не нужно ничего делать вручную. Откройте финансовую модель и увидите обновлённые цены.

**Схема:**
```
КЛ: Победитель = 1000 руб./шт
         ↓ автоматически
ФМ: Себестоимость = 1000 руб./шт → Маржа пересчитана
```

## Статусы конкурентного листа

| Статус | Цвет | Что означает | Что можно делать |
|--------|------|-------------|-----------------|
| **Черновик** | Серый | Только создан, настройка | Добавлять позиции, редактировать |
| **Сбор предложений** | Синий | Активная работа с поставщиками | Добавлять/редактировать предложения |
| **Оценка** | Оранжевый | Предложения собраны, анализ | Ранжировать, выбирать победителей |
| **Решение принято** | Зелёный | Победители выбраны | Финализация, подготовка к утверждению |
| **Утверждён** | Фиолетовый | Финальная версия | Только просмотр, изменения запрещены |

**Порядок переходов:**
```
Черновик → Сбор предложений → Оценка → Решение принято → Утверждён
```

На каждом шаге можно вернуться на предыдущий (кроме «Утверждён» — это финальный статус).

> **Важно:** При переходе в «Решение принято» система проверяет, что по каждой позиции есть минимум 3 предложения (или столько, сколько установлено в настройках КЛ). Если не хватает — покажет ошибку.

## Показатели (карточки вверху страницы КЛ)

| Показатель | Что значит |
|-----------|-----------|
| **Позиции** | Сколько позиций спецификации в этом КЛ |
| **Предложения** | Сколько всего предложений собрано + сколько победителей выбрано |
| **Лучшая цена** | Минимальная цена за единицу среди всех предложений |
| **Экономия** | Разница между плановой стоимостью и лучшими ценами. Зелёное — экономия, красное — перерасход |

Под карточкой «Экономия» видна строка: «Покрытие: 8/10 позиций (80%), экономия 12%» — показывает, сколько позиций уже обеспечены ценами и какова общая экономия.

---

## Статус обеспечения позиции

Для каждой позиции спецификации показывается статус обеспечения:

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Обеспечена** | Зелёный | Победитель выбран, цена есть |
| **Частично** | Оранжевый | Предложения есть, но победитель не выбран |
| **Не обеспечена** | Красный | Предложений меньше, чем требуется (по умолчанию < 3) |

---

## Ценовой индикатор

В таблице предложений рядом с ценой каждого поставщика стоит цветная точка:
- **Зелёная** — самая низкая цена среди всех предложений по этой позиции
- **Красная** — самая высокая цена

Это помогает мгновенно увидеть, кто дешевле, а кто дороже.

---

## Применение к КП

Цены из КЛ можно передать не только в ФМ, но и напрямую в коммерческое предложение.

На странице КП (вкладка «Материалы» или «Работы») для каждой позиции доступна **панель конкурентного листа**, где видны предложения поставщиков с ценами. Можно выбрать конкретного поставщика — его цена станет себестоимостью позиции КП.

Подробнее — в разделе [Коммерческие предложения](commercial-proposals.md).

## Частые вопросы

**Зачем нужно минимум 3 поставщика?**
Для государственных строек — это требование закона (конкурентная процедура). Для коммерческих — хорошая практика: сравнение 3+ цен помогает получить лучшую стоимость и защищает от обвинений в завышении. Количество можно изменить в настройках КЛ (поле «Минимум предложений»).

**Можно ли выбрать победителя не по самой низкой цене?**
Да. Автовыбор берёт самую дешёвую цену, но вы можете выбрать любого поставщика вручную (иконка кубка). Например, если один поставщик дороже на 5%, но привозит за 3 дня вместо 30 — это может быть выгоднее. При ручном выборе можно указать причину.

**Что такое авторанжирование?**
Это автоматическая оценка всех предложений по 4 критериям: цена (50% веса), срок поставки (25%), гарантия (15%), условия оплаты (10%). Каждое предложение получает балл от 0 до 100. Ранжирование помогает выбрать, но не выбирает победителя автоматически — для этого есть отдельная кнопка «Автовыбор лучших».

**Как цена попадает в ФМ?**
Автоматически при каждом выборе победителя. Система находит связанную позицию бюджета и записывает в неё себестоимость = цена победителя. Ничего делать вручную не нужно.

**Можно ли вернуться и изменить победителя?**
Да, пока КЛ не в статусе «Утверждён». Просто выберите другого поставщика — предыдущий победитель сбросится, новая цена автоматически обновится в ФМ.

**Что означает «Утверждён»?**
Это финальный статус. После утверждения КЛ заблокирован для изменений. Это гарантирует, что выбранные цены не будут случайно изменены после согласования.

**Как создать КЛ для работ (а не для материалов)?**
Точно так же — КЛ создаётся из спецификации, а в спецификации могут быть как материалы, так и работы. Для работ вместо поставщиков указываете подрядчиков и их расценки.
