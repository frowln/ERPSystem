# Дефекты

---

## Что это и зачем

Дефект — это выявленное несоответствие выполненных работ проектной документации, нормам или стандартам. Проще говоря: что-то сделано не так, как нужно.

Дефекты важно фиксировать, потому что:
- **Безопасность** — незамеченный дефект конструкций может привести к аварии
- **Деньги** — чем раньше нашли, тем дешевле исправить
- **Ответственность** — зафиксированный дефект — это юридическое основание требовать от подрядчика переделки за его счёт
- **Приёмка** — объект с неустранёнными критическими дефектами не примут

Модуль «Дефекты» в системе ПРИВОД ведёт учёт всех обнаруженных дефектов, отслеживает их устранение и контролирует сроки (SLA).

## Как создать дефект

1. Перейдите **Дефекты → Список дефектов**
2. Нажмите **«+ Новый дефект»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Название** | Да | Краткое описание (например: «Трещина в плите 2 этажа у оси Б-3») |
| **Проект** | Да | К какому проекту |
| **Описание** | Да | Подробное описание: где, что, насколько серьёзно |
| **Серьёзность** | Да | Низкая / Средняя / Высокая / Критическая |
| **Подрядчик** | Нет | Кто допустил дефект (для предъявления претензии) |
| **Ответственный** | Нет | Кто должен устранить |
| **Срок устранения** | Нет | До когда нужно исправить |
| **Фото** | Рекомендуется | Фотофиксация дефекта |

4. **Отметка на чертеже** — можно поставить «булавку» на плане этажа, чтобы показать точное место дефекта (pinX/pinY). Это помогает бригаде быстро найти, где проблема.
5. Нажмите **«Создать»**

## Серьёзность дефектов

| Серьёзность | Цвет | Описание | Пример | SLA (типовой) |
|-------------|------|----------|--------|:-------------:|
| **Низкая** (LOW) | Зелёный | Косметический дефект, не влияет на функцию | Царапина на двери, сколы краски | 14 дней |
| **Средняя** (MEDIUM) | Жёлтый | Функциональный дефект, работа возможна | Неровная штукатурка, плитка с отклонением | 7 дней |
| **Высокая** (HIGH) | Оранжевый | Серьёзный дефект, влияет на эксплуатацию | Протечка кровли, неработающий вентканал | 3 дня |
| **Критическая** (CRITICAL) | Красный | Угроза безопасности или конструктиву | Трещина в несущей стене, провис плиты | 1 день |

**SLA** — это максимальное время на устранение. Система показывает обратный отсчёт и предупреждает, когда сроки поджимают.

## Статусы дефекта

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Открыт** (OPEN) | Красный | Дефект обнаружен и зафиксирован |
| **В работе** (IN_PROGRESS) | Жёлтый | Подрядчик устраняет дефект |
| **Исправлен** (FIXED) | Синий | Подрядчик сообщил, что исправил |
| **Проверен** (VERIFIED) | Зелёный | Инспектор проверил — действительно исправлен |
| **Закрыт** (CLOSED) | Серый | Дефект полностью закрыт |
| **Отклонён** (REJECTED) | Серый | Признан несущественным или ошибкой фиксации |

```
Открыт → В работе → Исправлен → Проверен → Закрыт
                  ↘                ↓
                   ←  (не исправлен, возврат)
                                Отклонён
```

**Важно:** Если при проверке оказалось, что дефект не устранён — его возвращают в «Открыт» или «В работе». Система считает количество повторных проверок (реинспекций).

## Отметка на чертеже

Одна из самых удобных функций — возможность поставить «булавку» прямо на плане этажа. Это работает так:

1. При создании дефекта нажмите **«Указать на чертеже»**
2. Загрузите план этажа (если ещё не загружен) или выберите из имеющихся
3. Кликните на место, где обнаружен дефект
4. Система сохранит координаты (pinX, pinY)
5. На плане появится маркер с номером дефекта

Бригадир открывает план на планшете и видит: красная точка у оси Б-3 — значит, там трещина, нужно исправить.

## Дашборд дефектов

**Путь:** Дефекты → **Дашборд**

Дашборд показывает общую картину по дефектам:

### Показатели

- **Всего дефектов** — общее количество
- **Открытых** — сколько ещё не устранено
- **Просроченных** — сколько вышло за SLA (красный индикатор)
- **Средний срок устранения** — сколько дней в среднем уходит на исправление

### Диаграмма Парето

Показывает: **какие типы дефектов встречаются чаще всего**. По принципу 80/20: 80% проблем вызваны 20% причин.

Например:
```
1. Штукатурные работы — 45 дефектов (35%)  ████████████
2. Сантехника — 28 дефектов (22%)          ████████
3. Электрика — 18 дефектов (14%)           █████
4. Бетонные работы — 12 дефектов (9%)      ███
5. Остальные — 25 дефектов (20%)           ██████
```

Это помогает понять, где системная проблема, и работать с подрядчиком или бригадой над корневой причиной.

## Типичные сценарии

### Сценарий 1: Трещина в стене

```
1. Инженер обходит объект и находит трещину в несущей стене

2. Создаёт дефект:
   → Название: «Трещина в несущей стене ось В/3-4, отм. +6.000»
   → Серьёзность: Критическая
   → Подрядчик: ООО «СтройМонтаж»
   → Ставит булавку на план 2 этажа
   → Прикладывает фото с линейкой (ширина трещины 1.5 мм)
   → SLA: 1 день

3. Подрядчик получает уведомление → статус «В работе»
   → Вызывает проектировщика для оценки
   → Проектировщик: усилить углеволокном
   → Подрядчик выполняет усиление → статус «Исправлен»

4. Инженер проверяет:
   → Усиление выполнено по рекомендации проектировщика ✅
   → Фото «после» прикреплено
   → Статус: «Проверен» → «Закрыт»

5. В карточке дефекта видно всю историю:
   срок устранения, фото до/после, кто устранил
```

### Сценарий 2: Массовые дефекты штукатурки

```
1. При приёмке 3 этажа найдено 12 дефектов штукатурки:
   → Неровности > 5мм (допуск 2мм)
   → Трещины в углах
   → Отслоение от основания

2. Все 12 дефектов зафиксированы с фото и булавками на плане

3. На дашборде видно: штукатурка — 35% всех дефектов (Парето)
   → Это системная проблема, а не единичный случай

4. Руководитель проекта вызывает бригадира штукатуров:
   → Показывает план с булавками
   → Обсуждает причины: нарушение технологии
   → Устанавливает контроль: чек-лист на каждое помещение

5. Подрядчик устраняет все 12 дефектов за 5 дней (SLA 7 дней) ✅
```

## Частые вопросы

**Чем дефект отличается от NCR?**
Дефект — это конкретное нарушение (трещина, протечка). NCR (несоответствие) — более формальный документ, обычно для серьёзных или системных проблем, с обязательным расследованием причин и корректирующими мерами.

**Кто может создавать дефекты?**
Любой сотрудник с ролью ENGINEER или выше: инженер по качеству, инженер-строитель, руководитель проекта. VIEWER может только просматривать.

**Что делать, если подрядчик не устраняет дефект в срок?**
Система показывает просроченные дефекты красным. Это основание для претензии подрядчику. При системных нарушениях — основание для штрафных санкций по договору.

**Можно ли удалить дефект?**
Удалить нельзя (чтобы не было манипуляций). Если дефект зафиксирован ошибочно — используйте статус «Отклонён» с указанием причины.

**Зачем отмечать дефект на чертеже?**
Чтобы бригада точно знала, ГДЕ проблема. «Трещина на 2 этаже» — неоднозначно. Булавка на плане — точное место.

**Как посмотреть все дефекты одного подрядчика?**
В списке дефектов используйте фильтр по подрядчику. На дашборде можно увидеть статистику по подрядчикам.
