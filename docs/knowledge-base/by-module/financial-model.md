# Финансовая модель (ФМ)

---

## Что это и зачем

Финансовая модель — это главная таблица проекта, в которой сведены **все цены**: себестоимость (сколько стоит нам), сметная стоимость (сколько стоит по нормативам) и цена заказчику (сколько платит клиент). По каждой позиции — от арматуры до монтажных работ — видна маржа: сколько компания зарабатывает.

ФМ создаётся автоматически при создании проекта. Данные в неё приходят из трёх источников:
- **Конкурентный лист (КЛ)** → себестоимость (реальные цены поставщиков)
- **Смета (ЛСР)** → сметная стоимость (нормативные расценки)
- **Вручную** → цена заказчику (с наценкой)

**Цепочка:** [Спецификация](specifications.md) → [КЛ](competitive-list.md) → **Финансовая модель** → [КП](commercial-proposals.md)

Это главный экран ФМ. Здесь собрана вся ценовая информация по проекту.

### Показатели (KPI-полоска вверху)

| Показатель | Цвет | Что значит |
|-----------|------|-----------|
| **Себестоимость** | Серый | Сумма всех закупочных цен (из КЛ): сколько тратим |
| **Сметная стоимость** | Серый | Сумма по нормативным расценкам (из ЛСР): сколько стоит по нормативам |
| **Цена заказчику** | Серый | Сумма цен для клиента: сколько выставляем |
| **Накладные (НР)** | Оранжевый | Накладные расходы (% от себестоимости) |
| **Сметная прибыль (СП)** | Бирюзовый | Сметная прибыль (% от себестоимости) |
| **Непредвиденные (НП)** | Янтарный | Резерв на непредвиденные расходы (% от себестоимость + НР) |
| **НДС 20%** | Фиолетовый | Налог на добавленную стоимость = Цена заказчику x 20% |
| **Маржа** | Цвет зависит от % | Разница: цена заказчику минус себестоимость |
| **Маржа %** | Цвет зависит от % | То же в процентах |
| **ROI** | Зелёный/красный | Рентабельность инвестиций |

**Цвет маржи:**
- Красный — маржа отрицательная (убыток)
- Оранжевый — маржа 0–5% (опасная зона)
- Жёлтый — маржа 5–15% (нормально)
- Зелёный — маржа больше 15% (хорошо)

### Вкладки фильтрации

| Вкладка | Что показывает |
|---------|---------------|
| **Все** | Все позиции |
| **Работы** | Только работы (монтаж, прокладка, установка...) |
| **Материалы** | Только материалы (кабель, трубы, арматура...) |
| **Оборудование** | Только оборудование (трансформаторы, насосы, щиты...) |
| **CVR** | Отслеживание стоимости по стадиям (план → договор → акт → оплата) |
| **Снимки** | Сохранённые версии ФМ для сравнения изменений |
| **ВЭ** | Анализ стоимостной инженерии (оптимизация затрат) |

## Три колонки цен

Главная особенность ФМ — **три уровня цен** по каждой позиции:

### 1. Себестоимость (из КЛ)

**Колонки:** «Себест./ед.» и «Себестоимость»

Это реальная закупочная цена — сколько мы платим поставщику или подрядчику.

**Откуда берётся:**
- Автоматически из **конкурентного листа** (при выборе победителя)
- Из **счёта поставщика** (при привязке через КП)
- Вручную (если КЛ не используется)

**Пример:** Арматура 12 мм — себестоимость 45 000 руб./тонна (цена от поставщика «МеталлТрейд»)

### 2. Сметная стоимость (из ЛСР)

**Колонка:** «Сметная»

Это цена по нормативным расценкам — сколько стоит работа или материал «по книжке» (по ГЭСН, ФЕР, ТЕР).

**Откуда берётся:**
- Автоматически при **импорте ЛСР** (загрузка xlsx из ГРАНД-Сметы)
- Из привязки к расценкам базы ценообразования

> **Зачем нужна сметная цена?** Для контроля. Если ваша цена заказчику выше сметной — система покажет предупреждение (красная обводка). Заказчик, особенно на госстройке, может потребовать обоснование.

### 3. Цена заказчику

**Колонки:** «Цена заказчику/ед.» и «Сумма заказчику»

Это финальная цена, которую платит клиент. Обычно выше себестоимости (иначе работаем в убыток) и может быть выше или ниже сметной.

**Откуда берётся:**
- Вводится вручную в таблице ФМ
- Рассчитывается через наценку

**Формула маржи:**
```
Маржа = Цена заказчику - Себестоимость
Маржа % = (Маржа / Цена заказчику) × 100%
```

## Как данные приходят из КЛ

Когда в конкурентном листе выбирается победитель, его цена **автоматически** записывается в ФМ:

```
КЛ: Позиция «Арматура 12 мм» → Победитель: «МеталлТрейд», 45 000 руб./т
         ↓ мгновенно
ФМ: «Арматура 12 мм» → Себестоимость = 45 000 руб./т
         ↓ пересчёт
ФМ: Маржа = Цена заказчику − 45 000 (обновлена)
```

Рядом с ценой отображается источник: **«Из конкурентного листа»** (фиолетовая метка).

Подробнее о работе с КЛ — в разделе [Конкурентный лист](competitive-list.md).

---

## Как данные приходят из ЛСР (сметы)

Нажмите кнопку **«Импорт ЛСР»** (зелёная, в шапке страницы ФМ).

1. **Загрузите файл** — перетащите xlsx-файл сметы из ГРАНД-Сметы (или любой Excel-сметы)
2. **Просмотрите результат** — система покажет распознанные позиции: код, наименование, ед.изм., количество, цена, итого
3. **Нажмите «Импортировать»**

Система сопоставит позиции по наименованию:
- Если позиция уже есть в ФМ → обновит колонку **«Сметная стоимость»**
- Если позиции нет → создаст новую

Подробнее об импорте сметы — в разделе [Сметы](estimates.md).

## Как редактировать наценку

### Цена заказчику (прямой ввод)

Кликните на ячейку в колонке **«Цена заказчику/ед.»** — она станет редактируемой. Введите число и нажмите Enter (или кликните вне ячейки).

Система мгновенно пересчитает:
- «Сумма заказчику» = цена × количество
- «Маржа» = сумма заказчику − себестоимость
- «Маржа %» — в процентах

### Накладные, прибыль, непредвиденные (НР, СП, НП)

Три дополнительные колонки с процентными надбавками:

| Колонка | Обозначение | Что это | Типичное значение |
|---------|------------|---------|------------------|
| **НР, %** | Накладные расходы | Косвенные затраты (аренда офиса, управление, связь) | 80–120% от ФОТ |
| **СП, %** | Сметная прибыль | Плановая прибыль подрядчика | 50–65% от ФОТ |
| **НП, %** | Непредвиденные расходы | Резерв на неожиданности | 3–5% |

Кликните на процент — он станет редактируемым. Под процентом мелким шрифтом показана сумма в рублях.

### Режим валидации

В шапке ФМ есть переключатель режима:
- **Мягкий** (щит зелёный) — если цена заказчику превышает сметную, показывается предупреждение (красная обводка ячейки), но сохранение разрешено
- **Строгий** (щит красный) — если цена заказчику превышает сметную, сохранение блокируется

## Маржа по разделам и общая

### По позициям

Для каждой строки таблицы видна маржа в рублях и в процентах:

| Себестоимость | Цена заказчику | Маржа | Маржа % |
|--------------|---------------|-------|---------|
| 45 000 руб. | 60 000 руб. | 15 000 руб. | 25% (зелёный) |
| 100 000 руб. | 95 000 руб. | −5 000 руб. | −5.3% (красный) |

### По разделам

Позиции группируются в разделы (например, «Электрика», «Вентиляция», «Фундамент»). У каждого раздела — итоговая строка с суммой себестоимости, сметной, цены заказчику, НДС и маржи.

Нажмите на название раздела, чтобы свернуть/развернуть его позиции.

### Общие итоги (подвал таблицы)

В нижней строке таблицы **«Итого»** — суммы по всем позициям:
- Итого себестоимость
- Итого сметная
- Итого заказчику
- Итого НДС (20%)
- Итого маржа и маржа %

Эти же суммы дублируются в KPI-полоске вверху.

---

## НДС (20%)

Для каждой позиции автоматически рассчитывается НДС:

```
НДС = Сумма заказчику × 20%
```

НДС отображается в отдельной колонке (фиолетовым цветом) и в KPI-полоске вверху.

> **Подсказка:** НДС применяется, если цены указаны без НДС. Если ваши цены уже включают НДС — учитывайте это при вводе.

## Кнопка «Создать КП»

В шапке ФМ есть кнопка **«Создать КП»** (фиолетовая). Нажмите её, чтобы сформировать коммерческое предложение на основе текущей финансовой модели.

1. Укажите название (по умолчанию: «КП — {название бюджета}»)
2. Нажмите **«Создать КП»**
3. Система создаст КП с копией всех позиций ФМ и перенаправит вас на страницу нового КП

## Дополнительные инструменты

### CVR (Cost/Value Realization)

Вкладка **CVR** показывает, как стоимость проходит через стадии:
- **Запланировано** → **Тендер** → **Договор** → **Акт подписан** → **Счёт выставлен** → **Оплачено**

Для каждой позиции виден прогресс-бар: какая доля стоимости на какой стадии.

### Снимки

Вкладка **Снимки** позволяет сохранить текущее состояние ФМ и потом сравнить с будущими версиями. Удобно, чтобы видеть, как менялись цены с течением проекта.

### Сценарное моделирование

Кнопка **«Сценарии»** открывает инструмент для моделирования «что если»:
- Укажите целевую маржу (например, 20%)
- Система рассчитает, какие цены заказчику нужно установить

### Привязка к графику (WBS)

Каждую позицию можно привязать к задаче из графика проекта (WBS). Это позволяет распределить стоимость по времени для расчёта Cash Flow.

## Частые вопросы

**Когда создаётся ФМ?**
Автоматически при создании нового проекта. Вы также можете создать ФМ вручную через Финансы → Бюджеты → «Новый бюджет».

**Откуда берётся себестоимость?**
Из конкурентного листа (при выборе поставщика-победителя), из привязанного счёта поставщика или вводится вручную. Источник показан цветной меткой рядом с ценой.

**Откуда берётся сметная стоимость?**
Из импорта ЛСР (локальной сметы). Нажмите кнопку «Импорт ЛСР» и загрузите xlsx-файл из ГРАНД-Сметы.

**Что означает красная обводка у цены заказчику?**
Цена заказчику превышает сметную стоимость. В мягком режиме это предупреждение (можно сохранить), в строгом — блокировка.

**Как понять, прибыльный ли проект?**
Посмотрите на показатель «Маржа %» в KPI-полоске. Зелёный (> 15%) — хорошо. Жёлтый (5–15%) — нормально. Оранжевый (0–5%) — рискованно. Красный (< 0%) — убыток.

**Можно ли менять цены, если бюджет утверждён?**
Зависит от статуса. В статусе «Черновик» — можно всё. В «Утверждён» и «Активный» — ограниченно. В «Заморожен» и «Закрыт» — нельзя.

**Что такое CVR?**
Cost/Value Realization — отслеживание стоимости по стадиям. Для каждой позиции видно, какая доля суммы уже на этапе договора, акта, счёта или оплаты. Полезно для контроля выполнения.
