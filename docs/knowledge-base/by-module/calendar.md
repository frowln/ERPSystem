# Календарь

---

## Что это и зачем

Календарь — это единое расписание для всей компании и каждого проекта. Совещания, дедлайны, поставки, проверки, вехи — всё в одном месте. Не нужно заглядывать в пять разных мест: открыли календарь — увидели, что сегодня важного.

Также в системе есть **Диаграмма Ганта** — это ленточный график всех задач по проекту с зависимостями и критическим путём. Гант помогает увидеть весь план работ целиком.

---

## Где найти

**Календарь:** Главное меню → **Календарь**

**Диаграмма Ганта:** Главное меню → **Планирование** → **Ганта**

---

## Календарь — основной вид

### Виды календаря

Два режима отображения (переключатель вверху):

| Вид | Что видно |
|-----|----------|
| **Месяц** | Сетка на весь месяц. Каждая клетка — день. Внутри — цветные полоски событий (до 3, остальные — «+N ещё»). Сегодняшний день выделен синим кружком |
| **Неделя** | 7 дней с часовой разбивкой (с 7:00 до 20:00). Удобно для планирования дня. Сверху — события на весь день. Сегодняшний столбец подсвечен голубым |

### Навигация

- **Стрелки влево/вправо** — перейти к предыдущему/следующему месяцу (или неделе)
- **Кнопка «Сегодня»** — вернуться к текущей дате

### Фильтры

- **Фильтр по типу** — цветные кнопки-чипсы над календарём. Показывают количество событий каждого типа. Нажмите, чтобы показать/скрыть
- **Все события** — показать всё разом
- **Показать дедлайны задач** — красная кнопка в шапке. Включает наложение сроков задач на календарь (берётся из ваших задач)

### Боковая панель

Справа от календаря — таблица **«Ближайшие события»** (10 штук):
- Дата
- Название + имя организатора
- Тип события (цветная метка)
- Время (или «весь день»)

Нажмите на событие — откроется карточка с подробностями.

### Легенда цветов

Внизу боковой панели — цветные точки с подписями всех типов событий.

---

## Типы событий

| Тип | Цвет | Для чего |
|-----|------|---------|
| **Совещание** | Синий | Планёрки, встречи с заказчиком, выезды на площадку |
| **Дедлайн** | Красный | Крайние сроки: сдача документации, подача отчётов, контрактные даты |
| **Проверка** | Оранжевый | Инспекции, аудиты, технический надзор |
| **Поставка** | Зелёный | Приход материалов, техники, оборудования на площадку |
| **Веха** | Фиолетовый | Завершение ключевого этапа: «фундамент готов», «кровля смонтирована» |
| **Выходной** | Серый | Праздничные дни, выходные компании |
| **Другое** | Серый | Всё остальное |

## Как создать событие

1. Нажмите кнопку **«+ Новое событие»** в правом верхнем углу (или кликните на дату в сетке календаря)
2. Заполните форму:

**Основная информация:**
- **Название** (обязательно) — например, «Планёрка с заказчиком» или «Поставка арматуры»
- **Тип события** (обязательно) — выберите из списка (совещание, дедлайн, проверка и т.д.)
- **Проект** — привяжите к проекту (необязательно, но удобно для фильтрации)
- **Организатор** — заполняется автоматически (вы)
- **Весь день** — поставьте галочку, если событие не привязано к конкретному времени

**Дата и время:**
- **Дата начала** (обязательно)
- **Время начала** (если не «весь день»)
- **Дата окончания** (обязательно)
- **Время окончания** (если не «весь день»)

**Место:**
- **Онлайн-совещание** — переключатель. Если включено — введите ссылку на видеозвонок
- **Место проведения** — если встреча очная, укажите адрес или кабинет

**Описание:**
- Свободный текст до 2000 символов. Можно описать повестку, приложить ссылки

3. Нажмите **«Сохранить»**

> **Подсказка:** Для регулярных совещаний (еженедельная планёрка) создайте событие один раз — в будущем система позволит настроить повторение.

## Карточка события

Нажмите на событие в календаре — откроется карточка:

- Тип события (цветная метка) + имя организатора
- Дата и время
- Место проведения (с иконкой карты) или ссылка на онлайн-встречу (с иконкой ссылки)
- Описание (если заполнено)
- Проект (если привязан)

## Производственный календарь

Помимо событий, в системе есть **Производственный (рабочий) календарь**. Он определяет, какие дни рабочие, а какие — выходные и праздничные.

**Зачем нужен:**
- Для правильного расчёта сроков на диаграмме Ганта (задачи считаются только в рабочих днях)
- Для табеля учёта рабочего времени (Т-13)
- Для расчёта зарплат

**Как настроить:**
1. Перейдите в настройки календаря
2. Инициализируйте год — система создаст стандартный производственный календарь РФ
3. Добавьте исключения: корпоративные выходные, рабочие субботы и т.д.

## Диаграмма Ганта

Диаграмма Ганта — это ленточный график, на котором каждая задача изображена полоской от даты начала до даты окончания.

**Путь:** Главное меню → **Планирование** → вкладка **Ганта**

### Что видно на графике

- **Полоски** — каждая задача. Длина полоски = продолжительность задачи
- **Красные полоски** — задачи на критическом пути (если они задержатся — весь проект задержится)
- **Синие полоски** — обычные задачи (есть запас по времени)
- **Стрелки** — зависимости между задачами (одна не может начаться, пока другая не закончена)
- **Вертикальная линия «Сегодня»** — текущая дата

### Показатели (карточки вверху)

| Показатель | Что значит |
|-----------|-----------|
| Всего задач | Количество задач в проекте |
| Критических задач | Сколько задач на критическом пути (+ процент от общего) |
| Длительность проекта | Сколько дней от первой задачи до последней |
| Длина критического пути | Сколько дней длится самая длинная цепочка зависимых задач |

### Масштаб (zoom)

Переключатель вверху:
- **Месяц** — подробный вид, видно каждый день
- **Квартал** — средний масштаб (по умолчанию)
- **Год** — общий вид, весь проект на одном экране

### Фильтры

- Поиск по названию или коду задачи
- «Только критические» — показать только задачи на критическом пути
- Развернуть/свернуть все группы

## WBS — Структура декомпозиции работ

**Путь:** Планирование → вкладка **WBS**

WBS (Work Breakdown Structure) — это дерево, в котором проект разбит на всё более мелкие части.

Пример:
```
Проект «ЖК Солнечный»
├── Подготовительные работы
│   ├── Разрешения и согласования
│   ├── Площадка и коммуникации
│   └── Мобилизация
├── Фундамент
│   ├── Земляные работы
│   ├── Монтаж опалубки
│   └── Бетонирование
├── Каркас
│   ├── Монтаж колонн
│   └── Монтаж перекрытий
└── ...
```

Каждый элемент WBS можно развернуть/свернуть. К каждому привязаны задачи, бюджет и сроки.

## Критический путь

**Путь:** Планирование → вкладка **Критический путь**

Критический путь — это самая длинная цепочка зависимых задач. Если любая задача на этом пути задержится — весь проект задержится.

**Что видно:**
- Список задач на критическом пути (выделены красным)
- Общая длительность цепочки
- Зависимости между задачами

> **Подсказка:** Обращайте особое внимание на задачи критического пути. Если нужно ускорить проект — именно эти задачи нужно ускорять в первую очередь.

## EVM — Метод освоенного объёма

**Путь:** Главное меню → **Планирование** → вкладка **EVM**

EVM сравнивает три числа:
1. Сколько **планировали** потратить к сегодняшнему дню
2. Сколько **фактически** потратили
3. Сколько работ **фактически** выполнено (в деньгах)

**Ключевые показатели:**

| Показатель | Значение | Что значит |
|-----------|---------|-----------|
| **CPI** | > 1.0 | Экономим бюджет. = 1.0 — точно по плану. < 1.0 — перерасход |
| **SPI** | > 1.0 | Опережаем график. = 1.0 — по графику. < 1.0 — отстаём |

> **Пример:** CPI = 0.8 означает: на каждый потраченный рубль мы получаем работ только на 80 копеек. Перерасход 20%. SPI = 1.1 означает: мы выполнили на 10% больше, чем планировали к этой дате. Опережаем.

## Связь с другими модулями

| Модуль | Как связан |
|--------|-----------|
| **Проект** | Календарные события привязываются к проекту. Ганта строится по задачам проекта |
| **Задачи** | Дедлайны задач отображаются на календаре. Задачи — основа для Ганта |
| **HR / Табель** | Производственный календарь определяет рабочие/выходные дни для табеля |
| **Безопасность** | Плановые инспекции и инструктажи могут отображаться как события |
| **Закупки** | Даты поставок материалов — как события типа «Поставка» |
| **Закрытие (КС-2)** | Сроки закрытия периодов — как дедлайны |

## Частые вопросы

**Как увидеть события только по одному проекту?**
В фильтрах календаря выберите нужный проект. Также на странице проекта (вкладка «Обзор») видны ближайшие события.

**Как наложить сроки задач на календарь?**
Нажмите красную кнопку **«Показать дедлайны задач»** в шапке календаря. Дедлайны из ваших задач появятся на сетке.

**Чем отличается календарь от Ганта?**
Календарь — это расписание событий (совещания, поставки, проверки). Ганта — это план работ с зависимостями. Календарь показывает «что произойдёт в этот день», Ганта показывает «как связаны задачи и где мы относительно плана».

**Что такое критический путь простыми словами?**
Представьте: нельзя класть кирпич, пока нет фундамента. Нельзя штукатурить, пока нет стен. Нельзя красить, пока нет штукатурки. Эта цепочка — и есть критический путь. Если задержать любое звено — задержится весь проект.

**Что означают красные и синие полоски на Ганте?**
Красные — задачи на критическом пути (задержка = задержка проекта). Синие — задачи с запасом времени (небольшая задержка не повлияет на срок проекта).

**Как посмотреть, укладываемся ли мы в план?**
Откройте **Планирование → EVM**. Если CPI >= 1.0 и SPI >= 1.0 — всё хорошо. Если какой-то показатель ниже 1.0 — есть проблема (перерасход или отставание).

**Можно ли настроить рабочие дни?**
Да. В производственном календаре можно инициализировать стандартный календарь РФ на год, а затем добавить исключения: рабочие субботы, корпоративные выходные и т.д.

**Как пригласить коллегу на совещание?**
Создайте событие типа «Совещание». После создания можно добавить участников — они получат уведомление и смогут подтвердить или отклонить приглашение.
