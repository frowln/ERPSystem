# Дашборды и отчёты

---

## Что это и зачем

**Аналитика** — это модуль, который собирает данные со всех модулей системы и показывает их в виде графиков, таблиц и KPI. Это «приборная панель» вашей компании.

Без аналитики:
- Директор спрашивает «как дела на объектах?» — менеджеры собирают данные полдня
- Финансовую картину видно только в конце месяца, когда бухгалтер сведёт всё вручную
- Проблемные проекты замечают, когда уже поздно что-то исправить
- Каждый руководитель ведёт свой Excel — цифры не сходятся

Модуль «Аналитика» в системе ПРИВОД:
- Дашборд руководителя — ключевые показатели на одном экране
- Конструктор отчётов — создавайте свои отчёты
- KPI сотрудников — оценка эффективности
- Бонусные расчёты — автоматический расчёт премий
- 16+ готовых шаблонов отчётов

## Часть 1: Дашборд руководителя

**Путь:** Аналитика → **Дашборд руководителя**

На одном экране видно **всё самое важное**:

| Виджет | Что показывает |
|--------|---------------|
| **Активные проекты** | Сколько проектов в работе прямо сейчас |
| **Выполнение плана** | Процент выполнения по всем проектам |
| **Финансовые показатели** | Доходы, расходы, прибыль за период |
| **Просроченные задачи** | Сколько задач не выполнено в срок |
| **Безопасность** | Количество инцидентов, дней без происшествий |
| **Загрузка ресурсов** | Насколько загружены люди и техника |

### Период

Все данные можно смотреть за разные периоды:
- Сегодня
- Эта неделя
- Этот месяц
- Этот квартал
- Этот год
- Произвольный диапазон дат

## Часть 2: Готовые отчёты

**Путь:** Аналитика → **Отчёты**

Система содержит **16+ готовых шаблонов**:

### Финансовые отчёты

| Отчёт | Что показывает |
|-------|---------------|
| **Сводка по проектам** | Бюджет, факт, отклонение — по каждому проекту |
| **Доходы и расходы** | Поступления vs. затраты за период |
| **Дебиторская задолженность** | Кто и сколько вам должен |
| **Кредиторская задолженность** | Кому и сколько должны вы |
| **Рентабельность проектов** | Прибыльность каждого объекта |

### Производственные отчёты

| Отчёт | Что показывает |
|-------|---------------|
| **Выполнение работ** | Факт vs. план по объёмам |
| **Отклонения от графика** | Какие работы отстают |
| **Загрузка техники** | Коэффициент использования |
| **Расход материалов** | Факт vs. нормы |

### Кадровые отчёты

| Отчёт | Что показывает |
|-------|---------------|
| **Табель** | Рабочее время сотрудников |
| **Текучесть кадров** | Принятые / уволенные за период |
| **Фонд оплаты труда** | Расходы на зарплату по подразделениям |

### Отчёты по безопасности

| Отчёт | Что показывает |
|-------|---------------|
| **Инциденты** | Количество и типы происшествий |
| **Обучение ТБ** | Кто прошёл / не прошёл инструктаж |
| **Предписания** | Выданные и выполненные предписания |

Каждый отчёт можно:
- Фильтровать по проектам, датам, подразделениям
- Экспортировать в **Excel** или **PDF**
- Распечатать

## Часть 3: Конструктор отчётов

**Путь:** Аналитика → **Конструктор отчётов**

Если готовых отчётов не хватает — создайте свой.

### Шаг 1: Выберите источник данных

| Источник | Что содержит |
|----------|-------------|
| **Проекты** | Статусы, даты, бюджеты |
| **Задачи** | Исполнители, сроки, статусы |
| **Финансы** | Счета, платежи, бюджеты |
| **Контракты** | Суммы, контрагенты, статусы |
| **Сотрудники** | Должности, подразделения, табели |
| **Техника** | ГСМ, пробег, ТО |
| **Материалы** | Склад, поступления, расход |
| **Закупки** | Заявки, заказы, поставки |
| **Безопасность** | Инциденты, обучение, инспекции |
| **Качество** | Дефекты, предписания, замечания |
| **Документы** | Типы, статусы, авторы |
| **CRM** | Лиды, сделки, активность |

### Шаг 2: Настройте параметры

- Какие **колонки** показать
- Как **группировать** (по проектам, по месяцам, по сотрудникам...)
- Какие **фильтры** применить
- Как **сортировать**

### Шаг 3: Выберите визуализацию

| Тип диаграммы | Для чего подходит |
|---------------|------------------|
| **Таблица** | Детальные данные с цифрами |
| **Столбчатая** | Сравнение значений (бюджет по проектам) |
| **Линейная** | Изменения во времени (расходы по месяцам) |
| **Круговая** | Доли целого (распределение бюджета) |
| **Область** | Накопительные графики |
| **Точечная** | Корреляция двух показателей |

### Шаг 4: Сохраните

Отчёт сохраняется и доступен в любой момент. Можно настроить **автоматическую отправку** на email по расписанию.

## Часть 4: KPI (ключевые показатели эффективности)

**Путь:** Аналитика → **KPI**

### Что такое KPI

KPI — это **числовые показатели**, по которым оценивается работа сотрудника или подразделения. Например:
- Сдача объекта в срок — да или нет
- Отклонение от бюджета — не более 5%
- Количество замечаний по качеству — не более 3

### Категории KPI

| Категория | Примеры показателей |
|-----------|-------------------|
| **Финансовые** | Выручка, прибыль, рентабельность, отклонение от бюджета |
| **Производственные** | Выполнение объёмов, соблюдение сроков, производительность |
| **Качество** | Количество дефектов, замечаний, переделок |
| **Безопасность** | Дни без инцидентов, процент обученных, выполнение предписаний |
| **Кадры** | Текучесть, удовлетворённость, стаж |

### Настройка KPI

Для каждого показателя задаётся:

| Параметр | Описание |
|----------|----------|
| **Название** | Что измеряем |
| **Формула** | Как считается |
| **Единица измерения** | %, руб., штуки, дни |
| **Целевое значение** | Какой результат — «хорошо» |
| **Пороги** | Зелёный / Жёлтый / Красный |
| **Период** | Месяц / Квартал / Год |
| **Ответственный** | Кто отвечает за этот KPI |

## Часть 5: Достижения KPI

**Путь:** Аналитика → **Достижения KPI**

Здесь видны **фактические значения** KPI за период:

| Показатель | Цель | Факт | Выполнение | Статус |
|-----------|------|------|-----------|--------|
| Сдача в срок | 100% | 85% | 85% | 🟡 |
| Рентабельность | 15% | 18% | 120% | 🟢 |
| Дефекты | ≤5 | 2 | 160% | 🟢 |
| Безопасность | 0 инцидентов | 1 | 0% | 🔴 |

Цветовая индикация:
- 🟢 **Зелёный** — план выполнен или перевыполнен
- 🟡 **Жёлтый** — близко к плану (80–99%)
- 🔴 **Красный** — план не выполнен (менее 80%)

## Часть 6: Бонусные расчёты

**Путь:** Аналитика → **Бонусы**

На основе KPI система автоматически рассчитывает **премии** сотрудников.

### Статусы расчёта

| Статус | Описание |
|--------|----------|
| **Черновик** (DRAFT) | Расчёт создан, можно редактировать |
| **На согласовании** (PENDING) | Отправлен руководителю на утверждение |
| **Утверждён** (APPROVED) | Руководитель согласовал |
| **Выплачен** (PAID) | Премия начислена |
| **Отклонён** (REJECTED) | Руководитель не согласовал |

### Как это работает

1. В конце периода система собирает фактические KPI
2. Рассчитывает процент выполнения
3. Применяет формулу расчёта бонуса (настраивается)
4. Формирует ведомость
5. После утверждения — передаёт в бухгалтерию

## Типичные сценарии

### Сценарий 1: Еженедельная планёрка у директора

```
1. Директор открывает дашборд перед планёркой:
   → Активных проектов: 12
   → Выполнение плана: 78%
   → 3 проекта отстают от графика (красный)
   → 2 просроченных счёта на 3,5 млн руб.

2. На планёрке обсуждает проблемные проекты:
   → Открывает отчёт «Отклонения от графика»
   → ЖК «Южный парк» — отставание 8 дней
   → Причина: задержка поставки арматуры

3. Принимает решение:
   → Ускорить закупку у альтернативного поставщика
   → Добавить вторую смену на монтаж

4. Через неделю:
   → ЖК «Южный парк» вернулся в график ✅
```

### Сценарий 2: Создание собственного отчёта

```
1. Финансовый директор хочет видеть ежемесячный отчёт:
   «Рентабельность по проектам за квартал»

2. Открывает Конструктор отчётов:
   → Источник: Финансы
   → Колонки: Проект, Выручка, Затраты, Прибыль, Рентабельность
   → Группировка: по проектам
   → Фильтр: период — последний квартал
   → Визуализация: столбчатая диаграмма

3. Сохраняет отчёт → он появляется в списке

4. Настраивает автоотправку:
   → Каждый понедельник → на email финдиректора
   → Формат: Excel-файл
```

### Сценарий 3: Квартальная оценка KPI

```
1. HR-менеджер открывает «Достижения KPI» за 1 квартал:

   Менеджер проектов Иванов:
   → Сдача в срок: 100% (цель 100%) → 🟢
   → Бюджет: отклонение 3% (цель ≤5%) → 🟢
   → Удовлетворённость клиента: 4.5/5 → 🟢
   → Итого: 115% выполнения → бонус 25 000 руб.

   Прораб Петров:
   → Сдача в срок: 70% (цель 100%) → 🔴
   → Качество: 8 замечаний (цель ≤5) → 🔴
   → Безопасность: 1 инцидент → 🔴
   → Итого: 45% выполнения → бонус 0 руб., беседа с руководителем

2. Ведомость отправляется на утверждение директору
3. После утверждения → бухгалтер начисляет премии
```

## Частые вопросы

**Кто видит дашборд руководителя?**
По умолчанию — роли ADMIN и MANAGER. Можно настроить доступ для других ролей в разделе «Роли и права».

**Можно ли создать свой дашборд?**
Да, через Конструктор отчётов. Вы можете создать любой набор графиков и таблиц и сохранить его.

**Откуда берутся данные?**
Автоматически из всех модулей системы. Не нужно ничего вводить специально для аналитики — данные собираются в реальном времени.

**Как часто обновляются данные?**
В реальном времени. Как только кто-то вносит данные в любой модуль — они сразу отражаются в аналитике.

**Можно ли выгрузить отчёт в Excel?**
Да. Все отчёты можно экспортировать в Excel и PDF. Кнопка экспорта есть на каждом отчёте.

**Что такое «предиктивная аналитика»?**
Это прогнозирование на основе исторических данных. Например, система может предсказать вероятность задержки проекта или перерасхода бюджета. Эта функция использует алгоритмы машинного обучения.
