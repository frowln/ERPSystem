# Завершение проекта

---

## Что это и зачем

Завершение проекта — это **не просто конец работ**. Это целый процесс, который может длиться месяцы: нужно устранить все замечания, собрать документы, пройти комиссии, получить разрешение на ввод в эксплуатацию и обеспечить гарантийное обслуживание.

Без системного управления завершением:
- Заказчик нашёл 50 замечаний, а подрядчик «потерял» половину
- Комплект исполнительной документации собран на 70% — а ввод в эксплуатацию через неделю
- Гарантийный срок начался, но непонятно, кто за что отвечает
- Документы для Стройнадзора готовы не все — получение ЗОС затягивается

Модуль «Завершение проекта» в системе ПРИВОД включает:
- **Приёмочные комиссии** — чек-листы проверки с подписями
- **Передача документов** — комплекты для заказчика и надзорных органов
- **Гарантийный период** — учёт обязательств и рекламаций
- **ЗОС** — заключение о соответствии (документ от Стройнадзора)
- **Пакет Стройнадзора** — полный комплект для получения разрешения на ввод

## Часть 1: Приёмочные комиссии

**Приёмочная комиссия** — это группа людей (представители заказчика, подрядчика, проектировщика, надзорных органов), которые осматривают объект и фиксируют, что можно принять, а что нет.

### Как это работает

1. **Создаётся чек-лист** — список всего, что нужно проверить
2. **Комиссия осматривает** — каждый пункт проверяется и оценивается
3. **Подписание** — члены комиссии подписывают акт приёмки
4. **Результат** — принято / принято с замечаниями / не принято

### Шаблоны чек-листов

Система содержит готовые шаблоны для разных видов работ:

| Шаблон | Количество пунктов | Для чего |
|--------|:------------------:|----------|
| Общестроительные работы | ~40 | Фундамент, стены, перекрытия, кровля |
| Электромонтажные работы | ~35 | Проводка, щиты, освещение, заземление |
| Сантехнические работы | ~30 | Водопровод, канализация, отопление |
| Вентиляция и кондиционирование | ~25 | Воздуховоды, оборудование, автоматика |
| Отделочные работы | ~45 | Полы, стены, потолки, покраска, плитка |
| Благоустройство | ~20 | Дорожки, озеленение, ограждение, МАФ |

### Статусы чек-листа

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Не начат** (NOT_STARTED) | Серый | Чек-лист создан, проверка ещё не проводилась |
| **В процессе** (IN_PROGRESS) | Жёлтый | Идёт осмотр и заполнение |
| **Завершён** (COMPLETED) | Зелёный | Все пункты проверены, акт подписан ✅ |
| **Не пройден** (FAILED) | Красный | Есть критические замечания, нужна повторная проверка |
| **Приостановлен** (ON_HOLD) | Оранжевый | Проверка приостановлена (не готовы документы, погода и т.д.) |

### Подписание акта

Каждый член комиссии должен подписать акт. Возможные решения при подписании:

| Решение | Описание |
|---------|----------|
| **Утверждаю** (APPROVED) | Всё в порядке, принимаю |
| **С замечаниями** (APPROVED_WITH_COMMENTS) | Принимаю, но есть замечания (не критические) |
| **Отклоняю** (REJECTED) | Не принимаю, нужно устранить и повторить |
| **Воздерживаюсь** (ABSTAINED) | Не в моей компетенции |

## Часть 2: Передача документов заказчику

При сдаче объекта заказчику передаётся комплект документов. Без полного комплекта заказчик **не примет объект**.

### Что входит в комплект

| Раздел | Документы |
|--------|-----------|
| **Проектная документация** | Рабочие чертежи, спецификации, расчёты |
| **Исполнительная документация** | АОСР, общий журнал работ, протоколы испытаний |
| **Сертификаты и паспорта** | На все материалы и оборудование |
| **Инструкции** | По эксплуатации оборудования и систем |
| **Гарантийные документы** | Гарантийные письма от всех подрядчиков |
| **Исполнительные схемы** | Чертежи «как построено» (as-built) |
| **Акты испытаний** | Давление трубопроводов, электрика, вентиляция |
| **Разрешительные документы** | Разрешение на строительство, заключение экспертизы |

### Статусы комплекта документов

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** (DRAFT) | Серый | Формируется список документов |
| **Комплектуется** (IN_PREPARATION) | Жёлтый | Документы собираются от подрядчиков |
| **Передан** (SUBMITTED) | Синий | Комплект передан заказчику на проверку |
| **Принят** (ACCEPTED) | Зелёный | Заказчик принял документы ✅ |
| **Возвращён** (REJECTED) | Красный | Заказчик вернул — не хватает документов |

### Процент готовности

Система показывает прогресс комплектации:

```
Исполнительная документация:  ████████████████████░░  90%
Сертификаты материалов:       █████████████████░░░░░  75%
Инструкции по эксплуатации:   ███████████░░░░░░░░░░  50%
Акты испытаний:               ████████████████████░░  95%
Исполнительные схемы:         ██████████████░░░░░░░  65%

Общая готовность:             ████████████████░░░░░  75%
```

## Часть 3: Гарантийный период

После приёмки объекта начинается **гарантийный период** — время, в течение которого подрядчик обязан бесплатно устранять дефекты.

### Гарантийные обязательства

| Поле | Описание |
|------|----------|
| **Подрядчик** | Кто отвечает |
| **Вид работ** | На что распространяется гарантия |
| **Срок гарантии** | Обычно 2–5 лет |
| **Дата начала** | Дата подписания акта приёмки |
| **Дата окончания** | Дата начала + срок гарантии |
| **Контракт** | К какому договору привязана гарантия |
| **Сумма обеспечения** | Гарантийное удержание или банковская гарантия |

### Статусы гарантийного обязательства

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Действует** (ACTIVE) | Зелёный | Гарантия в силе — подрядчик обязан устранять дефекты |
| **Скоро истекает** (EXPIRING_SOON) | Жёлтый | Осталось менее 60 дней — пора провести осмотр |
| **Истекла** (EXPIRED) | Серый | Гарантийный период закончился |
| **Аннулирована** (VOIDED) | Красный | Гарантия недействительна (например, подрядчик обанкротился) |

### Гарантийные рекламации

Если в гарантийный период обнаружен дефект — создаётся **рекламация** (гарантийный иск).

#### Типы дефектов

| Тип | Описание | Пример |
|-----|----------|--------|
| **Конструктивный** (STRUCTURAL) | Дефект несущих конструкций | Трещина в балке |
| **Кровля** (ROOFING) | Протечки, повреждение кровли | Течёт в месте примыкания |
| **Фасад** (FACADE) | Проблемы с наружной отделкой | Отслоение штукатурки |
| **Сантехника** (PLUMBING) | Водопровод, канализация | Подтекает стояк |
| **Электрика** (ELECTRICAL) | Электроснабжение | Не работает группа розеток |
| **ОВиК** (HVAC) | Вентиляция, кондиционирование | Не работает вытяжка |
| **Отделка** (FINISHING) | Внутренние работы | Отклеиваются обои, скрипит пол |

#### Статусы рекламации

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Открыта** (OPEN) | Красный | Дефект обнаружен и зафиксирован |
| **На рассмотрении** (IN_REVIEW) | Жёлтый | Подрядчик изучает проблему |
| **Принята** (APPROVED) | Синий | Подрядчик признал и начинает устранение |
| **Устраняется** (IN_REPAIR) | Оранжевый | Идут ремонтные работы |
| **Устранена** (RESOLVED) | Зелёный | Дефект устранён, ждёт проверку |
| **Закрыта** (CLOSED) | Серый | Проверено, всё в порядке ✅ |
| **Отклонена** (REJECTED) | Красный | Подрядчик не признаёт гарантийный случай |

```
Открыта → На рассмотрении → Принята → Устраняется → Устранена → Закрыта
                                ↓
                            Отклонена (→ спор → арбитраж)
```

## Часть 4: ЗОС (Заключение о соответствии)

**ЗОС** — заключение о соответствии построенного объекта проектной документации и техническим регламентам. Выдаётся органом **Государственного строительного надзора** (Стройнадзор).

Без ЗОС **невозможно** получить разрешение на ввод в эксплуатацию.

### Что проверяет Стройнадзор

- Соответствие объекта проектной документации
- Наличие полного комплекта исполнительной документации
- Выполнение предписаний (все устранены)
- Результаты итоговой проверки

### Статусы ЗОС

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** (DRAFT) | Серый | Документы готовятся |
| **На рассмотрении** (UNDER_REVIEW) | Жёлтый | Подано в Стройнадзор |
| **Выдано** (APPROVED) | Зелёный | ЗОС получено ✅ |
| **Отказано** (REJECTED) | Красный | Отказ (нужно устранить замечания) |

> **Важно:** ЗОС — один из финальных документов. После получения ЗОС подаётся заявление на **разрешение на ввод в эксплуатацию**.

## Часть 5: Пакет документов для Стройнадзора

Для получения ЗОС нужно подготовить и передать в Стройнадзор полный комплект документов. Система помогает собрать их все.

### Категории документов

| № | Категория | Что входит |
|---|-----------|------------|
| 1 | **Разрешительные** | Разрешение на строительство, заключение экспертизы |
| 2 | **Проектные** | Рабочая документация (все разделы) |
| 3 | **Исполнительная документация** | АОСР, общий журнал работ, специальные журналы |
| 4 | **Лабораторные** | Протоколы испытаний бетона, металла, грунтов |
| 5 | **Геодезические** | Исполнительные съёмки, геомониторинг |
| 6 | **Сертификаты** | На материалы, изделия, оборудование |
| 7 | **Акты испытаний** | Гидравлические, пневматические, электрические |
| 8 | **Пожарная безопасность** | Акты, сертификаты огнезащиты, протоколы |
| 9 | **Экология** | Паспорта отходов, акты рекультивации |
| 10 | **Кадровые** | Допуски работников, удостоверения, приказы |

### Статусы пакета

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** (DRAFT) | Серый | Формируется состав пакета |
| **Формируется** (GENERATING) | Жёлтый | Документы собираются |
| **Готов** (READY) | Зелёный | Полный комплект сформирован ✅ |
| **Передан** (SENT) | Синий | Пакет передан в Стройнадзор |
| **Ошибка** (ERROR) | Красный | Не хватает документов или есть ошибки |

Система показывает процент готовности каждой категории и подсвечивает, что ещё не собрано.

## Часть 6: Исполнительные схемы (As-Built)

**Исполнительные схемы** — это чертежи, показывающие, **как объект построен на самом деле** (в отличие от проектных чертежей, которые показывают, как планировалось).

### Типы схем

| Тип | Описание | Пример |
|-----|----------|--------|
| **План** (PLAN) | Горизонтальное сечение (вид сверху) | План фундаментов, план этажа |
| **Разрез** (SECTION) | Вертикальное сечение | Разрез по стене, разрез фундамента |
| **Узел** (DETAIL) | Увеличенный фрагмент | Узел примыкания кровли к стене |
| **Фасад** (FACADE) | Вид снаружи | Фасад здания с отметками |

Каждая схема содержит:
- Фактические размеры и отметки
- Отклонения от проекта (если есть)
- Привязки к осям
- Подпись ответственного лица
- Дату составления

### Зачем нужны исполнительные схемы

- Для будущего ремонта: чтобы знать, **где реально** проходят трубы и кабели
- Для приёмки: Стройнадзор требует как обязательный документ
- Для эксплуатации: чтобы эксплуатирующая организация знала реальное состояние

## Часть 7: Общий процесс завершения

Весь процесс завершения проекта можно представить как последовательность этапов:

```
1. Завершение строительных работ
   ↓
2. Устранение замечаний (punch list)
   → См. модуль «Дефекты»: docs/knowledge-base/by-module/punch-list.md
   ↓
3. Приёмочная комиссия
   → Осмотр, чек-лист, подписание акта
   ↓
4. Сбор исполнительной документации
   → АОСР, журналы, протоколы, сертификаты
   ↓
5. Подготовка пакета для Стройнадзора
   → 10 категорий документов
   ↓
6. Итоговая проверка Стройнадзора
   → Осмотр объекта + проверка документов
   ↓
7. Получение ЗОС
   → Заключение о соответствии
   ↓
8. Получение разрешения на ввод в эксплуатацию
   → Подача в администрацию города
   ↓
9. Передача документов заказчику
   → Полный комплект
   ↓
10. Начало гарантийного периода
    → 2–5 лет, устранение рекламаций
```

## Типичные сценарии

### Сценарий 1: Приёмка электромонтажных работ

```
1. Электромонтаж корпуса Б завершён
   → Подрядчик заявляет о готовности

2. Создаётся чек-лист «Электромонтажные работы»:
   → 35 пунктов проверки
   → Комиссия: инженер заказчика, подрядчик, проектировщик

3. Осмотр (2 дня):
   → Щитовая: всё по проекту ✅
   → Освещение: 3 светильника не работают ❌
   → Розетки: 2 розетки не закреплены ❌
   → Заземление: сопротивление в норме ✅
   → Маркировка кабелей: частично отсутствует ❌

4. Результат: Не пройден (3 замечания)
   → Замечания передаются подрядчику (см. punch list)

5. Подрядчик устраняет за 3 дня → повторный осмотр

6. Повторная проверка:
   → Все 35 пунктов: пройдено ✅
   → Акт подписан всеми членами комиссии
   → Статус: Завершён
```

### Сценарий 2: Получение ЗОС

```
1. Все работы завершены, все замечания устранены
   → Дашборд показывает: готовность к ЗОС — 92%

2. Не хватает:
   → 3 протокола испытаний трубопроводов
   → 1 акт огнезащитной обработки
   → 2 исполнительные схемы

3. За 2 недели всё собрано → готовность 100%

4. Пакет Стройнадзора:
   → 10 категорий документов: все зелёные ✅
   → Статус: Готов → Передан

5. Итоговая проверка Стройнадзора:
   → Осмотр объекта: соответствует проекту ✅
   → Документы: полный комплект ✅
   → Предписания: все исполнены ✅

6. ЗОС выдано: Утверждено ✅
   → Номер: ЗОС-77-2026/234
   → Дата: 20.11.2026

7. На основании ЗОС → заявление на разрешение на ввод
```

### Сценарий 3: Гарантийная рекламация

```
1. Через 8 месяцев после сдачи объекта:
   → Заказчик обнаружил протечку кровли на 5 этаже

2. Создаётся гарантийная рекламация:
   → Тип дефекта: Кровля
   → Описание: «Протечка в районе оси Г/12, 5 этаж»
   → Фото: 4 снимка (пятно на потолке, вид снаружи)
   → Подрядчик: ООО «КровляМонтаж»
   → Гарантия: действует до 15.12.2028

3. Подрядчик рассматривает:
   → Выезжает на осмотр → подтверждает гарантийный случай
   → Статус: Принята → Устраняется

4. Ремонтные работы (3 дня):
   → Устранена негерметичность примыкания
   → Восстановлен потолок

5. Проверка заказчиком:
   → Протечка устранена ✅
   → Статус: Устранена → Закрыта

6. Стоимость устранения: 0 ₽ (за счёт подрядчика по гарантии)
```

## Частые вопросы

**Что такое ЗОС простыми словами?**
ЗОС — это «диплом» для здания. Государственный строительный надзор подтверждает, что объект построен по проекту и безопасен. Без ЗОС нельзя получить разрешение на ввод в эксплуатацию.

**Сколько длится гарантийный период?**
По закону — не менее 5 лет для конструктивных элементов (фундамент, стены, кровля). Для инженерных систем — обычно 2–3 года. Конкретные сроки прописаны в договоре.

**Что если подрядчик отказывается устранять гарантийный дефект?**
Статус рекламации «Отклонена». Далее: претензия по договору → досудебное урегулирование → суд. Гарантийное удержание может быть использовано для оплаты ремонта другим подрядчиком.

**Сколько времени занимает получение ЗОС?**
Обычно 1–3 месяца после подачи документов (зависит от региона и сложности объекта). Если документы неполные — отказ, доработка и повторная подача.

**Можно ли вводить объект по частям?**
Да, если проектом предусмотрены очереди строительства. Каждая очередь проходит отдельную процедуру приёмки и получения ЗОС.

**Кто входит в приёмочную комиссию?**
Обычно: представитель заказчика, представитель генподрядчика, представитель проектной организации (авторский надзор), представитель эксплуатирующей организации. Для госзаказов — также представители надзорных органов.

**Что делать, если заказчик не принимает документы?**
Узнать причину отказа (она фиксируется в системе). Обычно не хватает конкретных документов — система покажет, каких именно. Дополнить и передать повторно.
