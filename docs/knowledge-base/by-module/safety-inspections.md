# Проверки по охране труда

---

## Что это и зачем

Проверка (инспекция) — это плановый или внеплановый осмотр строительной площадки инженером по ОТ или инспектором. Цель — найти нарушения **до того**, как они приведут к инциденту.

Проверки обязательны по ТК РФ и нормативным актам. Регулярные проверки снижают количество инцидентов, помогают соответствовать требованиям Ростехнадзора и ГИТ.

## Виды проверок

| Тип | Описание | Когда проводится |
|-----|---------|-----------------|
| **Плановая** | Регулярная проверка по графику | По расписанию (еженедельно, ежемесячно) |
| **Внеплановая** | Проверка по причине (жалоба, инцидент, требование надзора) | По необходимости |
| **Повторная** | Проверка устранения ранее выявленных нарушений | После срока устранения нарушений |
| **Надзорная** | Проверка внешними органами (Ростехнадзор, ГИТ, МЧС) | По плану надзорного органа |

## Как создать проверку

1. Перейдите **Безопасность → Проверки**
2. Нажмите **«+ Новая проверка»**

3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Дата проверки** | Да | Когда проводится |
| **Тип** | Да | Плановая / Внеплановая / Повторная / Надзорная |
| **Проект** | Нет | На каком объекте |
| **Инспектор** | Нет | Кто проводит (выбор из сотрудников) |
| **Примечание** | Нет | Дополнительная информация |

4. Нажмите **«Создать»** → проверка в статусе «Запланирована»

---

## Статусы проверки

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Запланирована** | Синий | Проверка создана, ожидает проведения |
| **В процессе** | Жёлтый | Инспектор на объекте, проверка идёт |
| **Завершена** | Зелёный | Проверка завершена, результаты зафиксированы |
| **Отменена** | Серый | Проверка отменена |

```
Запланирована → В процессе → Завершена
      ↓
  Отменена (из любого статуса)
```

## Как провести проверку

### Шаг 1: Начать проверку

1. Откройте запланированную проверку
2. Нажмите **«Начать проверку»**
3. Статус меняется на «В процессе»

### Шаг 2: Обход площадки

Инспектор обходит объект и проверяет:
- Состояние ограждений, лесов, подмостей
- Наличие и исправность СИЗ у работников
- Соблюдение технологии работ
- Электробезопасность (кабели, щитки, заземление)
- Пожарная безопасность (огнетушители, пути эвакуации)
- Чистота и порядок на площадке
- Наличие допусков у работников (удостоверения, инструктажи)

### Шаг 3: Фиксация нарушений

Если обнаружено нарушение — добавьте его прямо из проверки:

1. В детальной странице проверки нажмите **«Добавить нарушение»**
2. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Описание** | Да | Что именно нарушено |
| **Тяжесть** | Да | Низкая / Средняя / Высокая / Критическая |
| **Срок устранения** | Нет | До какой даты нужно исправить |
| **Ответственный** | Нет | Кто должен устранить |

### Шаг 4: Завершить проверку

1. После обхода нажмите **«Завершить проверку»**
2. Выставите **общую оценку**:

| Оценка | Цвет | Что означает |
|--------|------|-------------|
| **Удовлетворительно** | Зелёный | Всё в порядке или мелкие замечания |
| **Требует улучшения** | Жёлтый | Есть нарушения, но не критичные |
| **Неудовлетворительно** | Оранжевый | Серьёзные нарушения, требуется немедленное исправление |
| **Критическое** | Красный | Опасная ситуация, работы необходимо остановить |

3. Заполните поля:
   - **Выводы** — что обнаружено
   - **Рекомендации** — что нужно сделать
   - **Дата следующей проверки** — когда провести повторную

4. Статус меняется на «Завершена»

## Нарушения

### Что такое нарушение

Нарушение — это конкретный факт несоблюдения требований безопасности, выявленный при проверке или в результате инцидента. Каждое нарушение отслеживается до полного устранения.

**Путь:** Безопасность → **Нарушения**

### Тяжесть нарушений

| Уровень | Цвет | Описание | Пример |
|---------|------|---------|--------|
| **Низкая** | Зелёный | Мелкое замечание, не влияет на безопасность напрямую | Мусор на проходе, отсутствие таблички |
| **Средняя** | Жёлтый | Нарушение, которое может привести к проблемам | Работник без каски в неопасной зоне |
| **Высокая** | Оранжевый | Серьёзное нарушение, есть риск для жизни | Неисправные леса, отсутствие ограждения у проёма |
| **Критическая** | Красный | Непосредственная угроза жизни, работы нужно остановить | Работа на высоте без страховки, оголённые провода |

### Статусы нарушения

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Открыто** | Красный | Нарушение обнаружено, ждёт устранения |
| **В работе** | Жёлтый | Назначен ответственный, идёт устранение |
| **Устранено** | Зелёный | Нарушение исправлено и проверено |
| **Просрочено** | Тёмно-красный | Срок устранения прошёл, нарушение не устранено |

```
Открыто → В работе → Устранено
              ↓
         Просрочено (если срок прошёл)
```

### Жизненный цикл нарушения

1. **Обнаружение** — инспектор находит нарушение при проверке → статус «Открыто»
2. **Назначение** — указывает ответственного и срок устранения → статус «В работе»
3. **Устранение** — ответственный исправляет проблему
4. **Проверка** — инспектор проверяет, что нарушение устранено → статус «Устранено»
5. Если срок прошёл, а нарушение не устранено → автоматически «Просрочено»

### Просроченные нарушения

Система автоматически отслеживает просроченные нарушения. На дашборде безопасности отдельный блок показывает количество просроченных нарушений — они требуют немедленного внимания.

## Соответствие (Compliance)

**Путь:** Безопасность → **Соответствие**

Дашборд соответствия показывает общую картину: насколько организация соответствует требованиям охраны труда.

### Показатели дашборда

| Показатель | Что означает |
|-----------|-------------|
| **Всего сотрудников** | Общее число работников |
| **Соответствуют** | Все инструктажи пройдены, сертификаты действуют |
| **Не соответствуют** | Есть просроченные инструктажи или сертификаты |
| **Скоро истекают** | Сертификаты, истекающие в ближайшие 90 дней |
| **Инструктажи по плану** | Запланированные инструктажи |
| **Просроченные инструктажи** | Повторные инструктажи, срок которых прошёл |
| **Блокировки доступа** | Работники, заблокированные из-за несоответствия |

### Блокировка доступа

Если у работника:
- Истёк обязательный сертификат (ОТ и ТБ, электробезопасность и т.д.)
- Просрочен повторный инструктаж
- Не пройдено обязательное обучение

→ Система может заблокировать доступ к работе. Руководитель видит **блокировку** на дашборде и должен:
1. Обеспечить прохождение инструктажа / обновление сертификата
2. Нажать **«Снять блокировку»** после устранения причины

### Автоматическое планирование

Кнопка **«Автопланирование»** — система автоматически:
- Создаёт повторные инструктажи для сотрудников, у которых подходит срок
- Планирует обучение на основе правил (периодичность, тип работ)

## Оценка рисков (Risk Scoring)

Система автоматически рассчитывает **оценку риска** для каждого проекта по 8 факторам:

| Фактор | Вес | Описание |
|--------|-----|---------|
| **Частота инцидентов** | Высокий | Сколько инцидентов за последние 30 дней |
| **Тренд нарушений** | Высокий | Растёт или падает количество нарушений |
| **Соответствие обучения** | Средний | Какой % работников прошёл обучение |
| **Количество субподрядчиков** | Средний | Больше субподрядчиков = больше риск |
| **Истечение сертификатов** | Средний | Какой % сертификатов скоро истекает |
| **Опыт бригады** | Средний | Средний стаж работников |
| **Сложность работ** | Низкий | Оценка сложности текущих работ |
| **Погодные риски** | Низкий | Влияние погоды на безопасность |

### Уровни риска

| Балл | Уровень | Цвет | Действие |
|------|---------|------|---------|
| 0–25 | **Низкий** | Зелёный | Стандартный режим |
| 26–50 | **Средний** | Жёлтый | Усиленный контроль |
| 51–75 | **Высокий** | Оранжевый | Дополнительные проверки, инструктажи |
| 76–100 | **Критический** | Красный | Немедленные меры, возможна остановка работ |

## Типичные сценарии

### Сценарий 1: Плановая еженедельная проверка

```
Понедельник утро:
1. Инженер по ОТ открывает запланированную проверку INS-00042
   → Нажимает «Начать проверку»
   → Статус: В процессе

2. Обходит объект «Ленина, 15»:
   → Леса — ОК ✅
   → Ограждения проёмов — ОК ✅
   → Электрощитки — НАРУШЕНИЕ: открытая дверца щитка без замка
   → СИЗ — ОК ✅
   → Чистота площадки — НАРУШЕНИЕ: строительный мусор на лестнице

3. Добавляет 2 нарушения:
   → Нарушение 1: «Открытый электрощиток без замка», тяжесть: Высокая, срок: 1 день, ответственный: электрик Петров
   → Нарушение 2: «Мусор на лестнице», тяжесть: Низкая, срок: сегодня, ответственный: бригадир Сидоров

4. Завершает проверку:
   → Оценка: «Требует улучшения»
   → Рекомендации: «Установить замки на все электрощитки, назначить ответственного за уборку»
   → Следующая проверка: через неделю

5. Через 1 день — проверяет устранение:
   → Щиток с замком → Нарушение 1: Устранено ✅
   → Лестница чистая → Нарушение 2: Устранено ✅
```

### Сценарий 2: Надзорная проверка ГИТ

```
1. Приехал инспектор ГИТ (государственная инспекция труда)

2. Инженер по ОТ создаёт проверку:
   → Тип: Надзорная
   → Инспектор: Инспектор ГИТ Козлов А.В.
   → Статус: В процессе

3. По результатам — 3 нарушения:
   → Критическое: Работа на высоте без страховки (бригада электриков)
   → Высокое: Просроченные удостоверения у 2 стропальщиков
   → Среднее: Нет журнала инструктажей по пожарной безопасности

4. Инспектор выписывает предписание
   → В системе: оценка «Неудовлетворительно»
   → Сроки устранения: 3 дня / 10 дней / 10 дней

5. Через 10 дней — повторная проверка (тип: Повторная):
   → Все нарушения устранены → Оценка: «Удовлетворительно»
```

## СОУТ (Специальная оценка условий труда)

**Путь:** Безопасность → **Карта СОУТ**

СОУТ — обязательная процедура (каждые 5 лет). Аттестованная организация оценивает каждое рабочее место и присваивает класс условий труда.

### Классы условий труда

| Класс | Название | Что означает | Последствия |
|-------|---------|-------------|-------------|
| **1** | Оптимальные | Безопасные условия | Нет дополнительных компенсаций |
| **2** | Допустимые | Приемлемые условия | Нет дополнительных компенсаций |
| **3.1** | Вредные (1 степень) | Лёгкие отклонения | Надбавка 4% к зарплате |
| **3.2** | Вредные (2 степень) | Заметные отклонения | Надбавка 4% + сокращённый день |
| **3.3** | Вредные (3 степень) | Значительные отклонения | Надбавка 4% + сокращённый день + доп. отпуск 7 дней |
| **3.4** | Вредные (4 степень) | Сильные отклонения | Все компенсации выше + молоко |
| **4** | Опасные | Угроза жизни | Работы в таких условиях не допускаются (нужно устранить) |

### Карточка СОУТ

| Поле | Описание |
|------|---------|
| Номер карточки | Уникальный номер |
| Рабочее место | Название (например, «Рабочее место электросварщика») |
| Номер рабочего места | Код |
| Подразделение | Отдел или бригада |
| Должность | Какая профессия |
| Количество работников | Сколько человек работают на этом месте |
| **Класс условий** | 1 / 2 / 3.1 / 3.2 / 3.3 / 3.4 / 4 |
| Дата оценки | Когда проведена СОУТ |
| Следующая оценка | Через 5 лет |
| Вредные факторы | Что выявлено (шум, вибрация, пыль, химия и т.д.) |
| Компенсации | Какие надбавки положены |
| Рекомендации по СИЗ | Какие средства защиты нужны |

## Частые вопросы

**Как часто нужно проводить проверки?**
Зависит от опасности работ. Обычно: еженедельно на активных объектах. Для особо опасных работ — ежедневно.

**Кто может проводить проверку?**
Инженер по ОТ, руководитель проекта, представитель профсоюза. Для надзорных — инспектор Ростехнадзора или ГИТ.

**Что делать, если найдено критическое нарушение?**
Остановить работы в опасной зоне, устранить нарушение, провести внеплановый инструктаж, зафиксировать в системе.

**Можно ли удалить проверку?**
Технически — мягкое удаление. Но завершённые проверки рекомендуется не удалять — это аудиторский след.

**Что такое СОУТ и обязательна ли она?**
СОУТ — Специальная оценка условий труда. Обязательна для всех работодателей, проводится раз в 5 лет. Штраф за отсутствие: до 80 000 ₽ для организации.

**Как связаны нарушения и инциденты?**
Нарушение — это потенциальная проблема (ещё не произошло). Инцидент — это уже произошедшее событие. Неустранённое нарушение может привести к инциденту.

**Что показывает оценка риска?**
Числовой балл (0–100), рассчитанный по 8 факторам. Чем выше балл — тем опаснее проект. Помогает приоритизировать проверки.
