# КС-2 (Акт о приёмке выполненных работ)

---

## Что это и зачем

КС-2 — это официальный акт, подтверждающий, что подрядчик выполнил определённые работы. Без подписанного КС-2 подрядчик не может потребовать оплату — это юридическое основание для расчётов.

Форма КС-2 утверждена Госкомстатом России (унифицированная форма №КС-2). Она обязательна для всех строительных организаций.

**В акте указано:** какие работы выполнены, в каком объёме, по какой цене и на какую сумму. Акт подписывают две стороны — подрядчик (кто выполнил) и заказчик (кто принял).

**Цепочка:** Журнал работ → **КС-2** → [КС-3](ks3.md) → [Счёт](invoices.md) → [Оплата](payments.md)

---

## Где найти

**Путь:** Главное меню → **Закрытие** → **КС-2**

---

## Список КС-2

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Номер | Номер акта (например, «P-202603-0001») |
| Наименование | Автоматическое: «КС-2 №... от ...» |
| Дата | Дата составления акта |
| Объект | К какому проекту относится |
| Договор | Номер договора |
| Статус | Черновик / На рассмотрении / Подписан / Закрыт |
| Сумма | Итого по акту (руб.) |
| Позиции | Количество строк (работ) в акте |

**Вкладки-фильтры:**
- **Все** — все акты
- **Черновик** — в работе, можно редактировать
- **На рассмотрении** — отправлен на проверку
- **Подписан** — обе стороны подписали
- **Закрыт** — финальный статус

**Поиск:** по наименованию, номеру или проекту.

## Как создать КС-2

### Вручную

1. Перейдите в **Закрытие → КС-2**
2. Нажмите **«Новый КС-2»**

3. Заполните шапку:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Номер** | Да | Уникальный номер акта (до 50 символов) |
| **Дата** | Да | Дата составления |
| **Объект** | Да | К какому проекту относится |
| **Договор** | Да | По какому договору выполнены работы |
| **Примечание** | Нет | Дополнительные комментарии |

4. Нажмите **«Создать»** — акт создаётся в статусе «Черновик»

### Из журнала работ (пайплайн)

Система может **автоматически** сформировать КС-2 из записей журнала работ за месяц:

1. Перейдите в **Закрытие → Пайплайн КС-2** (`/ks2/pipeline`)
2. Выберите **объект**, **договор** и **месяц**
3. Нажмите **«Предпросмотр»** — система покажет, какие работы будут включены
4. Нажмите **«Сгенерировать»** — система создаст КС-2 с заполненными позициями

**Как это работает:**
- Система берёт записи из журнала работ (дневные отчёты) за выбранный месяц
- Группирует по наименованию работ и единице измерения
- Подбирает цену из спецификации (если есть привязка)
- Создаёт КС-2 с готовыми строками

> **Подсказка:** Кнопка «Пакетная генерация» создаст КС-2 сразу по всем договорам проекта за месяц — удобно, когда много субподрядчиков.

## Позиции (строки) КС-2

После создания акта добавьте позиции — конкретные работы.

Нажмите **«Добавить позицию»** и заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Наименование работ** | Да | Что выполнено (например, «Кладка стен из кирпича М-150») |
| **Единица измерения** | Нет | м³, м², шт, т, п.м. и т.д. |
| **Количество** | Да | Объём выполненных работ (больше 0) |
| **Цена за единицу** | Да | Стоимость одной единицы (руб.) |
| **Сумма** | Авто | Считается автоматически: количество × цена |
| **Ставка НДС** | Нет | 20% (по умолчанию), 10% или 0% |
| **Сумма НДС** | Авто | Считается: сумма × ставка НДС |
| **Примечание** | Нет | Комментарий к строке |

Можно добавить сколько угодно строк. Итоги по акту пересчитываются автоматически при каждом изменении.

### Пример позиций КС-2

| № | Наименование | Ед. изм. | Кол-во | Цена | Сумма |
|---|-------------|----------|--------|------|-------|
| 1 | Кладка стен из кирпича М-150 | м³ | 120 | 4 500 | 540 000 |
| 2 | Устройство монолитного перекрытия | м² | 300 | 3 200 | 960 000 |
| 3 | Штукатурка стен | м² | 450 | 800 | 360 000 |
| | | | | **Итого:** | **1 860 000** |
| | | | | **НДС 20%:** | **372 000** |
| | | | | **Всего с НДС:** | **2 232 000** |

## Итоги акта

На карточке КС-2 отображаются 4 итоговых показателя:

| Показатель | Что значит |
|-----------|-----------|
| Итого без НДС | Сумма всех позиций (до налога) |
| НДС | Сумма налога по всем позициям |
| Итого с НДС | Полная стоимость работ |
| Общий объём | Суммарное количество по всем позициям |

Итоги пересчитываются автоматически при добавлении, изменении или удалении любой позиции.

---

## Статусы КС-2

| Статус | Цвет | Что означает | Что можно делать |
|--------|------|-------------|-----------------|
| **Черновик** | Серый | Акт создан, позиции заполняются | Редактировать, добавлять/удалять позиции |
| **На рассмотрении** | Синий | Отправлен на проверку и согласование | Проверять, согласовывать |
| **Подписан** | Зелёный | Обе стороны подписали | Формировать КС-3, выставлять счёт |
| **Закрыт** | Фиолетовый | Документооборот завершён | Только просмотр |

### Порядок переходов

```
Черновик → На рассмотрении → Подписан → Закрыт
                ↓
            Черновик (возврат на доработку)
```

> **Важно:** Для отправки на рассмотрение в акте должна быть хотя бы одна позиция. Пустой акт отправить нельзя.

> **Важно:** При подписании КС-2 система автоматически обновляет бюджет проекта — фактические объёмы отражаются в колонке «Акты подписаны».

## Согласование КС-2

Страница **«Согласование КС-2»** (`/ks2/approvals`) показывает маршрут утверждения акта.

### Этапы согласования

| Этап | Кто согласует | Что проверяет |
|------|-------------|--------------|
| 1. **Подрядчик** | Исполнитель работ | Объёмы и цены верны |
| 2. **Технический контроль** | Технический менеджер | Работы фактически выполнены |
| 3. **Бухгалтерия** | Бухгалтер | Суммы и НДС корректны |
| 4. **Заказчик** | Представитель заказчика | Принимает работы |

На каждом этапе ответственный может:
- **Утвердить** — акт переходит на следующий этап
- **Отклонить** (с обязательным комментарием) — акт возвращается на доработку

## Проверка объёмов

Страница **«Проверка объёмов»** (`/ks2/volume-check`) позволяет сравнить объёмы в КС-2 с плановыми данными из сметы.

**Что показывается по каждой позиции:**

| Колонка | Что значит |
|---------|-----------|
| Работа | Наименование |
| Ед. изм. | Единица измерения |
| По смете | Плановое количество |
| Сдано ранее | Сколько уже было в предыдущих КС-2 |
| В этом акте | Сколько сдаётся сейчас |
| Остаток | Сколько ещё осталось сдать |
| Статус | В норме / Внимание / Превышение |

**Цветовая индикация:**
- **Зелёный** (в норме) — объём в пределах сметы
- **Жёлтый** (внимание) — использовано 75–100% от сметного объёма
- **Красный** (превышение) — объём превышает смету

## Печатная форма КС-2

Нажмите кнопку **«Печать»** на карточке акта — откроется печатная форма, соответствующая официальному формату.

### Структура печатной формы

| Раздел | Что содержит |
|--------|-------------|
| **Шапка** | «Форма КС-2» — «АКТ ВЫПОЛНЕННЫХ РАБОТ», номер и дата |
| **Стороны** | Слева — подрядчик (название, ИНН, адрес). Справа — заказчик (название, ИНН, адрес) |
| **Объект** | Наименование объекта строительства, адрес |
| **Договор** | Номер и дата договора |
| **Таблица работ** | № п/п, Наименование, Ед. изм., Кол-во, Цена, Сумма |
| **Итоги** | Итого без НДС → НДС (20%) → Всего с НДС |
| **Примечание** | Свободный текст (если есть) |
| **Подписи** | Две колонки: «Подписано подрядчиком» и «Подписано заказчиком» с местами для подписи и ФИО |

Суммы отображаются в российском формате: `1 234 567,89` (пробел — разделитель тысяч, запятая — десятичный разделитель).

> Печатную форму можно также скачать как PDF (`/api/pdf-reports/ks2/{id}/pdf`).

## Пайплайн КС-2

**Путь:** Закрытие → **Пайплайн КС-2** (`/ks2/pipeline`)

Пайплайн — это инструмент автоматической генерации КС-2 из данных журнала работ.

**Как пользоваться:**

1. Выберите **объект** (проект)
2. Выберите **договор** (или оставьте пустым для пакетной генерации)
3. Выберите **месяц** (например, «2026-03»)
4. Нажмите **«Предпросмотр»** — увидите список работ с объёмами
5. Нажмите **«Сгенерировать»** — система создаст КС-2

**Показатели предпросмотра:**
- Количество позиций работ
- Общая сумма
- Количество уникальных видов работ

> **Внимание:** Если по позиции нет цены (работа не привязана к спецификации), в колонке «Цена» будет 0 и жёлтое предупреждение. Заполните цену вручную после генерации.

## КС-6а (Журнал учёта выполненных работ)

**Путь:** Закрытие → **КС-6а** (`/ks6a`)

КС-6а — это накопительная ведомость, где по каждой работе помесячно записано: сколько выполнено в январе, в феврале и т.д.

**Выберите объект и год**, чтобы увидеть:

| Колонка | Что показывает |
|---------|---------------|
| Месяц | Календарный месяц |
| Работа | Наименование |
| Ед. изм. | Единица |
| План | Плановый объём (из сметы) |
| Нарастающий итог | Сколько всего выполнено к этому месяцу |
| За месяц | Сколько выполнено именно в этом месяце |
| Прогресс | Визуальная полоска (% выполнения от плана) |

## Корректировочные акты

**Путь:** Закрытие → **Корректировочные акты** (`/correction-acts`)

Если в подписанном КС-2 обнаружена ошибка (например, неверный объём), вместо отмены всего акта создаётся корректировочный акт.

**Пример:** Написали «100 м³ бетона», а на самом деле — 95 м³. Корректировочный акт: −5 м³.

| Поле | Описание |
|------|----------|
| Исходный акт | Ссылка на оригинальный КС-2 |
| Причина | Почему нужна корректировка |
| Позиции | Работа, исходное кол-во, корректировка (может быть отрицательной) |

Статусы: Черновик → Утверждён → Применён.

## Интеграции

### 1С
При подписании КС-2 система может автоматически выгрузить акт в 1С как «Реализация товаров и услуг». Статус синхронизации виден на карточке акта:
- **Не отправлен** — синхронизация не производилась
- **Отправлен в 1С** — данные переданы
- **Проведён в 1С** — документ проведён в 1С
- **Ошибка** — произошла ошибка при передаче

### ЭДО (электронный документооборот)
КС-2 можно отправить через СБИС или другую систему ЭДО. Статус электронной подписи также отображается на карточке.

### Портал заказчика
Заказчик может создавать черновики КС-2 через [портал](portal.md) и отправлять на проверку. После утверждения черновик конвертируется в полноценный КС-2.

## Частые вопросы

**Зачем нужен КС-2?**
Это обязательный документ в строительстве. Без подписанного КС-2 подрядчик не может выставить счёт, а заказчик — не обязан платить. КС-2 фиксирует факт: какие именно работы выполнены и на какую сумму.

**Можно ли создать КС-2 без сметы?**
Да. Позиции можно заполнить вручную — указать наименование, объём и цену. Но рекомендуется привязывать к спецификации или смете, чтобы цены были обоснованы.

**Что такое пайплайн?**
Это автоматическая генерация КС-2 из журнала работ. Система берёт записи за месяц, группирует их и создаёт акт. Это экономит время, особенно когда много мелких работ.

**Можно ли изменить подписанный КС-2?**
Нет. После подписания акт нельзя редактировать. Если нашли ошибку — создайте корректировочный акт.

**Как КС-2 связан с КС-3?**
КС-3 — это итоговая справка, которая собирает суммы из одного или нескольких КС-2 за период. Один КС-3 может включать несколько КС-2 (например, все акты за квартал).

**Как выставить счёт на основании КС-2?**
После подписания КС-2 создайте [счёт](invoices.md), привязанный к тому же договору. Сумма счёта = сумма КС-2 (или КС-3, если актов несколько).

**Что такое проверка объёмов?**
Это сравнение: сколько заявлено в КС-2 vs сколько заложено в смете. Если объём в акте превышает сметный — система покажет красным. Это защита от переплаты.

**Кто подписывает КС-2?**
Две стороны: подрядчик (кто выполнил работы) и заказчик (кто принял). В системе это отражается в маршруте согласования: подрядчик → техконтроль → бухгалтерия → заказчик.
