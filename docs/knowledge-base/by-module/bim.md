# BIM

---

## Что это и зачем

**BIM** (Building Information Modeling) — это **цифровая модель здания**. Не просто 3D-картинка, а «умная» модель, где каждый элемент (стена, колонна, труба) содержит информацию: материал, размеры, стоимость, кто проектировал, когда монтировать.

Без BIM:
- Проектировщики чертят в 2D — коллизии (пересечения труб и конструкций) находят только на стройке
- Переделки стоят в 10 раз дороже, чем исправления на этапе проектирования
- Архитектор, конструктор и инженер работают в разных программах — их проекты не стыкуются
- Заказчику показывают плоские чертежи — он не понимает, что получит

Модуль BIM в системе ПРИВОД:
- Хранение и просмотр 3D-моделей (IFC, Revit, ArchiCAD)
- Автоматическое обнаружение коллизий (пересечений)
- Проектные разделы (архитектура, конструкции, инженерные сети)
- BCF-задачи — замечания, привязанные к точке модели
- 4D-планирование — связь модели с графиком строительства
- Тепловая карта дефектов — где на объекте больше всего проблем

## Часть 1: Модели

BIM-модель — это файл, содержащий 3D-геометрию и данные обо всех элементах здания.

### Поддерживаемые форматы

| Формат | Программа | Описание |
|--------|-----------|----------|
| **IFC** | Универсальный | Открытый стандарт обмена BIM-данными |
| **RVT** | Autodesk Revit | Самая популярная BIM-программа |
| **DWG** | AutoCAD | 2D/3D чертежи |
| **PLN** | ArchiCAD | Модели Graphisoft ArchiCAD |
| **NWD** | Navisworks | Сводные модели |
| **SKP** | SketchUp | 3D-эскизы |

### Статусы модели

| Статус | Описание |
|--------|----------|
| **Загружена** (UPLOADED) | Файл загружен, ещё не обработан |
| **Обрабатывается** (PROCESSING) | Система анализирует модель |
| **Активна** (ACTIVE) | Модель готова к работе |
| **Устарела** (OUTDATED) | Появилась новая версия |
| **Архивная** (ARCHIVED) | Сохранена для истории |

### Версии

Каждая модель может иметь несколько **версий**. Когда проектировщик обновляет модель — загружается новая версия, старая становится «Устарела». Можно сравнить версии и увидеть, что изменилось.

## Часть 2: Проектные разделы

**Путь:** BIM → **Проектные разделы**

Проектная документация состоит из **разделов** (дисциплин):

| Раздел | Код | Что содержит |
|--------|-----|-------------|
| **Архитектура** | АР | Планировки, фасады, разрезы |
| **Конструкции** | КР | Фундаменты, колонны, перекрытия |
| **Отопление и вентиляция** | ОВ | Системы отопления, вентиляции, кондиционирования |
| **Водоснабжение** | ВК | Водопровод и канализация |
| **Электрика** | ЭО | Электроснабжение, освещение |
| **Слаботочные системы** | СС | Связь, видеонаблюдение, пожарная сигнализация |
| **Генплан** | ГП | Благоустройство, дороги, сети |

### Статусы раздела

| Статус | Описание |
|--------|----------|
| **В работе** (IN_PROGRESS) | Проектировщик разрабатывает |
| **На проверке** (REVIEW) | ГИП проверяет раздел |
| **Утверждён** (APPROVED) | Раздел согласован, можно строить |
| **Отклонён** (REJECTED) | Нужна доработка |

Каждый раздел привязан к **BIM-модели** (или нескольким), имеет ответственного проектировщика и срок разработки.

## Часть 3: Обнаружение коллизий

**Путь:** BIM → **Обнаружение коллизий**

**Коллизия** — это пересечение элементов из разных разделов. Например: труба отопления проходит через несущую колонну. На чертеже это не видно, а в BIM-модели — обнаруживается автоматически.

### Серьёзность коллизий

| Уровень | Цвет | Описание | Пример |
|---------|------|----------|--------|
| **Критическая** | Красный | Невозможно построить | Труба проходит через колонну |
| **Серьёзная** | Оранжевый | Требует переделки | Воздуховод слишком близко к кабелю |
| **Незначительная** | Жёлтый | Можно решить на месте | Мелкое пересечение кабельных лотков |
| **Информационная** | Серый | Для сведения | Элементы касаются, но не пересекаются |

### Статусы коллизии

| Статус | Описание |
|--------|----------|
| **Обнаружена** (NEW) | Система нашла пересечение |
| **В работе** (ACTIVE) | Назначен ответственный, исправляет |
| **Исправлена** (RESOLVED) | Проектировщик устранил |
| **Проверена** (REVIEWED) | ГИП подтвердил исправление |
| **Допустима** (APPROVED) | Пересечение допустимо (не нужно исправлять) |

### Проверка коллизий

Система проверяет пересечения **между разделами**:
- Архитектура × Конструкции
- Конструкции × Отопление/вентиляция
- Электрика × Водоснабжение
- И все остальные комбинации

Результат: список коллизий с точными координатами в модели.

## Часть 4: BCF-задачи

**Путь:** BIM → **BCF-задачи**

**BCF** (BIM Collaboration Format) — стандарт для обмена замечаниями к BIM-модели. Каждая задача привязана к **точке в модели** — с видом камеры, чтобы было понятно, о чём речь.

### Что содержит BCF-задача

| Поле | Описание |
|------|----------|
| **Тема** | Суть замечания |
| **Описание** | Подробности |
| **Приоритет** | Критический / Высокий / Средний / Низкий |
| **Статус** | Открыт / В работе / Решён / Закрыт |
| **Автор** | Кто создал замечание |
| **Ответственный** | Кто должен исправить |
| **Скриншот** | Вид модели с указанием проблемы |
| **Координаты** | Точка в 3D-модели |

### Связь с другими модулями

BCF-задачу можно связать с:
- **Дефектом** (модуль Качество) — если обнаружено на стройке
- **RFI** (запрос информации) — если нужно уточнение от проектировщика
- **Изменением** (модуль Изменения) — если требуется изменение проекта

## Часть 5: 4D — ход строительства

**Путь:** BIM → **Ход строительства**

**4D BIM** — это 3D-модель + время. Каждый элемент модели привязан к работе из графика строительства.

### Что это даёт

- **Визуализация строительства** — можно «проиграть» строительство как видео: вот ставится фундамент, потом растут стены, потом крыша
- **Контроль прогресса** — на модели цветом показано: что уже построено (зелёный), что строится сейчас (жёлтый), что ещё не начато (серый), что отстаёт (красный)
- **Презентация для заказчика** — наглядно показать, на какой стадии объект

### Тепловая карта дефектов

На 3D-модели можно наложить **тепловую карту** — зоны, где зафиксировано больше всего дефектов или замечаний. Это помогает:
- Выявить проблемные участки
- Обратить внимание на качество работ в этих зонах
- Усилить контроль

## Типичные сценарии

### Сценарий 1: Проверка коллизий перед строительством

```
1. Все проектировщики загрузили свои разделы:
   → Архитектура (АР) — v3.0
   → Конструкции (КР) — v2.1
   → Отопление (ОВ) — v1.5
   → Электрика (ЭО) — v1.3

2. ГИП запускает проверку коллизий:
   → Найдено 47 коллизий:
     — 3 критические (красные)
     — 8 серьёзных (оранжевые)
     — 36 незначительных (жёлтые)

3. Критические:
   → Труба Ø200 проходит через колонну К-15
     (ОВ × КР) → назначена на инженера ОВ
   → Кабельный лоток совпадает с воздуховодом
     (ЭО × ОВ) → назначена на координатора

4. Через неделю:
   → Все 3 критические — исправлены ✅
   → 6 из 8 серьёзных — исправлены ✅
   → 2 серьёзные — в работе (срок: пятница)

5. После исправления всех коллизий:
   → Разделы утверждены ✅
   → Можно начинать строительство
```

### Сценарий 2: BCF-замечание на стройке

```
1. Инженер ПТО на объекте замечает:
   «Проём в стене не совпадает с проектом»

2. Открывает BIM-модель на планшете:
   → Находит стену
   → Создаёт BCF-задачу:
     — Тема: «Проём ПР-12 сдвинут на 150 мм вправо»
     — Скриншот: вид модели с проёмом
     — Приоритет: Высокий
     — Ответственный: проектировщик АР

3. Проектировщик получает уведомление:
   → Проверяет модель и чертежи
   → Ответ: «Ошибка в чертеже, правильный проём
     по оси 5. Обновляю раздел АР до v3.1»

4. Загружает новую версию модели → BCF закрыт ✅
```

### Сценарий 3: Показ заказчику хода строительства

```
1. Заказчик приезжает на еженедельную планёрку

2. Руководитель проекта открывает 4D-модель:
   → Включает анимацию строительства
   → Заказчик видит: 1–3 этажи — зелёные (построены)
   → 4 этаж — жёлтый (монтируется сейчас)
   → 5–9 этажи — серые (ещё не начаты)
   → Фасад — серый (по графику — октябрь)

3. Заказчик: «А где будет детская площадка?»
   → Руководитель вращает модель → показывает двор
   → «Вот тут, по генплану. Благоустройство — декабрь»

4. Заказчик доволен — видит реальный прогресс
   в понятном 3D-виде, а не таблицы с процентами
```

## Частые вопросы

**Нужен ли специальный компьютер для BIM?**
Для просмотра моделей в системе ПРИВОД — нет, модели открываются в браузере. Для создания моделей (в Revit, ArchiCAD) — нужен мощный компьютер с хорошей видеокартой.

**Что такое IFC?**
IFC (Industry Foundation Classes) — открытый международный стандарт для обмена BIM-данными. Файл .ifc можно открыть в любой BIM-программе. Это «общий язык» между разными программами.

**Обязательно ли использовать BIM?**
С 2022 года BIM обязателен для объектов, финансируемых из бюджета (Постановление Правительства РФ №331). Для коммерческих объектов — не обязателен, но настоятельно рекомендуется.

**Что делать, если проектировщик не работает в BIM?**
Можно загрузить 2D-чертежи (DWG). Проверка коллизий и 4D-планирование будут недоступны, но хранение и версионирование — будут работать.

**Сколько весит BIM-модель?**
От 50 МБ до 2 ГБ в зависимости от размера и детализации объекта. Система оптимизирует модели для просмотра в браузере.

**Что такое LOD (Level of Detail)?**
Уровень детализации модели. LOD 100 — концепция (примерные формы), LOD 200 — общий проект, LOD 300 — рабочая документация (точные размеры), LOD 400 — изготовление и монтаж, LOD 500 — как построено (as-built).
