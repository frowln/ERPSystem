# Сотрудники

---

## Что это и зачем

Сотрудник — это карточка человека, работающего в организации. Содержит всё: ФИО, должность, контакты, паспортные данные, СНИЛС, ИНН, дату приёма, оклад, сертификаты и привязку к проектам.

Карточка сотрудника — основа кадрового учёта. Пока человек не добавлен в систему, его нельзя:
- Назначить в [бригаду](crews.md) или на проект
- Включить в [табель](timesheets.md)
- Оформить [отпуск](leave.md) или больничный
- Выписать [кадровый приказ](hr-russian.md)

---

## Где найти

**Путь:** Главное меню → **Кадры** → **Сотрудники**

## Список сотрудников

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Табельный номер | Уникальный номер (EMP-00001) |
| ФИО | Полное имя (Фамилия Имя Отчество) |
| Должность | Текущая позиция |
| Отдел | Подразделение |
| Статус | Цветной бейдж: Активен / В отпуске / Отстранён / Уволен |
| Сертификаты | Количество действующих сертификатов |
| Дата приёма | Когда принят на работу |

**Фильтры:**
- **Поиск** — по ФИО, табельному номеру или должности
- **Статус** — вкладки: Все / Активные / В отпуске / Уволенные / Отстранённые

**Действия:**
- Нажмите на строку → карточка сотрудника
- Кнопка **«+ Новый сотрудник»** → форма создания

## Как добавить сотрудника

1. Перейдите **Кадры → Сотрудники**
2. Нажмите **«+ Новый сотрудник»**

3. Заполните форму:

### Личные данные

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Фамилия** | Да | До 255 символов |
| **Имя** | Да | До 255 символов |
| **Отчество** | Нет | До 255 символов |
| **Дата рождения** | Нет | Для расчёта возраста |
| **Email** | Нет | Рабочая электронная почта |
| **Телефон** | Нет | Контактный номер |

### Документы

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Паспорт** | Нет | Серия и номер (до 20 символов) |
| **ИНН** | Нет | Индивидуальный налоговый номер (до 12 символов) |
| **СНИЛС** | Нет | Страховой номер (до 14 символов) |

### Работа

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Должность** | Да | Например: «Прораб», «Электрик», «Инженер ПТО» (до 200 символов) |
| **Отдел** | Нет | Подразделение (например: «Строительство», «Финансы») |
| **Дата приёма** | Да | Дата начала работы |
| **Дата увольнения** | Нет | Заполняется только при увольнении |

### Оплата

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Часовая ставка** | Нет | Оплата за час (₽), например: 450.50 |
| **Месячный оклад** | Нет | Фиксированная месячная зарплата (₽), например: 75 000 |

### Примечания

| Поле | Описание |
|------|---------|
| **Примечание** | Свободный текст до 2000 символов — любая дополнительная информация |

4. Нажмите **«Создать»** — система автоматически присвоит табельный номер (EMP-00001 и далее).

> Сотрудник создаётся в статусе **«Активен»**.

---

## Статусы сотрудника

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Активен** | Зелёный | Работает, может назначаться на проекты |
| **В отпуске** | Синий | Временно отсутствует (отпуск, больничный). Вернётся |
| **Отстранён** | Жёлтый | Дисциплинарное отстранение. Нельзя назначать на проекты |
| **Уволен** | Красный | Трудовой договор расторгнут. Карточка остаётся для истории |

> При увольнении сотрудник остаётся в системе со статусом «Уволен». Все данные (табели, проекты, сертификаты) сохраняются для аудита.

## Карточка сотрудника (детальная страница)

При нажатии на строку в списке открывается полная карточка.

### Заголовок
- ФИО (крупно)
- Должность
- Статус (бейдж)
- Если есть истекающие сертификаты → красное предупреждение

### Вкладка «Информация»

**Личные данные:**
- ФИО, дата рождения, возраст
- Телефон, email
- Паспорт, ИНН, СНИЛС

**Работа:**
- Должность, отдел
- Дата приёма, стаж (рассчитывается автоматически: сегодня − дата приёма)
- Часовая ставка / месячный оклад

### Вкладка «Сертификаты»

Таблица всех профессиональных сертификатов:

| Название | Номер | Кем выдан | Дата выдачи | Действует до | Статус |
|----------|-------|----------|-------------|-------------|--------|
| Охрана труда (общий) | СТ-001 | ООО «Учцентр» | 01.03.2025 | 01.03.2026 | ⚠️ Истекает |
| Работы на высоте | ВР-045 | ГАСН | 15.06.2024 | 15.06.2027 | ✅ Действует |
| Электробезопасность | ЭБ-12 | Ростехнадзор | 10.01.2023 | 10.01.2024 | ❌ Истёк |

**Цвета статуса:**
- 🟢 **Действует** — до истечения > 90 дней
- 🟡 **Истекает** — до истечения ≤ 90 дней
- 🔴 **Истёк** — срок действия прошёл

**Кнопка «Добавить сертификат»** — форма:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Тип** | Да | Охрана труда (общий) / Работы на высоте / Электробезопасность / Пожарная безопасность / Медосмотр / Квалификация / Водительское удостоверение / Сварка / Прочее |
| **Название** | Да | Полное наименование (до 500 символов) |
| **Номер** | Нет | Номер сертификата |
| **Дата выдачи** | Да | Когда получен |
| **Действует до** | Нет | Если не указано — бессрочный |
| **Кем выдан** | Нет | Организация, выдавшая сертификат |

> **Важно:** Система проверяет сроки автоматически каждый день. Администратор получает оповещения об истекающих сертификатах.

### Вкладка «Проекты»

Список проектов, на которые назначен сотрудник:

| Проект | Роль | Дата начала | Дата окончания | Ставка |
|--------|------|------------|---------------|--------|
| Дом на Ленина, 15 | Прораб | 01.01.2026 | — (бессрочно) | 500 ₽/час |
| Офис на Мира, 8 | Консультант | 15.02.2026 | 30.04.2026 | 450 ₽/час |

### Вкладка «Часы»

Сводка по отработанному времени: часы за месяц, переработка, рабочие дни. Ссылки на [табели](timesheets.md).

## Роли и права доступа

Система ПРИВОД использует ролевую модель доступа. Роль определяет, что пользователь может видеть и делать.

### Роли системы

| Роль | Описание |
|------|---------|
| **ADMIN** | Полный доступ ко всему. Управление пользователями, настройками, всеми модулями |
| **MANAGER** | Руководитель проекта. Согласование, управление командой, бюджеты |
| **ENGINEER** | Инженер / специалист. Работа с документами, задачами, отчётами |
| **ACCOUNTANT** | Бухгалтер. Финансы, счета, оплаты, проводки |
| **VIEWER** | Наблюдатель. Только просмотр, без права изменений |

### Кто что может с сотрудниками

| Действие | ADMIN | MANAGER | ENGINEER | ACCOUNTANT | VIEWER |
|----------|:-----:|:-------:|:--------:|:----------:|:------:|
| Просмотр списка | ✓ | ✓ | ✓ | ✓ | ✓ |
| Создание | ✓ | ✓ | — | — | — |
| Редактирование | ✓ | ✓ | — | — | — |
| Удаление | ✓ | — | — | — | — |
| Добавление сертификатов | ✓ | ✓ | — | — | — |
| Назначение на проект | ✓ | ✓ | — | — | — |
| Утверждение табелей | ✓ | ✓ | — | ✓ | — |

> **Роль** — это роль в системе (ADMIN, MANAGER и т.д.), а **должность** — это позиция сотрудника (прораб, электрик). Это разные вещи: прораб может иметь роль ENGINEER в системе.

---

## Мягкое удаление

Сотрудники **никогда** не удаляются полностью. При удалении:
- Карточка помечается как удалённая
- Исчезает из списков и поиска
- Но все данные (табели, назначения, сертификаты) сохраняются
- Можно восстановить через администратора

## Частые вопросы

**Как уволить сотрудника?**
Откройте карточку → измените статус на «Уволен» → укажите дату увольнения. Также рекомендуется оформить [кадровый приказ](hr-russian.md) об увольнении.

**Можно ли назначить сотрудника на несколько проектов?**
Да. Сотрудник может одновременно работать на нескольких проектах с разными ролями и ставками.

**Как узнать, что сертификат истекает?**
В карточке сотрудника — вкладка «Сертификаты». Жёлтый статус = истекает в ближайшие 90 дней. Красный = уже истёк. Также есть общий дашборд сертификаций (**Кадры → Матрица сертификаций**).

**Что такое «стаж» и как он считается?**
Стаж = сегодня минус дата приёма. Считается автоматически и показывается в годах, месяцах и днях.

**Паспорт, ИНН, СНИЛС обязательны?**
Нет, в системе они необязательные. Но для кадрового учёта по ТК РФ их нужно иметь — рекомендуется заполнять.

**Где посмотреть всех сотрудников на проекте?**
Двумя способами: **Кадры → Бригады** (фильтр по проекту) или в карточке проекта — вкладка «Команда».

**Чем отличается часовая ставка от оклада?**
Часовая ставка используется для сдельной/почасовой оплаты. Оклад — для фиксированной месячной зарплаты. Можно указать оба — система использует нужное в зависимости от настроек расчёта зарплаты.
