# Управление стоимостью (Cost Management)

---

## Что это и зачем

Модуль «Управление стоимостью» — это продвинутый инструмент финансового контроля проекта. Если [бюджет](budgets.md) отвечает на вопрос «сколько у нас денег», то управление стоимостью отвечает на вопросы:

- **Сколько мы обязались потратить?** (обязательства/контракты)
- **Укладываемся ли мы в бюджет?** (статьи затрат + план/факт)
- **Сколько будет стоить проект к завершению?** (прогноз EAC)
- **Рентабелен ли проект?** (маржа, profit fade, риски)

Модуль состоит из 6 блоков:

| Блок | Что делает |
|------|-----------|
| **Статьи затрат** | Структура расходов (иерархия) |
| **Обязательства** | Контракты и заказы (сколько обязались заплатить) |
| **Бюджетные строки** | Детальный план/факт по каждой статье |
| **Прогноз (EVM)** | Earned Value — сколько будет стоить к завершению |
| **Прогноз Cash Flow** | Сценарии движения денег |
| **Рентабельность** | Маржа, profit fade, риск-уровни по портфелю |

## 1. Статьи затрат (Cost Codes)

**Путь:** Управление стоимостью → **Статьи затрат**

Статья затрат — это категория расходов с уникальным кодом. Статьи организованы в **иерархию до 4 уровней**.

### Пример иерархии

```
01 — Строительные работы (Уровень 1)
  01.01 — Земляные работы (Уровень 2)
    01.01.01 — Выемка грунта (Уровень 3)
    01.01.02 — Обратная засыпка (Уровень 3)
  01.02 — Бетонные работы (Уровень 2)
  01.03 — Кладочные работы (Уровень 2)
02 — Материалы (Уровень 1)
  02.01 — Кирпич (Уровень 2)
  02.02 — Цемент (Уровень 2)
03 — Механизмы (Уровень 1)
04 — Накладные расходы (Уровень 1)
```

### Колонки таблицы

| Колонка | Что показывает |
|---------|---------------|
| Код | Иерархический код (например, «01.02.01») |
| Наименование | Название статьи |
| Бюджет | Выделенная сумма |
| Обязательства | Сколько уже законтрактовано |
| Факт | Сколько фактически потрачено |
| Прогноз | Ожидаемая итоговая сумма |
| Отклонение | Разница бюджет − прогноз (зелёное — экономия, красное — перерасход) |
| Освоение | Прогресс-бар (% факта от бюджета) |

**Показатели (вверху):**
- Общий бюджет
- Общие обязательства
- Общий факт
- Количество статей с перерасходом

## 2. Обязательства (Commitments)

**Путь:** Управление стоимостью → **Обязательства**

Обязательство — это контракт или заказ, по которому вы **обязались заплатить** определённую сумму. Пока обязательство не закрыто — деньги «зарезервированы».

### Типы обязательств

| Тип | Описание |
|-----|----------|
| **Заказ на закупку** | Заказ материалов у поставщика |
| **Субподряд** | Договор с субподрядчиком |
| **Профессиональные услуги** | Проектирование, консалтинг, экспертиза |

### Колонки таблицы

| Колонка | Что показывает |
|---------|---------------|
| Номер | Уникальный номер обязательства |
| Наименование | Описание + название контрагента |
| Статус | Черновик / Выпущен / Утверждён / Закрыт / Аннулирован |
| Тип | Закупка / Субподряд / Услуги |
| Скорр. сумма | Исходная сумма + утверждённые изменения |
| Выставлено | Сколько по обязательству уже выставлено счетов |
| Остаток | Скорр. сумма − выставлено |

### Показатели (вверху)

| Показатель | Что значит |
|-----------|-----------|
| Всего обязательств | Количество записей |
| Исходная сумма | Общая сумма при создании |
| Скорректированная | С учётом всех изменений (change orders) |
| Выставлено счетов | Сколько уже предъявлено к оплате |
| Остаток | Сколько ещё не выставлено |

### Статусы обязательств

```
Черновик → Выпущен → Утверждён → Закрыт
    ↓          ↓          ↓
 Аннулирован  Аннулирован  Аннулирован
```

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** | Серый | Создано, ещё не действует |
| **Выпущен** | Синий | Отправлено контрагенту |
| **Утверждён** | Синий | Контрагент подтвердил |
| **Закрыт** | Фиолетовый | Все работы/поставки выполнены |
| **Аннулирован** | Серый | Отменено (конечный статус) |

### Карточка обязательства

На детальной странице видно:
- **Финансовая сводка:** исходная сумма, утверждённые изменения, скорректированная сумма, выставлено, оплачено, прогресс (%)
- **Даты:** начало и окончание
- **Позиции:** таблица строк (описание, статья, кол-во, ед., цена, сумма)
- **Изменения (Change Orders):** история корректировок суммы

## 3. Бюджетные строки (Budget Lines)

Бюджетные строки — это детализация бюджета по статьям затрат.

**Что содержит каждая строка:**

| Поле | Описание |
|------|----------|
| Статья затрат | К какой статье относится |
| Описание | Что именно (например, «Кирпич М-150 для стен 1 этажа») |
| Исходный бюджет | Первоначально запланированная сумма |
| Утверждённые изменения | Корректировки (+ или −) |
| Скорр. бюджет | Авто: исходный + изменения |
| Обязательства | Сумма законтрактованного |
| Факт | Фактически потрачено |
| Прогноз | Ожидаемая итоговая стоимость |
| Отклонение | Авто: скорр. бюджет − прогноз |

**Сводные показатели** доступны через API:
- Итого исходный бюджет
- Итого скорректированный бюджет
- Итого факт
- Итого отклонение

## 4. Прогноз (Earned Value Management)

**Путь:** Управление стоимостью → **Прогноз**

EVM (Earned Value Management) — метод прогнозирования стоимости проекта к завершению. Используется на крупных стройках для раннего обнаружения перерасхода.

### Ключевые показатели

| Показатель | Сокращение | Что значит | Формула |
|-----------|-----------|-----------|---------|
| Бюджет при завершении | **BAC** | Сколько планировали потратить на весь проект | Из бюджета |
| Освоенный объём | **EV** | Сколько «заработано» по факту выполнения | % выполнения × BAC |
| Плановый объём | **PV** | Сколько должны были освоить к этой дате | Из графика |
| Фактические затраты | **AC** | Сколько реально потратили | Из оплат |
| Оценка при завершении | **EAC** | Прогноз: сколько будет стоить проект | BAC ÷ CPI |
| Оценка до завершения | **ETC** | Прогноз: сколько ещё нужно потратить | EAC − AC |
| Отклонение при завершении | **VAC** | Будем ли в бюджете? | BAC − EAC |

### Индексы эффективности

| Индекс | Формула | Что означает |
|--------|---------|-------------|
| **CPI** (затраты) | EV ÷ AC | > 1 = экономия, < 1 = перерасход |
| **SPI** (сроки) | EV ÷ PV | > 1 = опережаем, < 1 = отстаём |

**Пример:**
```
BAC = 10 000 000 руб. (бюджет)
EV  =  4 000 000 руб. (освоили 40%)
PV  =  5 000 000 руб. (по плану должны были освоить 50%)
AC  =  4 500 000 руб. (реально потратили)

CPI = 4 000 000 ÷ 4 500 000 = 0.89 → перерасход 11%
SPI = 4 000 000 ÷ 5 000 000 = 0.80 → отставание 20%
EAC = 10 000 000 ÷ 0.89 = 11 236 000 руб. → проект обойдётся дороже на 1 236 000
```

### Страница прогноза

**Показатели (4 карточки):**
- Бюджет (BAC)
- Прогноз EAC
- Отклонение VAC (со стрелкой: в бюджете или перерасход)
- Проблемные статьи (количество)

**Таблица:** Прогноз по статьям затрат — код, бюджет, прогноз, отклонение (цветом)

**Диаграмма:** Горизонтальные бары отклонений — красные (перерасход) и зелёные (экономия)

### Методы прогнозирования

| Метод | Описание |
|-------|----------|
| **Ручной** | Эксперт сам оценивает стоимость |
| **Освоенный объём** | Автоматически по CPI |
| **Тренд** | На основе динамики затрат за последние месяцы |
| **Регрессия** | Статистическая модель |

## 5. Cash Flow по затратам

**Путь:** Управление стоимостью → **Cash Flow по затратам**

Показывает движение денег с точки зрения **расходов** (не доходов/расходов как в общем Cash Flow, а именно затрат проекта).

**Показатели (4 карточки):**
- Плановые расходы
- Фактические расходы
- Прогнозные расходы
- Отклонение от плана

**Два графика:**

1. **Помесячные расходы** — 3 столбца за каждый месяц: план, факт, прогноз
2. **Кумулятивная S-кривая** — нарастающие линии: план, факт, прогноз

## 6. Сценарии прогноза Cash Flow

**Путь:** Управление стоимостью → **Прогноз Cash Flow**

Позволяет создавать **сценарии** — «что если»:

| Параметр сценария | Описание |
|------------------|----------|
| Название | Имя сценария (например, «Оптимистичный») |
| Горизонт | На сколько месяцев вперёд (по умолчанию 12) |
| Рост затрат | % роста затрат в месяц |
| Задержка оплат | Среднее время задержки оплат (дни) |
| Удержание | % удержания |
| Учитывать НДС | Да/Нет |

**Для каждого сценария** система генерирует таблицу:

| Колонка | Что показывает |
|---------|---------------|
| Период | Месяц |
| Прогноз доход | Ожидаемые поступления |
| Прогноз расход | Ожидаемые выплаты |
| Прогноз нетто | Доход − расход |
| Факт доход | Реальные поступления |
| Факт расход | Реальные выплаты |
| Отклонение | Прогноз − факт |
| Кумулятивный прогноз | Нарастающий итог |

## 7. Хаб прогнозирования

**Путь:** Управление стоимостью → **Хаб прогнозирования**

Единая панель со всеми прогнозными данными:

**Показатели портфеля:**
- Общий BAC, EV, AC
- CPI и SPI по всему портфелю
- Сравнение методов расчёта EAC (по CPI, по SPI+CPI, bottom-up, интегрированный)

**Распределение рисков:** Круговая диаграмма — сколько проектов в каждом уровне риска (низкий, средний, высокий, критический)

**Таблицы:**
- EVM-тренд по проектам: BAC, AC, EV, % выполнения, CPI, SPI
- Сравнение методов EAC по каждому проекту
- Сводка рентабельности: сумма контракта, маржа, % маржи, уровень риска

## 8. Рентабельность

**Путь:** Управление стоимостью → **Рентабельность**

Показывает, зарабатывает ли каждый проект деньги или теряет.

### Вкладки

| Вкладка | Что показывает |
|---------|---------------|
| **Все** | Все проекты |
| **В зоне риска** | Проекты с высоким или критическим риском |
| **Убыточные** | Проекты с отрицательной маржой |
| **Портфель** | Общая сводка по всем проектам |

### Колонки таблицы

| Колонка | Что показывает |
|---------|---------------|
| Проект | Название |
| Сумма контракта | Стоимость контракта с заказчиком |
| EAC | Прогноз итоговой стоимости |
| ETC | Сколько ещё нужно потратить |
| Прогноз маржи | Контракт − EAC (руб.) |
| % маржи | Маржа ÷ контракт × 100% |
| Profit fade | Потеря маржи относительно первоначальной оценки |
| % выполнения | Сколько проекта сделано |
| Уровень риска | Низкий / Средний / Высокий / Критический |

### Уровни риска

| Уровень | Цвет | Когда назначается |
|---------|------|------------------|
| **Низкий** | Зелёный | Маржа стабильна, CPI > 0.95 |
| **Средний** | Жёлтый | Есть небольшой перерасход или снижение маржи |
| **Высокий** | Оранжевый | Значительный перерасход, маржа падает |
| **Критический** | Красный | Проект убыточен или близок к убытку |

### Profit fade (потеря прибыли)

Profit fade — это разница между первоначально запланированной маржой и текущим прогнозом.

**Пример:**
```
Контракт:          10 000 000 руб.
Первонач. бюджет:   8 000 000 руб.  → Маржа: 2 000 000 (20%)
Текущий прогноз:    9 200 000 руб.  → Маржа: 800 000 (8%)

Profit fade = 2 000 000 − 800 000 = 1 200 000 руб. (12%)
```

Это значит: первоначально планировали заработать 2 млн, а по текущим прогнозам заработаем только 800 тыс. Потеря — 1.2 млн.

### Портфельная сводка

Вкладка «Портфель» показывает общую картину:
- Общая сумма контрактов
- Общая прогнозная маржа
- Средний % маржи
- Общий profit fade
- Количество проектов в зоне риска
- Количество убыточных проектов
- Круговая диаграмма распределения по уровням риска

## Частые вопросы

**Чем управление стоимостью отличается от бюджета?**
Бюджет — это план: «выделено 10 млн». Управление стоимостью — это контроль: «из 10 млн уже законтрактовано 7 млн, потрачено 4 млн, прогноз на завершение — 11 млн, перерасход 1 млн». Управление стоимостью даёт раннее предупреждение о проблемах.

**Что такое обязательство?**
Это юридическое обещание заплатить. Когда вы подписали договор с субподрядчиком на 5 млн — у вас появилось обязательство. Даже если ни одного счёта ещё не выставлено — деньги «заняты». Без учёта обязательств можно случайно раздать весь бюджет.

**Зачем нужен EVM?**
EVM позволяет ответить на вопрос: «Сколько будет стоить проект к завершению?». Если CPI = 0.85 — значит, на каждый запланированный рубль вы тратите 1.18 руб. При бюджете 10 млн это перерасход ~1.8 млн. Чем раньше вы это увидите — тем больше шансов скорректировать.

**Что означает CPI < 1?**
Перерасход. CPI = 0.9 значит: на каждый заработанный рубль вы потратили 1.11 руб. Нужно или сократить расходы, или пересмотреть бюджет.

**Что такое profit fade?**
Это разница между первоначально запланированной прибылью и текущим прогнозом. Если при подписании договора планировали маржу 20%, а сейчас прогноз показывает 8% — profit fade = 12%. Чем выше profit fade, тем серьёзнее ситуация.

**Кто может видеть данные управления стоимостью?**
Администратор, Руководитель проекта, Менеджер затрат (Cost Manager), Финансовый менеджер и Бухгалтер. Обычный инженер или просмотрщик не имеет доступа.

**Как создать статью затрат?**
Управление стоимостью → Статьи затрат → кнопка «Создать». Укажите код (уникальный), название, уровень (1–4) и родительскую статью (если это подстатья). Статьи создаются в привязке к конкретному проекту.
