# Реестр документов

---

## Что это и зачем

На любом строительном объекте — сотни документов: чертежи, сметы, договоры, акты, разрешения, протоколы, фотографии. Если документы хранятся в папках на рабочем столе, в мессенджерах и на флешках — найти нужный документ невозможно, а потерять — легко.

Модуль «Документы» в системе ПРИВОД — это **единое хранилище** всех документов организации:
- Загрузка файлов любого формата (PDF, чертежи, фото, Excel)
- Категоризация (договор, смета, чертёж, акт и т.д.)
- Согласование (черновик → на рассмотрении → утверждён)
- Версионность (хранится история изменений)
- Совместный доступ (кто видит, кто редактирует)
- Комментарии к документам
- Разметка на чертежах (пометки, замечания прямо на PDF)

Дополнительно система поддерживает **CDE** (Common Data Environment / Общая среда данных) — стандарт ISO 19650 для управления проектной документацией с ревизиями и трансмиталами.

## Часть 1: Реестр документов

### Категории документов

| Категория | Что входит | Примеры |
|-----------|-----------|---------|
| **Договор** | Контракты, дополнительные соглашения | Договор подряда №123, допсоглашение №2 |
| **Смета** | Сметная документация | Локальная смета на кладку, сводный сметный расчёт |
| **Спецификация** | Ведомости материалов и оборудования | Спецификация оборудования ОВ, ведомость арматуры |
| **Чертёж** | Проектные и рабочие чертежи | Чертёж КР-12, план 3 этажа, разрез 1-1 |
| **Разрешение** | Разрешительная документация | Разрешение на строительство, ордер ГАТИ |
| **Акт** | Акты выполненных работ, приёмки | Акт КС-2, акт скрытых работ |
| **Счёт** | Счета на оплату | Счёт №456 от поставщика |
| **Протокол** | Протоколы совещаний, испытаний | Протокол совещания от 01.03, протокол испытания бетона |
| **Корреспонденция** | Письма, уведомления | Письмо заказчику, уведомление о готовности |
| **Фото** | Фотофиксация | Фото хода работ, фото дефекта |
| **Отчёт** | Аналитические отчёты | Отчёт о ходе работ, финансовый отчёт |
| **Прочее** | Всё остальное | Инструкции, регламенты, графики |

### Как загрузить документ

1. Перейдите **Документы → Список документов**
2. Нажмите **«+ Новый документ»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Название** | Да | Название документа (до 500 символов) |
| **Номер документа** | Нет | Регистрационный номер (например: «ДП-2026-0045») |
| **Категория** | Да | Из списка выше (12 категорий) |
| **Проект** | Нет | К какому проекту относится |
| **Договор** | Нет | К какому договору (если применимо) |
| **Описание** | Нет | Пояснение к документу |
| **Раздел ПД** | Нет | Раздел проектной документации (АР, КР, ОВиК, ВК, ЭОМ, СС, ТХ, ГП, ПОС, ЭН) |
| **Теги** | Нет | Метки для быстрого поиска |
| **Срок действия** | Нет | До какой даты документ действителен |
| **Файл** | Рекомендуется | Загрузите файл (PDF, JPEG, XLSX, DWG и др.) |

4. Нажмите **«Создать»** — документ в статусе «Черновик»

### Статусы документа

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** (DRAFT) | Серый | Создан, можно редактировать |
| **На рассмотрении** (UNDER_REVIEW) | Синий | Отправлен на согласование |
| **Утверждён** (APPROVED) | Зелёный | Согласован ответственным лицом |
| **Действующий** (ACTIVE) | Зелёный | Документ в работе, используется |
| **В архиве** (ARCHIVED) | Серый | Устарел или заменён новой версией |
| **Отменён** (CANCELLED) | Красный | Отменён (ошибка, дубликат и т.п.) |

```
Черновик → На рассмотрении → Утверждён → Действующий → В архиве
     ↘                   ↘                                ↗
      Отменён ← (из любого статуса кроме архива)
```

### Версии документа

Когда документ обновляется (например, вышла новая редакция чертежа), система сохраняет историю:

- **Версия 1** — первоначальный файл
- **Версия 2** — исправленный файл (предыдущая версия сохраняется)
- **Версия 3** — ещё одно обновление

Все предыдущие версии доступны для просмотра. Это важно: если возник спор — можно посмотреть, что было раньше.

### Совместный доступ

К каждому документу можно настроить доступ:

| Уровень доступа | Что можно делать |
|-----------------|-----------------|
| **Просмотр** | Только смотреть и скачивать |
| **Редактирование** | Смотреть + загружать новые версии |
| **Полный доступ** | Всё, включая удаление и управление доступом |

### Комментарии

К любому документу можно оставить комментарий — это удобно для обсуждения:
- «Прошу проверить расчёт на стр. 5» — инженер
- «Расчёт проверен, замечаний нет» — ГИП
- «Утверждаю» — руководитель

### Разметка на чертежах

Для документов-чертежей (PDF, изображения) доступен инструмент разметки:
- **Точка** — поставить метку в конкретном месте
- **Линия** — провести линию (указать размер, направление)
- **Прямоугольник** — выделить область
- **Круг** — обвести проблемное место
- **Произвольная линия** — нарисовать от руки
- **Текст** — добавить текстовый комментарий

Каждая пометка имеет цвет, автора и статус (активная / решённая / в архиве). Это заменяет бумажные «красные линии» (redline) при проверке чертежей.

## Часть 2: CDE — Общая среда данных

**CDE** (Common Data Environment) — это расширенный модуль для управления проектной документацией по стандарту **ISO 19650**. Если обычный реестр документов — это «папка с файлами», то CDE — это полноценная система управления проектной информацией.

### Зачем CDE, если есть обычный реестр?

| Обычный реестр | CDE |
|---------------|-----|
| Один файл = один документ | Контейнер документа с множеством ревизий |
| Статусы: черновик → утверждён | Жизненный цикл по ISO 19650: WIP → Shared → Published → Archived |
| Нет формальной передачи | Трансмиталы: официальная передача с подтверждением |
| Для внутреннего использования | Для обмена между организациями (заказчик ↔ проектировщик ↔ подрядчик) |

CDE нужен на крупных объектах, где:
- Участвуют несколько организаций
- Требуется строгий контроль версий
- Заказчик требует ISO 19650
- Нужна аудиторская след: кто, когда, что передал

### Контейнер документа

В CDE каждый документ — это **контейнер**, который содержит ревизии (версии).

| Поле | Описание |
|------|----------|
| **Номер документа** | Уникальный код (например: «АР-01-001») |
| **Название** | Полное название |
| **Классификация** | Проектная / Строительная / Пусконаладка / Административная / Договорная / Нормативная |
| **Дисциплина** | Архитектура / Конструктивный / Инженерные сети / Генплан / Электрика / Сантехника / Пожарная безопасность |
| **Зона** | Зона/блок здания |
| **Этаж** | Уровень/этаж |
| **Метаданные** | Дополнительные поля (расширяемые) |

### Жизненный цикл по ISO 19650

| Состояние | Цвет | Описание |
|-----------|------|----------|
| **В работе** (WIP) | Жёлтый | Документ в разработке. Видит только автор и его команда |
| **Общий доступ** (SHARED) | Синий | Открыт для рассмотрения другими участниками |
| **Опубликован** (PUBLISHED) | Зелёный | Выпущен для производства работ. Это «рабочий» документ |
| **Архив** (ARCHIVED) | Серый | Заменён новой версией или устарел |

```
В работе → Общий доступ → Опубликован → Архив
```

**Важно:** Переход только вперёд. Нельзя вернуть опубликованный документ в «В работе» — нужно создать новую ревизию.

### Ревизии

Каждая ревизия — это конкретная версия файла внутри контейнера:

| Поле | Описание |
|------|----------|
| **Код ревизии** | Например: P01, P02, C01 (P = preliminary, C = construction) |
| **Статус** | Текущая / Заменена / Отозвана |
| **Файл** | Загруженный файл (PDF, DWG и т.д.) |
| **Описание** | Что изменилось в этой ревизии |
| **Кто загрузил** | Автор ревизии |
| **Кто утвердил** | Кто одобрил (если утверждена) |

### Трансмиталы

**Трансмитал** — это официальная передача документов от одной организации другой. Аналог сопроводительного письма с перечнем документов.

| Поле | Описание |
|------|----------|
| **Номер** | Уникальный номер (например: «TRN-2026-015») |
| **Тема** | О чём эта передача |
| **Назначение** | Для информации / Для рассмотрения / Для утверждения / Для производства работ / Исполнительная |
| **От кого** | Организация-отправитель |
| **Кому** | Организация-получатель |
| **Дата выпуска** | Когда отправлен |
| **Срок ответа** | До когда нужен ответ |
| **Состав** | Список документов (ревизий) в трансмитале |

### Статусы трансмитала

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Черновик** (DRAFT) | Серый | Формируется состав, ещё не отправлен |
| **Выпущен** (ISSUED) | Синий | Отправлен получателю |
| **Подтверждён** (ACKNOWLEDGED) | Зелёный | Получатель подтвердил получение |
| **С ответом** (RESPONDED) | Зелёный | Получатель дал ответ (замечания, согласование) |
| **Закрыт** (CLOSED) | Серый | Процесс завершён |

## Часть 3: Согласование документов (Workflow)

Система ПРИВОД поддерживает **настраиваемые маршруты согласования** документов. Это значит, что для каждого типа документа можно настроить цепочку: кто и в каком порядке должен согласовать.

### Как работает согласование

1. Документ отправляется на согласование
2. Система создаёт **цепочку согласования** — список шагов
3. На каждом шаге указан ответственный (роль или конкретный человек)
4. Ответственный видит документ во **Входящих согласованиях** и принимает решение:
   - **Утвердить** — документ переходит на следующий шаг
   - **Отклонить** — документ возвращается автору с комментарием
   - **Делегировать** — передать решение другому человеку
5. Когда все шаги пройдены — документ утверждён

### Пример цепочки согласования договора

```
Шаг 1: Юрист → проверяет условия → Утверждено ✅
Шаг 2: Финансовый директор → проверяет суммы → Утверждено ✅
Шаг 3: Генеральный директор → финальное утверждение → Утверждено ✅

Результат: Договор согласован
```

### Входящие согласования

**Путь:** Главное меню → **Бизнес-процессы** → **Входящие согласования**

Это личный «почтовый ящик» — здесь видны все документы, которые ждут вашего решения:

- Название документа
- Тип (договор, счёт, заявка и т.д.)
- Текущий шаг
- Срок (SLA) — до когда нужно принять решение
- Признак просрочки (красный индикатор)

Можно утверждать документы по одному или пакетно (выбрать несколько и утвердить разом).

### SLA и эскалация

Для каждого шага согласования можно установить **SLA** — максимальное время на решение (в часах). Если срок истёк, а решение не принято:
- Документ помечается как **просроченный**
- Может быть **эскалирован** — передан вышестоящему руководителю

## Типичные сценарии

### Сценарий 1: Загрузка и согласование чертежа

```
1. Проектировщик загружает чертёж КР-15 (план армирования 3 этажа)
   → Категория: Чертёж
   → Раздел ПД: КР (конструктивный)
   → Проект: Жилой дом по ул. Ленина
   → Статус: Черновик

2. Отправляет на рассмотрение → статус: На рассмотрении
   → Цепочка: ГИП → Руководитель проекта

3. ГИП открывает чертёж, ставит пометку на узле У-3:
   → «Несовпадение армирования — проверить расчёт»
   → Решение: Отклонено

4. Проектировщик исправляет, загружает версию 2
   → Повторная отправка на рассмотрение

5. ГИП проверяет → Утверждено ✅
   → Руководитель проекта → Утверждено ✅
   → Статус: Утверждён → Действующий
```

### Сценарий 2: Передача документации через трансмитал

```
1. Проектировщик подготовил комплект чертежей раздела КР:
   → 15 чертежей, ревизия P02

2. Создаёт трансмитал:
   → Номер: TRN-2026-023
   → Тема: «Раздел КР, ревизия P02 — для производства работ»
   → Назначение: Для производства работ
   → От: ООО «ПроектСтрой»
   → Кому: ООО «ГенПодряд»
   → Добавляет 15 документов

3. Выпускает трансмитал → статус: Выпущен
   → Подрядчик получает уведомление

4. Подрядчик подтверждает получение → статус: Подтверждён
   → В системе зафиксировано: кто, когда получил

5. При необходимости подрядчик оставляет замечания
   → статус: С ответом
   → Проектировщик видит замечания
```

### Сценарий 3: Поиск документа при проверке

```
1. Инспектор запрашивает: «Покажите разрешение на строительство»

2. Инженер открывает Документы → фильтр:
   → Категория: Разрешение
   → Проект: Жилой дом

3. Находит: «Разрешение на строительство №RNS-2025-0234»
   → Статус: Действующий ✅
   → Срок действия: до 31.12.2027 ✅
   → Версия 1 (без изменений)

4. Скачивает PDF и предъявляет инспектору ✅
```

## Частые вопросы

**Какие форматы файлов можно загружать?**
Любые: PDF, JPEG, PNG, XLSX, DOCX, DWG, IFC и другие. Система хранит файлы в защищённом хранилище (MinIO).

**Можно ли восстановить удалённый документ?**
Документы не удаляются физически — используется «мягкое удаление». Администратор может восстановить.

**Кто видит мои документы?**
По умолчанию — все сотрудники организации. Для ограничения доступа настройте права (Просмотр / Редактирование / Полный доступ).

**Зачем нужна CDE, если есть обычный реестр?**
CDE — для работы с проектной документацией по стандарту ISO 19650. Нужен на крупных объектах с несколькими организациями. Для внутренних документов достаточно обычного реестра.

**Что такое трансмитал простыми словами?**
Это «сопроводительное письмо» с пакетом документов. Вы отправляете комплект чертежей подрядчику, а система фиксирует: что отправили, когда, кому. Получатель подтверждает получение.

**Как найти нужный документ?**
Используйте поиск (по названию, номеру, имени файла), фильтры (по категории, статусу, проекту) или теги.

**Сколько версий документа хранится?**
Все. Каждая загрузка нового файла — новая версия. Предыдущие версии не удаляются.
