# Подбор персонала

---

## Что это и зачем

**Подбор персонала** — модуль для управления вакансиями и кандидатами. От публикации вакансии до приёма на работу — весь процесс в одном месте.

В строительстве постоянно нужны люди: бригады на новые объекты, замена уволившихся, сезонное расширение. Без системы подбора — хаос в Excel и потерянные резюме.

---

## Где найти

**Путь:** Главное меню → **Подбор персонала**

| Страница | URL | Что делает |
|----------|-----|-----------|
| **Кандидаты** | /recruitment/applicants | Список всех кандидатов |
| **Доска кандидатов** | /recruitment/applicants/board | Канбан по этапам отбора |
| **Вакансии** | /recruitment/jobs | Открытые вакансии |

---

## Часть 1: Вакансии

| Поле | Описание |
|------|----------|
| **Должность** | Название (прораб, электрик, сварщик 5 разряда) |
| **Отдел** | К какому подразделению |
| **Проект** | На какой объект нужен (если конкретный) |
| **Количество** | Сколько человек нужно |
| **Ставка** | Оклад или почасовая, диапазон (от–до) |
| **Требования** | Опыт, сертификаты, допуски |
| **Срочность** | Обычная / Срочная / Критическая |
| **Статус** | Открыта → В работе → Закрыта / Отменена |

---

## Часть 2: Кандидаты

### Карточка кандидата

| Поле | Описание |
|------|----------|
| **ФИО** | Полное имя |
| **Телефон / Email** | Контакты |
| **Вакансия** | На какую вакансию претендует |
| **Источник** | Откуда пришёл (HeadHunter, рекомендация, прямой контакт) |
| **Резюме** | Загруженный файл |
| **Ожидаемая ЗП** | Желаемая зарплата |
| **Комментарии** | Заметки рекрутера |

### Воронка подбора (канбан-доска)

| Этап | Описание |
|------|----------|
| **Новый** | Резюме получено, ещё не рассмотрено |
| **Скрининг** | Первичная проверка (опыт, сертификаты подходят?) |
| **Собеседование** | Назначена встреча |
| **Тестовое задание** | Практическая проверка (для инженеров, сметчиков) |
| **Предложение** | Отправлен оффер |
| **Принят** | Кандидат согласился → создаётся карточка сотрудника |
| **Отклонён** | Не подошёл (с указанием причины) |

---

## Часть 3: Приём на работу

Когда кандидат принят:
1. Нажмите **«Принять на работу»** в карточке кандидата
2. Система создаст карточку сотрудника с заполненными данными
3. Далее — стандартный процесс: документы → инструктаж → бригада

→ Подробнее: [Сотрудник от приёма до увольнения](../workflows/hr-lifecycle.md)

## Частые вопросы

**Можно ли создать сотрудника без модуля подбора?**
Да. Модуль подбора — для систематизации. Сотрудника можно создать напрямую в Кадры → Сотрудники.

**Как отследить, сколько людей нужно на проект?**
В вакансиях укажите проект. Потом фильтруйте по проекту — увидите все открытые позиции и их статус.
