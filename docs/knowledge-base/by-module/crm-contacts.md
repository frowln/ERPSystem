# Контрагенты

---

## Что это и зачем

**Контрагент** — это любая организация, с которой вы работаете: заказчик, подрядчик, поставщик, проектировщик. Это ваш «справочник компаний» со всеми реквизитами, контактами и историей.

Без единого справочника контрагентов:
- Менеджер вбивает реквизиты вручную при каждом новом договоре
- ИНН или расчётный счёт введены с ошибкой — оплата не проходит
- Кто-то уже работал с этой компанией, но вы об этом не знаете
- Подрядчик сменил банк, а у вас старые реквизиты — платёж ушёл не туда

Модуль «Контрагенты» в системе ПРИВОД — это:
- Единый справочник всех организаций
- Автозаполнение реквизитов по ИНН (через DaData)
- Типизация: заказчик, подрядчик, поставщик, субподрядчик, проектировщик
- Связь с контрактами, счетами, проектами

## Часть 1: Типы контрагентов

Каждый контрагент может иметь **одну или несколько ролей** одновременно:

| Тип | Описание | Пример |
|-----|----------|--------|
| **Заказчик** | Кто заказывает вам работу и платит | ООО «Застройщик Юг» |
| **Подрядчик** | Кто выполняет работы для вас | ООО «МонтажСтрой» |
| **Субподрядчик** | Специализированный подрядчик | ООО «ЭлектроСервис» (электрика) |
| **Поставщик** | Кто продаёт вам материалы и оборудование | ООО «СтройМатериалы» |
| **Проектировщик** | Кто делает проектную документацию | ООО «ПроектБюро» |

> Одна организация может быть одновременно, например, и подрядчиком, и поставщиком — если она и выполняет работы, и поставляет материалы.

## Часть 2: Создание контрагента

**Путь:** Контрагенты → **«+ Новый контрагент»**

### Автозаполнение по ИНН

Самый быстрый способ создать контрагента:
1. Введите **ИНН** организации
2. Система автоматически найдёт компанию и заполнит все реквизиты: название, КПП, ОГРН, юридический адрес

> Это работает через сервис DaData — он подтягивает данные из ЕГРЮЛ (Единого государственного реестра юридических лиц).

### Реквизиты организации

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Полное название** | Да | Например: ООО «Строительная компания "ЮгСтрой"» |
| **Краткое название** | Нет | Например: ЮгСтрой |
| **ИНН** | Да | 10 цифр (юрлицо) или 12 (ИП) |
| **КПП** | Нет | Код причины постановки на учёт (9 цифр) |
| **ОГРН** | Нет | Основной государственный регистрационный номер (15 цифр) |
| **Юридический адрес** | Нет | Адрес по учредительным документам |
| **Фактический адрес** | Нет | Где реально находится офис |

### Банковские реквизиты

| Поле | Описание |
|------|----------|
| **Расчётный счёт** | Номер счёта (20 цифр) |
| **БИК** | Банковский идентификационный код (9 цифр) |
| **Корреспондентский счёт** | Корсчёт банка (20 цифр) |
| **Название банка** | Например: ПАО «Сбербанк» |

### Контактная информация

| Поле | Описание |
|------|----------|
| **Контактное лицо** | ФИО ответственного человека |
| **Телефон** | Номер для связи |
| **Email** | Электронная почта |
| **Сайт** | Адрес веб-сайта |

### Типы (галочки)

| Галочка | Что означает |
|---------|-------------|
| ☑ Заказчик | Эта организация — ваш клиент |
| ☑ Поставщик | Поставляет вам материалы |
| ☑ Подрядчик | Выполняет для вас работы |
| ☑ Субподрядчик | Специализированный подрядчик |
| ☑ Проектировщик | Проектная/изыскательская организация |

## Часть 4: Карточка контрагента

На карточке контрагента видна вся информация:

### Основная информация
- Все реквизиты (ИНН, КПП, ОГРН, адреса)
- Банковские реквизиты
- Контакты
- Типы (заказчик/подрядчик/поставщик...)
- Примечания

### Связанные данные

Карточка показывает, где этот контрагент встречается в системе:
- **Контракты** — какие договоры заключены с этой организацией
- **Счета** — какие счета выставлены или получены
- **Проекты** — в каких проектах участвует
- **Лиды CRM** — связанные лиды

---

## Часть 5: Статусы

| Статус | Что означает |
|--------|-------------|
| **Активен** | Организация работает, можно заключать договоры |
| **Неактивен** | Организация деактивирована (ликвидирована, прекратили сотрудничество) |

Деактивированный контрагент остаётся в системе (для истории), но не отображается в списках выбора при создании новых договоров.

## Типичные сценарии

### Сценарий 1: Добавление нового подрядчика

```
1. Нашли подрядчика на электромонтаж — ООО «ЭлектроСервис»

2. Создаём контрагента:
   → Вводим ИНН: 7707083893
   → Система автоматически заполняет:
     — Название: ООО «ЭлектроСервис»
     — КПП: 770701001
     — ОГРН: 1027700132195
     — Юридический адрес: г. Москва, ул. Ленина, 15

3. Дополняем вручную:
   → Банковские реквизиты (расчётный счёт, БИК)
   → Контактное лицо: Иванов С.П., тел. +7-985-123-4567
   → Тип: ☑ Субподрядчик

4. Сохраняем → контрагент в системе

5. Теперь при создании контракта на электромонтаж:
   → Выбираем контрагента из справочника
   → Реквизиты подставляются автоматически
```

### Сценарий 2: Изменение банковских реквизитов

```
1. Подрядчик ООО «МонтажСтрой» сменил банк

2. Бухгалтер открывает карточку контрагента
   → Редактирование → Банковские реквизиты:
     — Новый расчётный счёт: 40702810...
     — Новый БИК: 044525974
     — Новый банк: АО «Альфа-Банк»

3. Сохраняет → все новые платежи пойдут по новым реквизитам

4. Старые платежи в истории сохранены с прежними реквизитами
```

## Частые вопросы

**Что такое ИНН?**
Идентификационный номер налогоплательщика. 10 цифр для юрлица, 12 для ИП. По ИНН можно однозначно определить организацию.

**Что такое КПП?**
Код причины постановки на учёт в налоговой. 9 цифр. У ИП нет КПП, только у юрлиц.

**Что такое ОГРН?**
Основной государственный регистрационный номер. 13 цифр для юрлица, 15 для ИП. Подтверждает, что организация зарегистрирована.

**Как работает автозаполнение по ИНН?**
Система обращается к сервису DaData, который берёт данные из ЕГРЮЛ (реестр Федеральной налоговой службы). Данные актуальны и верифицированы.

**Можно ли удалить контрагента?**
Нет, только деактивировать. Это сделано специально: если с контрагентом есть контракты и платежи, удаление нарушит целостность данных. Деактивированный контрагент остаётся для истории.

**Может ли одна организация быть и заказчиком, и подрядчиком?**
Да. Типы — это флаги (галочки), и можно отметить несколько одновременно. Например, генподрядчик может быть вашим заказчиком на одном проекте и подрядчиком на другом.
