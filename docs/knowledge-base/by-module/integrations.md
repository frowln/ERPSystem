# Интеграции

---

## Что это и зачем

**Интеграции** — это связь системы ПРИВОД с другими программами: 1С, ИСУП, Telegram, электронный документооборот (ЭДО) и другие.

Без интеграций:
- Бухгалтер вручную переносит данные из ПРИВОД в 1С — ошибки, потеря времени
- КС-2 формируются в ПРИВОД, а потом заново вбиваются в 1С — двойная работа
- Уведомления о задачах приходят только в систему — а сотрудники сидят в Telegram
- Отчёты в ИСУП Минстроя отправляются вручную — рутина и риск пропустить срок

Модуль «Интеграции» в системе ПРИВОД:
- **1С:Бухгалтерия** — двусторонний обмен: КС-2, КС-3, платежи, контрагенты, номенклатура
- **ИСУП** — автоматическая передача данных в информационную систему Минстроя
- **Telegram** — уведомления и команды через бот
- **СБИС / Диадок** — электронный документооборот с контрагентами
- **Погода** — прогноз погоды на объектах
- **SMS** — отправка уведомлений по SMS
- **WebDAV** — подключение сетевых папок
- **API-ключи** — интеграция с любыми внешними системами

## Часть 1: Интеграция с 1С

**Путь:** Интеграции → **1С**

Это **главная интеграция** для строительных компаний. Система ПРИВОД ведёт производственный учёт, а 1С — бухгалтерский. Обмен данными между ними избавляет от двойного ввода.

### Что передаётся в 1С

| Данные | Направление | Описание |
|--------|-------------|----------|
| **КС-2 (акты)** | ПРИВОД → 1С | Акты выполненных работ |
| **КС-3 (справки)** | ПРИВОД → 1С | Справки о стоимости работ |
| **Платежи** | ПРИВОД → 1С | Исходящие платежи подрядчикам |
| **Контрагенты** | ПРИВОД ↔ 1С | Синхронизация справочника организаций |
| **Номенклатура** | ПРИВОД ↔ 1С | Синхронизация справочника материалов |
| **Банковские выписки** | 1С → ПРИВОД | Входящие платежи от заказчиков |

### Экспорт КС-2 / КС-3 в 1С

1. В модуле «Закрытие» формируете КС-2
2. Нажимаете **«Выгрузить в 1С»**
3. Система формирует файл обмена (XML)
4. Файл загружается в 1С автоматически (или вручную)
5. В 1С появляется документ «Реализация» с теми же суммами

### Экспорт платежей

1. В модуле «Финансы» создаёте платёж подрядчику
2. Нажимаете **«Выгрузить в 1С»**
3. В 1С появляется платёжное поручение
4. Бухгалтер проверяет и отправляет в банк

### Импорт банковских выписок

1. В 1С загружается банковская выписка (ежедневно)
2. Нажимаете **«Импорт из 1С»** в ПРИВОД
3. Система подтягивает входящие платежи
4. Автоматически привязывает к счетам и проектам

### Синхронизация справочников

| Справочник | Как работает |
|-----------|-------------|
| **Контрагенты** | Новый контрагент в ПРИВОД → появляется в 1С, и наоборот |
| **Номенклатура** | Материалы и услуги синхронизируются между системами |

### Настройка подключения

| Параметр | Описание |
|----------|----------|
| **Адрес сервера 1С** | URL вашей базы 1С |
| **Имя базы** | Название информационной базы |
| **Логин / Пароль** | Учётные данные для обмена |
| **Расписание синхронизации** | Автоматически каждые N минут/часов |

## Часть 2: Интеграция с ИСУП

**Путь:** Интеграции → **ИСУП**

**ИСУП** — информационная система управления проектами Минстроя России. Передача данных в ИСУП обязательна для объектов с государственным финансированием.

### Типы передач

| Тип | Описание |
|-----|----------|
| **Ход строительства** | Процент готовности, фото, акты |
| **Отчёт о выполнении** | Объёмы работ за период |
| **Финансовый отчёт** | Освоение бюджета |
| **Исполнительная документация** | Акты скрытых работ, протоколы испытаний |
| **Фотоотчёт** | Фотографии хода работ |
| **Разрешительная документация** | Разрешения, лицензии, допуски |

### Статусы передачи

| Статус | Описание |
|--------|----------|
| **Черновик** (DRAFT) | Подготавливается, ещё не отправлена |
| **На проверке** (PENDING) | Проверяется перед отправкой |
| **Отправлена** (SENT) | Передана в ИСУП |
| **Принята** (ACCEPTED) | ИСУП принял данные |
| **Отклонена** (REJECTED) | Ошибки в данных, нужно исправить |
| **Ошибка** (ERROR) | Техническая ошибка при передаче |

### Как это работает

1. Данные собираются автоматически из модулей ПРИВОД
2. Формируется пакет в формате ИСУП (XML)
3. Вы проверяете данные и нажимаете **«Отправить»**
4. Система передаёт данные в ИСУП
5. Результат: «Принято» или «Отклонено» (с комментариями)

### Привязка проектов

Каждый проект в ПРИВОД можно привязать к объекту в ИСУП:
- Код объекта в ИСУП
- Номер государственного контракта
- Этап строительства

## Часть 3: Telegram-бот

**Путь:** Настройки → Интеграции → **Telegram**

### Что умеет бот

| Функция | Описание |
|---------|----------|
| **Уведомления** | Новые задачи, комментарии, изменения статусов |
| **Быстрые действия** | Принять задачу, отметить выполнение |
| **Запросы** | «Мои задачи», «Статус проекта» |
| **Фото** | Отправить фото с объекта — оно привяжется к задаче |

### Настройка

1. Создайте бот в Telegram (через @BotFather)
2. Введите **токен бота** в настройках интеграции
3. Каждый сотрудник привязывает свой Telegram-аккаунт
4. Готово — уведомления приходят в Telegram

### Какие уведомления приходят

- Новая задача назначена на вас
- Комментарий к вашей задаче
- Приближается срок задачи
- Задача просрочена
- Изменён статус проекта
- Требуется ваше согласование

## Часть 4: СБИС / ЭДО

Электронный документооборот — обмен документами с контрагентами **без бумаги**.

### Что передаётся через ЭДО

| Документ | Описание |
|----------|----------|
| **Счета-фактуры** | Выставление и получение |
| **Акты** | КС-2, акты сверки |
| **Договоры** | Подписание с ЭЦП |
| **Накладные** | Товарные накладные (ТОРГ-12) |
| **УПД** | Универсальный передаточный документ |

### Преимущества

- Документы доходят за **секунды** (не за дни по почте)
- **Юридическая значимость** — электронная подпись равна бумажной
- Не нужно хранить бумагу — всё в электронном архиве
- Автоматическая проверка — система не пропустит ошибку в реквизитах

## Часть 5: Другие интеграции

### Погода

Система автоматически загружает **прогноз погоды** для каждого объекта:
- Температура, осадки, ветер
- Предупреждения о неблагоприятных условиях
- Запись фактической погоды в журнал работ

Зачем: при температуре ниже −20°C бетонные работы запрещены. Система предупредит заранее.

### SMS-уведомления

Для срочных уведомлений — когда Telegram и email недостаточно:
- Аварийная ситуация на объекте
- Критическая просрочка
- Требуется срочное решение руководителя

### WebDAV

Подключение **сетевых папок** (NAS, облачное хранилище):
- Синхронизация документов
- Общий доступ к файлам проекта
- Резервное копирование

### Государственные реестры

Автоматическая проверка данных:
- **ЕГРЮЛ** — проверка контрагентов по ИНН
- **Реестр СРО** — проверка допусков подрядчиков
- **Реестр лицензий** — проверка действующих лицензий

### API-ключи

Для интеграции с **любыми внешними системами**:
- Создайте API-ключ в настройках
- Передайте ключ разработчику внешней системы
- Используйте REST API ПРИВОД для обмена данными

## Типичные сценарии

### Сценарий 1: Ежемесячная выгрузка КС-2 в 1С

```
1. Инженер ПТО сформировал КС-2 за март:
   → 5 актов по 5 подрядчикам
   → Общая сумма: 12 450 000 руб.

2. Нажимает «Выгрузить в 1С» для каждого акта:
   → Формируются XML-файлы
   → Автоматическая загрузка в 1С

3. Бухгалтер в 1С:
   → Видит 5 новых документов «Реализация»
   → Суммы, контрагенты, номенклатура — всё подтянулось
   → Проверяет → проводит

4. Экономия: 2 часа работы бухгалтера (вместо ручного ввода)
   + исключены ошибки при переносе сумм
```

### Сценарий 2: Передача данных в ИСУП

```
1. Конец месяца — нужно отчитаться в ИСУП:
   → Проект «Школа №15» — бюджетное финансирование

2. Система автоматически собирает:
   → Объём выполненных работ (из модуля Закрытие)
   → Освоение бюджета (из модуля Финансы)
   → Фотографии (из фотоотчётов)

3. Формируется пакет → инженер проверяет:
   → Всё корректно ✅
   → Нажимает «Отправить»

4. ИСУП:
   → Статус: «Принято» ✅
   → Данные зарегистрированы в системе Минстроя
```

### Сценарий 3: Telegram-уведомление

```
1. Руководитель проекта на совещании, не за компьютером

2. В это время в системе:
   → Подрядчик загрузил акт КС-2 на портале
   → Требуется согласование руководителя

3. В Telegram приходит сообщение от бота:
   «📋 Новый акт КС-2 ожидает согласования
    Проект: ЖК Южный парк
    Подрядчик: ООО МонтажСтрой
    Сумма: 2 350 000 руб.
    [Перейти к акту]»

4. Руководитель переходит по ссылке:
   → Проверяет акт → согласовывает
   → Не нужно ждать возвращения к компьютеру
```

## Частые вопросы

**Обязательно ли подключать 1С?**
Нет. ПРИВОД работает самостоятельно. Интеграция с 1С — для удобства, чтобы избежать двойного ввода данных.

**Какие версии 1С поддерживаются?**
1С:Бухгалтерия 3.0 и 1С:Управление строительной организацией. Обмен через стандартные форматы XML.

**ИСУП — это обязательно?**
Только для объектов с государственным финансированием. Для коммерческих объектов — не требуется.

**Безопасен ли Telegram-бот?**
Да. Бот не хранит данные. Каждый пользователь привязан к своему аккаунту. Конфиденциальная информация (суммы контрактов, персональные данные) в уведомления не включается.

**Можно ли подключить свою систему через API?**
Да. В настройках можно создать API-ключ и использовать REST API ПРИВОД. Документация API доступна по адресу /api/docs.

**Что делать, если передача в ИСУП отклонена?**
Посмотрите причину отклонения в статусе передачи. Обычно это ошибки в данных (неправильные коды, несоответствие форматов). Исправьте и отправьте повторно.
