# Проектные разделы и ревизии

---

## Что это и зачем

**Проектные разделы** — модуль для управления проектной документацией по разделам (архитектура, конструктив, инженерные сети) и контроля ревизий (версий) чертежей.

В строительном проекте десятки разделов и сотни чертежей. Каждый чертёж проходит несколько ревизий (изменений). Без системы контроля — путаница: «Какая версия актуальная? Кто утвердил? Когда изменили?»

## Часть 1: Разделы проекта

Стандартные разделы строительного проекта:

| Код | Раздел | Содержание |
|-----|--------|-----------|
| **АР** | Архитектурные решения | Планировки, фасады, разрезы |
| **КР** | Конструктивные решения | Фундаменты, каркас, перекрытия |
| **ОВ** | Отопление и вентиляция | Системы отопления, вентиляции, кондиционирования |
| **ВК** | Водоснабжение и канализация | Трубопроводы, сантехника |
| **ЭО** | Электрооборудование | Электроснабжение, освещение |
| **СС** | Слаботочные системы | Связь, видеонаблюдение, СКУД |
| **ПБ** | Пожарная безопасность | Сигнализация, пожаротушение |
| **ГП** | Генеральный план | Благоустройство, дороги |
| **ПОС** | Проект организации строительства | Стройгенплан, графики |

Каждый раздел содержит набор документов (чертежи, спецификации, расчёты).

---

## Часть 2: Ревизии (версии)

Каждый документ может иметь несколько ревизий:

| Ревизия | Дата | Описание изменений | Автор | Статус |
|---------|------|-------------------|-------|--------|
| Rev.0 | 01.02.2026 | Первоначальный выпуск | Архитектор | Архив |
| Rev.1 | 15.02.2026 | Изменена планировка 2 этажа | Архитектор | Архив |
| Rev.2 | 01.03.2026 | По замечаниям экспертизы | Архитектор | **Актуальная** |

> **Правило:** на объекте работают ТОЛЬКО по актуальной ревизии. Все предыдущие — в архиве.

---

## Часть 3: Рецензии

Процесс проверки проектной документации:

1. Проектировщик загружает новую ревизию
2. Назначает рецензентов (ГИП, заказчик, экспертиза)
3. Каждый рецензент оставляет замечания или утверждает
4. При замечаниях — проектировщик вносит правки → новая ревизия
5. При утверждении всеми → документ получает статус «Утверждён»

→ Подробнее о процессе: [Согласование документа](../workflows/document-approval.md)

## Частые вопросы

**Как узнать актуальную версию чертежа?**
Актуальная ревизия всегда помечена. Все предыдущие — в статусе «Архив». Система не позволит скачать устаревшую версию без предупреждения.

**Кто может загружать новые ревизии?**
Автор документа (проектировщик) или пользователь с правом редактирования раздела.
