# Бюджеты

---

## Что это и зачем

Бюджет — это финансовый план проекта. В нём записано: сколько денег выделено, на что запланированы расходы и сколько ожидаем получить от заказчика. Бюджет состоит из позиций — каждая позиция это конкретная статья затрат (материал, работа, оборудование).

Бюджет помогает не выйти за рамки: если подрядчик просит больше, чем заложено в бюджете — система покажет перерасход. Если заказчик платит меньше, чем планировали — видно отклонение.

В системе ПРИВОД бюджет и финансовая модель (ФМ) — это **один и тот же объект**, но с разными представлениями:
- **Бюджет** — вид «сверху»: плановые и фактические суммы, отклонения, статус
- **[Финансовая модель](financial-model.md)** — вид «изнутри»: три колонки цен (себестоимость, сметная, клиентская), маржа, НДС

---

## Где найти

**Путь:** Главное меню → **Финансы** → **Бюджеты**

## Список бюджетов

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Название | Имя бюджета (например, «Бюджет — ЖК Солнечный, корпус 1») |
| Объект | К какому проекту привязан |
| Статус | Черновик, утверждён, активный, заморожен, закрыт |
| План выручка | Сколько планируем получить от заказчика |
| План затраты | Сколько планируем потратить |
| Факт выручка | Сколько уже получили (с прогресс-баром) |
| Факт затраты | Сколько уже потратили (красный прогресс-бар, если > 100%) |
| Отклонение | Разница между планом и фактом в процентах |

**Вкладки-фильтры:**
- **Все** — все бюджеты
- **Черновик** — в разработке
- **Утверждён** — согласован руководством
- **Активный** — в работе, идёт учёт фактических затрат
- **Заморожен** — временно приостановлен
- **Закрыт** — проект завершён

**Поиск:** по названию или объекту.

## Как создать бюджет

### Автоматически (рекомендуется)

При создании нового проекта бюджет создаётся **автоматически**. Вам не нужно делать ничего дополнительно — откройте проект, перейдите во вкладку «Финансы» и увидите готовый бюджет.

### Вручную

1. Перейдите в **Финансы → Бюджеты**
2. Нажмите **«Новый бюджет»**
3. Заполните форму:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Название** | Да | Например: «Бюджет — Реконструкция цеха №3» |
| **Проект** | Нет | Выберите проект (можно создать бюджет без привязки к проекту) |
| **Договор** | Нет | Привязка к конкретному договору |
| **Примечания** | Нет | Свободный текст |

4. Нажмите **«Создать»**

Бюджет создаётся в статусе **«Черновик»** с настройками по умолчанию:
- Накладные расходы: 12%
- Непредвиденные расходы: 5%
- Временные сооружения: 3%

## Как добавить позиции

### Вручную

На странице бюджета (или финансовой модели) нажмите **«Добавить позицию»** и заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Наименование** | Да | Что это: «Арматура А500С d12», «Монтаж электрики» |
| **Категория** | Да | Тип: материалы, работы, оборудование, субподряд, накладные, прочее |
| **Плановая сумма** | Да | Сколько выделено на эту позицию (руб.) |
| **Количество** | Нет | Сколько единиц (15 тонн, 5000 метров) |
| **Единица измерения** | Нет | шт, т, м, м², п.м., комплект и т.д. |
| **Себестоимость за ед.** | Нет | Закупочная цена за единицу |
| **Цена заказчику за ед.** | Нет | Цена для клиента за единицу |

### Из спецификации

Если у проекта есть спецификация — нажмите кнопку **«Передать в ФМ»** на странице спецификации. Все позиции автоматически скопируются в бюджет с правильными наименованиями, количествами и единицами измерения.

Подробнее — в разделе [Спецификации](specifications.md).

### Из сметы (ЛСР)

Нажмите кнопку **«Импорт ЛСР»** на странице финансовой модели. Загрузите xlsx-файл сметы из ГРАНД-Сметы — позиции заполнят бюджет с нормативными ценами.

Подробнее — в разделе [Сметы](estimates.md).

### Разделы (группировка)

Позиции можно объединить в **разделы** — например, «Электрика», «Вентиляция», «Фундамент». Раздел — это заголовочная строка, под которой сгруппированы позиции. У каждого раздела автоматически считается итого по всем колонкам.

Нажмите на название раздела, чтобы свернуть/развернуть его.

## Страница бюджета (детальный вид)

### Карточки показателей (вверху)

| Показатель | Что значит |
|-----------|-----------|
| **Плановая выручка** | Сколько планируем получить от заказчика |
| **Плановая себестоимость** | Сколько планируем потратить |
| **Плановая маржа** | Разница: выручка − себестоимость (сколько зарабатываем) |
| **Фактическая выручка** | Сколько уже получили |
| **Фактическая себестоимость** | Сколько уже потратили |
| **Фактическая маржа** | Разница: факт выручки − факт затрат |

### Вкладки

| Вкладка | Что внутри |
|---------|-----------|
| **Обзор** | 6 карточек показателей |

| **График** | Столбчатая диаграмма: план vs факт vs зарезервировано |
| **Финансовая модель** | Ссылка на полную ФМ с тремя колонками цен |

## План vs факт — как вносить фактические затраты

### Откуда берётся «факт»

Фактические суммы заполняются **автоматически** из связанных документов:

| Источник | Что обновляет | Когда |
|----------|--------------|-------|
| **Договор с подрядчиком** | Зарезервировано (committedAmount) | При подписании договора |
| **Акт КС-2** | Акт подписан (actSignedAmount) | При приёмке работ |
| **Счёт от поставщика** | Выставлено (invoicedAmount) | При получении счёта |
| **Оплата** | Оплачено (paidAmount) | При фактической оплате |

Таким образом, вам **не нужно вводить факт вручную** — он подтягивается из документооборота.

### Стадии прохождения суммы

Для каждой позиции бюджета видно, на какой стадии находятся деньги:

```
Запланировано → Тендер → Договор → Акт подписан → Счёт выставлен → Оплачено
```

Это отражается в колонке **«Статус»** (цветная метка) и в полоске **CVR** (прогресс-бар):

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Запланировано** | Серый | Позиция в бюджете, закупка не начата |
| **Тендер** | Синий | Идёт выбор поставщика |
| **Договор** | Фиолетовый | Договор с поставщиком подписан |
| **Акт подписан** | Оранжевый | Работы приняты / товар получен |
| **Счёт выставлен** | Пурпурный | Поставщик прислал счёт |
| **Оплачено** | Зелёный | Деньги перечислены |

### Пересчёт фактических данных

Нажмите кнопку **«Пересчитать»** в шапке бюджета — система обновит все фактические суммы из связанных документов (договоров, актов, счетов, оплат).

## Отклонение: план − факт

Система автоматически рассчитывает отклонение — разницу между тем, что планировали, и тем, что фактически получилось.

### По выручке

```
Отклонение выручки = Фактическая выручка − Плановая выручка
```

- **Положительное** (зелёное) — получили больше, чем планировали. Хорошо
- **Отрицательное** (красное) — получили меньше. Нужно разобраться

### По затратам

```
Отклонение затрат = Фактические затраты − Плановые затраты
```

- **Отрицательное** (зелёное) — потратили меньше плана. Экономия
- **Положительное** (красное) — перерасход. Опасно

### По марже

```
Отклонение маржи = Фактическая маржа − Плановая маржа
```

### Отклонение в процентах

```
Отклонение % = (Отклонение / План) × 100%
```

В списке бюджетов отклонение показано в колонке **«Отклонение»** с цветовой индикацией:
- Зелёный — в рамках плана или лучше
- Красный — перерасход или недополучение

### Остаток по позиции

Для каждой позиции бюджета рассчитывается:

```
Остаток = Плановая сумма − Фактически потрачено − Зарезервировано
```

Если остаток отрицательный — по этой позиции перерасход.

## Лимиты: как бюджет ограничивает закупки

### Мягкий контроль (по умолчанию)

Система **не блокирует** закупки, если бюджет превышен, но **предупреждает** визуально:

- **Прогресс-бар затрат** становится красным, если факт > 100% от плана
- **Отклонение** отображается красным цветом
- **Маржа** меняет цвет при снижении ниже пороговых значений:
  - Красный — маржа < 0% (убыток)
  - Оранжевый — маржа 0–5% (опасная зона)
  - Жёлтый — маржа 5–15% (нормально)
  - Зелёный — маржа > 15% (хорошо)

### Строгий контроль (опция)

На странице финансовой модели можно включить **строгий режим валидации** (переключатель в шапке):
- В строгом режиме система **не позволяет** установить цену заказчику выше сметной стоимости
- Это защищает от случайного завышения цен

### Заморозка бюджета

Если нужно полностью запретить изменения — переведите бюджет в статус **«Заморожен»**. В этом статусе:
- Нельзя добавлять или удалять позиции
- Нельзя менять цены
- Можно только просматривать

Чтобы разморозить — верните в статус «Активный».

---

## Статусы бюджета

| Статус | Что означает | Что можно делать |
|--------|-------------|-----------------|
| **Черновик** | В разработке | Всё: добавлять, удалять, менять позиции и цены |
| **Утверждён** | Согласован руководством | Структура зафиксирована. Можно вернуть в черновик или активировать |
| **Активный** | В работе | Идёт учёт фактических затрат. Ограниченное редактирование |
| **Заморожен** | Приостановлен | Только просмотр. Можно вернуть в активный или закрыть |
| **Закрыт** | Завершён | Только просмотр. Финальный статус |

**Порядок переходов:**
```
Черновик ←→ Утверждён → Активный ←→ Заморожен
                              ↓              ↓
                          Закрыт ←───────────┘
```

> **Подсказка:** Утвердите бюджет перед началом работ — это зафиксирует базовый план, с которым потом будете сравнивать факт.

## Категории позиций

Каждая позиция бюджета имеет категорию:

| Категория | Что включает | Примеры |
|-----------|-------------|---------|
| **Материалы** | Строительные материалы | Арматура, бетон, кабель, трубы |
| **Работы** | Строительно-монтажные работы | Монтаж, сварка, прокладка, покраска |
| **Оборудование** | Техника и приборы | Трансформаторы, насосы, щиты |
| **Субподряд** | Работы подрядчиков | Монтаж вентиляции (подрядчик ООО «КлиматСервис») |
| **Накладные** | Косвенные затраты | Аренда офиса, связь, охрана, транспорт |
| **Прочее** | Остальное | Экспертизы, согласования, непредвиденные |

## Дополнительные инструменты

### Снимки (Snapshots)

Можно сохранить текущее состояние бюджета как «снимок» и потом сравнить с будущими версиями. Удобно, чтобы видеть, как менялись суммы с течением проекта.

- Кнопка **«Создать снимок»** на вкладке «Снимки»
- Выбрать два снимка для сравнения
- Система покажет, какие позиции изменились и на сколько

### График (план vs факт)

На вкладке **«График»** — столбчатая диаграмма:
- **Синие столбцы** — плановые суммы
- **Зелёные столбцы** — фактические суммы
- **Серые столбцы** — зарезервированные суммы

Наглядно видно, где перерасход, а где экономия.

### Привязка к графику (WBS)

Каждую позицию бюджета можно привязать к задаче из графика проекта. Это позволяет:
- Распределить стоимость по времени
- Построить **Cash Flow** (прогноз движения денежных средств)
- Видеть, когда какие расходы ожидаются

### Экспорт

Бюджет можно выгрузить в Excel (xlsx) или PDF для печати или отправки руководству.

### ROI

Система рассчитывает **ROI** (рентабельность инвестиций) — показатель того, сколько прибыли приносит каждый вложенный рубль.

### Сценарное моделирование

На странице ФМ доступен инструмент **«Сценарии»**: укажите целевую маржу (например, 20%) — система рассчитает, какие цены заказчику нужно установить для её достижения.

## Частые вопросы

**Когда создаётся бюджет?**
Автоматически при создании нового проекта. Также можно создать вручную: Финансы → Бюджеты → «Новый бюджет».

**Чем бюджет отличается от финансовой модели?**
Это один и тот же объект, но с разными представлениями. Бюджет — вид «сверху» (план vs факт, отклонения, статусы). ФМ — вид «изнутри» (три колонки цен, маржа по позициям, НДС). В списке бюджетов отображаются итоговые суммы, а в ФМ — детальная таблица с каждой позицией.

**Откуда берутся фактические суммы?**
Автоматически из документооборота: договоры → зарезервировано, акты КС-2 → принято, счета → выставлено, оплаты → оплачено. Вручную ничего вводить не нужно — система считает сама по мере оформления документов.

**Можно ли выйти за рамки бюджета?**
По умолчанию — да, система не блокирует, но предупреждает (красный цвет, негативное отклонение). Если нужна жёсткая блокировка — включите строгий режим валидации в ФМ.

**Что делать при перерасходе?**
Посмотрите, по каким позициям перерасход (красные значения в колонке «Отклонение»). Варианты: пересмотреть цены, найти более дешёвого поставщика, согласовать допсоглашение с заказчиком, урезать другие статьи.

**Можно ли изменить утверждённый бюджет?**
Можно вернуть его в статус «Черновик», внести изменения и утвердить заново. Но лучше использовать **снимки** — сохранить текущую версию и сравнивать с изменённой.

**Что такое CVR?**
Cost Value Reconciliation — отслеживание стоимости по стадиям. Для каждой позиции видно, сколько денег на стадии «договор», «акт», «счёт», «оплата». Это прогресс-бар, показывающий движение денег.

**Как бюджет связан с Cash Flow?**
Если позиции бюджета привязаны к задачам графика (WBS), система распределяет суммы по времени и строит прогноз Cash Flow — когда и сколько денег понадобится.
