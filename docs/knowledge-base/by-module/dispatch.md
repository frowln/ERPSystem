# Отгрузки и доставки

---

## Что это и зачем

Отгрузка (Dispatch) — это факт отправки материала поставщиком в наш адрес. Когда поставщик подтвердил [заказ](purchase-orders.md) и загружает машину — это начало отгрузки. Когда материал прибыл на склад — это завершение доставки.

Модуль отвечает на вопрос: **«Что сейчас в пути и когда будет?»**

Без отслеживания отгрузок склад не знает, когда готовить площадку для разгрузки, а прораб не знает, когда материалы появятся на объекте.

---

## Где найти

В системе есть два связанных раздела:

| Раздел | Путь | Для чего |
|--------|------|---------|
| **Отгрузки (Закупки)** | Закупки → Отгрузки | Отслеживание поставок от поставщиков |
| **Диспетчеризация (Операции)** | Операции → Диспетчеризация | Управление транспортом на объекте |

## Часть 1: Отгрузки от поставщиков

Когда поставщик отправляет материалы по [заказу на закупку](purchase-orders.md), создаётся запись об отгрузке.

### Как создать отгрузку

1. Откройте **подтверждённый** заказ поставщику (PO)
2. Нажмите **«Создать отгрузку»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Заказ (PO)** | Да | К какому заказу относится (подтянется автоматически) |
| **Маршрут** | Нет | Предопределённый маршрут (если есть) |
| **Транспорт** | Нет | Номер машины (например, «КАМАЗ А123ВС 77») |
| **Водитель** | Нет | ФИО водителя |
| **Планируемая отправка** | Да | Когда поставщик загрузит машину |
| **Планируемое прибытие** | Да | Когда ждать на складе |
| **Трек-номер** | Нет | Номер отслеживания (для транспортных компаний) |

4. Добавьте позиции — что именно везут:

| Поле | Описание |
|------|---------|
| Материал | Из позиций заказа |
| Количество | Сколько единиц в этой отгрузке |
| Ед. изм. | Единица измерения |
| Вес | Масса груза |

5. Нажмите **«Создать»** — отгрузка в статусе «Запланирована».

### Статусы отгрузки

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Запланирована** | Серый | Отгрузка создана, машина ещё не загружена |
| **Загрузка** | Жёлтый | Материалы грузятся на транспорт |
| **В пути** | Синий | Машина выехала, материалы в дороге |
| **Доставлена** | Зелёный | Материалы прибыли на склад |
| **Отменена** | Красный | Отгрузка не состоялась |

```
Запланирована → Загрузка → В пути → Доставлена
      ↓            ↓          ↓
   Отменена    Отменена   Отменена
```

### Жизненный цикл

**Шаг 1. Запланирована**
Поставщик сообщил, что готов отгрузить. Вы создали запись с датами и транспортом.

**Шаг 2. Загрузка**
Поставщик начал грузить машину. Нажмите **«Начать загрузку»**.

**Шаг 3. В пути**
Машина выехала от поставщика. Нажмите **«Отметить отправку»**. Система зафиксирует фактическое время отправления.

**Шаг 4. Доставлена**
Машина прибыла на склад. Нажмите **«Отметить доставку»**. Система зафиксирует фактическое время прибытия.

**Что происходит при доставке:**
- На [складе](warehouse.md) автоматически создаётся **приход** (движение типа RECEIPT)
- Остатки материалов обновляются
- Статус [заказа](purchase-orders.md) обновляется:
  - Если всё доставлено → «Доставлен»
  - Если частично → «Частично доставлен»

## Часть 2: Маршруты доставки

Маршрут — это типовой путь от поставщика до склада. Если вы часто заказываете у одного поставщика, создайте маршрут один раз — и используйте его при каждой отгрузке.

### Как создать маршрут

| Поле | Описание |
|------|---------|
| **Код** | Уникальный код (например, «МТ→ЦС-01») |
| **Название** | Описание маршрута (например, «ООО МеталлТрейд → Центральный склад») |
| **Откуда** | Адрес загрузки (склад поставщика) |
| **Куда** | Адрес выгрузки (наш склад) |
| **Расстояние** | Километры |
| **Время в пути** | Ожидаемое время (часы) |
| **Тип транспорта** | Грузовик / Фургон / Прицеп / Бортовой / Цистерна / Кран |
| **Активен** | Вкл/выкл |

**Пример:**
```
Код: ЗЖБ→ПЛ-01
Название: Завод ЖБИ → Площадка «Дом на Ленина»
Откуда: г. Москва, Промышленная, 25 (Завод ЖБИ)
Куда: г. Москва, Ленина, 15 (стройплощадка)
Расстояние: 35 км
Время в пути: 1.5 часа
Тип транспорта: Бортовой (плиты, блоки)
```

При создании отгрузки можно выбрать маршрут — адреса и расстояние подставятся автоматически.

## Часть 3: Диспетчеризация (Операции)

Диспетчеризация — это управление **собственным транспортом** компании: куда направить машину, кого назначить водителем, что везти.

**Путь:** Операции → **Диспетчеризация**

### Отличие от отгрузок

| | Отгрузки | Диспетчеризация |
|---|---------|----------------|
| **Кто везёт** | Поставщик (или транспортная компания) | Наш транспорт |
| **Что отслеживаем** | Когда привезут заказанное | Куда отправить нашу машину |
| **Связь** | С заказами поставщикам (PO) | С проектами и рабочими заданиями |

### Как создать заявку на диспетчеризацию

| Поле | Описание |
|------|---------|
| **Номер** | Уникальный (например, «ТР-2026-001») |
| **Проект** | Для какого объекта |
| **Транспорт** | Машина из автопарка |
| **Водитель** | Кого назначить |
| **Откуда** | Точка загрузки |
| **Куда** | Точка выгрузки |
| **Груз** | Что перевозим (описание) |
| **Количество и вес** | Сколько и какой массы |
| **Дата и время** | Когда выполнить |

### Статусы диспетчеризации

| Статус | Описание |
|--------|---------|
| **Запланировано** | Заявка создана, машина назначена |
| **Отправлено** | Машина выехала с точки загрузки |
| **В пути** | Машина в дороге |
| **Доставлено** | Прибыли на место, выгрузка завершена |
| **Выполнено** | Задание полностью завершено |
| **Отменено** | Задание отменено |

```
Запланировано → Отправлено → В пути → Доставлено → Выполнено
```

### Что фиксируется

При выполнении заявки система записывает:
- Фактическое время отправления
- Фактическое время прибытия
- Пройденное расстояние
- Расход топлива

Эти данные используются для контроля [автопарка](fleet.md).

### Календарь диспетчеризации

**Путь:** Операции → **Календарь диспетчеризации**

Все заявки на транспорт отображаются на календаре: видно, какие машины заняты, какие свободны, и где «окна» для новых задач.

## Часть 4: Рейтинг поставщиков

После завершения доставки можно оценить поставщика по трём критериям:

| Критерий | Что оцениваем | Шкала |
|----------|-------------|-------|
| **Качество** | Соответствие материалов заказу, наличие брака | 0–5 |
| **Доставка** | Пунктуальность, сохранность груза | 0–5 |
| **Цена** | Соотношение цены и качества | 0–5 |

**Общий рейтинг** = (Качество + Доставка + Цена) ÷ 3

**Пример:**
```
Поставщик: ООО «МеталлТрейд»
Период: Март 2026

Качество: 4.5 (арматура соответствует ГОСТ, сертификаты есть)
Доставка: 3.0 (задержка 2 дня от плана)
Цена: 4.0 (рыночная, без завышения)

Общий рейтинг: 3.83 из 5.0

Комментарий: «Хороший материал, но доставку нужно контролировать»
```

Рейтинги накапливаются и помогают выбирать поставщиков для будущих заказов.

## Часть 5: Резервирование материалов

Можно зарезервировать материалы на складе под конкретный проект — чтобы их не выдали на другой объект.

| Поле | Описание |
|------|---------|
| Материал | Что резервируем |
| Проект | Для какого объекта |
| Количество | Сколько зарезервировать |
| Рабочее задание | Для какой конкретной работы (необязательно) |
| Срок резерва | До какой даты (необязательно) |

**Статусы резерва:**
- **Активен** — материал зарезервирован, не доступен для других проектов
- **Снят** — резерв отменён, материал снова доступен
- **Истёк** — срок резерва закончился, материал снова доступен

> Зарезервированный материал виден в [остатках](warehouse.md) в колонке «Зарезервировано». Доступный остаток = Всего − Зарезервировано.

---

## Полный пример: от заказа до склада

```
1. Снабженец создаёт PO:
   «10 т арматуры от ООО МеталлТрейд, доставка 20 марта»
   → Статус PO: ПОДТВЕРЖДЁН

2. 19 марта — поставщик сообщает: «Завтра отгружаем»
   Снабженец создаёт отгрузку:
   Транспорт: КАМАЗ А123ВС 77
   Водитель: Смирнов А.В.
   Планируемое прибытие: 20 марта 14:00
   → Статус отгрузки: ЗАПЛАНИРОВАНА

3. 20 марта 08:00 — поставщик начал загрузку
   → Статус отгрузки: ЗАГРУЗКА

4. 20 марта 10:00 — машина выехала
   → Статус отгрузки: В ПУТИ

5. 20 марта 13:30 — машина прибыла на склад
   → Статус отгрузки: ДОСТАВЛЕНА

6. Кладовщик проверяет: арматура d12, 10 т — всё совпадает ✓
   Регистрирует приёмку на PO
   → Статус PO: ДОСТАВЛЕН
   → На складе: +10 т арматуры d12
   → Движение: приход SM-000456

7. Снабженец оценивает поставщика:
   Качество: 5, Доставка: 4 (чуть раньше плана), Цена: 4
   → Рейтинг МеталлТрейд: 4.33 / 5.0
```

## Частые вопросы

**Как отследить, где сейчас мой заказ?**
Откройте отгрузку, привязанную к вашему PO. Статус покажет: «Запланирована» (ещё не отправлено), «В пути» (едет), «Доставлена» (на складе).

**Обязательно ли создавать отгрузку?**
Нет. Можно зарегистрировать доставку напрямую на PO (кнопка «Зарегистрировать доставку»). Но с отгрузкой — лучше контроль: вы знаете, что именно едет и когда ждать.

**Что если машина задерживается?**
Статус остаётся «В пути». Плановое и фактическое время прибытия фиксируются отдельно — разница покажет задержку. Это учитывается в рейтинге поставщика.

**Чем отличается отгрузка от диспетчеризации?**
Отгрузка — это **входящая** логистика (от поставщика к нам). Диспетчеризация — это **внутренняя** логистика (наш транспорт между нашими объектами).

**Как оценить поставщика?**
После завершения доставки откройте рейтинг поставщика и выставите оценки по трём критериям: качество, доставка, цена. Комментарий — по желанию.

**Можно ли отменить отгрузку?**
Да, из любого статуса кроме «Доставлена». Нажмите «Отменить» — статус станет «Отменена», склад не затрагивается.

**Зачем нужны маршруты?**
Чтобы не вводить адреса каждый раз. Создайте маршрут «Завод ЖБИ → Площадка» один раз — и выбирайте его при каждой отгрузке. Расстояние и время подставятся автоматически.

**Как узнать, какие материалы зарезервированы?**
В [остатках на складе](warehouse.md) — колонка «Зарезервировано». Или в карточке [материала](materials.md) — секция «Остатки по складам».
