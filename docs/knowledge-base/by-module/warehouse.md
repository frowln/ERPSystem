# Складской учёт

---

## Что это и зачем

Складской учёт — это модуль, который управляет всей логистикой материалов: где они хранятся, сколько их, кто отвечает, и когда нужно заказывать ещё. Это «диспетчерская» стройки — всё, что приходит и уходит, проходит через склад.

Модуль решает три главные задачи:
1. **Знать, что есть** — текущие остатки по каждому складу
2. **Контролировать движение** — приход, расход, перемещение между складами
3. **Не допускать дефицита** — лимиты, оповещения, прогноз потребности

## Остатки

**Путь:** Склад → **Остатки**

Эта страница показывает, **сколько чего есть прямо сейчас** на каждом складе.

**Что видно:**

| Колонка | Что показывает |
|---------|---------------|
| Материал | Наименование и код |
| Склад/Локация | Где хранится |
| Всего | Общее количество |
| Зарезервировано | Выделено для проектов, но ещё не выдано |
| Доступно | Всего − Зарезервировано (можно выдавать) |
| Цена за единицу | Последняя цена поступления |
| Общая стоимость | Количество × Цена |

**Формула:**
```
Доступно = Всего − Зарезервировано
```

**Фильтры:**
- По материалу (поиск по названию)
- По складу/локации
- По проекту
- Показать только с низким остатком

> **Важно:** Зарезервированный материал числится на складе физически, но система не позволит выдать его на другой проект.

## Складские локации

**Путь:** Склад → **Локации**

Локация — это место хранения: центральный склад, площадка на объекте, временное хранилище.

### Типы локаций

| Тип | Описание | Пример |
|-----|---------|--------|
| **Центральный склад** | Основной склад компании | «Склад на ул. Промышленной» |
| **Объектный склад** | Склад на стройплощадке | «Площадка "Дом на Ленина"» |
| **Транзитный** | Временное хранение при перевозке | «Транзит: Площадка → Склад» |
| **Склад контрагента** | Склад поставщика или субподрядчика | «Склад ООО "Бетон Плюс"» |

### Иерархия

Локации могут быть вложенными:

```
Центральный склад
├── Зона А (открытая, крупногабарит)
│   ├── Стеллаж 1
│   │   ├── Полка 1
│   │   └── Полка 2
│   └── Стеллаж 2
└── Зона Б (закрытая, мелкий крепёж)
    └── Стеллаж 1
        ├── Ячейка A1
        └── Ячейка A2
```

### Как создать локацию

| Поле | Описание |
|------|---------|
| **Код** | Уникальный код (например, «WH-01» или «SITE-A») |
| **Наименование** | Человеческое название |
| **Тип** | Центральный / Объектный / Транзитный / Контрагента |
| **Адрес** | Физический адрес |
| **Ответственный** | Кто управляет этой локацией |
| **Родительская локация** | Если это зона внутри склада — укажите родителя |

## Складские ордера

**Путь:** Склад → **Складские ордера**

Ордер — это первичный документ для склада, как чек в магазине. Два основных типа:
- **М-4 (приходный ордер)** — материал приходит на склад
- **М-11 (расходный ордер)** — материал уходит со склада

### Типы ордеров

| Тип | Направление | Когда используется |
|-----|------------|-------------------|
| **Приход (М-4)** | → на склад | Поставщик привёз материалы |
| **Расход (М-11)** | со склада → | Выдача на объект, бригаде |
| **Внутреннее перемещение** | склад ↔ склад | Между зонами или складами |
| **Возврат** | → на склад | Бригада вернула неиспользованное |

### Статусы ордера

| Статус | Цвет | Что можно делать |
|--------|------|-----------------|
| **Черновик** | Серый | Редактировать, добавлять позиции, удалять |
| **Проведён** | Зелёный | Ордер проведён, остатки обновлены. Нельзя менять |
| **Отменён** | Красный | Ордер аннулирован, остатки не затронуты |

### Как создать ордер

1. Перейдите **Склад → Складские ордера**
2. Нажмите **«Новый ордер»**
3. Заполните:
   - Тип (приход / расход / перемещение / возврат)
   - Дата
   - Склад
   - Контрагент (поставщик или получатель)
   - Ответственный
4. Добавьте позиции: материал, количество, цена (для прихода)
5. Нажмите **«Провести»** — ордер будет проведён, остатки обновятся

> **Важно:** После проведения ордер нельзя изменить. Если ошибка — создайте обратный ордер (возврат или корректировку).

## Инвентаризация

**Путь:** Склад → **Инвентаризация**

Инвентаризация — это пересчёт всего, что есть на складе, и сравнение с данными системы.

### Зачем нужна
- Обязательна минимум 1 раз в год (требование законодательства)
- Выявляет недостачи (украли, потеряли, разбили)
- Выявляет излишки (не оформленный приход, пересортица)
- Приводит данные системы в соответствие с реальностью

### Как проводить

**Шаг 1. Создать проверку**
- Выберите склад (или зону)
- Укажите дату и ответственного
- Статус: **«Запланирована»**

**Шаг 2. Начать подсчёт**
- Нажмите **«Начать»**
- Система загрузит все материалы этого склада с ожидаемыми количествами
- Статус: **«В процессе»**

**Шаг 3. Внести фактические данные**
Для каждого материала введите, сколько посчитали на самом деле:

| Материал | Ед. изм. | По системе | Факт | Расхождение |
|----------|---------|----------:|-----:|------------:|
| Кирпич М-150 | шт | 10 000 | 9 800 | **−200** (недостача) |
| Арматура d12 | м | 500 | 500 | 0 (совпадает) |
| Цемент М400 | кг | 2 000 | 2 100 | **+100** (излишек) |

**Шаг 4. Завершить проверку**
- Нажмите **«Завершить»**
- Система **автоматически создаёт корректировочные движения** для всех расхождений
- Недостача 200 кирпичей → списание
- Излишек 100 кг цемента → оприходование
- Статус: **«Завершена»**

### Статусы инвентаризации

| Статус | Что происходит |
|--------|---------------|
| **Запланирована** | Создана, ещё не начата |
| **В процессе** | Идёт подсчёт |
| **Завершена** | Расхождения исправлены |
| **Отменена** | Проверка аннулирована |

## Лимиты запасов и оповещения

### Лимиты

**Путь:** Склад → **Лимиты запасов**

Лимит — это правило: «Если арматуры на Центральном складе меньше 500 кг — сигнал тревоги».

**4 типа лимитов:**

| Тип | Что значит | Пример |
|-----|-----------|--------|
| **Минимум** | Ниже нельзя — срочно заказать | Бетон: минимум 20 м³ |
| **Максимум** | Больше — перерасход на хранение | Арматура: максимум 100 т |
| **Точка перезаказа** | Пора размещать заказ (с учётом срока доставки) | Цемент: при 50 кг — заказать (доставка 10 дней) |
| **Страховой запас** | Неприкосновенный резерв на случай форс-мажора | Кабель: 5% всегда в запасе |

### Оповещения

**Путь:** Склад → **Оповещения**

Когда остаток выходит за пределы лимита, система создаёт оповещение.

**Уровни:**

| Уровень | Цвет | Что делать |
|---------|------|-----------|
| **Информация** | Синий | Принять к сведению |
| **Предупреждение** | Жёлтый | Запланировать действие в ближайшие дни |
| **Критический** | Красный | Действовать немедленно! |

**Жизненный цикл оповещения:**
1. Остаток упал ниже минимума → создано оповещение (активное)
2. Кладовщик увидел → нажал «Принято» (подтверждённое)
3. Заказ размещён, материал поступил → оповещение снимается (решённое)

## Лимитно-заборные ведомости

**Путь:** Склад → **Лимитно-заборные ведомости**

Лимитно-заборная ведомость (ЛЗВ, форма М-8) — это документ, который устанавливает лимит выдачи материала на объект за период. Как бюджет на материалы: «В марте бригаде можно получить не более 20 тонн цемента».

### Зачем нужна

Без ЛЗВ прораб может набрать материалов сколько хочет. С ЛЗВ — система не даст превысить лимит.

### Как работает

| Поле | Описание |
|------|---------|
| Номер | Уникальный номер ведомости |
| Проект | На какой объект выдаётся |
| Материал | Какой именно материал |
| Ед. изм. | Единица измерения |
| Лимит | Максимальное количество за период |
| Период | С какой по какую дату действует |
| Склад | Откуда выдавать |

**Пример:**

```
Материал: Цемент М400
Проект: Жилой дом, ул. Ленина
Лимит: 20 тонн
Период: 1 марта — 31 марта 2026

5 марта:  Выдано 5 т  → Остаток лимита: 15 т
12 марта: Выдано 8 т  → Остаток лимита: 7 т
15 марта: Возврат 2 т → Остаток лимита: 9 т
20 марта: Запрос 12 т → ОТКАЗ: лимит всего 9 т!
```

### Статусы ЛЗВ

| Статус | Описание |
|--------|---------|
| **Активная** | Действует, можно выдавать |
| **Исчерпана** | Лимит использован полностью (0 осталось) |
| **Закрыта** | Период истёк или закрыта вручную |
| **Отменена** | Аннулирована |

### Операции

- **Выдача** — уменьшает остаток лимита
- **Возврат** — увеличивает остаток лимита (бригада вернула неиспользованное)
- **Закрытие** — прекращает выдачу досрочно

## Отчёт М-29

**Путь:** Склад → **Отчёт М-29**

М-29 — это обязательная форма отчётности, которая показывает: сколько материала **положено** по нормам и сколько **фактически потрачено** на объекте за месяц.

### Что показывает

| Материал | Ед. изм. | По нормам | Факт | Отклонение | % |
|----------|---------|----------:|-----:|-----------:|--:|
| Бетон В25 | м³ | 500 | 480 | −20 | −4% (экономия) |
| Арматура d12 | т | 50 | 52 | +2 | +4% (перерасход) |
| Гвозди | кг | 100 | 90 | −10 | −10% (экономия) |

### Параметры отчёта

- Выберите **проект**
- Выберите **месяц** и **год**
- Система сама соберёт данные

### Откуда берутся данные

- **По нормам** — из спецификаций и смет проекта
- **Факт** — из складских движений (приход → расход на объект)

### Зачем нужен

- Обязательный документ для контролирующих органов
- Показывает, где перерасход (прораб должен объяснить)
- Показывает экономию (хорошо!)
- Помогает скорректировать бюджет на следующий месяц

## Адресное хранение

**Путь:** Склад → **Адресное хранение**

Адресное хранение — это карта склада. У каждой ячейки есть адрес (зона-стеллаж-полка-ячейка), и система знает, что где лежит.

### Как устроено

```
Зона А
├── Стеллаж 01
│   ├── Полка 1 → [Арматура d10, 500 м]
│   ├── Полка 2 → [Арматура d12, 300 м]
│   └── Полка 3 → [СВОБОДНО]
└── Стеллаж 02
    ├── Полка 1 → [Кабель ВВГ 3×2.5, 200 м]
    └── Полка 2 → [Кабель ВВГ 3×1.5, 150 м]
```

### Что видно на экране

- **Сетка ячеек** с цветовой кодировкой:
  - 🟢 Зелёный — свободная ячейка
  - 🟠 Оранжевый — занятая ячейка
  - 🔵 Синий — найденная по поиску
- **Поиск** — по названию материала или адресу ячейки
- **Показатели**: всего ячеек, занято, свободно, загрузка в %

### Зачем нужно

- Быстро найти, где лежит нужный материал
- Планировать размещение новых поступлений
- Не тратить время на блуждание по складу
- Обучать новых кладовщиков

## Штрих-код сканер

**Путь:** Склад → **Сканер штрих-кодов**

Сканер позволяет быстро работать с материалами через штрих-код или QR-код.

### Режимы работы

| Режим | Что делает |
|-------|-----------|
| **Приём** | Отсканировал → ввёл количество → материал оприходован |
| **Выдача** | Отсканировал → ввёл количество → материал списан со склада |
| **Инвентаризация** | Отсканировал → ввёл фактическое количество → зафиксировано |

### Как работает

1. Откройте страницу сканера
2. Выберите режим (приём / выдача / инвентаризация)
3. Отсканируйте штрих-код с упаковки или наклейки
4. Система мгновенно покажет:
   - Название материала
   - Категория
   - Единица измерения
   - Текущий остаток на складе
   - Основное место хранения
5. Подтвердите количество → операция выполнена

### История сканирований

Внизу страницы — список последних 20 сканирований: что сканировали, когда, успешно ли.

> **Совет:** Штрих-коды нужно предварительно присвоить материалам в [справочнике](materials.md#штрих-код). Без привязки сканирование покажет «Материал не найден».

## Потребность в материалах

**Путь:** Склад → **Потребность в материалах**

Система анализирует, какие материалы нужны для работ на ближайшие дни, и сравнивает с текущими остатками.

### Три статуса

| Статус | Цвет | Что значит | Что делать |
|--------|------|-----------|-----------|
| **Достаточно** | Зелёный | Остаток ≥ 120% от потребности | Ничего |
| **Мало** | Жёлтый | Остаток 80–120% от потребности | Запланировать заказ |
| **Дефицит** | Красный | Остаток < 80% от потребности | Срочно заказать! |

### Пример

```
Проект: Офисное здание, неделя 15–22 марта

Бетон В25:   Нужно 100 м³, на складе 20 м³ → ДЕФИЦИТ 🔴
Арматура:    Нужно 50 т, на складе 55 т     → ДОСТАТОЧНО ✅
Окна ПВХ:    Нужно 80 шт, на складе 30 шт   → ДЕФИЦИТ 🔴
Цемент:      Нужно 10 т, на складе 11 т      → МАЛО ⚠️
```

Для материалов с дефицитом — кнопка **«Создать заявку на закупку»**, которая автоматически формирует заявку в модуле снабжения.

## Межпроектные перемещения

**Путь:** Склад → **Межпроектные перемещения**

Когда на одном объекте излишки, а на другом — дефицит, дешевле переместить, чем заказывать новое.

### Как работает

```
Проект А (излишки бетона)
        ↓
Запрос на перемещение
        ↓
Утверждение руководителем проекта
        ↓
В пути (транспорт)
        ↓
Проект Б (получил и принял)
```

### Статусы

| Статус | Описание |
|--------|---------|
| **Черновик** | Запрос создан |
| **На согласовании** | Ждёт утверждения руководителя |
| **Утверждён** | Одобрено, готово к отправке |
| **В пути** | Материал в дороге |
| **Получен** | Доставлен и принят |
| **Отменён** | Отказ |

### Что указывается

- Откуда (проект-отправитель)
- Куда (проект-получатель)
- Материалы и количества
- Транспорт (номер машины, водитель)
- Ожидаемая дата прибытия

## Быстрый приход и быстрое подтверждение

### Быстрый приход

Упрощённая версия приёмки — 2 шага вместо полного оформления ордера:

1. Отсканируйте штрих-код или выберите материал из списка
2. Введите количество → выберите склад → подтвердите

Результат: материал сразу появляется в остатках.

### Быстрое подтверждение

Массовое подтверждение черновиков перемещений:

- На экране — все черновики, ожидающие подтверждения
- Галочками выберите нужные
- Нажмите **«Подтвердить выбранные»**
- Все выбранные перемещения проводятся одним действием

## Что если выдать больше, чем есть?

**Система не позволит.** При попытке создать расход или перемещение, превышающее доступный остаток:

```
Запрос: выдать 300 кг цемента
Доступно на складе: 250 кг
→ ОШИБКА: «Недостаточно материала. Доступно только 250 кг»
→ Движение остаётся в статусе «Черновик»
→ Нужно: уменьшить количество ИЛИ дождаться прихода
```

Проверка происходит при **подтверждении** движения (не при создании черновика). В черновике можно указать любое количество — ошибка появится только при попытке провести.

```
Потребность (из спецификации)
        ↓
Заявка на закупку (снабжение)
        ↓
Заказ поставщику (PO)
        ↓
Приход на склад (складской ордер М-4)
        ↓
Хранение (адресное хранение, лимиты)
        ↓
Расход на объект (складской ордер М-11)
        ↓
Использование (КС-2, М-29)
        ↓
Учёт (проводка, бюджет, cash flow)
```

## Частые вопросы

**Где посмотреть текущий остаток?**
Склад → Остатки. Фильтруйте по материалу или складу. Также остаток виден в карточке каждого [материала](materials.md).

**Можно ли выдать материал без ЛЗВ?**
Да. Лимитно-заборные ведомости — это инструмент контроля, а не обязательное условие. Расход можно оформить обычным движением. Но с ЛЗВ — лучше контроль.

**Как провести инвентаризацию?**
Склад → Инвентаризация → «Новая проверка» → выбрать склад → «Начать» → ввести фактические данные → «Завершить». Расхождения исправляются автоматически.

**Что такое М-29 и кому он нужен?**
М-29 — это обязательный отчёт: план vs факт расхода материалов за месяц. Нужен бухгалтерии и контролирующим органам. Показывает, где экономия, а где перерасход.

**Как настроить оповещение о нехватке?**
Склад → Лимиты запасов → создайте лимит для нужного материала и склада → укажите минимальное количество. Когда остаток упадёт ниже — система создаст оповещение.

**Что такое «зарезервировано»?**
Материал, выделенный для конкретного проекта. Он на складе, но его нельзя выдать другому проекту. Доступный остаток = Всего − Зарезервировано.

**Как работает штрих-код сканер?**
Отсканируйте код → система покажет материал и остаток → выберите операцию (приём/выдача) → подтвердите. Штрих-коды предварительно привязываются к материалам в [справочнике](materials.md#штрих-код).

**Можно ли отменить проведённый ордер?**
Нет. Проведённый ордер нельзя изменить. Если ошибка — создайте обратную операцию: возврат (если был расход) или списание (если был лишний приход).

**Как переместить материал между объектами?**
Склад → Межпроектные перемещения → создайте запрос → укажите проект-отправитель, проект-получатель, материалы → отправьте на согласование.
