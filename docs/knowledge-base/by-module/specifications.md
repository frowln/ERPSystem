# Спецификации

---

## Что это и зачем

Спецификация — это перечень материалов и оборудования для проекта. Берётся из проектной документации (ПД). В спецификации указано ЧТО и СКОЛЬКО нужно: наименование, тип/марка, код, завод-изготовитель, единица измерения, количество, вес. Цен в спецификации НЕТ — это чисто технический документ.

Спецификация — отправная точка для закупок и формирования цены. Из неё создаётся конкурентный лист (КЛ) для сбора цен от поставщиков, а позиции передаются в финансовую модель (ФМ).

**Цепочка:** Проектная документация → **Спецификация** → КЛ → ФМ

---

## Где найти

**Путь:** Главное меню → **Спецификации**

## Список спецификаций

**Что видно в таблице:**

| Колонка | Что показывает |
|---------|---------------|
| Название | Имя спецификации + ID мелким шрифтом |
| Проект | К какому объекту относится |
| Статус | Черновик, на проверке, согласована, активная |
| Версия | Номер версии (v1, v2 и т.д.) |
| Кол-во позиций | Сколько позиций в спецификации |
| Дата создания | Когда создана |

**Вкладки-фильтры:**
- **Все** — все спецификации
- **Черновик** — в разработке
- **На проверке** — отправлена на согласование
- **Согласована** — утверждена, но не в работе
- **Активная** — действующая

**Кнопки:**
- **Аналоги** — перейти к списку материальных аналогов
- **Запросы на аналоги** — перейти к запросам на замену материалов
- **Реестр КЛ** — реестр всех конкурентных листов
- **Новая спецификация** — создать

## Как создать спецификацию

### Шаг 1: Заполните основные данные

1. Нажмите **«Новая спецификация»**
2. Заполните форму:
   - **Проект** (обязательно) — выберите проект
   - **Название** (обязательно) — например, «Спецификация электрооборудования, корпус 3»
   - **Статус** — черновик (по умолчанию)
   - **Примечания** — свободный текст

### Шаг 2: Загрузите позиции

**Вариант А: Импорт из Excel (рекомендуется)**

1. Перетащите xlsx-файл в зону загрузки (или нажмите «Выбрать файл»)
2. Система автоматически определит колонки:

| Колонка | Что ищет в заголовке |
|---------|---------------------|
| Наименование | «наименование», «название», «name» |
| Тип/Марка | «тип», «марка», «модель», «brand» |
| Код | «код», «артикул», «code» |
| Завод-изготовитель | «завод», «производитель», «изготовитель» |
| Единица измерения | «ед.изм.», «единица», «unit» |
| Количество | «кол-во», «количество», «qty» |
| Вес | «вес», «масса», «weight» |
| Примечание | «примечание», «комментарий», «note» |

3. Проверьте распознанные позиции в таблице ниже
4. При необходимости отредактируйте прямо в таблице

> **Подсказка:** Если заголовки в вашем Excel не распознались — система использует стандартный порядок колонок ПД: №, Наименование, Тип/Марка, Код, Завод, Ед.изм., Кол-во, Вес, Примечание.

**Вариант Б: Импорт из PDF**

1. Перетащите PDF-файл в зону загрузки
2. Система распознает таблицу с позициями (OCR + парсинг)
3. Проверьте результат и при необходимости поправьте

**Вариант В: Ввод вручную**

Добавляйте позиции по одной прямо в таблице на форме.

### Шаг 3: Настройте автоматические действия

Под таблицей позиций — два флажка:

- **Автоматически передать в ФМ** — позиции спецификации сразу попадут в финансовую модель проекта. Система сгруппирует их по разделам и создаст соответствующую структуру в ФМ
- **Автоматически создать КП** — на основе ФМ создастся коммерческое предложение

> **Подсказка:** Рекомендуется включить оба флажка — это сэкономит время. ФМ и КП можно потом отредактировать.

### Шаг 4: Сохраните

Нажмите **«Создать спецификацию»**. Система автоматически:
- Сохранит все позиции
- Передаст их в ФМ (если флажок включён)
- Создаст КП (если флажок включён)
- Создаст конкурентный лист (КЛ) для сбора цен от поставщиков

## Структура спецификации

Колонки таблицы позиций (формат проектной документации):

| Колонка | Описание | Пример |
|---------|----------|--------|
| № п/п | Порядковый номер | 1, 2, 3... |
| Наименование | Название материала или оборудования | «Трансформатор ТМ-630/10» |
| Тип/Марка | Модель, марка, модификация | «ТМ-630/10-У1» |
| Код | Код оборудования или артикул | «ОКПД2 27.11.41.110» |
| Завод-изготовитель | Кто производит | «АББ», «Schneider Electric» |
| Ед. изм. | Единица измерения | шт, м, кг, комплект |
| Кол-во | Количество | 3 |
| Вес, кг | Вес единицы | 1 850 |
| Примечание | Дополнительная информация | «С маслорасширителем» |

**Типы позиций:**
- **Оборудование** — трансформаторы, насосы, щиты, приборы
- **Материалы** — кабель, трубы, арматура, краска, цемент
- **Работы** — монтаж, установка, прокладка, сварка, наладка

> Система автоматически определяет тип по названию: если в названии есть «монтаж», «установка», «прокладка» — это работа; «труба», «кабель», «арматура» — материал; остальное — оборудование.

## Страница спецификации (детальный вид)

Нажмите на спецификацию в списке — откроется детальная страница.

**Показатели (карточки вверху):**

| Показатель | Что значит |
|-----------|-----------|
| Позиций | Общее количество позиций |
| Оборудование | Сколько позиций типа «оборудование» |
| Материалы | Сколько позиций типа «материалы» |
| Работы | Сколько позиций типа «работы» |
| Покрытие | Статус обеспечения: сколько позиций обеспечено ценами из КЛ |
| Лучшая цена | Минимальная цена из конкурентных предложений |
| Общая стоимость | Сумма плановых стоимостей всех позиций |

**Таблица позиций:**
- Все колонки спецификации (наименование, марка, код, завод, количество, ед.изм., вес)
- Тип позиции (цветная метка: оборудование/материал/работа)
- Статус обеспечения (полностью / частично / не обеспечено)
- Лучшая цена (из КЛ)
- Лучший поставщик (из КЛ)
- Плановая стоимость
- Привязка к ФМ (есть/нет)

> **Подсказка:** Все ячейки можно редактировать прямо в таблице — дважды кликните на ячейку.

**Кнопки действий:**
- **Передать в ФМ** — отправить выбранные позиции в финансовую модель
- **Импорт ЛСР** — загрузить сметные данные для привязки расценок
- **Авто-ранжирование КЛ** — автоматически выбрать лучшие предложения в конкурентном листе
- **Создать КЛ** — создать конкурентный лист для сбора цен
- **Новая позиция** — добавить пустую строку
- **Экспорт PDF** — скачать спецификацию в PDF

## Конкурентный лист (КЛ)

Конкурентный лист — это таблица сравнения цен от разных поставщиков на позиции спецификации.

**Путь:** Спецификации → откройте спецификацию → кнопка «Создать КЛ» (или из реестра КЛ)

**Как работает:**
1. Из спецификации создаётся КЛ (автоматически или кнопкой)
2. По каждой позиции собираются предложения от 3+ поставщиков
3. Для каждого поставщика указывается: цена за единицу, условия поставки, сроки
4. Система автоматически ранжирует предложения и выбирает лучшее
5. Лучшая цена из КЛ = себестоимость в ФМ

**Действия с КЛ:**
- **Добавить предложение** — ввести цену от поставщика
- **Авто-ранжирование** — система расставит места по цене
- **Авто-выбор лучшего** — автоматически выберет самые дешёвые предложения
- **Выбрать победителя** — вручную выбрать поставщика
- **Применить к КП** — передать выбранные цены в коммерческое предложение
- **Изменить статус** — черновик → на рассмотрении → утверждён → закрыт

### Аналоги материалов

Если указанного в спецификации материала нет на рынке (или слишком дорогой), можно запросить замену:

**Путь:** Спецификации → Аналоги (или Запросы на аналоги)

- Создайте запрос на аналог — укажите оригинальную позицию и предложите замену
- Запрос проходит согласование (согласовать / отклонить)
- После согласования аналог подставляется в спецификацию

## Связь с другими модулями

| Модуль | Как связан |
|--------|-----------|
| **Проект** | Спецификация привязывается к проекту |
| **Финансовая модель (ФМ)** | Позиции спецификации передаются в ФМ (кнопка «Передать в ФМ») |
| **КП** | Коммерческое предложение формируется из ФМ, куда попали позиции спецификации |
| **Конкурентный лист (КЛ)** | Создаётся из спецификации для сбора цен от поставщиков |
| **Смета (ЛСР)** | Смета может создаваться на основе спецификации |
| **Закупки** | Заявки на закупку формируются из позиций спецификации |
| **Склад** | Поступления на склад сверяются со спецификацией |

**Цепочка данных:**
```
Проектная документация → Спецификация → КЛ (цены поставщиков) → ФМ (себестоимость)
                                                                    ↑
                                                          Смета (ЛСР) → сметная цена
                                                                    ↓
                                                                   КП → Заказчик
```

## Частые вопросы

**Какие форматы файлов поддерживает импорт?**
Excel (.xlsx, .xls) и PDF. Для Excel система автоматически определяет колонки по заголовкам. Для PDF используется OCR-распознавание.

**Почему в спецификации нет цен?**
Спецификация — это технический документ из проектной документации. Цены собираются отдельно: через конкурентный лист (от поставщиков) и через смету (по нормативам). Это позволяет разделить техническую и финансовую работу.

**Что такое «обеспечение» (покрытие) позиции?**
Это статус, показывающий, есть ли цена на эту позицию. «Полностью обеспечена» — есть цена в КЛ. «Частично» — цена есть не на всё количество. «Не обеспечена» — поставщики ещё не прислали предложения.

**Как передать позиции в финансовую модель?**
На детальной странице спецификации нажмите кнопку **«Передать в ФМ»**. Выберите бюджет (или система выберет единственный автоматически). Позиции сгруппируются по разделам и появятся в ФМ.

**Зачем нужно минимум 3 поставщика в конкурентном листе?**
Для государственных строек — это требование закона (конкурентная процедура). Для коммерческих — хорошая практика: сравнение 3+ цен помогает получить лучшую стоимость и защищает от обвинений в завышении.

**Можно ли добавить свой тип позиции?**
Сейчас доступны три типа: оборудование, материалы, работы. Система определяет тип автоматически по названию, но вы можете изменить его вручную в выпадающем списке.

**Как создать новую версию спецификации?**
На детальной странице нажмите кнопку **«Копировать»** — создастся копия с увеличенным номером версии. Оригинал останется без изменений.
