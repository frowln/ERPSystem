# Управление изменениями

---

## Что это и зачем

На любой стройке неизбежны изменения: заказчик хочет другую планировку, проектировщик нашёл ошибку, на площадке обнаружились грунтовые воды. Каждое изменение влияет на **стоимость** и **сроки** проекта.

Без управления изменениями:
- Подрядчик делает дополнительные работы «по устной договорённости», а потом не может получить оплату
- Заказчик не понимает, почему бюджет вырос на 15%
- Никто не знает, сколько реально стоят все изменения

Модуль «Управление изменениями» в системе ПРИВОД фиксирует каждое изменение от момента обнаружения до оплаты:

1. **Событие изменения** (Change Event) — «Что случилось?»
2. **Запрос на изменение** (Change Order Request) — «Сколько это стоит?»
3. **Дополнительное соглашение** (Change Order) — «Оформляем официально»

## Часть 1: События изменений (Change Event)

Событие — это факт обнаружения потенциального изменения. Его создаёт тот, кто первым заметил, что «что-то не так» или «нужно сделать по-другому».

### Источники изменений

| Источник | Описание | Пример |
|----------|----------|--------|
| **Запрос информации (RFI)** | Вопрос по проекту выявил проблему | RFI показал противоречие в чертежах |
| **Изменение проекта** | Проектировщик выпустил корректировку | Изменена конструкция перекрытия |
| **Полевые условия** | На площадке обнаружено непредвиденное | Грунтовые воды, скальный грунт, старый фундамент |
| **Запрос заказчика** | Заказчик хочет изменения | Другая планировка, дополнительный этаж |
| **Нормативные требования** | Изменились нормы или требования | Новые требования пожарной безопасности |
| **Стоимостная оптимизация** | Предложение удешевить без потери качества | Замена материала на более дешёвый аналог |
| **Прочее** | Всё остальное | Форс-мажор, погодные условия |

### Как создать событие

1. Перейдите **Управление изменениями → События**
2. Нажмите **«+ Новое событие»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Название** | Да | Краткое описание (например: «Обнаружены грунтовые воды на отм. -3.000») |
| **Проект** | Да | К какому проекту |
| **Источник** | Да | Откуда пришло изменение (7 вариантов) |
| **Описание** | Нет | Подробности |
| **Оценка влияния на стоимость** | Нет | Сколько примерно будет стоить (₽) |
| **Оценка влияния на сроки** | Нет | На сколько дней задержка |
| **Связанный RFI** | Нет | Если событие связано с RFI |
| **Связанный дефект** | Нет | Если событие связано с дефектом |

4. Нажмите **«Создать»**

### Статусы события

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Выявлен** (IDENTIFIED) | Синий | Событие обнаружено и зафиксировано |
| **На рассмотрении** (UNDER_REVIEW) | Жёлтый | Руководство рассматривает: нужно ли оформлять? |
| **Утверждён для расценки** (APPROVED_FOR_PRICING) | Оранжевый | Да, нужно — отправить на расценку подрядчику |
| **Расценён** (PRICED) | Синий | Подрядчик предоставил стоимость |
| **Утверждён** (APPROVED) | Зелёный | Изменение одобрено, создаётся change order |
| **Отклонён** (REJECTED) | Красный | Изменение не принято |
| **Аннулирован** (VOID) | Серый | Потеряло актуальность |

```
Выявлен → На рассмотрении → Утверждён для расценки → Расценён → Утверждён
                ↓                    ↓                    ↓
            Отклонён           Отклонён              Отклонён
                                                         ↓
Аннулирован ← (из любого активного статуса)
```

## Часть 2: Дополнительные соглашения (Change Order)

Когда событие одобрено — создаётся **дополнительное соглашение** (Change Order). Это юридический документ, который меняет условия договора.

### Типы change order

| Тип | Описание | Влияние на сумму |
|-----|----------|:----------------:|
| **Дополнительные работы** (ADDITION) | Работы, которых не было в договоре | + (увеличение) |
| **Исключение работ** (DEDUCTION) | Работы отменяются или уменьшаются | − (уменьшение) |
| **Замена** (SUBSTITUTION) | Замена одного решения другим | ± (может быть любое) |
| **Продление сроков** (TIME_EXTENSION) | Только сроки, без изменения стоимости | 0 (только дни) |
| **Комбинированный** (MIXED) | Несколько изменений разного типа | ± |

### Как создать change order

1. Перейдите **Управление изменениями → Дополнительные соглашения**
2. Нажмите **«+ Новое»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Название** | Да | Описание изменения |
| **Проект** | Да | К какому проекту |
| **Договор** | Нет | К какому договору |
| **Тип** | Да | Из списка выше (5 типов) |
| **Сумма** | Нет | Общая сумма изменения |
| **Влияние на сроки** | Нет | На сколько дней сдвигается график |
| **Описание** | Нет | Подробное обоснование |

4. Добавьте **позиции** (line items):

| Поле позиции | Описание |
|-------------|----------|
| **Описание** | Что именно меняется |
| **Количество** | Объём |
| **Единица** | м³, м², шт, т |
| **Цена за единицу** | Стоимость |
| **Итого** | Автоматически: кол-во × цена |
| **Код затрат** | К какой статье бюджета относится |

5. Нажмите **«Создать»** — change order в статусе «Черновик»

### Статусы change order

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** (DRAFT) | Серый | Создан, можно редактировать |
| **Ожидает утверждения** (PENDING_APPROVAL) | Жёлтый | Отправлен на утверждение |
| **Утверждён** (APPROVED) | Зелёный | Утверждён, можно выполнять работы |
| **Исполнен** (EXECUTED) | Синий | Работы выполнены, оплата произведена |
| **Аннулирован** (VOID) | Серый | Отменён |

```
Черновик → Ожидает утверждения → Утверждён → Исполнен
                                              Аннулирован ← (из любого)
```

### Влияние на договор

При утверждении change order система рассчитывает:
- **Первоначальная сумма договора** — сколько было
- **Сумма изменения** — сколько добавляется/вычитается
- **Пересмотренная сумма** — сколько стало
- **Процент изменения** — на сколько процентов изменился бюджет

Пример:
```
Первоначальная сумма: 50 000 000 ₽
Change Order №1: + 3 200 000 ₽ (доп. работы по фундаменту)
Change Order №2: - 800 000 ₽ (замена материала фасада)
Change Order №3: + 1 500 000 ₽ (доп. этаж парковки)

Пересмотренная сумма: 53 900 000 ₽
Изменение: +7.8%
```

### Канбан-доска

**Путь:** Управление изменениями → **Доска**

Change orders разложены по колонкам-статусам. Удобно видеть общую картину: сколько в работе, сколько ожидает, сколько утверждено.

## Часть 3: Аналитика

**Путь:** Управление изменениями → **Дашборд**

Дашборд показывает влияние всех изменений на проект:

### Влияние на бюджет
- Первоначальная сумма договора
- Общая сумма утверждённых дополнений
- Общая сумма утверждённых вычетов
- Чистое изменение (нетто)
- Пересмотренная сумма
- Процент изменения
- Ожидающие утверждения (сумма)

### Влияние на сроки
- Общее влияние на график (дней)
- Влияние на критический путь
- Количество change orders на критическом пути

### Тенденции
- Ежемесячный тренд (сколько изменений в месяц)
- Распределение по источникам (откуда приходят изменения)
- Распределение по типам (дополнения / вычеты / замены)
- Накопительная стоимость

## Типичные сценарии

### Сценарий 1: Грунтовые воды

```
1. При рытье котлована обнаружены грунтовые воды
   на отметке -3.000 (по проекту не предусмотрено)

2. Прораб создаёт событие изменения:
   → Источник: Полевые условия
   → Оценка стоимости: ~2 500 000 ₽ (водопонижение)
   → Оценка сроков: +14 дней

3. Руководитель проекта рассматривает:
   → Статус: На рассмотрении → Утверждён для расценки

4. Подрядчик предоставляет расценку:
   → Водопонижение иглофильтрами: 2 300 000 ₽
   → Статус: Расценён

5. Заказчик утверждает → Статус: Утверждён

6. Создаётся Change Order:
   → Тип: Дополнительные работы
   → Позиции:
     - Водопонижение: 2 300 000 ₽
     - Допнасыпь щебня: 200 000 ₽
   → Итого: 2 500 000 ₽
   → Влияние на сроки: +14 дней
   → Договор: 50 000 000 → 52 500 000 ₽ (+5%)

7. Утверждение → Выполнение работ → Исполнен
```

### Сценарий 2: Заказчик меняет планировку

```
1. Заказчик хочет объединить 2 офиса в один на 3 этаже

2. Событие изменения:
   → Источник: Запрос заказчика
   → Описание: «Объединение помещений 305 и 306 (снос перегородки,
     перенос розеток, перенос вентканала)»

3. Расценка от подрядчика:
   → Демонтаж перегородки: 45 000 ₽
   → Перенос 4 розеток: 28 000 ₽
   → Перенос вентканала: 120 000 ₽
   → Восстановление пола: 35 000 ₽
   → Итого: 228 000 ₽

4. Change Order:
   → Тип: Комбинированный (демонтаж + новые работы)
   → Сумма: 228 000 ₽
   → Влияние на сроки: +3 дня

5. Заказчик утверждает → работы выполняются
```

## Частые вопросы

**Что такое change order простыми словами?**
Это дополнительное соглашение к договору. Когда нужно сделать что-то сверх первоначального контракта — оформляется change order с указанием стоимости и сроков.

**Обязательно ли фиксировать каждое изменение?**
Да. Неоформленные изменения — главная причина споров на стройке. «Вы сказали, мы сделали» без документа = нет оплаты.

**Кто утверждает change order?**
Зависит от суммы и организации. Обычно: до 500 000 ₽ — руководитель проекта, свыше — генеральный директор или заказчик.

**Что если заказчик не утверждает change order?**
Работы по этому изменению не выполняются. Если подрядчик выполнит без утверждения — оплата не гарантирована.

**Нормально ли, что change orders есть на каждом проекте?**
Да. На строительных проектах изменения неизбежны. Нормальный уровень: 5–15% от суммы контракта. Если больше 20% — проблема с качеством проектной документации.

**Как посмотреть все изменения по проекту?**
На дашборде управления изменениями видна полная картина: все события, все change orders, общее влияние на бюджет и сроки.
