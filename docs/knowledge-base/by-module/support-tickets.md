# Поддержка (тикеты)

---

## Что это и зачем

**Тикет** — это обращение в службу поддержки. Кто-то не может войти в систему, нашёл ошибку, хочет новую функцию — всё это фиксируется как тикет.

Без системы тикетов:
- Обращения приходят по телефону, почте, в мессенджере — и теряются
- Никто не знает, кто чем занимается
- Одну и ту же проблему решают два человека
- Нет статистики: сколько обращений, какие проблемы самые частые

Модуль «Поддержка» в системе ПРИВОД:
- Все обращения в одном месте
- Каждый тикет имеет статус, приоритет, ответственного
- Канбан-доска для наглядного управления
- Комментарии и внутренние заметки
- Статистика: время решения, нагрузка, категории

## Часть 1: Создание тикета

### Поля тикета

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Тема** | Да | Краткое описание проблемы |
| **Описание** | Да | Подробности: что произошло, что ожидали, что получили |
| **Категория** | Да | Тип обращения (из списка ниже) |
| **Приоритет** | Да | Насколько срочно |
| **Срок** | Нет | До когда нужно решить |

---

## Часть 2: Категории обращений

| Категория | Описание | Пример |
|-----------|----------|--------|
| **Техническая** (TECHNICAL) | Проблемы с работой системы | Страница не загружается, ошибка при сохранении |
| **Доступ** (ACCESS) | Проблемы со входом и правами | Не могу войти, нет доступа к модулю |
| **Документы** (DOCUMENTS) | Вопросы по документам | Не могу скачать файл, ошибка в генерации |
| **Оборудование** (EQUIPMENT) | Проблемы с оборудованием | Сканер не работает, принтер не печатает |
| **Безопасность** (SAFETY) | Вопросы безопасности | Подозрительная активность в аккаунте |
| **График** (SCHEDULE) | Вопросы по календарю и срокам | Неправильные даты, не обновляется Гантт |
| **Ошибка** (BUG) | Явная ошибка в программе | Кнопка не работает, неправильный расчёт |
| **Вопрос** (QUESTION) | Как сделать что-то в системе | Как создать КС-2? Как экспортировать отчёт? |
| **Запрос на доработку** (FEATURE_REQUEST) | Хочу новую функцию | Добавить фильтр по дате, новый тип отчёта |
| **Прочее** (OTHER) | Всё, что не попадает в другие категории | Предложение по улучшению |

---

## Часть 3: Приоритеты

| Приоритет | Цвет | Описание | Пример |
|-----------|------|----------|--------|
| **Низкий** | Серый | Не срочно, можно подождать | Хочу поменять цвет графика |
| **Средний** | Жёлтый | Стандартное обращение | Не могу найти нужный отчёт |
| **Высокий** | Оранжевый | Мешает работе | Не могу создать КС-2 |
| **Критический** | Красный | Полностью блокирует работу | Система не загружается, данные пропали |

---

## Часть 4: Статусы тикета

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Открыт** (OPEN) | Синий | Тикет создан, ещё никому не назначен |
| **Назначен** (ASSIGNED) | Голубой | Назначен специалисту поддержки |
| **В работе** (IN_PROGRESS) | Оранжевый | Специалист занимается решением |
| **Ожидание ответа** (WAITING_RESPONSE) | Жёлтый | Ждём дополнительной информации от пользователя |
| **Решён** (RESOLVED) | Зелёный | Решение найдено и применено |
| **Закрыт** (CLOSED) | Серый | Тикет закрыт, проблема решена |

### Порядок прохождения

```
Открыт → Назначен → В работе → Решён → Закрыт
                        ↓
                  Ожидание ответа → (получили ответ) → В работе
```

## Часть 5: Канбан-доска

**Путь:** Поддержка → **Доска**

Тикеты раскладываются по **6 колонкам** (по статусам):

```
| Открыт | Назначен | В работе | Ожидание ответа | Решён | Закрыт |
|--------|----------|----------|-----------------|-------|--------|
| ■ ■    | ■ ■ ■    | ■ ■      | ■               | ■ ■   | ■ ■ ■  |
```

Каждая карточка показывает: номер, тему, приоритет, ответственного.

Можно **перетаскивать** карточки между колонками — статус меняется автоматически.

## Часть 6: Карточка тикета

На детальной странице тикета:

### Информация

| Блок | Содержание |
|------|-----------|
| **Заголовок** | Номер (TK-001), тема, описание |
| **Статус и приоритет** | С цветными индикаторами |
| **Автор** | Кто создал тикет |
| **Ответственный** | Кто решает (можно переназначить) |
| **Дата создания** | Когда создан |
| **Срок** | До какой даты нужно решить |
| **Дата решения** | Когда решён (если решён) |

### Действия

- **Назначить** — выбрать ответственного
- **Изменить статус** — перевести на следующий этап
- **Добавить комментарий** — написать ответ или заметку
- **Оценка** — после решения пользователь может поставить оценку (1–5 ⭐)

### Комментарии

Под карточкой — лента комментариев:
- Кто написал, когда
- Текст комментария
- **Внутренний комментарий** — виден только команде поддержки (пользователь не видит)

| **Последние тикеты** | 5 последних обращений |

## Часть 8: SLA (время реакции)

Для каждой **категории** тикетов можно настроить **SLA** — максимальное время реакции:

| Категория | Время реакции (пример) |
|-----------|------------------------|
| Критическая ошибка | 1 час |
| Техническая проблема | 4 часа |
| Доступ | 2 часа |
| Вопрос | 8 часов |
| Запрос на доработку | 24 часа |

Если SLA нарушен — тикет подсвечивается как просроченный.

## Типичные сценарии

### Сценарий 1: Пользователь не может войти

```
1. Бухгалтер Сидорова не может войти в систему

2. Звонит в поддержку → создаётся тикет:
   → Тема: «Не могу войти — ошибка "неверный пароль"»
   → Категория: Доступ
   → Приоритет: Высокий
   → Статус: Открыт (TK-047)

3. Тикет назначен на администратора Петрова:
   → Статус: Назначен → В работе

4. Петров проверяет:
   → Внутренний комментарий: «Аккаунт заблокирован
     после 5 неудачных попыток входа. Разблокировал,
     сбросил пароль.»

5. Комментарий пользователю:
   «Пароль сброшен. Новый пароль отправлен на почту.
    При первом входе система попросит его изменить.»

6. Статус: Решён ✅
   → Время решения: 25 минут (SLA = 2 часа ✅)

7. Сидорова подтверждает → Закрыт
   → Оценка: ⭐⭐⭐⭐⭐ (5 из 5)
```

### Сценарий 2: Ошибка при создании КС-2

```
1. Инженер ПТО сообщает об ошибке:
   → Тема: «При создании КС-2 ошибка 500»
   → Категория: Ошибка (BUG)
   → Приоритет: Критический
   → Описание: «При нажатии "Создать КС-2" для проекта
     ЖК Южный парк — появляется ошибка.
     Скриншот прилагается.»

2. Тикет: TK-048, статус: Открыт

3. Назначен на разработчика Козлова:
   → Статус: В работе
   → Внутренний комментарий: «Проблема в расчёте НДС
     при нулевой сумме строки. Исправляю.»

4. Исправлено → Статус: Решён
   → Комментарий: «Ошибка исправлена. Попробуйте создать
     КС-2 повторно.»

5. Инженер подтверждает: работает → Закрыт ✅
   → Время решения: 3 часа (SLA = 1 час ❌ — просрочен)
```

## Частые вопросы

**Кто может создавать тикеты?**
Любой пользователь системы. Тикеты — это способ сообщить о проблеме или задать вопрос.

**Кто видит внутренние комментарии?**
Только команда поддержки (администраторы и назначенные специалисты). Обычные пользователи видят только публичные комментарии.

**Можно ли переназначить тикет другому специалисту?**
Да. На карточке тикета нажмите «Назначить» и выберите нового ответственного.

**Что значит «Ожидание ответа»?**
Это значит, что специалист задал уточняющий вопрос и ждёт ответа от пользователя. Пока пользователь не ответит — работа приостановлена.

**Зачем нужна оценка?**
Оценка помогает понять качество работы поддержки. Если средняя оценка низкая — нужно что-то улучшать.

**Как быстро должны ответить на мой тикет?**
Зависит от категории и приоритета. Критические ошибки — обычно в течение часа. Обычные вопросы — в течение рабочего дня.
