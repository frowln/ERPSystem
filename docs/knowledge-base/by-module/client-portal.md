# Портал клиента / подрядчика

---

## Что это и зачем

**Портал** — это отдельный вход в систему ПРИВОД для **внешних пользователей**: заказчиков, подрядчиков, субподрядчиков и поставщиков. Они видят только то, что вы им разрешили — и ничего лишнего.

Без портала:
- Заказчик звонит каждый день: «Как дела на объекте?» — отвлекает команду
- Подрядчик присылает акт по почте — теряется в потоке писем
- Заказчик хочет посмотреть документы — нужно копировать и отправлять вручную
- Субподрядчик не знает, какие задачи ему назначены — звонит прорабу

Портал решает все эти проблемы:
- Заказчик **сам** смотрит прогресс, документы, фото с объекта
- Подрядчик **сам** отправляет акты КС-2 через портал
- Все видят свои задачи, дефекты, запросы информации
- Общение через встроенные сообщения — всё в одном месте

## Часть 1: Роли в портале

Каждый пользователь портала имеет свою **роль**:

| Роль | Описание | Что обычно видит |
|------|----------|-----------------|
| **Заказчик** (CUSTOMER) | Тот, кто заказывает и оплачивает работу | Прогресс, документы, счета, контракты, дефекты |
| **Подрядчик** (CONTRACTOR) | Тот, кто выполняет работы | Задачи, КС-2, документы, дефекты, RFI |
| **Субподрядчик** (SUBCONTRACTOR) | Специализированный подрядчик | Задачи, свои документы, дефекты |
| **Поставщик** (SUPPLIER) | Поставщик материалов | Счета, документы на поставку |

---

## Часть 2: Уровни доступа

Помимо роли, каждому пользователю назначается **уровень доступа**:

| Уровень | Описание |
|---------|----------|
| **Только просмотр** (VIEW_ONLY) | Может смотреть, но не может создавать/редактировать |
| **Ограниченный** (LIMITED) | Может создавать некоторые записи (например, дефекты) |
| **Полный** (FULL) | Полный доступ ко всем разрешённым разделам |

Дополнительно для каждого проекта настраиваются **точечные разрешения**:

| Разрешение | Что даёт |
|-----------|----------|
| ☑ Финансы | Видит бюджет, счета, оплаты |
| ☑ Документы | Видит проектную и исполнительную документацию |
| ☑ График | Видит календарный план и вехи |
| ☑ Фото | Видит фотоотчёты с объекта |

## Часть 3: Что видит клиент

### ✅ Что ВИДИТ пользователь портала

| Раздел | Что доступно |
|--------|-------------|
| **Дашборд** | Метрики: проекты, документы, открытые RFI, дефекты, ожидающие подписи |
| **Проекты** | Только проекты, к которым есть доступ. Статус, прогресс, бюджет (если разрешено) |
| **Документы** | Документы, которыми с ним поделились. Скачивание |
| **Контракты** | Свои контракты: сумма, статус, оплаченная доля |
| **Счета** | Счета по своим контрактам |
| **Задачи** | Только задачи, назначенные этому пользователю |
| **График** | Вехи проекта и их статус (если разрешено) |
| **Сообщения** | Переписка с командой проекта |
| **КС-2** | Черновики актов — создание, отправка, согласование |
| **RFI** | Свои запросы информации и ответы на них |
| **Дефекты** | Свои замечания и рекламации |
| **Подписи** | Запросы на подпись документов |
| **Фото** | Фотоотчёты с объекта (если разрешено) |
| **Дневные отчёты** | Ежедневные отчёты (если разрешено) |

### ❌ Что НЕ ВИДИТ пользователь портала

| Раздел | Почему скрыт |
|--------|-------------|
| **Внутренние задачи** | Задачи для своих сотрудников — внутреннее дело |
| **Себестоимость** | Ваши закупочные цены и маржа — коммерческая тайна |
| **Зарплаты и HR** | Кадровые данные — конфиденциально |
| **Внутренние дефекты** | Дефекты, найденные вашей командой (до передачи заказчику) |
| **Внутренние комментарии** | Внутренние примечания к дефектам и RFI |
| **Склад** | Складские остатки и движения |
| **Безопасность** | Инциденты и инструктажи |
| **Другие проекты** | Проекты, к которым нет доступа |
| **Другие контрагенты** | Данные других заказчиков и подрядчиков |
| **CRM** | Воронка продаж и лиды — чисто внутреннее |
| **Настройки системы** | Администрирование, интеграции, пользователи |

> **Принцип:** Портал показывает **только то, что касается конкретного пользователя** по конкретным проектам. Всё остальное недоступно.

## Часть 5: Дефекты и рекламации

Через портал заказчик может **сообщить о дефекте** (замечании), не звоня по телефону.

### Категории дефектов

| Категория | Описание |
|-----------|----------|
| **Конструктивные** | Трещины, деформации несущих конструкций |
| **Сантехника** | Протечки, неисправности водопровода/канализации |
| **Электрика** | Проблемы с проводкой, розетками, освещением |
| **ОВиК** | Вентиляция, кондиционирование, отопление |
| **Отделка** | Дефекты покраски, плитки, обоев |
| **Фасад** | Проблемы с наружной отделкой |
| **Лифты** | Неисправности лифтового оборудования |
| **Пожарная безопасность** | Сигнализация, оповещение, тушение |
| **Прочее** | Всё остальное |

### Приоритеты

| Приоритет | Цвет | Описание |
|-----------|------|----------|
| **Низкий** | Серый | Косметический дефект, не влияет на эксплуатацию |
| **Средний** | Жёлтый | Требует устранения, но не срочно |
| **Высокий** | Оранжевый | Существенный дефект, влияет на использование |
| **Критический** | Красный | Угрожает безопасности или делает невозможным использование |

### Статусы дефекта

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Подан** (SUBMITTED) | Синий | Заказчик зафиксировал дефект |
| **Распределён** (TRIAGED) | Жёлтый | Ваша команда увидела и оценила |
| **Назначен** (ASSIGNED) | Оранжевый | Назначен подрядчик для устранения |
| **В работе** (IN_PROGRESS) | Синий | Идёт устранение |
| **На проверке** (VERIFICATION) | Фиолетовый | Устранено, заказчик проверяет |
| **Закрыт** (CLOSED) | Зелёный | Заказчик подтвердил устранение ✅ |
| **Отклонён** (REJECTED) | Красный | Не признан дефектом |

### Что видит заказчик при создании дефекта

| Поле | Описание |
|------|----------|
| **Заголовок** | Краткое описание проблемы |
| **Категория** | Из списка выше |
| **Приоритет** | Низкий — Критический |
| **Описание** | Подробности: что, где, когда обнаружено |
| **Местоположение** | Где именно (этаж, помещение, ось) |
| **Фото** | До 10 фотографий дефекта |

> Ваша команда может добавлять **внутренние комментарии**, которые заказчик НЕ видит.

## Часть 6: КС-2 через портал

Подрядчик может **создавать черновики актов КС-2** и отправлять их на согласование через портал.

### Статусы черновика КС-2

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** (DRAFT) | Серый | Подрядчик готовит акт |
| **Подан** (SUBMITTED) | Синий | Отправлен на проверку |
| **На проверке** (UNDER_REVIEW) | Жёлтый | Ваша команда проверяет |
| **Утверждён** (APPROVED) | Зелёный | Акт принят ✅ |
| **Отклонён** (REJECTED) | Красный | Есть замечания, нужна доработка |
| **Конвертирован** (CONVERTED) | Серый | Создан официальный КС-2 в системе |

### Что содержит черновик

| Поле | Описание |
|------|----------|
| **Номер черновика** | Автоматический |
| **Проект** | К какому проекту |
| **Отчётный период** | Начало и конец периода (месяц) |
| **Описание работ** | Что было выполнено |
| **Сумма** | Общая стоимость выполненных работ |

## Часть 7: RFI (запросы информации)

Подрядчик или заказчик может задать **официальный вопрос** по проекту через RFI (Request for Information).

### Статусы RFI в портале

| Статус | Цвет | Описание |
|--------|------|----------|
| **Открыт** (OPEN) | Красный | Вопрос задан, ждёт ответа |
| **Назначен** (ASSIGNED) | Жёлтый | Назначен ответственный |
| **Отвечен** (ANSWERED) | Зелёный | Ответ дан |
| **Закрыт** (CLOSED) | Серый | Вопрос решён |

Каждый RFI может содержать:
- Тему и подробный вопрос
- Приоритет (низкий / обычный / высокий / критический)
- Раздел проекта (АР, КР, ОВ и т.д.)
- Влияние на стоимость и сроки
- Дискуссию: вопросы-ответы (треды)

## Часть 8: Подписи документов

Через портал можно **запросить электронную подпись** на документ.

| Статус | Цвет | Описание |
|--------|------|----------|
| **Ожидает** (PENDING) | Жёлтый | Документ ждёт вашей подписи |
| **Подписан** (SIGNED) | Зелёный | Вы подписали ✅ |
| **Отклонён** (REJECTED) | Красный | Вы отклонили (с указанием причины) |
| **Просрочен** (EXPIRED) | Серый | Срок подписания истёк |

## Часть 9: Администрирование портала

**Путь:** Портал → **Администрирование** (доступно только роли ADMIN)

### Управление пользователями

| Действие | Описание |
|----------|----------|
| **Пригласить** | Создать пользователя портала (имя, email, телефон, компания, роль) |
| **Назначить проект** | Дать доступ к конкретному проекту с уровнем доступа |
| **Отозвать доступ** | Убрать доступ к проекту |
| **Приостановить** | Временно заблокировать пользователя |
| **Деактивировать** | Полностью заблокировать |

### Статусы пользователя портала

| Статус | Цвет | Описание |
|--------|------|----------|
| **Активен** (ACTIVE) | Зелёный | Может входить и работать |
| **Приглашён** (INVITED) | Жёлтый | Приглашение отправлено, ещё не вошёл |
| **Приостановлен** (SUSPENDED) | Оранжевый | Временно заблокирован |
| **Деактивирован** (DEACTIVATED) | Серый | Полностью заблокирован |

### Настройка видимости

Администратор может включить/выключить разделы портала для всех пользователей:
- Прогресс проекта
- Документы
- Счета
- График
- Контракты
- RFI
- Дефекты
- Подписи
- Фото
- Ежедневные отчёты

## Часть 10: Фотоотчёты и ежедневные отчёты

### Фотоотчёты

Через портал можно загружать и просматривать **фото с объекта**:

| Категория | Описание |
|-----------|----------|
| **Прогресс** | Фото хода работ |
| **Дефект** | Фото обнаруженного дефекта |
| **Материалы** | Фото поступивших материалов |
| **Безопасность** | Фото нарушений или соответствия ОТ |
| **Общее** | Прочие фото |

Каждое фото содержит: название, описание, категорию, дату, автора, GPS-координаты (если есть).

### Ежедневные отчёты

Подрядчик отправляет **ежедневный отчёт** о работе:

| Поле | Описание |
|------|----------|
| **Дата** | За какой день |
| **Погода** | Солнечно / Облачно / Дождь / Снег / Ветер / Туман |
| **Температура** | Минимум и максимум (°C) |
| **Рабочие** | Количество рабочих на площадке |
| **Техника** | Количество единиц техники |
| **Выполненные работы** | Описание того, что сделано за день |
| **Использованные материалы** | Что израсходовано |
| **Проблемы** | Если что-то мешало работе |
| **Охрана труда** | Замечания по ОТ |
| **Фото** | Фотографии за день |

Статусы отчёта: **Черновик** → **Подан** → **Утверждён** / **Отклонён**

## Типичные сценарии

### Сценарий 1: Заказчик проверяет прогресс

```
1. Директор застройщика ООО «ЮгСтрой» заходит в портал

2. Дашборд показывает:
   → 2 проекта доступны
   → 3 документа новых
   → 1 подпись ожидает
   → 0 дефектов открыто

3. Открывает проект «ЖК Южный парк»:
   → Прогресс: 62%
   → Фаза: Надземная часть
   → Следующая веха: «Монтаж 5 этажа» — 20.05.2026

4. Смотрит фотоотчёт:
   → 15 фото за последнюю неделю
   → Видно: каркас 4 этажа завершён

5. Проверяет документы:
   → Новый акт АОСР на армирование перекрытия
   → Скачивает → подписывает на портале → Подписано ✅

6. Всё это — без единого звонка прорабу
```

### Сценарий 2: Подрядчик подаёт акт КС-2

```
1. Подрядчик ООО «МонтажСтрой» входит в портал

2. Переходит в «КС-2 черновики» → Новый черновик:
   → Проект: ЖК Южный парк
   → Период: 01.04 — 30.04.2026
   → Описание: «Монтаж металлоконструкций 4 этаж»
   → Сумма: 4 200 000 ₽

3. Нажимает «Подать» → статус: Подан

4. Ваш ПТО получает уведомление → проверяет:
   → Объёмы соответствуют? Да ✅
   → Цены по договору? Да ✅
   → Статус: Утверждён ✅

5. На основании утверждённого черновика создаётся
   официальный КС-2 в системе → далее оплата
```

### Сценарий 3: Заказчик фиксирует дефект

```
1. Заказчик обнаружил протечку в подвале

2. Создаёт дефект через портал:
   → Категория: Сантехника
   → Приоритет: Высокий
   → Описание: «Протечка в подвале, ось В/3,
     вода на полу, примерно 0.5 м²»
   → Место: Подвал, ось В/3
   → Фото: 3 снимка

3. Ваша команда видит новый дефект:
   → Статус: Подан → Распределён
   → Назначают субподрядчику по сантехнике
   → Статус: Назначен → В работе

4. Внутренний комментарий (заказчик НЕ видит):
   «Вероятно, недогерметизация стыка трубы.
    Нужно вскрыть гипсокартон и проверить»

5. Через 2 дня устранено:
   → Статус: На проверке
   → Заказчик проверяет → Закрыт ✅

6. Заказчик ставит оценку: ⭐⭐⭐⭐ (4 из 5)
```

## Частые вопросы

**Кто может создать пользователя портала?**
Только администратор (роль ADMIN) через раздел «Администрирование портала».

**Безопасно ли давать доступ заказчику?**
Да. Пользователь портала видит **только** то, что ему разрешено. Он не может зайти в основную систему, не видит себестоимость, зарплаты, внутренние задачи и другие проекты.

**Можно ли дать доступ к одному проекту, а к другому — нет?**
Да. Доступ настраивается отдельно для каждого проекта. Можно дать доступ к проекту А с полным доступом, а к проекту Б — только на просмотр.

**Видит ли заказчик нашу маржу?**
Нет. Заказчик видит суммы контракта и выставленные счета. Себестоимость, закупочные цены и маржа — скрыты.

**Может ли подрядчик создавать документы?**
Зависит от уровня доступа. С уровнем «Полный» — может создавать КС-2 черновики, RFI, дефекты. С уровнем «Только просмотр» — только смотреть.

**Что такое SLA для дефектов?**
Это максимальный срок устранения дефекта. Если дедлайн нарушен — система помечает дефект как просроченный (SLA breached).

**Как заказчик получает уведомления?**
При создании новых документов, ответах на RFI, смене статуса дефекта — заказчик видит обновления на дашборде при следующем входе в портал.
