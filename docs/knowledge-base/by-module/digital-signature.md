# Электронная подпись (КЭП)

---

## Что это и зачем

**КЭП** (квалифицированная электронная подпись) — это цифровой аналог собственноручной подписи. По Федеральному закону №63-ФЗ «Об электронной подписи» документ, подписанный КЭП, **юридически равнозначен** бумажному документу с собственноручной подписью и печатью.

Зачем нужна КЭП на стройке:
- **Подписание АОСР** — вместо сбора подписей 4-5 человек в разных офисах
- **Утверждение КС-2, КС-3** — акты выполненных работ подписываются электронно
- **Согласование договоров** — не нужно ездить с бумагами
- **Подача отчётности** — в ИСУП (информационная система управления проектами Минстроя)
- **Скорость** — документ подписывается за минуту, а не за неделю

В системе ПРИВОД модуль «КЭП» позволяет:
- Хранить электронные сертификаты
- Подписывать любые документы системы
- Отправлять запросы на подпись другим людям
- Проверять подлинность подписей
- Вести реестр машиночитаемых доверенностей (МЧД)

## Часть 1: Что такое КЭП простыми словами

Представьте, что у вас есть **электронный ключ** (как ключ от квартиры, только цифровой). Этот ключ:
- Выдан **удостоверяющим центром** (аналог нотариуса)
- Привязан к вашему ИНН и ОГРН организации
- Имеет **срок действия** (обычно 1 год)
- Хранится на **защищённом носителе** (Рутокен, JaCarta) или в облаке

Когда вы «подписываете» документ этим ключом — создаётся математическое доказательство, что:
1. Документ подписал именно вы (а не кто-то другой)
2. Документ не изменялся после подписания
3. Подпись поставлена в определённое время

### Виды электронных подписей (по 63-ФЗ)

| Вид | Юридическая сила | Где применяется |
|-----|:----------------:|----------------|
| **Простая (ПЭП)** | Ограниченная | Логин/пароль на сайте госуслуг |
| **Усиленная неквалифицированная (УНЭП)** | По соглашению сторон | Внутренний документооборот |
| **Усиленная квалифицированная (КЭП)** | Полная (= собственноручная) | Госзакупки, отчётность, ИСУП, юридические документы |

Система ПРИВОД работает с **КЭП** — самым надёжным видом, который имеет полную юридическую силу.

## Часть 2: Сертификаты

**Сертификат** — это электронный документ, который подтверждает, что ключ подписи принадлежит конкретному человеку или организации.

### Что содержит сертификат

| Поле | Описание |
|------|----------|
| **Серийный номер** | Уникальный номер сертификата |
| **Владелец (CN)** | ФИО владельца |
| **Организация** | Название компании |
| **ИНН** | Идентификационный номер налогоплательщика |
| **ОГРН** | Основной государственный регистрационный номер |
| **Удостоверяющий центр** | Кто выдал сертификат |
| **Действует с** | Дата начала действия |
| **Действует до** | Дата окончания (обычно 1 год) |
| **Отпечаток** | Хэш сертификата для идентификации |

### Статусы сертификата

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Действующий** (ACTIVE) | Зелёный | Сертификат актуален, можно подписывать |
| **Истёк** (EXPIRED) | Серый | Срок действия закончился — нужно получить новый |
| **Отозван** (REVOKED) | Красный | Сертификат отозван удостоверяющим центром (утеря, компрометация) |
| **Приостановлен** (SUSPENDED) | Жёлтый | Временно приостановлен |

### Поддерживаемые криптопровайдеры

| Провайдер | Описание |
|-----------|----------|
| **КриптоПро** | Основной российский криптопровайдер (ГОСТ Р 34.10-2012) |
| **ViPNet** | Альтернативный криптопровайдер |
| **Рутокен** | Аппаратный токен (USB-ключ) |
| **JaCarta** | Аппаратный токен |

## Часть 3: Как подписать документ

**Путь:** ЭЦП → **Подписание**

Подписание проходит в **4 шага** (мастер подписания):

### Шаг 1: Выбор документа

Выберите, что подписываем:
- Акт КС-2
- Акт КС-3
- Договор
- Счёт
- АОСР
- Любой другой документ из системы

### Шаг 2: Выбор сертификата

Выберите свой сертификат из списка. Система показывает:
- Владелец
- Удостоверяющий центр
- Серийный номер
- Срок действия

> ⚠️ Если сертификат истёк — система предупредит и не позволит подписать.

### Шаг 3: Ввод ПИН-кода

Введите ПИН-код контейнера закрытого ключа. Это пароль, который защищает ваш ключ подписи.

> **Важно:** ПИН-код знаете только вы. Не сообщайте его никому. Система не хранит ваш ПИН-код.

### Шаг 4: Результат

Если всё успешно — документ подписан:
- Подписант: ФИО
- Дата подписания: точное время
- Удостоверяющий центр: кто выдал сертификат
- Подпись действительна: ✅

Если произошла ошибка (неверный ПИН, истёк сертификат) — система сообщит причину.

## Часть 4: Запросы на подпись

Не обязательно подписывать самому — можно **отправить запрос на подпись** другому человеку. Например, руководитель отправляет договор генеральному директору на подпись.

### Как отправить запрос

1. Выберите документ
2. Укажите, кому отправить запрос на подпись
3. Установите **срок** (до когда нужна подпись)
4. Установите **приоритет**: Обычный / Высокий / Срочный
5. Нажмите **«Отправить»**

Получатель увидит запрос в своём списке и сможет подписать или отклонить.

### Статусы запроса

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Ожидает подписания** (PENDING) | Жёлтый | Запрос отправлен, ждём |
| **Подписан** (SIGNED) | Зелёный | Подписано ✅ |
| **Отклонён** (REJECTED) | Красный | Подписант отказался (с указанием причины) |
| **Просрочен** (EXPIRED) | Серый | Срок истёк, подпись не получена |
| **Отменён** (CANCELLED) | Серый | Запрос отменён отправителем |

## Часть 5: Проверка подписи

**Путь:** ЭЦП → **Проверка подписи**

Если вы получили подписанный документ и хотите убедиться, что подпись подлинная — используйте проверку.

Система проверяет:
1. **Сертификат действителен?** — не истёк, не отозван
2. **Подпись математически верна?** — документ не изменялся после подписания
3. **OCSP-проверка** — онлайн-запрос в удостоверяющий центр для подтверждения статуса сертификата

Результаты проверки:

| Результат | Цвет | Что означает |
|-----------|------|-------------|
| **Действительна** (GOOD) | Зелёный | Подпись подлинная, сертификат действует |
| **Отозвана** (REVOKED) | Красный | Сертификат подписанта отозван |
| **Неизвестно** (UNKNOWN) | Жёлтый | Не удалось проверить (нет связи с УЦ) |
| **Ошибка** (ERROR) | Красный | Техническая ошибка при проверке |

## Часть 6: МЧД (Машиночитаемая доверенность)

**МЧД** — это электронная доверенность, которая позволяет одному человеку подписывать документы от имени другого (или от имени организации).

По закону (с 01.09.2023) для подписания документов от имени юридического лица сотрудник должен иметь МЧД, если он не является руководителем.

### Что содержит МЧД

| Поле | Описание |
|------|----------|
| **Номер** | Регистрационный номер доверенности |
| **Доверитель** | Кто выдал доверенность (организация, ИНН) |
| **Представитель** | Кому выдана (ФИО, ИНН) |
| **Полномочия** | Что разрешено делать (подписывать акты, договоры и т.д.) |
| **Действует с** | Дата начала |
| **Действует до** | Дата окончания |
| **Регистрационный номер** | Номер в реестре ФНС |

### Статусы МЧД

| Статус | Описание |
|--------|----------|
| **Действует** | МЧД активна |
| **Истёк срок** | Срок действия закончился |
| **Отозвана** | Доверитель отозвал доверенность |
| **Приостановлена** | Временно приостановлена |

## Типичные сценарии

### Сценарий 1: Подписание АОСР электронной подписью

```
1. Прораб оформил АОСР на армирование плиты
   → Нужны подписи 4 человек:
   — Представитель подрядчика (прораб)
   — Авторский надзор (проектировщик)
   — Технический надзор (заказчик)
   — Представитель застройщика

2. Без КЭП: нужно привезти бумажный акт в 3 разных офиса
   → Минимум 2-3 дня

3. С КЭП в системе ПРИВОД:
   → Прораб подписывает на планшете (1 минута)
   → Отправляет запрос на подпись остальным трём

4. Каждый получает уведомление → открывает систему →
   проверяет акт → подписывает КЭП:
   → Представитель подрядчика: Подписано ✅ (10:15)
   → Авторский надзор: Подписано ✅ (11:30)
   → Технический надзор: Подписано ✅ (14:00)
   → Представитель застройщика: Подписано ✅ (15:20)

5. Все 4 подписи за 1 день вместо 3 дней ✅
   → АОСР закрыт → можно бетонировать
```

### Сценарий 2: Проверка подписи на присланном документе

```
1. Заказчик прислал подписанный акт КС-2
   → Нужно убедиться, что подпись подлинная

2. Открываем проверку подписи:
   → Сертификат: действителен ✅
   → Подпись: математически верна ✅
   → OCSP: сертификат не отозван ✅
   → Результат: Подпись действительна ✅

3. Можно принимать документ в работу
```

### Сценарий 3: Истекший сертификат

```
1. Главный инженер пытается подписать договор
   → Система: «Срок действия сертификата истёк 15.02.2026»
   → Подписание невозможно ❌

2. Нужно:
   → Обратиться в удостоверяющий центр
   → Получить новый сертификат
   → Загрузить в систему

3. Системный администратор видит в реестре:
   → 2 сертификата с истёкшим сроком (красные)
   → Отправляет уведомление владельцам: «Обновите сертификат»
```

## Частые вопросы

**Где получить КЭП?**
В аккредитованном удостоверяющем центре (УЦ). Для руководителей — в ФНС (бесплатно). Для сотрудников — в коммерческих УЦ (Контур, СКБ Контур, Тензор и др.).

**Сколько стоит КЭП?**
Для руководителей — бесплатно (через ФНС). Для сотрудников — от 1 500 до 5 000 ₽ в год в зависимости от УЦ и типа носителя.

**Нужен ли специальный носитель (токен)?**
Рекомендуется. Рутокен или JaCarta — USB-ключ, на котором хранится закрытый ключ. Это безопаснее, чем хранение на компьютере.

**Может ли один человек иметь несколько сертификатов?**
Да. Например, один для организации «ООО СтройМонтаж», другой — для ИП.

**Что делать, если токен потерян?**
Немедленно обратиться в удостоверяющий центр для отзыва сертификата. Затем получить новый. Отозванным сертификатом подписать ничего нельзя.

**Обязательна ли КЭП на стройке?**
Для подачи отчётности в ИСУП (с 2024 года для госзаказов) — да. Для внутреннего документооборота — по решению организации, но значительно ускоряет работу.

**Можно ли подписать документ с телефона?**
Зависит от криптопровайдера. КриптоПро поддерживает мобильную подпись через облачный сервис. Рутокен — через NFC-считыватель.

**Что такое ГОСТ Р 34.10-2012?**
Это российский стандарт криптографии, по которому работает КЭП. Все аккредитованные УЦ выдают сертификаты по этому стандарту.
