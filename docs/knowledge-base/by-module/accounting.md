# Бухгалтерия

---

## Что это и зачем

Модуль «Бухгалтерия» — это полноценный бухгалтерский учёт внутри системы ПРИВОД. Здесь ведутся проводки, хранится план счетов, учитываются основные средства, формируются налоговые декларации и контролируется Единый налоговый счёт (ЕНС).

Модуль состоит из **17 сущностей**, объединённых в 6 блоков:

| Блок | Что входит |
|------|-----------|
| **План счетов** | Справочник бухгалтерских счетов (активные, пассивные, активно-пассивные) |
| **Проводки** | Журнальные записи: дебет, кредит, сумма |
| **Журналы** | Финансовые журналы по типам операций (банк, касса, продажи, закупки) |
| **Основные средства** | Учёт имущества, амортизация |
| **ЕНС и налоги** | Единый налоговый счёт, операции, платежи, сверки, декларации |
| **Контрагенты и центры затрат** | Справочники для аналитики |

**Бухгалтерия работает поверх финансового модуля:** [счета](invoices.md), [оплаты](payments.md), КС-2/КС-3 — это первичные документы. А проводки — это их отражение на бухгалтерских счетах.

## 1. План счетов

**Путь:** Бухгалтерия → **План счетов**

План счетов — это справочник всех бухгалтерских счетов организации. Каждый счёт имеет номер (код) и название.

### Стандартные счета (примеры)

| Код | Название | Тип |
|-----|----------|-----|
| 01 | Основные средства | Активный |
| 02 | Амортизация основных средств | Пассивный |
| 10 | Материалы | Активный |
| 20 | Основное производство | Активный |
| 26 | Общехозяйственные расходы | Активный |
| 41 | Товары | Активный |
| 51 | Расчётные счета | Активный |
| 60 | Расчёты с поставщиками | Активно-пассивный |
| 62 | Расчёты с покупателями | Активно-пассивный |
| 68 | Расчёты по налогам | Активно-пассивный |
| 70 | Расчёты с персоналом по оплате труда | Пассивный |
| 80 | Уставный капитал | Пассивный |
| 90 | Продажи | Активно-пассивный |
| 99 | Прибыли и убытки | Активно-пассивный |

> При создании организации в системе автоматически загружаются ~30 стандартных счетов российского плана счетов.

### Типы счетов

| Тип | Цвет | Что означает |
|-----|------|-------------|
| **Активный** | Синий | Учитывает имущество и средства (то, что есть у компании) |
| **Пассивный** | Фиолетовый | Учитывает обязательства и капитал (откуда деньги пришли) |
| **Активно-пассивный** | Оранжевый | Может быть и тем, и другим (например, расчёты с контрагентами — нам могут быть должны или мы должны) |

### Иерархия счетов

Счета могут быть **синтетическими** (верхний уровень, например «10 — Материалы») и **аналитическими** (детализация, например «10.1 — Сырьё», «10.3 — Топливо»). Аналитические счета привязаны к родительскому через поле `parentId`.

В таблице счета отображаются деревом — раскрывайте родительский счёт, чтобы увидеть аналитику.

### Колонки таблицы

| Колонка | Что показывает |
|---------|---------------|
| Код | Номер счёта (моноширинный шрифт), кнопка раскрытия для дочерних |
| Наименование | Название счёта (жирным — верхний уровень) |
| Тип | Активный / Пассивный / Активно-пассивный (цветная метка) |
| Уровень | Аналитический (зелёная метка) или Синтетический (серая) |

**Поиск:** по коду или названию счёта.

## 2. Проводки (журнальные записи)

**Путь:** Бухгалтерия → **Проводки**

Проводка — это бухгалтерская запись: откуда списано (дебет), куда зачислено (кредит) и какая сумма.

### Пример проводки

```
Дт 10 (Материалы)  —  Кт 60 (Поставщики)  —  500 000 руб.
```

Это означает: получили материалы на 500 000 руб. от поставщика. На счёте «Материалы» прибавилось, на счёте «Расчёты с поставщиками» — задолженность.

### Список проводок — колонки

| Колонка | Что показывает |
|---------|---------------|
| № | Номер проводки (автогенерация, формат «JE-00001») |
| Дата | Дата операции |
| Операция | Описание + документ-основание (если указан) |
| Дебет | Счёт списания (код — название) |
| Кредит | Счёт зачисления (код — название) |
| Сумма | Сумма проводки (руб.) |
| Автор | Кто создал |

**Подзаголовок:** количество проводок и отфильтрованная сумма.

### Фильтры

| Фильтр | Что делает |
|--------|-----------|
| Поиск | По номеру, описанию, типу документа или коду счёта |
| Счёт | Показать проводки только по конкретному счёту |
| Период | Показать проводки за конкретный учётный период |
| Журнал | Показать проводки из конкретного журнала |

### Массовые действия

Можно выделить несколько проводок и удалить их одной кнопкой (с подтверждением).

### Как создать проводку

1. Нажмите **«Новая проводка»**
2. Заполните форму:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Дата** | Да | Дата операции |
| **Период** | Да | Учётный период (должен быть открыт) |
| **Журнал** | Да | В какой журнал записать (банковский, кассовый и т.д.) |
| **Описание** | Да | Что за операция (например, «Оплата поставщику за цемент») |
| **Счёт дебета** | Да | Откуда списать (выбор из плана счетов) |
| **Счёт кредита** | Да | Куда зачислить (выбор из плана счетов) |
| **Сумма** | Да | Сумма операции (больше 0) |
| **Тип документа** | Нет | Тип первичного документа (счёт, акт, накладная) |
| **UUID документа** | Нет | Ссылка на документ в системе |
| **Примечание** | Нет | Дополнительный комментарий |

> **Подсказка:** Форма автоматически сохраняет черновик. Если вы закроете страницу и вернётесь — данные восстановятся. Черновик хранится 7 дней.

> **Важно:** Если нет открытых периодов, система покажет предупреждение и предложит открыть текущий период одной кнопкой.

3. Нажмите **«Создать проводку»**

### Карточка проводки

При нажатии на проводку открывается детальная страница:

- **Описание операции** — текст
- **Корреспонденция счетов** — таблица: Сторона (Дебет/Кредит) | Счёт | Сумма
- **Документ-основание** — тип и UUID связанного документа
- **Боковая панель** — номер, дата, автор, время создания, ID периода, ID журнала
- **Сумма** — выделена крупным шрифтом

**Действия:** Редактировать, Удалить.

## 3. Финансовые журналы

**Путь:** Бухгалтерия → **Финансовые журналы**

Журнал — это группировка проводок по типу операций. Каждая проводка записывается в один из журналов.

### Типы журналов

| Тип | Что содержит | Пример проводок |
|-----|-------------|----------------|
| **Банковский** | Операции по расчётному счёту | Оплата поставщику, получение от заказчика |
| **Кассовый** | Наличные операции | Выдача под отчёт, приём наличных |
| **Продажи** | Реализация работ/услуг | Выставление акта заказчику |
| **Закупки** | Приобретение материалов/услуг | Получение материалов от поставщика |
| **Общий** | Всё остальное | Начисление амортизации, зарплаты |

### Колонки таблицы журналов

| Колонка | Что показывает |
|---------|---------------|
| Код | Уникальный код журнала (например, «BANK», «CASH») |
| Название | Наименование (например, «Банковский журнал») |
| Тип | Банковский / Кассовый / Продажи / Закупки / Общий |
| Статус | Активный (зелёный) или Неактивный (серый) |
| Дата создания | Когда создан |
| Действия | Активировать / Деактивировать |

### Как создать журнал

В верхней части страницы — форма создания:

| Поле | Описание |
|------|----------|
| **Код** | Уникальный код (2–20 символов, только буквы, цифры, «_» и «-») |
| **Название** | Наименование журнала (2–500 символов) |
| **Тип** | Выбрать из списка: Банковский, Кассовый, Продажи, Закупки, Общий |

> При создании организации автоматически создаются стандартные журналы (BANK, CASH и др.) и открывается первый учётный период.

**Фильтры:** поиск по коду/названию, фильтр по типу.

## 4. Учётные периоды

Учётный период — это месяц, в который можно создавать проводки. Периоды бывают **открытые** и **закрытые**.

| Статус | Что означает |
|--------|-------------|
| **Открыт** | Можно создавать и редактировать проводки за этот месяц |
| **Закрыт** | Месяц закрыт, изменения запрещены |

### Как работать с периодами

- **Открыть период:** На странице проводок, если нет открытых периодов, система предложит открыть текущий месяц одной кнопкой
- **Закрыть период:** Через API (`POST /api/accounting/periods/{id}/close`). После закрытия проводки за этот месяц нельзя изменить

> **Важно:** Закрытие периода — необратимое действие. Перед закрытием убедитесь, что все проводки за месяц корректны.

## 5. Основные средства

**Путь:** Бухгалтерия → **Основные средства**

Основные средства (ОС) — это дорогостоящее имущество организации со сроком использования больше года: здания, техника, автомобили, оборудование, компьютеры.

### Показатели (4 карточки)

| Показатель | Что значит |
|-----------|-----------|
| Первоначальная стоимость | Общая стоимость всех ОС на момент покупки |
| Текущая стоимость | Сколько ОС стоят сейчас (с учётом амортизации) |
| Накопленная амортизация | Сколько уже списано |
| Ежемесячная амортизация | Сколько списывается каждый месяц |

### Вкладки-фильтры

| Вкладка | Что показывает |
|---------|---------------|
| **Все** | Все основные средства |
| **Черновик** | Только что добавленные, ещё не введены в эксплуатацию |
| **В эксплуатации** | Активно используются, амортизация начисляется |
| **Выбыло** | Списано, продано или утилизировано |

### Колонки таблицы

| Колонка | Что показывает |
|---------|---------------|
| Инв. номер | Инвентарный номер (уникальный) |
| Наименование | Что это (с кодом внизу) |
| Статус | Черновик (серый) / В эксплуатации (зелёный) / Выбыло (красный) |
| Дата покупки | Когда приобретено |
| Первоначальная стоимость | За сколько купили |
| Текущая стоимость | Сколько стоит сейчас |
| Ежемесячная амортизация | Сколько списывается в месяц |
| Износ | Процент износа (начислено / первоначальная × 100%) |

### Статусы основных средств

```
Черновик → В эксплуатации → Выбыло
```

| Статус | Что означает | Что можно делать |
|--------|-------------|-----------------|
| **Черновик** | Объект зарегистрирован, но ещё не используется | Редактировать, ввести в эксплуатацию |
| **В эксплуатации** | Используется, амортизация начисляется ежемесячно | Списать (вывести из эксплуатации) |
| **Выбыло** | Списано или продано | Только просмотр |

### Методы амортизации

| Метод | Как считает |
|-------|-----------|
| **Линейный** | Равными долями: стоимость ÷ срок (мес.) = ежемесячная сумма |
| **Уменьшаемого остатка** | От остаточной стоимости: 2 ÷ срок × текущая стоимость |
| **Суммы чисел лет** | Убывающая формула: оставшиеся годы ÷ сумма лет × стоимость ÷ 12 |

**Пример (линейный метод):**
```
Стоимость:  1 200 000 руб.
Срок:       60 месяцев (5 лет)
Ежемесячно: 1 200 000 ÷ 60 = 20 000 руб.

Через 2 года (24 месяца):
  Начислено:  24 × 20 000 = 480 000 руб.
  Остаток:    1 200 000 − 480 000 = 720 000 руб.
  Износ:      480 000 ÷ 1 200 000 = 40%
```

> Амортизация начисляется автоматически раз в месяц для всех ОС в статусе «В эксплуатации».

### Фильтры

- Поиск по наименованию, коду или инвентарному номеру
- Фильтр по методу амортизации

## 6. ЕНС (Единый налоговый счёт)

ЕНС — это единый счёт в Федеральной налоговой службе, куда организация перечисляет все налоги одним платежом. ФНС сама распределяет деньги по видам налогов. С 2023 года — обязательный порядок для всех организаций в России.

### Что входит в блок ЕНС

#### 6.1. Счёт ЕНС

Учётная запись единого налогового счёта организации.

| Поле | Описание |
|------|----------|
| ИНН | Идентификационный номер налогоплательщика (уникальный) |
| Номер счёта | Номер ЕНС в ФНС |
| Баланс | Текущий остаток на ЕНС |
| Последнее обновление | Когда в последний раз синхронизировали баланс |
| Статус | Активный / Неактивный |

**Синхронизация баланса:** Кнопка «Синхронизировать» пересчитывает баланс по всем подтверждённым платежам.

#### 6.2. Операции по ЕНС

Каждая операция — это начисление или списание на едином налоговом счёте.

| Тип операции | Что означает |
|-------------|-------------|
| **Уплата налога** | Перечисление денег на ЕНС |
| **Пеня** | Начисление пени за просрочку |
| **Штраф** | Начисление штрафа |
| **Возврат** | Возврат переплаты |
| **Зачёт** | Зачёт между налогами |
| **Корректировка** | Ручная корректировка |

**Статусы операций:**
- **Ожидает** — создана, ещё не обработана
- **Обработан** — проведена
- **Отменён** — отменена

#### 6.3. Платежи по ЕНС

Конкретные платёжные поручения на уплату налогов.

| Поле | Описание |
|------|----------|
| Сумма | Сумма платежа |
| Дата | Дата платежа |
| Вид налога | НДС, Налог на прибыль, Налог на имущество, Транспортный, НДФЛ, Страховые взносы, Прочие |
| Статус | Черновик → Подтверждён → Сверен |
| Квитанция | Ссылка на платёжный документ |

#### 6.4. Сверка ЕНС

Сверка — это проверка: совпадает ли ваш учёт с данными ФНС.

| Поле | Описание |
|------|----------|
| Входящий остаток | Остаток на начало периода |
| Ожидаемая сумма | Сколько должно быть начислено |
| Фактическая сумма | Сколько начислено по вашим данным |
| Обороты (дебет/кредит) | Суммы начислений и оплат |
| Исходящий остаток | Остаток на конец периода |
| Расхождение | Разница (если есть) |

**Статусы сверки:**

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **Черновик** | Серый | Создана, данные заполняются |
| **В процессе** | Жёлтый | Сверка идёт |
| **Совпадает** | Зелёный | Расхождений нет — всё в порядке |
| **Расхождение** | Красный | Найдены расхождения — нужно разобраться |
| **Урегулировано** | Зелёный | Расхождение найдено и исправлено |

## 7. Налоговые декларации

Налоговая декларация — это обязательный отчёт в ФНС за период.

### Типы деклараций

| Тип | Периодичность | Описание |
|-----|-------------|----------|
| **НДС** | Ежеквартально | Налог на добавленную стоимость |
| **Налог на прибыль** | Ежеквартально | Налог на прибыль организации |
| **Налог на имущество** | Ежегодно | Налог на основные средства |
| **УСН** | Ежегодно | Упрощённая система налогообложения |

### Статусы декларации

```
Черновик → Рассчитана → Подана → Принята
```

| Статус | Что означает |
|--------|-------------|
| **Черновик** | Создана, данные заполняются |
| **Рассчитана** | Суммы рассчитаны, готова к подаче |
| **Подана** | Отправлена в ФНС (фиксируется дата подачи) |
| **Принята** | ФНС приняла декларацию (фиксируется дата принятия) |

## 8. Контрагенты

Контрагент — это любая организация, с которой вы работаете: поставщик, заказчик, подрядчик, субподрядчик, проектировщик.

| Поле | Описание |
|------|----------|
| Название | Полное и краткое наименование |
| ИНН / КПП / ОГРН | Реквизиты организации |
| Юридический адрес | Адрес регистрации |
| Фактический адрес | Адрес офиса |
| Банковские реквизиты | Расчётный счёт, БИК, корр. счёт, банк |
| Контактное лицо | ФИО, телефон, email |
| Роли | Поставщик / Заказчик / Подрядчик / Субподрядчик / Проектировщик (можно несколько) |

Контрагенты используются в проводках (аналитика по партнёрам), в счетах, оплатах, договорах и закупках.

## 9. Центры затрат

Центр затрат — это подразделение или проект, на который относятся расходы. Например: «Площадка №1», «Офис», «Администрация».

| Поле | Описание |
|------|----------|
| Код | Уникальный код (например, «CC-001») |
| Название | Наименование центра |
| Проект | Привязка к проекту (необязательно) |
| Родительский | Иерархия (центр может быть частью другого центра) |

Центры затрат позволяют видеть, куда именно уходят деньги — по проектам и подразделениям.

## 10. Акт сверки

Акт сверки — это документ, который подтверждает взаиморасчёты с контрагентом за период.

| Поле | Описание |
|------|----------|
| Контрагент | С кем сверяемся |
| Период | За какой период |
| Наш дебет / кредит | Сколько мы начислили / оплатили |
| Их дебет / кредит | Сколько контрагент начислил / оплатил |
| Расхождение | Разница между нашими и их данными |

**Статусы:**

```
Черновик → Отправлен → Подтверждён или Оспорен
```

| Статус | Что означает |
|--------|-------------|
| **Черновик** | Создан, готовится к отправке |
| **Отправлен** | Отправлен контрагенту на проверку |
| **Подтверждён** | Контрагент согласен — расхождений нет |
| **Оспорен** | Контрагент не согласен — нужно разбираться |

## Журнал проводок — как читать

Журнал проводок (он же «Книга учёта хозяйственных операций») — это хронологический список всех бухгалтерских записей. Каждая строка — одна проводка.

### Как читать проводку

```
JE-00042 | 15.03.2026 | Получены материалы | Дт 10 (Материалы) | Кт 60 (Поставщики) | 500 000 руб.
```

**Расшифровка:**
- **JE-00042** — номер проводки (автоматический)
- **15.03.2026** — когда произошла операция
- **Получены материалы** — описание
- **Дт 10** — на счёте «Материалы» появилось (дебет = приход на активном счёте)
- **Кт 60** — возникла задолженность перед поставщиком (кредит = приход на пассивном счёте)
- **500 000 руб.** — сумма операции

### Типичные проводки в строительстве

| Операция | Дебет | Кредит | Пример |
|----------|-------|--------|--------|
| Получены материалы | 10 (Материалы) | 60 (Поставщики) | Привезли кирпич на 500 000 |
| Оплатили поставщику | 60 (Поставщики) | 51 (Расчётный счёт) | Перевели 500 000 за кирпич |
| Заказчик оплатил | 51 (Расчётный счёт) | 62 (Покупатели) | Получили 1 200 000 от заказчика |
| Начислена зарплата | 20 (Производство) | 70 (Зарплата) | Начислили 300 000 бригаде |
| Выплачена зарплата | 70 (Зарплата) | 51 (Расчётный счёт) | Перевели 261 000 (после НДФЛ) |
| Списаны материалы | 20 (Производство) | 10 (Материалы) | Использовали кирпич на 200 000 |
| Начислена амортизация | 20 (Производство) | 02 (Амортизация) | Списано 20 000 за экскаватор |
| Начислен НДС | 90 (Продажи) | 68 (Налоги) | НДС 200 000 с выручки |

## Бухгалтерские отчёты

На основании проводок система позволяет формировать стандартные бухгалтерские отчёты:

| Отчёт | Что показывает | Откуда данные |
|-------|---------------|--------------|
| **Оборотно-сальдовая ведомость (ОСВ)** | Обороты и сальдо по всем счетам за период | Сумма проводок по каждому счёту |
| **Анализ счёта** | Обороты конкретного счёта с разбивкой по корреспондирующим счетам | Все проводки по выбранному счёту |
| **Карточка счёта** | Все операции по счёту в хронологическом порядке | Проводки с дебетом или кредитом по счёту |
| **Журнал-ордер** | Обороты по кредиту счёта с разбивкой по дебетуемым счетам | Проводки из финансового журнала |
| **Баланс** | Активы = Пассивы на дату | Сальдо всех счетов |

> Отчёты формируются автоматически на основании проводок. Чем аккуратнее ведутся проводки — тем точнее отчётность.

## Связь с финансовым модулем

Бухгалтерия и финансовый модуль работают в паре:

| Финансовый модуль | Бухгалтерия |
|------------------|-------------|
| [Счёт](invoices.md) (первичный документ) | Проводка (отражение на счетах) |
| [Оплата](payments.md) (факт перечисления) | Проводка по банку (Дт 60 — Кт 51) |
| [КС-2](ks2.md) (акт выполненных работ) | Проводка по реализации (Дт 62 — Кт 90) |
| [Бюджет](budgets.md) (план/факт) | ОСВ и баланс (факт по счетам) |
| [Cash Flow](cash-flow.md) (движение денег) | Анализ счёта 51 (расчётный счёт) |

**Как связаны данные:**

```
Первичный документ (счёт, акт, накладная)
        ↓
Проводка (Дт — Кт — Сумма)
        ↓
Финансовый журнал (группировка по типу)
        ↓
Отчёты (ОСВ, баланс, анализ счёта)
```

> **На текущий момент** проводки создаются **вручную** — бухгалтер сам отражает операции на счетах. Автоматическое создание проводок из счетов и оплат — в планах развития.

## Интеграция с 1С

Система ПРИВОД поддерживает двустороннюю синхронизацию с **1С:Предприятие** через протокол OData.

### Где настроить

| Страница | Путь | Описание |
|----------|------|----------|
| Дашборд 1С | `/settings/1c` | Общий статус подключения, последние синхронизации |
| Настройка подключения | `/integrations/1c` | Параметры сервера, логин/пароль, база |
| Экспорт КС-2/КС-3 | `/settings/1c/ks-export` | Выгрузка актов в 1С |
| Обмен данными | `/data-exchange/1c-config` | Управление конфигурациями обмена |

### Что можно синхронизировать

| Данные | Направление | Описание |
|--------|-------------|----------|
| **Контрагенты** | ↔ Двустороннее | Справочник поставщиков, заказчиков, подрядчиков |
| **Номенклатура** | ↔ Двустороннее | Справочник материалов |
| **Сотрудники** | ← Из 1С | Кадровый справочник |
| **Счета** | ↔ Двустороннее | Счета на оплату |
| **Платёжные поручения** | → В 1С | Платежи |
| **КС-2 / КС-3** | → В 1С | Акты выполненных работ (как «Реализация товаров и услуг») |
| **Банковские выписки** | ← Из 1С | Импорт выписок для сверки |

### Как работает синхронизация

1. **Настройте подключение:** укажите адрес сервера 1С, имя базы, логин и пароль
2. **Проверьте связь:** кнопка «Тест подключения» — система проверит доступность
3. **Выберите направление:**
   - **Из 1С** — только загрузка данных в ПРИВОД
   - **В 1С** — только выгрузка из ПРИВОД
   - **Двустороннее** — обмен в обе стороны
4. **Запустите синхронизацию:**
   - **Полная** — все данные с нуля
   - **Инкрементальная** — только изменения с последней синхронизации
   - **Ручная** — по кнопке

### Автоматическая синхронизация

Можно настроить интервал автосинхронизации (по умолчанию — каждые 60 минут). Система сама будет обмениваться данными в фоне.

### Конфликты

Если одна и та же запись изменена и в ПРИВОД, и в 1С — система покажет конфликт. Вы можете выбрать:
- **Приоритет ПРИВОД** — данные из ПРИВОД перезапишут 1С
- **Приоритет 1С** — данные из 1С перезапишут ПРИВОД
- **Ручное решение** — вы сами выберете, какие данные оставить

### Журнал обмена

Все синхронизации записываются в журнал:

| Колонка | Что показывает |
|---------|---------------|
| Конфигурация | Какое подключение 1С |
| Тип данных | Контрагенты, материалы, счета и т.д. |
| Направление | Импорт / Экспорт / Двустороннее |
| Статус | Ожидает / В процессе / Завершён / Ошибка / Частично |
| Записи | Сколько обработано |
| Ошибки | Сколько ошибок |
| Время | Начало, окончание, длительность |
| Кто запустил | Имя пользователя |

### Экспорт КС-2/КС-3

При подписании акта КС-2 или справки КС-3 система может **автоматически** выгрузить документ в 1С как «Реализация товаров и услуг». Это работает в фоне — вам ничего не нужно делать вручную.

## Частые вопросы

**Зачем бухгалтерия, если есть модуль финансов?**
Финансовый модуль (счета, оплаты, бюджеты) — это управленческий учёт: «сколько потратили, сколько заработали». Бухгалтерия — это регламентированный учёт: двойная запись на счетах, для налоговой и отчётности. Они дополняют друг друга.

**Проводки создаются автоматически?**
На текущий момент проводки создаются вручную. Бухгалтер создаёт проводку, указывая дебет, кредит и сумму. В планах — автоматическое создание проводок при оплате счетов и подписании актов.

**Что такое учётный период и зачем его закрывать?**
Учётный период — это месяц. Пока период открыт — можно создавать и редактировать проводки. После закрытия — изменения запрещены. Закрытие гарантирует, что данные за месяц не будут случайно изменены после сдачи отчётности.

**Как связана бухгалтерия с 1С?**
Через протокол OData. Система может загружать из 1С контрагентов, материалы, сотрудников. И выгружать в 1С счета, платежи, акты КС-2/КС-3. Настройка — в разделе «Интеграции → 1С». Синхронизация может быть автоматической (по расписанию) или ручной.

**Что такое ЕНС?**
Единый налоговый счёт — с 2023 года все налоги платятся одним платежом на один счёт в ФНС. Модуль ЕНС помогает вести учёт этих платежей, отслеживать баланс и проводить сверку с данными налоговой.

**Можно ли вести учёт только в ПРИВОД, без 1С?**
Да. Модуль бухгалтерии самодостаточен: план счетов, проводки, журналы, ОС, налоги. Интеграция с 1С — опциональная, для тех, кто уже работает в 1С и хочет синхронизировать данные.

**Как начислить амортизацию?**
Амортизация начисляется автоматически раз в месяц для всех ОС в статусе «В эксплуатации». При создании ОС укажите метод (линейный, уменьшаемого остатка или суммы чисел лет) и срок полезного использования в месяцах. Система сама рассчитает ежемесячную сумму.

**Что означает «аналитический» и «синтетический» счёт?**
Синтетический счёт — это верхний уровень (например, «10 — Материалы»). Аналитический — это детализация (например, «10.1 — Сырьё», «10.3 — Топливо»). Синтетический показывает общую картину, аналитический — подробности. В плане счетов они связаны иерархией.

**Где посмотреть оборотно-сальдовую ведомость?**
ОСВ формируется из проводок. На странице проводок используйте фильтр по периоду — подзаголовок покажет общую сумму. Для полноценной ОСВ в разрезе счетов используйте группировку по счёту дебета/кредита.
