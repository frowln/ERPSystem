# Отпуска и отсутствия

---

## Что это и зачем

Модуль отпусков управляет всеми видами отсутствия сотрудников: ежегодный отпуск, больничный, отпуск без содержания, учебный, декретный. Система ведёт баланс дней, согласует заявления и показывает, кто когда отсутствует.

По ТК РФ каждому сотруднику положен минимум **28 календарных дней** ежегодного оплачиваемого отпуска. Без учёта отпусков — штрафы от инспекции труда.

---

## Где найти

**Путь:** Главное меню → **Отпуска**

| Страница | Что делает |
|----------|-----------|
| **Заявления** | Список всех заявлений на отпуск |
| **Доска** | Визуальная доска: кто когда в отпуске |
| **Распределение** | Баланс дней: выделено / использовано / осталось |
| **Виды отпусков** | Справочник типов отпусков |

---

## Виды отпусков

| Тип | Оплачивается? | Макс. дней | Нужно согласование? |
|-----|:------------:|:----------:|:-------------------:|
| **Ежегодный оплачиваемый** | Да | 28 | Да |
| **Дополнительный** | Да | Зависит от условий | Да |
| **Без содержания** | Нет | Без ограничений | Да |
| **Учебный** | Зависит | 15 | Да |
| **Декретный (по беременности)** | Да | 140–160 | Нет (по справке) |
| **Больничный** | Да (ФСС) | Без ограничений | Нет (по больничному листу) |

Каждый тип имеет свой цвет на доске отпусков для быстрого визуального различия.

> Типы отпусков настраиваются администратором на странице **Отпуска → Виды отпусков**.

---

## Как подать заявление на отпуск

1. Перейдите **Отпуска → Заявления**
2. Нажмите **«+ Новое заявление»**
3. Заполните:

| Поле | Обязательно? | Описание |
|------|-------------|----------|
| **Сотрудник** | Да | Кто уходит в отпуск |
| **Вид отпуска** | Да | Ежегодный / Без содержания / Учебный и т.д. |
| **Дата начала** | Да | Первый день отпуска |
| **Дата окончания** | Да | Последний день |
| **Кол-во дней** | Авто | Рассчитывается автоматически |
| **Причина** | Нет | Пояснение (например: «Семейный отдых», «Сессия в университете») |

4. Нажмите **«Создать»** — заявление в статусе «Черновик»
5. Нажмите **«Отправить»** — заявление уходит руководителю на согласование

> **Важно:** Система проверяет пересечение дат. Если у сотрудника уже есть утверждённый отпуск на эти даты — ошибка.

---

## Статусы заявления

| Статус | Цвет | Что происходит |
|--------|------|---------------|
| **Черновик** | Серый | Создано, можно редактировать |
| **На рассмотрении** | Синий | Отправлено руководителю |
| **Утверждён** | Зелёный | Одобрено. Дни списываются из баланса |
| **Отказано** | Красный | Руководитель отказал (с причиной) |
| **Отменён** | Серый | Отменён сотрудником или руководителем |

```
Черновик → На рассмотрении → Утверждён
                ↓
            Отказано

Отменён ← (из любого статуса)
```

### Что происходит при утверждении

1. Статус меняется на «Утверждён»
2. Система находит баланс сотрудника по этому типу отпуска
3. **Использовано** увеличивается на количество дней
4. **Остаток** уменьшается

**Пример:**
```
Баланс Иванова: 28 дней (ежегодный, 2026)
Заявление: 14 дней (1–14 июня)
→ Утверждено
→ Использовано: 0 → 14
→ Остаток: 28 → 14
```

### Что происходит при отмене

Если утверждённый отпуск отменяется — дни **возвращаются** в баланс:
```
Остаток: 14 → 28 (вернулись 14 дней)
```

| Колонка | Что показывает |
|---------|---------------|
| Номер | Уникальный номер заявления |
| Сотрудник | ФИО |
| Вид отпуска | Тип с цветным бейджем |
| Даты | С — По |
| Кол-во дней | Длительность |
| Статус | Цветной бейдж |

**Вкладки:** Все / Черновики / На рассмотрении / Утверждённые / Отказанные

**Показатели (вверху):**
- Всего заявлений
- Ожидают утверждения

**Действия (по строке):**
- Если «На рассмотрении»: кнопки **«Утвердить»** и **«Отказать»**
- Если «Черновик»: **«Редактировать»**, **«Удалить»**
- Если «Утверждён»: **«Отменить»**

## Доска отпусков

**Путь:** Отпуска → **Доска**

Канбан-доска, где заявления разложены по колонкам-статусам:

```
| Черновик (2) | На рассмотрении (5) | Утверждён (12) | Отказано (1) |
|  [Иванов]    |   [Петров]           |  [Сидоров]     | [Козлов]     |
|  [Смирнов]   |   [Фёдоров]          |  [Николаев]    |              |
```

Каждая карточка показывает:
- Инициалы сотрудника
- ФИО и отдел
- Тип отпуска (цветной бейдж)
- Даты и количество дней
- Кто утвердил (если утверждён)

**Можно перетаскивать** карточки между колонками для быстрого утверждения.

## Распределение (баланс дней)

**Путь:** Отпуска → **Распределение**

Здесь видно, сколько дней выделено каждому сотруднику и сколько он уже использовал.

**Что видно в таблице:**

| Сотрудник | Вид отпуска | Год | Статус | Выделено | Использовано | Остаток | Прогресс |
|-----------|------------|-----|--------|:--------:|:------------:|:-------:|:--------:|
| Иванов И.И. | Ежегодный | 2026 | Утверждён | 28 | 14 | 14 | ████░░░░ 50% |
| Петров П.П. | Ежегодный | 2026 | Утверждён | 28 | 0 | 28 | ░░░░░░░░ 0% |
| Сидоров С.С. | Ежегодный | 2026 | Утверждён | 28 | 28 | 0 | ████████ 100% |

**Показатели (вверху):**
- Всего сотрудников
- Всего выделено дней
- Всего использовано
- Всего осталось

**Цвета остатка:**
- Зелёный — есть дни
- Красный — дни исчерпаны (или отрицательный баланс)

### Как создать распределение

В начале года HR-менеджер создаёт распределение для каждого сотрудника:

1. Нажмите **«+ Новое распределение»**
2. Заполните:

| Поле | Описание |
|------|---------|
| Сотрудник | Кому выделяем |
| Вид отпуска | Ежегодный / Дополнительный и т.д. |
| Год | На какой год |
| Кол-во дней | Сколько дней выделено (например, 28) |
| Примечание | Например: «Перенос 5 дней с 2025 года» |

3. Нажмите **«Создать»** → статус «Черновик»
4. Утвердите → статус «Утверждён» — сотрудник может подавать заявления

### Проверка баланса при утверждении

Когда руководитель утверждает заявление, система проверяет:
- Есть ли утверждённое распределение на этот год и тип отпуска?
- Хватает ли остатка дней?

Если остатка не хватает → ошибка (если тип отпуска не позволяет отрицательный баланс).

## Типичные сценарии

### Сценарий 1: Ежегодный отпуск

```
Январь 2026: HR создаёт распределение — 28 дней ежегодного для Иванова
→ Статус: Утверждён | Выделено: 28 | Остаток: 28

Май 2026: Иванов подаёт заявление на 14 дней (1–14 июня)
→ Статус заявления: Черновик → На рассмотрении

Май 2026: Руководитель утверждает
→ Статус заявления: Утверждён
→ Баланс: Использовано 14, Остаток 14

Сентябрь 2026: Иванов подаёт ещё на 14 дней (1–14 октября)
→ Утверждено
→ Баланс: Использовано 28, Остаток 0

Ноябрь 2026: Иванов хочет ещё 5 дней
→ ОШИБКА: Остаток 0 дней, не хватает
```

### Сценарий 2: Больничный

```
5 марта: Иванов заболел
→ Создаёт заявление: Тип «Больничный», 5–7 марта (3 дня)
→ Отправляет на рассмотрение
→ Руководитель утверждает (больничный обычно не отклоняют)
→ В табеле Т-13: код «Б» на 5, 6, 7 марта
→ Оплата: 3 дня работодатель, далее — ФСС
```

### Сценарий 3: Отпуск за свой счёт

```
Сотруднику нужно 3 дня по семейным обстоятельствам
→ Создаёт заявление: Тип «Без содержания», 3 дня
→ Причина: «Семейные обстоятельства»
→ Руководитель утверждает
→ Зарплата за эти дни не начисляется
→ В табеле: код «О» (или специальный код для неоплачиваемого)
```

## Частые вопросы

**Сколько дней отпуска положено?**
По ТК РФ — минимум 28 календарных дней ежегодного оплачиваемого. Дополнительные дни могут быть за вредность, ненормированный день, стаж.

**Можно ли перенести отпуск с прошлого года?**
Да. Создайте распределение на текущий год с увеличенным количеством дней. В примечании укажите: «Перенос X дней с 2025 года».

**Что если руководитель отказал?**
Заявление получает статус «Отказано» с указанием причины. Можно подать новое заявление на другие даты.

**Как отменить уже утверждённый отпуск?**
Нажмите **«Отменить»** на утверждённом заявлении. Дни вернутся в баланс. Это может потребовать дополнительного согласования.

**Где увидеть, кто сейчас в отпуске?**
На **Доске отпусков** — колонка «Утверждён» показывает всех, кто уходит или уже в отпуске. Фильтруйте по датам.

**Нужно ли создавать распределение для больничного?**
Обычно нет — больничный может иметь неограниченный баланс (`allowNegative = true`). Но администратор может настроить это по-другому.

**Как связаны отпуска и табель?**
Когда отпуск утверждён, в табеле Т-13 за эти дни нужно поставить код «О» (отпуск). Это делается вручную или подтягивается автоматически.

**Кто может утверждать отпуска?**
Руководитель проекта (MANAGER), HR-менеджер и Администратор (ADMIN).
