# Портфель проектов и тендеры

---

## Что это и зачем

**Портфель** — это управление всеми возможностями компании: тендеры, заявки на подряд, предквалификации. Здесь решается, за какие проекты бороться и какие пропустить.

Без портфеля:
- Компания хватается за все тендеры подряд — и не успевает подготовить качественные заявки
- Нет общей картины: сколько проектов в работе, какая загрузка, где пробелы
- Проигранные тендеры не анализируются — ошибки повторяются

## Часть 1: Возможности (Opportunities)

Возможность — это потенциальный проект, который компания рассматривает.

### Карточка возможности

| Поле | Описание |
|------|----------|
| **Название** | Краткое описание (например, «ЖК Южный — фасадные работы») |
| **Заказчик** | Кто объявил (контрагент из базы) |
| **Сумма** | Ориентировочная стоимость контракта |
| **Тип** | Тендер / Прямое обращение / Госзакупка / Внутренний |
| **Срок подачи** | Дедлайн для тендерной документации |
| **Статус** | Выявлена → Оценивается → Участвуем → Выиграна / Проиграна / Отклонена |
| **Вероятность** | Оценка шансов (%) |
| **Ответственный** | Кто ведёт эту возможность |

### Оценка «Участвовать / Не участвовать» (Go / No-Go)

Перед тем как тратить ресурсы на подготовку тендера, руководство оценивает:

| Критерий | Вопрос |
|----------|--------|
| **Компетенции** | Есть ли у нас опыт такого типа работ? |
| **Ресурсы** | Хватит ли людей и техники? |
| **Финансы** | Маржа будет достаточной? |
| **Сроки** | Успеем подготовить заявку? |
| **Риски** | Заказчик надёжный? Нет ли скрытых рисков? |
| **Конкуренция** | Сколько конкурентов? Есть ли шансы? |

## Часть 2: Тендеры

**Путь:** Портфель → **Тендеры**

### Статусы тендера

| Статус | Описание |
|--------|----------|
| **Выявлен** | Нашли тендер, ещё не решили участвовать |
| **Подготовка** | Готовим документацию |
| **Подан** | Заявка отправлена |
| **Рассмотрение** | Заказчик рассматривает |
| **Выигран** | Мы победили → можно создать проект |
| **Проигран** | Не прошли (зафиксировать причину!) |
| **Отклонён** | Решили не участвовать |

### Тендерный пакет (Bid Package)

Для каждого тендера готовится пакет документов:

| Раздел | Что включает |
|--------|-------------|
| **Техническое предложение** | Описание подхода, методология, график |
| **Ценовое предложение** | Смета / КП с расценками |
| **Квалификация** | Опыт компании, SRO, лицензии, рекомендации |
| **Ресурсы** | Ключевой персонал, техника, субподрядчики |
| **Финансовая устойчивость** | Бухгалтерская отчётность, банковские гарантии |

## Часть 3: Здоровье портфеля

Матрица здоровья показывает все проекты по 7 метрикам (RAG: красный/жёлтый/зелёный):

| Метрика | Зелёный | Жёлтый | Красный |
|---------|---------|--------|---------|
| **Бюджет** | В рамках | Отклонение <10% | Перерасход >10% |
| **Сроки** | По графику | Задержка <2 нед. | Задержка >2 нед. |
| **Качество** | Дефектов мало | Среднее | Много дефектов |
| **Безопасность** | 0 инцидентов | Мелкие инциденты | Серьёзные инциденты |
| **Документация** | Актуальна | Есть отставание | Критические пробелы |
| **Команда** | Укомплектована | Есть вакансии | Нехватка ключевых |
| **Заказчик** | Доволен | Есть замечания | Конфликт |

## Типичный сценарий

```
1. Менеджер по продажам находит тендер на портале госзакупок
   → Создаёт возможность в Портфеле

2. Руководство проводит оценку Go/No-Go:
   → Опыт есть ✅, ресурсы есть ✅, маржа 18% ✅
   → Решение: участвуем

3. Сметчик готовит ценовое предложение:
   → Смета → Спецификация → КЛ → ФМ → КП

4. Менеджер собирает тендерный пакет:
   → Техническое + ценовое + квалификация + ресурсы

5. Подача → Рассмотрение → Выиграли! ✅

6. Конвертация в проект → начало работ
```

## Частые вопросы

**Обязательно ли заполнять портфель?**
Нет, но рекомендуется. Без него вы не видите общую картину: сколько тендеров в работе, какая конверсия, какие причины проигрышей.

**Можно ли создать проект без тендера?**
Да. Если заказчик обратился напрямую — создавайте проект через CRM → Лиды или напрямую в модуле Проекты.

**Как анализировать проигранные тендеры?**
В карточке проигранного тендера укажите причину (цена, сроки, опыт, конкуренты). Со временем система покажет статистику — и вы поймёте, что улучшить.
