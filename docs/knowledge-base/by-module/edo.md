# Электронный документооборот (ЭДО)

---

## Что это и зачем

**Электронный документооборот** (ЭДО) — это система, которая управляет движением документов внутри организации: создание → согласование → утверждение → исполнение → архив.

Без ЭДО документы «гуляют» между кабинетами: распечатали, подписали, отсканировали, отправили по почте, потеряли. На это уходят дни и недели.

С ЭДО в системе ПРИВОД:
- **Маршруты согласования** — документ автоматически идёт к нужным людям в правильном порядке
- **Входящие согласования** — у каждого сотрудника свой «почтовый ящик» с документами на подпись
- **SLA и контроль сроков** — система следит, чтобы документы не «зависали»
- **Автоматизация** — правила, которые выполняются без участия человека
- **Аудит** — полная история: кто, когда, что сделал с документом

## Часть 1: Шаблоны процессов

**Шаблон процесса** — это многоразовый маршрут согласования. Создаёте его один раз, а потом применяете ко всем документам одного типа.

### К каким документам применяются процессы

| Тип документа | Пример маршрута |
|--------------|----------------|
| **Договор** | Юрист → Финдиректор → Гендиректор |
| **Счёт на оплату** | Руководитель проекта → Бухгалтерия → Финдиректор |
| **Заявка на закупку** | Инженер → Руководитель проекта → Снабженец |
| **Бюджет** | Экономист → Финдиректор → Гендиректор |
| **Change Order** | Инженер → Руководитель проекта → Заказчик |
| **Документ** | Любой настраиваемый маршрут |

### Как создать шаблон

1. Перейдите **Бизнес-процессы → Шаблоны процессов**
2. Нажмите **«+ Новый процесс»**
3. Заполните:

| Поле | Описание |
|------|----------|
| **Название** | Например: «Согласование договора подряда» |
| **Тип документа** | Договор / Счёт / Заявка / Бюджет / Change Order / Документ |
| **Описание** | Пояснение: для чего этот маршрут |

4. Добавьте **шаги процесса** (в конструкторе):

| Параметр шага | Описание |
|--------------|----------|
| **Название** | Например: «Проверка юристом» |
| **Статус: откуда** | Из какого статуса документ приходит (например: DRAFT) |
| **Статус: куда** | В какой статус переходит (например: LEGAL_REVIEW) |
| **Роль ответственного** | Кто должен принять решение (ADMIN, MANAGER, ACCOUNTANT) |
| **Конкретные согласующие** | Или конкретные люди (Иванов И.И., Петров П.П.) |
| **SLA (часы)** | Максимальное время на решение |
| **Условия** | Когда шаг применяется (например: если сумма > 1 000 000) |

5. Нажмите **«Сохранить»** → шаблон в статусе «Активный»

### Действия на каждом шаге

| Действие | Что происходит |
|----------|---------------|
| **Изменить статус** | Документ переходит в новый статус |
| **Отправить уведомление** | Уведомление ответственному (email, push) |
| **Назначить ответственного** | Автоматическое назначение исполнителя |
| **Создать задачу** | Автоматически создаётся задача в модуле «Задачи» |
| **Отправить email** | Письмо на почту |
| **Вебхук** | Уведомление внешней системы |

### Условия (когда применять шаг)

Можно настроить условия для шагов:

| Поле | Оператор | Пример |
|------|---------|--------|
| **Статус** | Равен / Не равен | Если статус = «Новый» |
| **Сумма** | Больше / Меньше | Если сумма > 500 000 ₽ |
| **Приоритет** | Равен | Если приоритет = «Срочный» |
| **Ответственный** | Равен / Содержит | Если назначен конкретный человек |
| **Подразделение** | В списке | Если отдел = «Финансовый» |

Это позволяет создавать разветвлённые маршруты: мелкие счета утверждает бухгалтер, крупные — финдиректор.

## Часть 2: Входящие согласования

**Путь:** Бизнес-процессы → **Входящие согласования**

Это главная рабочая страница для каждого согласующего. Здесь видны все документы, которые ждут вашего решения.

### Что видно во входящих

| Колонка | Описание |
|---------|----------|
| **Номер** | Номер документа |
| **Тип** | Договор, счёт, заявка и т.д. |
| **Текущий шаг** | На каком этапе (например: «Проверка юристом») |
| **Статус** | В процессе / Утверждено / Отклонено / Эскалировано |
| **Срок (SLA)** | До когда нужно принять решение |
| **Создано** | Когда запущен процесс |

### Индикаторы SLA

| Индикатор | Цвет | Что означает |
|-----------|------|-------------|
| **Нормально** | Зелёный | Срок ещё не подошёл |
| **Предупреждение** | Жёлтый | Осталось мало времени |
| **Срочно** | Оранжевый | Срок почти истёк |
| **Просрочено** | Красный | Срок истёк |

### Что можно сделать с документом

| Действие | Описание |
|----------|----------|
| **Утвердить** | Согласовать документ → переход к следующему шагу |
| **Отклонить** | Вернуть документ автору с комментарием |
| **Делегировать** | Передать решение другому человеку |
| **Отменить** | Отменить весь процесс согласования |

При утверждении или отклонении можно (и рекомендуется) добавить **комментарий** — например, «Утверждаю при условии корректировки п. 3.2 договора».

### Пакетное согласование

Если у вас 10 счетов на утверждение — не нужно открывать каждый. Выберите несколько и нажмите **«Утвердить выбранные»** — все будут утверждены разом.

### Цепочка согласования

Для каждого документа можно посмотреть полную цепочку: кто уже согласовал, кто отклонил, кто ещё не смотрел.

```
Шаг 1: Юрист Иванов — Утверждено ✅ (01.03, 14:30)
        Комментарий: «Без замечаний»
Шаг 2: Финдиректор Петрова — Утверждено ✅ (02.03, 09:15)
        Комментарий: «Бюджет согласован»
Шаг 3: Гендиректор Сидоров — Ожидает ⏳ (SLA: до 04.03)
```

## Часть 3: Правила автоматизации

Помимо ручных маршрутов, система поддерживает **правила автоматизации** — действия, которые выполняются автоматически при наступлении определённых условий.

### Типы триггеров (когда срабатывает)

| Триггер | Описание |
|---------|----------|
| **При создании** | Документ создан |
| **При смене статуса** | Статус изменился (например: стал «Утверждён») |
| **При изменении поля** | Изменилось значение поля (например: сумма) |
| **По расписанию** | В определённое время (ежедневно, еженедельно) |
| **Вручную** | Запускается пользователем |

### Примеры автоматизации

| Правило | Триггер | Действие |
|---------|---------|---------|
| **Автоприёмка мелких счетов** | Счёт < 10 000 ₽ создан | Автоматически утвердить |
| **Уведомление об инциденте** | Инцидент безопасности создан | Отправить email директору |
| **Назначение инженера** | Заявка на закупку создана | Назначить снабженца из отдела |
| **Эскалация просрочки** | SLA просрочен | Передать вышестоящему руководителю |
| **Контроль бюджета** | Change order > 500 000 ₽ | Отправить на согласование финдиректору |

### Готовые шаблоны правил

Система предлагает готовые шаблоны для типичных сценариев:

| Шаблон | Что делает |
|--------|-----------|
| **Автосогласование** | Автоматически утверждает документы по условиям |
| **Эскалация просрочки** | При просрочке SLA передаёт руководителю |
| **Уведомление о безопасности** | При инциденте уведомляет ответственных |
| **Автоназначение** | Автоматически назначает исполнителя |
| **Порог бюджета** | При превышении суммы запускает доп. согласование |

## Часть 4: Процессы (экземпляры)

**Путь:** Бизнес-процессы → **Все процессы**

Здесь видны все запущенные и завершённые процессы согласования:

### Статусы процесса

| Статус | Цвет | Что означает |
|--------|------|-------------|
| **В процессе** (IN_PROGRESS) | Синий | Документ проходит по маршруту |
| **Согласовано** (APPROVED) | Зелёный | Все шаги пройдены успешно |
| **Отклонено** (REJECTED) | Красный | На одном из шагов отклонён |
| **Отменено** (CANCELLED) | Серый | Процесс отменён |
| **Эскалировано** (ESCALATED) | Оранжевый | SLA нарушен, передан на эскалацию |

## Типичные сценарии

### Сценарий 1: Согласование договора подряда

```
1. Менеджер загружает проект договора подряда в систему
   → Запускает маршрут «Согласование договора»

2. Шаг 1: Юрист (SLA: 48 часов)
   → Юрист открывает документ во «Входящих»
   → Проверяет условия, вносит замечания
   → Решение: Утверждено с комментарием
     «П. 5.3 — изменить формулировку неустойки»
   → Время: 6 часов ✅

3. Шаг 2: Финансовый директор (SLA: 24 часа)
   → Проверяет суммы, условия оплаты
   → Решение: Утверждено ✅
   → Время: 3 часа ✅

4. Шаг 3: Генеральный директор (SLA: 24 часа)
   → Финальное утверждение
   → Решение: Утверждено ✅

5. Результат: Договор согласован за 1.5 дня
   (раньше на бумаге — 5-7 дней)
```

### Сценарий 2: Срочный счёт с эскалацией

```
1. Поступает счёт на 250 000 ₽ за аренду крана (срочная оплата)
   → Маршрут: Руководитель проекта → Бухгалтерия

2. Шаг 1: Руководитель проекта (SLA: 4 часа)
   → Руководитель на совещании, не смотрит систему
   → Через 4 часа: SLA просрочен ⚠️
   → Автоматическая эскалация → уведомление заместителю

3. Заместитель видит эскалированный документ:
   → Статус: Эскалировано (оранжевый)
   → Решение: Утверждено (за руководителя)

4. Шаг 2: Бухгалтерия
   → Утверждено → Счёт оплачен
```

### Сценарий 3: Автосогласование мелких закупок

```
1. Администратор настроил правило:
   «Если заявка на закупку < 50 000 ₽ → автоматически утвердить»

2. Инженер создаёт заявку: «Перчатки строительные, 100 пар — 15 000 ₽»

3. Система автоматически:
   → Проверяет условие: 15 000 < 50 000 ✅
   → Утверждает заявку
   → Создаёт задачу снабженцу: «Закупить перчатки»

4. Инженер получает уведомление: «Заявка утверждена»
   → Вся процедура заняла 0 секунд
```

## Частые вопросы

**Обязательно ли настраивать маршруты согласования?**
Нет. Можно работать без них — тогда документы согласовываются вручную (как на бумаге). Но маршруты экономят время и исключают «потерю» документов.

**Кто может настраивать маршруты?**
Администратор системы (роль ADMIN). Обычные пользователи используют готовые маршруты.

**Что если согласующий в отпуске?**
Используйте **делегирование**: согласующий (или администратор) передаёт полномочия заместителю. Или настройте автоматическую эскалацию при просрочке SLA.

**Можно ли изменить маршрут после запуска?**
Запущенный процесс идёт по изначальному маршруту. Но можно отменить процесс и запустить заново по другому шаблону.

**Как узнать, где «застрял» документ?**
Откройте процесс — видна цепочка согласования с текущим шагом. Если шаг просрочен — красный индикатор.

**Что означает «Эскалировано»?**
Срок согласования (SLA) истёк, и документ автоматически передан вышестоящему руководителю. Это гарантирует, что документы не «зависают» бесконечно.

**Сколько маршрутов можно создать?**
Без ограничений. Можно создать отдельный маршрут для каждого типа документа или даже для разных сумм (мелкие счета — короткий маршрут, крупные — длинный).
