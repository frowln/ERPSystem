# Настройки системы

---

## Что это и зачем

**Настройки** — это раздел для администратора системы. Здесь настраивается всё, что определяет работу ПРИВОД: данные организации, почтовый сервер, безопасность, интеграции, уведомления и резервное копирование.

Обычным пользователям не нужно заходить в настройки — они работают с модулями. Настройки — для **администратора** и **руководителя**.

---

## Часть 3: Безопасность

### Парольная политика

| Параметр | Описание | Рекомендуемое значение |
|----------|----------|----------------------|
| **Минимальная длина** | Минимум символов в пароле | 8 |
| **Обязательные символы** | Буквы + цифры + спецсимволы | Да |
| **Срок действия пароля** | Через сколько дней менять | 90 дней |
| **История паролей** | Нельзя повторять последние N паролей | 5 |
| **Блокировка** | После N неудачных попыток | 5 попыток → блокировка на 15 минут |

### Двухфакторная аутентификация (2FA)

**2FA** — это дополнительный уровень защиты. Помимо пароля, при входе нужно ввести код из приложения (Google Authenticator, Яндекс.Ключ).

| Параметр | Описание |
|----------|----------|
| **Обязательная 2FA** | Для всех / только для администраторов / отключена |
| **Метод** | TOTP (приложение-аутентификатор) |

### Управление сессиями

| Параметр | Описание |
|----------|----------|
| **Время жизни сессии** | Через сколько минут неактивности — выход из системы |
| **Максимум сессий** | Сколько устройств одновременно |
| **Логирование** | Запись всех входов/выходов |

## Часть 5: Уведомления

Настройка того, **какие события** и **по каким каналам** уведомляют пользователей.

### Каналы уведомлений

| Канал | Описание |
|-------|----------|
| **В системе** | Колокольчик в правом верхнем углу |
| **Email** | Письмо на рабочую почту |
| **Telegram** | Сообщение от бота |
| **SMS** | Текстовое сообщение на телефон |
| **Push** | Уведомление в браузере (PWA) |

### Типы событий

| Группа | Примеры событий |
|--------|----------------|
| **Задачи** | Новая задача, приближение срока, просрочка |
| **Согласования** | Требуется ваше согласование, документ утверждён/отклонён |
| **Финансы** | Новый счёт, платёж получен, просроченная оплата |
| **Безопасность** | Инцидент на объекте, просроченный инструктаж |

## Часть 6: Резервное копирование

### Автоматическое копирование

| Параметр | Описание |
|----------|----------|
| **Расписание** | Ежедневно в 03:00 (или другое время) |
| **Хранить копий** | Последние 30 копий |
| **Куда сохранять** | Локально / Внешнее хранилище (S3, NAS) |

### Что сохраняется

- **База данных** — все данные системы
- **Файлы** — загруженные документы, фото, вложения
- **Настройки** — конфигурация системы

### Восстановление

Если что-то пошло не так:
1. Выберите копию из списка (по дате)
2. Нажмите **«Восстановить»**
3. Система вернётся к состоянию на дату копии

> **Внимание:** восстановление заменяет **все** текущие данные данными из копии. Все изменения после даты копии будут потеряны.

## Типичные сценарии

### Сценарий 1: Первоначальная настройка системы

```
1. Администратор заполняет данные организации:
   → Название: ООО «СтройКомпания»
   → ИНН: 7707083893
   → Загружает логотип

2. Настраивает почту:
   → Сервер: imap.yandex.ru / smtp.yandex.ru
   → Тестовое письмо — получено ✅

3. Безопасность:
   → Минимальный пароль: 8 символов
   → 2FA: обязательно для администраторов
   → Блокировка: после 5 неудачных попыток

4. Уведомления:
   → Включает email и Telegram для руководителей
   → SMS — только для критических событий

5. Резервное копирование:
   → Ежедневно в 03:00
   → Хранить 30 копий
   → Сохранять на NAS (Synology)

6. Система готова к работе ✅
```

### Сценарий 2: Подключение Telegram-бота

```
1. Администратор:
   → Создаёт бот в Telegram через @BotFather
   → Получает токен: 1234567890:ABCdef...

2. В настройках ПРИВОД:
   → Интеграции → Telegram
   → Вводит токен бота
   → Проверка: бот активен ✅

3. Рассылает ссылку на бота сотрудникам:
   → Каждый сотрудник пишет боту /start
   → Бот привязывает Telegram к аккаунту ПРИВОД

4. Теперь уведомления приходят в Telegram ✅
```

## Частые вопросы

**Кто может менять настройки?**
Только пользователи с ролью **Администратор** (ADMIN).

**Что будет, если неправильно настроить почту?**
Модуль «Почта» не сможет отправлять и получать письма. Используйте кнопку «Тестовое письмо» для проверки.

**Обязательна ли двухфакторная аутентификация?**
Рекомендуется. Как минимум — для администраторов. Для остальных пользователей — на усмотрение руководителя.

**Как часто делать резервные копии?**
Рекомендуется — ежедневно. Для критически важных данных — дважды в день.

**Можно ли восстановить отдельный модуль?**
Нет, восстановление — целиком. Если нужно восстановить часть данных — обратитесь в службу поддержки.

**Где хранятся резервные копии?**
По умолчанию — на сервере. Рекомендуется настроить внешнее хранилище (NAS, облако) для защиты от выхода сервера из строя.
