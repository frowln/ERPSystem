# От сметы до получения оплаты

> Главный бизнес-процесс строительной компании — полная цепочка от расчёта стоимости работ до получения денег от заказчика.

---

## Обзор процесса

```
Смета (ЛСР) → Спецификация → КЛ → ФМ → КП → Договор → Работы → КС-2 → КС-3 → Счёт → Оплата
```

| | |
|---|---|
| **Участники** | Сметчик, Снабженец (ОПС), Финансист, Коммерческий директор, Прораб, Бухгалтер |
| **Время** | От 2 недель (расчёт + КП) до нескольких месяцев (весь цикл) |
| **Результат** | Деньги на расчётном счёте компании |

### Схема потока данных

```
┌─────────┐    ┌──────────────┐    ┌────┐    ┌────┐    ┌────┐    ┌────────┐
│  Смета  │───→│ Спецификация │───→│ КЛ │───→│ ФМ │───→│ КП │───→│ Договор│
│  (ЛСР)  │    │              │    │    │    │    │    │    │    │        │
└─────────┘    └──────────────┘    └────┘    └────┘    └────┘    └────────┘
     │                                         ↑                      │
     └─────────────── сметная цена ───────────→┘                      │
                                                                      ↓
┌─────────┐    ┌─────────┐    ┌──────┐    ┌──────┐              ┌──────────┐
│ Оплата  │←───│  Счёт   │←───│ КС-3 │←───│ КС-2 │←─────────── │  Работы  │
│         │    │         │    │      │    │      │              │          │
└─────────┘    └─────────┘    └──────┘    └──────┘              └──────────┘
```

## Шаг 1: Загрузка сметы (ЛСР)

| | |
|---|---|
| **Кто** | Сметчик |
| **Где** | Главное меню → **Сметы** → **Импорт ЛСР** |
| **Что делать** | Загрузить xlsx-файл из ГРАНД-Сметы. Система разберёт его на разделы, позиции и ресурсы. Каждая позиция получит **сметную цену** (по нормативам ГЭСН/ФЕР/ТЕР). |
| **Результат** | Смета создана со статусом **«Черновик»**. Все позиции с расценками загружены. |
| **Подробнее** | [Сметы](../by-module/estimates.md) |

> **Зачем?** Сметная цена — это «потолок», выше которого заказчик не заплатит. Она основана на государственных нормативах и индексах Минстроя.

---

## Шаг 2: Создание спецификации

| | |
|---|---|
| **Кто** | Сметчик или Инженер ПТО |
| **Где** | Главное меню → **Спецификации** → **Создать** |
| **Что делать** | Создать список материалов и оборудования для проекта. Можно импортировать из xlsx. В спецификации **нет цен** — только: наименование, тип/марка, код, завод-изготовитель, единица измерения, количество, вес. |
| **Результат** | Спецификация создана. Позиции автоматически передаются в **ФМ** (финансовую модель). |
| **Подробнее** | [Спецификации](../by-module/specifications.md) |

> **Зачем?** Спецификация — это техническое задание для снабженцев. По ней они будут искать поставщиков и запрашивать цены.

---

## Шаг 3: Конкурентный лист (КЛ)

| | |
|---|---|
| **Кто** | Снабженец (ОПС) |
| **Где** | Главное меню → **Спецификации** → **Реестр КЛ** |
| **Что делать** | Для каждой позиции спецификации — собрать предложения минимум от **3 поставщиков**. Ввести: цену, срок поставки, гарантию, условия оплаты. Система автоматически ранжирует поставщиков (50% цена, 25% сроки, 15% гарантия, 10% условия оплаты). Выбрать победителя. |
| **Результат** | КЛ утверждён. **Себестоимость** (цена победителя) автоматически попадает в ФМ. |
| **Подробнее** | [Конкурентный лист](../by-module/competitive-list.md) |

> **Зачем?** КЛ определяет, сколько компания **реально потратит** на материалы. Это основа для расчёта прибыли.

---

## Шаг 4: Финансовая модель (ФМ)

| | |
|---|---|
| **Кто** | Финансист или Коммерческий директор |
| **Где** | Главное меню → **Финансы** → **Финансовые модели** |
| **Что делать** | Проверить три колонки цен по каждой позиции: **себестоимость** (из КЛ), **сметная цена** (из ЛСР), **цена заказчику** (с наценкой). Настроить накладные расходы (НР), сметную прибыль (СП), непредвиденные расходы (НП). Проверить маржу — она должна быть зелёной (>15%). |
| **Результат** | ФМ показывает полную картину: сколько потратим, сколько получим, сколько заработаем. Кнопка **«Создать КП»** готова. |
| **Подробнее** | [Финансовая модель](../by-module/financial-model.md) |

> **Зачем?** ФМ — это «калькулятор прибыли». Если маржа красная (<0%) — проект убыточный, нужно пересмотреть цены или отказаться.

### Три цены в ФМ

| Колонка | Откуда | Что означает |
|---------|--------|-------------|
| **Себестоимость** | Из КЛ (победитель) | Сколько мы заплатим поставщику |
| **Сметная цена** | Из ЛСР (нормативы) | Максимальная цена по государственным расценкам |
| **Цена заказчику** | Наценка на себестоимость | Сколько мы попросим у заказчика |

---

## Шаг 5: Коммерческое предложение (КП)

| | |
|---|---|
| **Кто** | Коммерческий директор |
| **Где** | Главное меню → **Финансы** → **Коммерческие предложения** |
| **Что делать** | Нажать **«Создать КП»** из ФМ. Проверить две вкладки: **Материалы** (позиции из спецификации с ценами) и **Работы** (с торговым коэффициентом). При необходимости — провести тендер подрядчиков для работ. Сформировать печатную форму (PDF). Отправить заказчику. |
| **Результат** | КП отправлено заказчику. Статус: **«Отправлено»**. Ждём решения. |
| **Подробнее** | [Коммерческое предложение](../by-module/commercial-proposals.md) |

> **Зачем?** КП — это официальное предложение цены заказчику. Если заказчик согласен — подписываем договор.

---

## Шаг 6: Подписание договора

| | |
|---|---|
| **Кто** | Коммерческий директор → Юрист → Директор |
| **Где** | Главное меню → **Контракты** → **Создать** |
| **Что делать** | Создать договор на основе принятого КП. Заполнить: номер, сумму, НДС (20%), аванс (%), удержание (%), сроки. Пройти внутреннее согласование (юрист → руководство → бухгалтерия). Подписать с заказчиком. |
| **Результат** | Договор в статусе **«Подписан»** → переходит в **«Активный»**. Можно начинать работы. |
| **Подробнее** | [Контракты](../by-module/contracts.md) |

> **Зачем?** Без подписанного договора нельзя начинать работы и выставлять акты. Договор фиксирует сумму, сроки и условия оплаты.

---

## Шаг 7: Выполнение работ

| | |
|---|---|
| **Кто** | Прораб и бригады |
| **Где** | Главное меню → **Задачи** (канбан-доска) |
| **Что делать** | Выполнять работы по плану. Вести ежедневный журнал (КС-6а): какие работы выполнены, объёмы, бригады. Прораб отмечает факт в задачах. Данные из журнала — основа для КС-2. |
| **Результат** | Работы выполнены. Объёмы зафиксированы в журнале КС-6а. |
| **Подробнее** | [Задачи](../by-module/tasks.md) |

> **Зачем?** Точный учёт объёмов — это защита от споров с заказчиком. Что записано в журнале — то попадёт в акт.

---

## Шаг 8: Акт выполненных работ (КС-2)

| | |
|---|---|
| **Кто** | Инженер ПТО → Техконтроль → Бухгалтер → Заказчик |
| **Где** | Главное меню → **Закрытие** → **КС-2** |
| **Что делать** | Создать КС-2 из данных журнала КС-6а. Указать: какие работы, в каком объёме, по какой цене выполнены за отчётный период. Пройти 4 этапа согласования: подрядчик → техконтроль → бухгалтерия → заказчик. |
| **Результат** | КС-2 подписан всеми сторонами. Статус: **«Подписан»**. |
| **Подробнее** | [КС-2](../by-module/ks2.md) |

> **Зачем?** КС-2 — это юридическое подтверждение выполненных работ. Без подписанного КС-2 нельзя выставить счёт.

### 4 этапа согласования КС-2

| Этап | Кто проверяет | Что проверяет |
|------|--------------|---------------|
| 1 | Подрядчик (прораб) | Объёмы соответствуют факту |
| 2 | Техконтроль | Качество работ |
| 3 | Бухгалтерия | Цены и суммы по договору |
| 4 | Заказчик | Финальное утверждение |

---

## Шаг 9: Справка о стоимости (КС-3)

| | |
|---|---|
| **Кто** | Бухгалтер |
| **Где** | Главное меню → **Закрытие** → **КС-3** |
| **Что делать** | Создать КС-3, включив в неё подписанные КС-2 за период. Система автоматически суммирует: общая стоимость, НДС (20%), удержание (обычно 5%), итого к оплате (нетто). |
| **Результат** | КС-3 подписана. Известна точная сумма к оплате с учётом удержания. |
| **Подробнее** | [КС-3](../by-module/ks3.md) |

> **Зачем?** КС-3 объединяет несколько КС-2 в один итоговый документ. Это основание для выставления счёта. Удержание (5%) — это гарантия качества, которую заказчик вернёт после гарантийного периода.

### Формула КС-3

```
Стоимость работ (из КС-2)     1 000 000 ₽
+ НДС 20%                       200 000 ₽
= Итого с НДС                 1 200 000 ₽
— Удержание 5%                  −60 000 ₽
= К оплате (нетто)            1 140 000 ₽
```

## Шаг 10: Выставление счёта

| | |
|---|---|
| **Кто** | Бухгалтер |
| **Где** | Главное меню → **Финансы** → **Счета** |
| **Что делать** | Создать счёт на оплату на основании подписанной КС-3. Указать: контрагент, договор, сумму, НДС, срок оплаты. Отправить заказчику. |
| **Результат** | Счёт выставлен. Статус: **«Отправлен»**. Бюджет обновлён — сумма попала в колонку «Выставлено». |
| **Подробнее** | [Счета](../by-module/invoices.md) |

## Шаг 11: Получение оплаты

| | |
|---|---|
| **Кто** | Бухгалтер |
| **Где** | Главное меню → **Финансы** → **Оплаты** |
| **Что делать** | Зафиксировать поступление денег на расчётный счёт. Привязать оплату к счёту и договору. Возможна частичная оплата — система отследит остаток. |
| **Результат** | Оплата получена. Счёт закрыт (или частично оплачен). Бюджет обновлён. Данные попали в **Движение денежных средств** (Cash Flow). |
| **Подробнее** | [Оплаты](../by-module/payments.md) |

> **Процесс завершён.** Деньги на счёте. Весь путь от сметы до оплаты — прозрачен и задокументирован.

## Что может пойти не так

| Проблема | Где обнаружится | Что делать |
|----------|----------------|------------|
| **Маржа отрицательная** | ФМ (шаг 4) — красный цвет | Пересмотреть цены в КЛ, найти более дешёвых поставщиков, или увеличить наценку для заказчика |
| **Заказчик отклонил КП** | КП (шаг 5) | Скорректировать цены, предложить альтернативные материалы, дать скидку |
| **Объёмы в КС-2 не совпадают со сметой** | КС-2 (шаг 8) — проверка техконтроля | Проверить журнал КС-6а, скорректировать объёмы, оформить доп. соглашение если объёмы выросли |
| **Заказчик не подписывает КС-2** | КС-2 (шаг 8) — этап 4 | Устранить замечания по качеству, предоставить дополнительные документы (акты скрытых работ, сертификаты) |
| **Счёт не оплачен в срок** | Оплаты (шаг 11) | Связаться с заказчиком, при необходимости — направить претензию через модуль Контракты |
| **Фактическая себестоимость выше плановой** | ФМ → раздел CVR | Разобраться в причинах: рост цен поставщиков, перерасход материалов, простои бригад |
