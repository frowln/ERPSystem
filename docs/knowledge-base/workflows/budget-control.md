# Контроль бюджета проекта

> Как следить за финансами проекта: план vs факт, откуда берутся отклонения и что с ними делать.

---

## Обзор процесса

```
Бюджет (план) → Закупки (факт) → Зарплата (факт) → Отклонение → Корректировка
```

| | |
|---|---|
| **Участники** | Руководитель проекта, Финансист, Бухгалтер, Директор |
| **Время** | Непрерывно — на протяжении всего проекта |
| **Результат** | Проект в рамках бюджета (или отклонение выявлено и управляемо) |

### Схема потока

```
┌─────────────┐
│ БЮДЖЕТ (план)│
│ Материалы    │
│ Работы       │
│ Техника      │
│ Субподряд    │
│ Накладные    │
└──────┬──────┘
       │
       ↓
┌──────────────────────────────────────────────────┐
│                    ФАКТ                          │
│  ┌──────────┐  ┌──────────┐  ┌────────────────┐ │
│  │ Закупки  │  │ Зарплата │  │ Прочие расходы │ │
│  │(PO,склад)│  │ (табель) │  │ (техника, ...)│ │
│  └────┬─────┘  └────┬─────┘  └───────┬────────┘ │
│       └──────────────┼────────────────┘          │
└──────────────────────┼───────────────────────────┘
                       ↓
              ┌────────────────┐
              │  ОТКЛОНЕНИЕ    │
              │ (план — факт)  │
              └───────┬────────┘
                      │
            ┌─────────┴──────────┐
            ↓                    ↓
    ┌──────────────┐    ┌───────────────┐
    │ В рамках     │    │ Перерасход    │
    │ (зелёный)    │    │ (красный)     │
    └──────────────┘    └───────┬───────┘
                                │
                        ┌───────┴────────┐
                        │ КОРРЕКТИРОВКА  │
                        │ (решение рук.) │
                        └────────────────┘
```

## Шаг 1: Формирование бюджета (план)

| | |
|---|---|
| **Кто** | Руководитель проекта + Финансист |
| **Где** | Главное меню → **Финансы** → **Бюджеты** |
| **Что делать** | Бюджет создаётся автоматически при создании проекта. Заполнить плановые суммы по категориям: материалы, работы, оборудование, субподряд, накладные расходы, прочее. Источники: смета (ЛСР), финансовая модель (ФМ), коммерческое предложение (КП). Утвердить бюджет. |
| **Результат** | Бюджет утверждён. По каждой позиции видна плановая сумма. Статус: **«Активный»**. |
| **Подробнее** | [Бюджеты](../by-module/budgets.md) |

### Структура бюджета

| Категория | Источник плана | Пример |
|-----------|---------------|--------|
| **Материалы** | ФМ → себестоимость из КЛ | 15 000 000 ₽ |
| **Работы** | ЛСР → нормативные расценки | 8 000 000 ₽ |
| **Оборудование** | Спецификация → аренда/покупка | 3 000 000 ₽ |
| **Субподряд** | КП → расценки подрядчиков | 5 000 000 ₽ |
| **Накладные** | % от общей суммы (ФМ) | 2 000 000 ₽ |
| **Прочее** | Непредвиденные (обычно 3–5%) | 1 000 000 ₽ |
| **ИТОГО** | | **34 000 000 ₽** |

## Шаг 2: Учёт фактических расходов

Факт наполняется автоматически из разных модулей:

### 2.1 Закупки (материалы и оборудование)

| | |
|---|---|
| **Кто** | Снабженец → Бухгалтер |
| **Где** | Снабжение → PO → Счёт → Оплата |
| **Что происходит** | Каждый заказ на закупку (PO) создаёт **резерв** в бюджете. Когда приходит счёт и производится оплата — сумма переходит из «зарезервировано» в «оплачено». |
| **Подробнее** | [Закупка материалов](procurement.md) |

### 2.2 Зарплата (ФОТ)

| | |
|---|---|
| **Кто** | Прораб (табель) → Бухгалтер (расчёт) |
| **Где** | Кадры → Табели → Расчёт зарплаты |
| **Что происходит** | Утверждённый табель → расчёт зарплаты → сумма ФОТ попадает в бюджет проекта в категорию «Работы» или отдельную статью «ФОТ». |
| **Подробнее** | [Сотрудник от приёма до увольнения](hr-lifecycle.md) |

### 2.3 Прочие расходы

| | |
|---|---|
| **Кто** | Бухгалтер |
| **Где** | Финансы → Оплаты |
| **Что происходит** | Аренда техники, ГСМ, командировки, прочие платежи — каждая оплата привязывается к проекту и статье бюджета. |

| **Результат** | Факт по всем статьям обновляется в реальном времени. |

## Шаг 3: Анализ отклонений

| | |
|---|---|
| **Кто** | Руководитель проекта + Финансист |
| **Где** | Несколько инструментов (см. ниже) |
| **Что делать** | Регулярно (еженедельно или по запросу) проверять: план vs факт по каждой статье. Выявлять отклонения. Определять причины. |

### Инструмент 1: Бюджет — таблица позиций

| Позиция | План | Зарезерв. | Оплачено | Остаток | Статус |
|---------|------|-----------|----------|---------|--------|
| Бетон М300 | 2 000 000 | 500 000 | 1 200 000 | 300 000 | OK |
| Арматура | 1 500 000 | 200 000 | 1 400 000 | −100 000 | Перерасход! |

> **Цвета остатка:** Зелёный = в рамках. Жёлтый = приближается к лимиту. Красный = перерасход.

### Инструмент 2: S-кривая (план vs факт)

S-кривая показывает **нарастающим итогом**: сколько планировали потратить к этой дате и сколько потратили реально.

| Метрика | Значение | Что означает |
|---------|---------|-------------|
| **CPI** (Cost Performance Index) | = 1.0 | Идеально по плану |
| **CPI** | > 1.0 | Тратим меньше плана (хорошо) |
| **CPI** | < 1.0 | Тратим больше плана (проблема!) |

### Инструмент 3: БДДС (бюджет движения денежных средств)

БДДС показывает помесячно: доходы (от заказчика) vs расходы (материалы, зарплата, субподряд...). Главное — **кумулятивный баланс** не должен уходить в минус (кассовый разрыв).

### Инструмент 4: Cash Flow

Движение денежных средств: приток (оплаты от заказчика) и отток (наши расходы). Если отток стабильно превышает приток — у проекта проблемы с ликвидностью.

## Шаг 4: Выявление причин перерасхода

| | |
|---|---|
| **Кто** | Руководитель проекта |
| **Где** | Анализ данных из бюджета, закупок, табелей |

### Типичные причины перерасхода

| Категория | Причина | Как обнаружить |
|-----------|---------|---------------|
| **Материалы** | Рост цен поставщика | Сравнить цену в PO с плановой в ФМ |
| **Материалы** | Перерасход на объекте (брак, потери) | Отчёт М-29: факт > норма |
| **Работы** | Много сверхурочных | Табели: часы > плана |
| **Работы** | Низкая производительность бригады | План-факт по объёмам работ |
| **Субподряд** | Доп. работы (не по договору) | Доп. соглашения к контракту |
| **Оборудование** | Простои техники | Путевые листы: техника стоит |
| **Накладные** | Незапланированные расходы | Платежи без привязки к статье бюджета |

## Шаг 5: Корректировка

| | |
|---|---|
| **Кто** | Руководитель проекта → Директор (при крупных суммах) |
| **Где** | Бюджет → редактирование + совещание |
| **Что делать** | Принять решение на основе анализа. Варианты: |

### Варианты корректировки

| Действие | Когда применять | Пример |
|----------|----------------|--------|
| **Перераспределить** | Одна статья перерасходована, другая недорасходована | Сэкономили на технике → перенесли на материалы |
| **Увеличить бюджет** | Объективный рост затрат (рыночные цены выросли) | Доп. соглашение с заказчиком на увеличение суммы |
| **Оптимизировать расходы** | Можно снизить затраты без потери качества | Найти более дешёвого поставщика, оптимизировать логистику |
| **Зафиксировать убыток** | Ничего нельзя сделать | Принять и учесть в общей рентабельности портфеля |
| **Заморозить бюджет** | Ситуация неопределённая | Временно остановить расходы, разобраться в причинах |

> **Важно:** Изменения в бюджете утверждаются руководством. Система сохраняет **снэпшоты** (снимки) бюджета — можно всегда сравнить с первоначальным планом.

## Контроль на уровне портфеля

Для директора важно видеть не один проект, а **все проекты сразу**:

| Проект | Бюджет | Факт | Отклонение | Маржа |
|--------|--------|------|-----------|-------|
| ЖК «Южный» | 34 000 000 | 31 500 000 | −2 500 000 (экономия) | 22% |
| Офисный центр | 18 000 000 | 19 200 000 | +1 200 000 (перерасход) | 8% |
| Склад логистический | 12 000 000 | 11 800 000 | −200 000 (экономия) | 18% |

> **Здоровая маржа:** 15–40%. Ниже 10% — проект рискованный. Ниже 0% — убыточный.

## Что может пойти не так

| Проблема | Где обнаружится | Что делать |
|----------|----------------|------------|
| **Бюджет не утверждён** | Шаг 1 — статус «Черновик» | Все расходы идут «мимо контроля». Срочно утвердить бюджет |
| **Расходы не привязаны к проекту** | Оплаты без привязки | Бухгалтер должен привязывать каждый платёж к проекту и статье |
| **Кассовый разрыв** | БДДС — кумулятивный баланс < 0 | Нужны деньги раньше, чем придёт оплата от заказчика. Варианты: ускорить выставление КС-2, попросить аванс, использовать кредитную линию |
| **Перерасход по всем статьям** | Бюджет — все позиции красные | Системная проблема: заниженный бюджет, или реальная инфляция. Нужно пересмотреть весь бюджет и обсудить с заказчиком |
| **Прогноз хуже плана** | S-кривая, CPI < 0.9 | Если тенденция не меняется — итоговые затраты превысят бюджет. Нужны экстренные меры сейчас, пока не поздно |

## Периодичность контроля

| Период | Что делать | Кто |
|--------|-----------|-----|
| **Ежедневно** | Проверить: нет ли новых незапланированных расходов | Руководитель проекта |
| **Еженедельно** | Обзор план/факт по основным статьям, CPI | Руководитель + Финансист |
| **Ежемесячно** | Полный отчёт: БДДС, S-кривая, прибыльность | Директор |
| **По событию** | При крупном PO, доп. соглашении, изменении цен | По ситуации |

## Итого: кто за что отвечает

| Роль | Что делает |
|------|-----------|
| **Руководитель проекта** | Контролирует бюджет, выявляет отклонения, принимает решения |
| **Финансист** | Формирует бюджет, строит аналитику (БДДС, S-кривая, CPI) |
| **Бухгалтер** | Привязывает каждый платёж к проекту и статье бюджета |
| **Снабженец** | Контролирует цены закупок, не допускает перерасхода по материалам |
| **Директор** | Утверждает корректировки, контролирует портфель в целом |
