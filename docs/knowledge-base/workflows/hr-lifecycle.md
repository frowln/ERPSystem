# Сотрудник от приёма до увольнения

> Полный путь сотрудника в системе: от оформления на работу до расчёта при увольнении.

---

## Обзор процесса

```
Приём → Документы → Инструктаж → Бригада → Табель → Зарплата → Отпуск → Увольнение
```

| | |
|---|---|
| **Участники** | HR-специалист, Инженер по ОТ, Прораб (бригадир), Бухгалтер, Руководитель |
| **Время** | Оформление: 1–2 дня. Полный цикл: весь период работы сотрудника |
| **Результат** | Сотрудник оформлен, работает, получает зарплату, уходит в отпуск, при необходимости — увольняется |

### Схема потока

```
┌────────┐    ┌───────────┐    ┌────────────┐    ┌─────────┐
│ Приём  │───→│ Документы │───→│ Инструктаж │───→│ Бригада │
│        │    │           │    │    (ОТ)    │    │         │
└────────┘    └───────────┘    └────────────┘    └─────────┘
                                                      │
┌────────────┐    ┌──────────┐    ┌────────┐         │
│ Увольнение │←───│  Отпуск  │←───│Зарплата│←──Табель←┘
│            │    │          │    │        │
└────────────┘    └──────────┘    └────────┘
```

## Шаг 1: Приём на работу

| | |
|---|---|
| **Кто** | HR-специалист |
| **Где** | Главное меню → **Кадры** → **Сотрудники** → **Создать** |
| **Что делать** | Заполнить карточку сотрудника: ФИО, дата рождения, паспортные данные, ИНН, СНИЛС, адрес. Указать должность, отдел, дату приёма, ставку (почасовая или оклад), руководителя. Создать учётную запись в системе (логин — email, временный пароль). |
| **Результат** | Сотрудник создан в статусе **«Активен»**. У него есть логин для входа в систему. |
| **Подробнее** | [Сотрудники](../by-module/employees.md) |

### Что заполняется при приёме

| Раздел | Поля | Обязательно? |
|--------|------|-------------|
| **Личные данные** | ФИО, дата рождения, телефон, email | Да |
| **Документы** | Паспорт, ИНН, СНИЛС | Да |
| **Работа** | Должность, отдел, дата приёма, руководитель | Да |
| **Оплата** | Тип (оклад/почасовая), ставка, надбавки | Да |
| **Доп. информация** | Адрес, образование, предыдущий опыт | Нет |

## Шаг 2: Оформление документов

| | |
|---|---|
| **Кто** | HR-специалист |
| **Где** | Карточка сотрудника → вкладка **«Документы»** |
| **Что делать** | Загрузить скан-копии документов: паспорт, ИНН, СНИЛС, диплом, трудовая книжка, медосмотр. Оформить кадровые документы: трудовой договор, приказ о приёме (Т-1), личная карточка (Т-2). |
| **Результат** | Все документы загружены и привязаны к карточке сотрудника. |
| **Подробнее** | [Сотрудники → Документы](../by-module/employees.md) |

> **Важно:** Без действующего медосмотра нельзя допускать к работе на строительном объекте. Система отслеживает сроки действия всех документов.

---

## Шаг 3: Инструктажи по охране труда

| | |
|---|---|
| **Кто** | Инженер по ОТ (охране труда) |
| **Где** | Главное меню → **Безопасность** → **Инструктажи** |
| **Что делать** | Провести обязательные инструктажи: **вводный** (правила компании) и **первичный** (правила на рабочем месте). Сотрудник подписывает электронную подпись. Без этих двух инструктажей **нельзя допускать к работе**. |
| **Результат** | Инструктажи пройдены и подписаны. Сотрудник допущен к работе. |
| **Подробнее** | [Обучение и инструктажи](../by-module/safety-trainings.md) |

### Обязательные инструктажи (по ГОСТ 12.0.004-2015)

| Тип | Когда | Кто проводит |
|-----|-------|-------------|
| **Вводный** | В первый день | Инженер по ОТ |
| **Первичный** | Перед началом работы | Непосредственный руководитель |
| **Повторный** | Каждые 6 месяцев | Непосредственный руководитель |
| **Внеплановый** | После инцидента или изменения правил | Инженер по ОТ |
| **Целевой** | Перед опасными работами | Ответственный руководитель работ |

## Шаг 4: Назначение в бригаду и на проект

| | |
|---|---|
| **Кто** | Прораб или Руководитель проекта |
| **Где** | Главное меню → **Кадры** → **Бригады** |
| **Что делать** | Назначить сотрудника в бригаду: выбрать бригаду, указать роль (бригадир, рабочий), дату начала. Или назначить на проект напрямую с указанием роли и ставки. |
| **Результат** | Сотрудник привязан к бригаде и проекту. Прораб видит его в составе бригады. |
| **Подробнее** | [Бригады](../by-module/crews.md) |

> **Зачем?** Привязка к бригаде позволяет вести табель по бригадам, считать эффективность и равномерно распределять нагрузку.

---

## Шаг 5: Ежедневный табель

| | |
|---|---|
| **Кто** | Прораб (ежедневно) → HR (ежемесячно) |
| **Где** | Главное меню → **Кадры** → **Табели** |
| **Что делать** | **Прораб** — ежедневно отмечать: кто работал, сколько часов, на каком проекте, сверхурочные. **HR** — в конце месяца формировать табель Т-13: свести все дни, проверить коды (Я — явка, Б — больничный, О — отпуск, В — выходной). Отправить на утверждение. |
| **Результат** | Табель утверждён. Данные готовы для расчёта зарплаты. |
| **Подробнее** | [Табель учёта](../by-module/timesheets.md) |

### Коды табеля Т-13

| Код | Значение | Часы |
|-----|---------|------|
| **Я** | Явка (работа) | 8 |
| **Н** | Ночная смена | 8 (ночные) |
| **К** | Командировка | 8 |
| **Б** | Больничный | 0 |
| **О / ОТ** | Отпуск | 0 |
| **НН** | Неявка | 0 |
| **В** | Выходной / праздник | 0 |

> **Сверхурочные:** Первые 2 часа — ×1,5, далее — ×2,0. Максимум 4 часа/день и 120 часов/год по ТК РФ.

## Шаг 6: Расчёт зарплаты

| | |
|---|---|
| **Кто** | Бухгалтер |
| **Где** | Главное меню → **Расчёт зарплаты** |
| **Что делать** | На основе утверждённого табеля — рассчитать зарплату: оклад (или часы × ставка), надбавки, сверхурочные, премии. Удержать: НДФЛ (13%), взносы в фонды. Сформировать ведомость. Выплатить. |
| **Результат** | Зарплата рассчитана и выплачена. Данные отражены в бюджете проекта (статья «ФОТ»). |

### Формула (для оклада)

```
Оклад                          80 000 ₽
+ Сверхурочные (12 ч × 500 ₽ × 1,5)  +9 000 ₽
+ Премия (10%)                  +8 000 ₽
= Начислено                     97 000 ₽
— НДФЛ 13%                    −12 610 ₽
= К выплате                    84 390 ₽
```

## Шаг 7: Отпуска и отсутствия

| | |
|---|---|
| **Кто** | Сотрудник → Руководитель → HR |
| **Где** | Главное меню → **Кадры** → **Отпуска** |
| **Что делать** | **Сотрудник** подаёт заявку на отпуск: тип (ежегодный, за свой счёт, учебный), даты, причина. **Руководитель** согласовывает. **HR** проверяет остаток дней. Если одобрено — дни списываются из баланса. |
| **Результат** | Отпуск утверждён. В табеле за эти дни стоит код «О». Баланс дней уменьшен. |
| **Подробнее** | [Отпуска](../by-module/leave.md) |

### Виды отпусков

| Тип | Дней/год | Оплата | Согласование |
|-----|----------|--------|-------------|
| **Ежегодный оплачиваемый** | 28 (минимум по ТК РФ) | Средний заработок | Руководитель |
| **Дополнительный** | По условиям труда | Средний заработок | Руководитель |
| **За свой счёт** | Без ограничений | Без оплаты | Руководитель |
| **Учебный** | До 15 дней | По справке | HR |
| **Больничный** | Без ограничений | ФСС (после 3-го дня) | Не требуется |
| **Декретный** | 140–160 дней | ФСС | Не требуется |

## Шаг 8: Увольнение

| | |
|---|---|
| **Кто** | HR-специалист → Руководитель → Бухгалтер |
| **Где** | Карточка сотрудника → **«Заблокировать»** |
| **Что делать** | 1. Оформить приказ об увольнении. 2. Рассчитать компенсацию за неиспользованный отпуск. 3. Сделать окончательный расчёт. 4. Заблокировать учётную запись в системе. 5. Переназначить задачи другим сотрудникам. |
| **Результат** | Сотрудник в статусе **«Заблокирован»**. Не может войти в систему. Все его данные (задачи, комментарии, документы) сохранены. |
| **Подробнее** | [Пользователи и роли](../by-module/users-roles.md) |

> **Важно:** Не удаляйте аккаунт — только блокируйте. Данные сотрудника нужны для отчётности, аудита и возможного возвращения. Удаление необратимо.

## Что может пойти не так

| Проблема | Где обнаружится | Что делать |
|----------|----------------|------------|
| **Сотрудник не прошёл инструктаж** | Шаг 3 — нет записи | Нельзя допускать к работе! Провести инструктаж немедленно |
| **Просрочен сертификат** | Карточка сотрудника → Сертификаты (красный статус) | Направить на переаттестацию. До получения нового сертификата — отстранить от данного вида работ |
| **Ошибка в табеле** | Шаг 5 — проверка HR | Если табель ещё не утверждён — исправить. Если утверждён — создать корректировочный табель |
| **Перерасход ФОТ** | Бюджет → статья «Зарплата» | Разобраться: много сверхурочных, лишние люди на объекте, или норматив был занижен |
| **Сотрудник уволился, а задачи не переданы** | Задачи → фильтр по исполнителю | Срочно переназначить все открытые задачи другим сотрудникам |
| **Не хватает людей на объекте** | Бригады → неукомплектованы | Перевести из другой бригады, нанять нового, или привлечь субподрядчика |
