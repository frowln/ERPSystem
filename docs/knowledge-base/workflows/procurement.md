# Закупка материалов

> От момента, когда на стройке нужен материал, до момента, когда он лежит на складе и готов к выдаче.

---

## Обзор процесса

```
Потребность → Заявка → Согласование → КЛ → PO → Отгрузка → Приёмка → Склад → Выдача
```

| | |
|---|---|
| **Участники** | Прораб, Снабженец (ОПС), Руководитель, Кладовщик |
| **Время** | От 1 дня (срочная закупка) до 2–4 недель (стандартная) |
| **Результат** | Материал на складе, готов к выдаче на объект |

### Схема потока

```
┌──────────┐    ┌─────────┐    ┌────────────┐    ┌────┐
│Потребность│───→│ Заявка  │───→│Согласование│───→│ КЛ │
│ (прораб) │    │         │    │ (рук-ль)   │    │    │
└──────────┘    └─────────┘    └────────────┘    └────┘
                                                    │
     ┌──────────┐    ┌─────────┐    ┌──────┐       │
     │  Выдача  │←───│  Склад  │←───│Приёмка│←──PO←┘
     │(бригаде) │    │         │    │      │
     └──────────┘    └─────────┘    └──────┘
```

## Шаг 1: Определение потребности

| | |
|---|---|
| **Кто** | Прораб |
| **Где** | На объекте: план работ → какие материалы нужны |
| **Что делать** | Посмотреть план работ на ближайший период. Сверить с остатками на складе (Склад → Остатки). Определить, чего не хватает. Учесть сроки доставки — заявку нужно подать заранее. |
| **Результат** | Прораб знает: что нужно, сколько, к какому числу. |
| **Подробнее** | [Складской учёт](../by-module/warehouse.md) |

> **Важно:** Система показывает **лимитно-заборные ведомости** (М-8) — сколько материала уже выдано на объект и сколько осталось по лимиту. Если лимит исчерпан — нужно обоснование для дополнительной закупки.

---

## Шаг 2: Создание заявки на закупку

| | |
|---|---|
| **Кто** | Прораб или Снабженец |
| **Где** | Главное меню → **Снабжение** → **Создать заявку** |
| **Что делать** | Заполнить: проект, материал, количество, единицу измерения, желаемую дату поставки, приоритет (обычный / срочный / критический). Можно добавить несколько позиций в одну заявку. |
| **Результат** | Заявка создана в статусе **«Новая»**. Уходит на согласование. |
| **Подробнее** | [Закупки и PO](../by-module/purchase-orders.md) |

> **Совет:** Канбан-доска заявок показывает все заявки по статусам. Удобно видеть, что на каком этапе.

---

## Шаг 3: Согласование заявки

| | |
|---|---|
| **Кто** | Руководитель проекта (или финансист — для крупных сумм) |
| **Где** | Главное меню → **Снабжение** → **Заявки** (канбан-доска) |
| **Что делать** | Проверить обоснованность: нужен ли материал, правильное ли количество, есть ли бюджет. Утвердить или отклонить (с комментарием). Для сумм выше порога — может потребоваться двойное согласование. |
| **Результат** | Заявка в статусе **«Утверждена»**. Снабженец может начинать закупку. |

> **Если бюджет превышен:** Система покажет предупреждение — остаток бюджета по статье меньше суммы заявки. Руководитель решает: одобрить с перерасходом или отклонить.

---

## Шаг 4: Выбор поставщика (КЛ)

| | |
|---|---|
| **Кто** | Снабженец (ОПС) |
| **Где** | Главное меню → **Спецификации** → **Реестр КЛ** |
| **Что делать** | Если поставщик уже выбран через КЛ ранее — использовать его. Если нет — создать конкурентный лист: запросить цены у 3+ поставщиков, сравнить, выбрать победителя. |
| **Результат** | Поставщик выбран. Цена зафиксирована. |
| **Подробнее** | [Конкурентный лист](../by-module/competitive-list.md) |

> **Зачем 3 поставщика?** Это требование внутреннего контроля. Конкурентная закупка снижает цены на 10–20% и защищает от «откатных» схем.

---

## Шаг 5: Создание заказа на закупку (PO)

| | |
|---|---|
| **Кто** | Снабженец |
| **Где** | Главное меню → **Снабжение** → **Заказы на закупку** → **Создать** |
| **Что делать** | Создать PO (Purchase Order) на основе утверждённой заявки. Указать: поставщик, позиции (материал, количество, цена), условия оплаты, сроки доставки, адрес доставки. Отправить поставщику. |
| **Результат** | PO создан в статусе **«Отправлен поставщику»**. Бюджет обновлён — сумма резервирована в колонке «Зарезервировано». |
| **Подробнее** | [Закупки и PO](../by-module/purchase-orders.md) |

---

## Шаг 6: Отгрузка и доставка

| | |
|---|---|
| **Кто** | Снабженец (контроль) + Логист |
| **Где** | Главное меню → **Отгрузки** → **Заказы** |
| **Что делать** | Отслеживать статус отгрузки: поставщик подтвердил → загрузка → в пути → доставлено. Если доставка собственным транспортом — создать диспетчерское задание (транспорт, водитель, маршрут). |
| **Результат** | Материал доставлен на объект или на склад. |
| **Подробнее** | [Отгрузки и доставки](../by-module/dispatch.md) |

> **Частичная доставка:** Поставщик может отгружать партиями. Система отслеживает: заказано 100 тонн, доставлено 60, осталось 40. PO закроется, когда всё будет доставлено.

---

## Шаг 7: Приёмка на склад

| | |
|---|---|
| **Кто** | Кладовщик |
| **Где** | Главное меню → **Склад** → **Складские ордера** |
| **Что делать** | Создать приходный ордер (М-4). Проверить: количество совпадает с накладной, качество соответствует (нет повреждений, сертификаты в порядке). Если есть расхождения — зафиксировать акт. Провести ордер. |
| **Результат** | Материал оприходован на склад. Остатки обновлены. PO обновлён — статус позиции «Доставлено». |
| **Подробнее** | [Складской учёт](../by-module/warehouse.md) |

### Трёхстороннее сопоставление (3-way matching)

При оплате входящего счёта от поставщика система проверяет:

| Документ | Что сверяется |
|----------|-------------|
| **PO** (заказ) | Что заказали и по какой цене |
| **Накладная** (приёмка) | Что реально получили |
| **Счёт** (от поставщика) | За что нас просят заплатить |

Если все три совпадают — оплата согласовывается автоматически. Если нет — расхождение передаётся на ручную проверку.

---

## Шаг 8: Выдача на объект

| | |
|---|---|
| **Кто** | Кладовщик по требованию Прораба |
| **Где** | Главное меню → **Склад** → **Складские ордера** → **Расход (М-11)** |
| **Что делать** | Создать расходный ордер (М-11). Указать: материал, количество, проект, кому выдаётся. Система проверит лимит по ведомости М-8 — если лимит не превышен, выдача разрешена. |
| **Результат** | Материал выдан на объект. Остатки на складе уменьшились. Данные для отчёта М-29 (расход материалов) обновлены. |

> **Процесс завершён.** Материал на объекте, прораб может начинать работы.

## Что может пойти не так

| Проблема | Где обнаружится | Что делать |
|----------|----------------|------------|
| **Заявка отклонена** | Шаг 3 — согласование | Обосновать необходимость, скорректировать количество или найти альтернативный материал |
| **Нет бюджета на закупку** | Шаг 3 — предупреждение | Запросить корректировку бюджета у руководителя → [Контроль бюджета](budget-control.md) |
| **Поставщик задерживает доставку** | Шаг 6 — просрочка | Связаться с поставщиком. Если критично — найти альтернативного поставщика для срочной партии |
| **При приёмке обнаружен брак** | Шаг 7 — проверка качества | Оформить акт расхождений, вернуть брак поставщику, заказать замену |
| **Количество не совпадает с накладной** | Шаг 7 — 3-way matching | Зафиксировать расхождение, уведомить снабженца, скорректировать приходный ордер |
| **Лимит выдачи исчерпан** | Шаг 8 — проверка М-8 | Запросить увеличение лимита у руководителя проекта с обоснованием |
| **Перерасход материала на объекте** | Отчёт М-29 | Разобраться в причинах: брак при работе, потери, хищение. Скорректировать нормы или провести инвентаризацию |
