# Жизненный цикл проекта

> От первого звонка потенциального заказчика до сдачи объекта и гарантийного обслуживания.

---

## Обзор процесса

```
CRM лид → Тендер → КП → Договор → Проект → Планирование → Строительство → Закрытие (КС) → Сдача → Гарантия
```

| | |
|---|---|
| **Участники** | Менеджер по продажам, Коммерческий директор, Сметчик, Руководитель проекта, Прораб, Инженер ПТО, Бухгалтер |
| **Время** | От нескольких месяцев до нескольких лет |
| **Результат** | Объект построен, сдан заказчику, на гарантии |

### Схема потока

```
┌──────┐    ┌────────┐    ┌────┐    ┌────────┐    ┌────────┐    ┌────────────┐
│ Лид  │───→│ Тендер │───→│ КП │───→│ Договор│───→│ Проект │───→│Планирование│
│(CRM) │    │        │    │    │    │        │    │        │    │            │
└──────┘    └────────┘    └────┘    └────────┘    └────────┘    └────────────┘
                                                                      │
┌──────────┐    ┌───────┐    ┌──────────┐    ┌──────────────┐        │
│ Гарантия │←───│ Сдача │←───│ Закрытие │←───│ Строительство│←───────┘
│          │    │       │    │  (КС-2)  │    │              │
└──────────┘    └───────┘    └──────────┘    └──────────────┘
```

## Шаг 1: Лид в CRM

| | |
|---|---|
| **Кто** | Менеджер по продажам |
| **Где** | Главное меню → **CRM** → **Лиды** |
| **Что делать** | Зафиксировать контакт с потенциальным заказчиком: компания, контактное лицо, телефон, email, описание объекта, ожидаемая сумма, источник (сайт, рекомендация, тендерная площадка). Лид попадает в воронку продаж на этап **«Новый»** (вероятность 10%). |
| **Результат** | Лид создан. Менеджер начинает квалификацию. |
| **Подробнее** | [CRM: Лиды](../by-module/crm-leads.md) |

### Воронка продаж

| Этап | Вероятность | Что происходит |
|------|------------|----------------|
| **Новый** | 10% | Первый контакт, сбор информации |
| **Квалификация** | 25% | Подходит ли нам этот заказчик и объект? |
| **Предложение** | 50% | Подготовка и отправка КП |
| **Переговоры** | 75% | Обсуждение условий, корректировка цен |
| **Выигран** | 100% | Заказчик согласен → конвертация в проект |
| **Проигран** | 0% | Отказ (с указанием причины) |

## Шаг 2: Участие в тендере

| | |
|---|---|
| **Кто** | Менеджер по продажам + Сметчик |
| **Где** | Главное меню → **Портфель** → **Тендеры** |
| **Что делать** | Если заказчик проводит тендер — зарегистрировать участие. Подготовить тендерную документацию: техническое предложение, ценовое предложение, подтверждение квалификации (SRO, опыт, ресурсы). Подать в срок. |
| **Результат** | Тендерная заявка подана. Ожидание результатов. |

> **Не все проекты через тендер.** Если заказчик обращается напрямую — этот шаг пропускается, переходите к КП (шаг 3).

---

## Шаг 3: Коммерческое предложение (КП)

| | |
|---|---|
| **Кто** | Сметчик + Коммерческий директор |
| **Где** | Главное меню → **Финансы** → **Коммерческие предложения** |
| **Что делать** | Рассчитать стоимость проекта. Идеальный путь: Смета → Спецификация → КЛ → ФМ → КП (подробно описано в процессе [От сметы до оплаты](estimate-to-payment.md)). Сформировать PDF, отправить заказчику. |
| **Результат** | КП отправлено. Лид в CRM переходит на этап **«Предложение»** (50%). |
| **Подробнее** | [Коммерческое предложение](../by-module/commercial-proposals.md) |

---

## Шаг 4: Подписание договора

| | |
|---|---|
| **Кто** | Коммерческий директор → Юрист → Директор |
| **Где** | Главное меню → **Контракты** → **Создать** |
| **Что делать** | После согласия заказчика — оформить договор. Заполнить: номер, сумму, НДС, аванс, удержание, гарантию, сроки, штрафные санкции. Пройти внутреннее согласование (юрист → руководство → бухгалтерия). Подписать. |
| **Результат** | Договор подписан и активен. Лид в CRM — **«Выигран»**. Можно конвертировать в проект. |
| **Подробнее** | [Контракты](../by-module/contracts.md) |

---

## Шаг 5: Создание проекта

| | |
|---|---|
| **Кто** | Руководитель проекта |
| **Где** | Главное меню → **Проекты** → **Создать** (или конвертация из лида CRM) |
| **Что делать** | Создать проект: шифр (код), название, тип строительства, адрес, заказчик, плановые даты, бюджет. При конвертации из CRM — данные заполнятся автоматически. Система автоматически создаст **бюджет** и **финансовую модель**. |
| **Результат** | Проект создан в статусе **«Планирование»**. Автоматически созданы: бюджет, ФМ. |
| **Подробнее** | [Проекты](../by-module/projects.md) |

> **Статусы проекта:** Черновик → Планирование → **Активный** → Приостановлен → Завершён → Отменён

---

## Шаг 6: Планирование

| | |
|---|---|
| **Кто** | Руководитель проекта + Прораб |
| **Где** | Главное меню → **Планирование** → **Диаграмма Ганта** |
| **Что делать** | Составить план: разбить проект на этапы и работы (WBS). Установить зависимости между работами (нельзя начать отделку, пока не готов каркас). Назначить ресурсы: бригады, технику, материалы. Установить контрольные точки (milestones). |
| **Результат** | Календарный план готов. Базовый план зафиксирован (для сравнения план/факт). Статус проекта → **«Активный»**. |
| **Подробнее** | [Планирование](../by-module/planning.md) |

### Что входит в планирование

| Раздел | Что определяет |
|--------|---------------|
| **WBS** (иерархия работ) | Структура проекта: этапы → пакеты работ → задачи |
| **Диаграмма Ганта** | Сроки каждой работы, зависимости, критический путь |
| **Ресурсы** | Кто и что будет задействовано (люди, техника, материалы) |
| **Базовый план** | «Фотография» плана для будущего сравнения с фактом |

---

## Шаг 7: Строительство

| | |
|---|---|
| **Кто** | Прораб, Бригады, Снабженец, Инженер по ОТ |
| **Где** | Несколько модулей одновременно |
| **Что делать** | Выполнять работы по плану. Параллельно ведутся связанные процессы: |

| Процесс | Модуль | Подробнее |
|---------|--------|-----------|
| Закупка материалов | Снабжение + Склад | [Закупка материалов](procurement.md) |
| Учёт сотрудников | Кадры + Табели | [Сотрудник от приёма до увольнения](hr-lifecycle.md) |
| Контроль качества | Качество + Дефекты | [Управление дефектами](defect-management.md) |
| Охрана труда | Безопасность | Инспекции, инструктажи |
| Ежедневный учёт | Журнал КС-6а | Что сделано за день |
| Контроль бюджета | Бюджет + Cash Flow | [Контроль бюджета](budget-control.md) |

| **Результат** | Работы выполняются. Объёмы фиксируются в журнале. Проект движется к завершению. |

---

## Шаг 8: Закрытие периодов (КС-2 / КС-3)

| | |
|---|---|
| **Кто** | Инженер ПТО + Бухгалтер |
| **Где** | Главное меню → **Закрытие** → **КС-2** / **КС-3** |
| **Что делать** | Ежемесячно (или по договору): сформировать КС-2 (акт выполненных работ), пройти согласование, получить подпись заказчика. Объединить в КС-3 (справку о стоимости). Выставить счёт. Получить оплату. |
| **Результат** | Период закрыт. Деньги получены. Подробно — в процессе [От сметы до оплаты](estimate-to-payment.md). |
| **Подробнее** | [КС-2](../by-module/ks2.md), [КС-3](../by-module/ks3.md) |

---

## Шаг 9: Сдача объекта

| | |
|---|---|
| **Кто** | Руководитель проекта + Инженер ПТО + Заказчик |
| **Где** | Главное меню → **Завершение** |
| **Что делать** | 1. Закрыть все замечания (Punch List — 100%). 2. Подготовить исполнительную документацию (АОСР, КС-6, журналы, сертификаты). 3. Провести приёмочную комиссию (чек-лист из 35–45 пунктов). 4. Подготовить пакет документов для Стройнадзора. 5. Получить ЗОС (Заключение о соответствии). |
| **Результат** | Объект принят. ЗОС получено. Проект переходит в гарантийный период. |
| **Подробнее** | [Завершение проекта](../by-module/closeout.md) |

### Чек-лист сдачи объекта

| Этап | Прогресс | Статус |
|------|---------|--------|
| Punch List (все замечания закрыты) | 100% | Обязательно |
| Исполнительная документация передана | 100% | Обязательно |
| Приёмочная комиссия проведена | Подписана | Обязательно |
| Пакет для Стройнадзора отправлен | Подтверждён | Обязательно |
| ЗОС получено | Выдано | Обязательно |

---

## Шаг 10: Гарантийный период

| | |
|---|---|
| **Кто** | Руководитель проекта (контроль) + Подрядчики (исполнение) |
| **Где** | Главное меню → **Завершение** → **Гарантии** |
| **Что делать** | Отслеживать гарантийные обязательства по каждому подрядчику: вид работ, срок гарантии, сумма обеспечения. При обнаружении дефекта — создать гарантийную претензию. Подрядчик обязан устранить за свой счёт. По истечении гарантии — вернуть удержание. |
| **Результат** | Гарантийный период завершён без инцидентов (или все дефекты устранены). Удержание возвращено. Статус проекта → **«Завершён»**. |

> **Проект полностью завершён.** Все финансовые обязательства закрыты, документация передана, гарантия исполнена.

## Что может пойти не так

| Проблема | Где обнаружится | Что делать |
|----------|----------------|------------|
| **Лид проигран** | CRM (шаг 1) | Зафиксировать причину (цена, сроки, конкуренты). Анализировать статистику потерь |
| **Тендер проигран** | Портфель (шаг 2) | Запросить обратную связь от заказчика. Скорректировать ценовую политику |
| **Проект отстаёт от графика** | Планирование → EVM (SPI<1) | Увеличить ресурсы, пересмотреть критический путь, договориться с заказчиком о сдвиге сроков |
| **Бюджет превышен** | Бюджет → факт > план | Подробно — в [Контроль бюджета](budget-control.md) |
| **Punch List не закрыт к сдаче** | Завершение (шаг 9) | Усилить бригады, привлечь дополнительных подрядчиков, работать в выходные |
| **Стройнадзор не выдал ЗОС** | Завершение (шаг 9) | Устранить замечания, доукомплектовать документацию, подать повторно |
| **Гарантийный дефект** | Гарантии (шаг 10) | Направить претензию подрядчику, контролировать устранение, при отказе — использовать банковскую гарантию |
