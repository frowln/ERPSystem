# Карта системы ПРИВОД — Потоки данных и связи модулей

> Версия: 2026-03-12

---

## Основные потоки данных

### 1. Коммерческий поток (от лида до проекта)

```
CRM Лид → Контрагент → Оценка площадки → Тендер/Оппортунити → КП → Договор → Проект
```

**Описание:** Лид приходит из CRM (звонок, заявка с сайта, рекомендация). Менеджер конвертирует лида в контрагента (с ИНН, реквизитами). Проводится оценка площадки (SiteAssessment). Формируется тендерная заявка (BidPackage) или возможность (Opportunity). На основе расчётов составляется коммерческое предложение (КП). После согласования заключается договор, и создаётся проект.

**Модули:** CRM → Контрагенты → Оценка площадок → Портфель/Тендеры → КП → Контракты → Проекты

---

### 2. Сметный поток (ценообразование)

```
Спецификация (из ПД) → Конкурентный лист (КЛ) → Финансовая модель (ФМ) ← Смета (ЛСР) → КП
```

**Описание:** Из проектной документации формируется спецификация материалов и работ (наименование, марка, количество, вес — БЕЗ цен). Для каждой позиции спецификации создаётся конкурентный лист — сравнение цен от 3+ поставщиков, выбирается лучшая цена (себестоимость). Параллельно загружается локальная смета (ЛСР) из ГРАНД-Сметы — она даёт сметные цены (по нормативам ГЭСН/ФЕР/ТЕР). Обе цены попадают в финансовую модель (ФМ): себестоимость из КЛ, сметная цена из ЛСР. ФМ рассчитывает цену для заказчика (с наценкой) и маржу. Из ФМ формируется КП для клиента.

**Модули:** Спецификации → Конкурентные листы → ФМ (Бюджеты) ← Сметы → Коммерческие предложения

---

### 3. Финансовый поток (деньги)

```
Бюджет/ФМ → Счёт исходящий → Оплата от заказчика
                                    ↕
Заявка на закупку → PO → Счёт входящий → Оплата поставщику → Cash Flow
```

**Описание:** На основе ФМ формируется бюджет проекта (план). Заказчику выставляются исходящие счета, он оплачивает — это приход денег. Параллельно отдел снабжения создаёт заявки на закупку, из них формируются заказы поставщикам (PO), поставщики выставляют входящие счета, мы оплачиваем — это расход. Разница между приходом и расходом — cash flow. Факт сравнивается с планом (план/факт).

**Модули:** Бюджеты → Счета → Платежи → Cash Flow → Бухгалтерия (проводки, журналы)

---

### 4. Строительный поток (работа на площадке)

```
Проект → Задачи/Стадии → Наряд-задание → Журнал работ (ежедневный) → КС-2 → КС-3
```

**Описание:** Проект разбивается на задачи и стадии. Прорабу выдаётся наряд-задание (WorkOrder) на конкретные работы. Каждый день заполняется журнал работ (DailyLog): что сделано, кто работал, какая погода, какая техника. По итогам месяца выполненные работы оформляются актом КС-2 (акт выполненных работ). Из нескольких КС-2 формируется КС-3 (справка о стоимости выполненных работ и затрат). КС-3 — основание для оплаты от заказчика.

**Модули:** Проекты → Задачи → Операции (наряды) → Журнал работ → Закрытие (КС-2/КС-3)

---

### 5. Закупочный поток (снабжение)

```
Спецификация → Потребность → Заявка на закупку → Тендер поставщиков → PO → Отгрузка → Приход на склад
```

**Описание:** Из спецификации проекта определяется потребность в материалах. Снабженец создаёт заявку на закупку (PurchaseRequest). Проводится тендер среди поставщиков (через конкурентный лист). Побеждает лучшее предложение, формируется заказ поставщику (PurchaseOrder). Поставщик отгружает материал (Dispatch), материал приходует на склад.

**Модули:** Спецификации → Закупки (заявки, PO) → Диспетчеризация → Склад

---

### 6. Складской поток (материалы)

```
Приход (от поставщика) → Остаток → Расход (на объект) / Перемещение (между складами) → Списание → М-29
```

**Описание:** Материал приходит на склад (поступление). Хранится как остаток. Выдаётся на объект (расход по лимитно-заборной карте). Может перемещаться между складами или объектами. В конце месяца формируется отчёт М-29 (отчёт о расходе материалов). Если материал пришёл в негодность — списание. Периодически проводится инвентаризация (сверка факта с учётом).

**Модули:** Склад (приход, расход, перемещения, инвентаризация, М-29, лимитно-заборные карты)

---

### 7. Кадровый поток (HR)

```
Вакансия → Кандидат → Приём (трудовой договор) → Сотрудник → Бригада → Табель (Т-13) → Зарплата
                                                        ↓
                                                  Отпуск / Больничный
                                                  Инструктаж / Обучение
                                                  Квалификация / Сертификаты
```

**Описание:** Открывается вакансия (Recruitment), приходят кандидаты. При приёме оформляется трудовой договор и кадровый приказ. Сотрудник включается в бригаду (Crew). Бригадир ведёт табель рабочего времени (форма Т-13). По табелю начисляется зарплата. Параллельно отслеживаются отпуска, больничные, инструктажи, обучения, квалификации.

**Модули:** Подбор → HR (сотрудники, бригады) → HR Российский (Т-13, приказы) → Зарплата → Отпуска

---

### 8. Поток охраны труда (безопасность)

```
Инструктаж (вводный/первичный/повторный/внеплановый/целевой) → Допуск к работе
                                                                        ↓
Инцидент → Расследование → Акт Н-1 → Корректирующие меры
                                                                        ↑
Проверка (инспекция) → Нарушение → Предписание → Устранение
                                                                        ↑
СОУТ → Карта условий труда → Выдача СИЗ → Медосмотр
```

**Описание:** Каждый работник проходит инструктажи 5 видов. Без инструктажа — нельзя на площадку. Если происходит инцидент — расследование, акт Н-1 (при несчастном случае). Параллельно проводятся проверки (инспекции), выявляются нарушения, выдаются предписания. СОУТ (специальная оценка условий труда) определяет вредность, от неё зависит выдача СИЗ и прохождение медосмотров.

**Модули:** Безопасность (инструктажи, инциденты, инспекции, СИЗ, СОУТ, Н-1, сертификации)

---

### 9. Поток качества

```
Входной контроль материалов → Сертификаты материалов
                                        ↓
Инспекция качества → Дефект / Несоответствие (NCR) → Корректирующее действие → Закрытие
                                        ↓
                             Punch List (при сдаче) → Устранение → Проверка → АОСР
```

**Описание:** Материалы проверяются при поступлении (входной контроль). В процессе строительства проводятся инспекции качества. Если найден дефект — фиксируется NCR (несоответствие), назначается ответственный, определяются корректирующие действия. При сдаче объекта формируется punch list (перечень замечаний). Каждое замечание устраняется и проверяется. По факту приёмки скрытых работ оформляется АОСР.

**Модули:** Качество (инспекции, NCR, чек-листы, допуски) → Дефекты → Пунч-листы → ПТО (АОСР, акты скрытых работ)

---

### 10. Документный поток

```
Документ → Версии → Согласование (workflow) → Подписание (КЭП) → Отправка (ЭДО/СБИС) → Архив (CDE)
```

**Описание:** Документ создаётся в системе, проходит версионирование. Отправляется на согласование по маршруту (workflow: инженер → ГИП → директор). После согласования подписывается электронной подписью (КЭП). Подписанный документ отправляется контрагенту через ЭДО (СБИС или Диадок). Все документы хранятся в CDE (единая среда данных) с контролем доступа.

**Модули:** Документы → Согласования (Workflow) → КЭП → ЭДО → CDE

---

### 11. Поток завершения проекта (Closeout)

```
Punch List → Устранение замечаний → Исполнительная документация (АОСР, журналы) → ЗОС → Пакет стройнадзора → Гарантия
```

**Описание:** Перед сдачей объекта закрываются все пункты punch list. Собирается полный комплект исполнительной документации (АОСР, журналы работ, акты скрытых работ, исполнительные схемы). Оформляется ЗОС (заключение о соответствии). Формируется пакет для стройнадзора. После ввода в эксплуатацию начинается гарантийный период.

**Модули:** Пунч-листы → Исполнительная документация → Closeout (ЗОС, стройнадзор, ввод в эксплуатацию, гарантия)

---

### 12. Поток техники (автопарк)

```
Транспортное средство → Путевой лист → Рейс → Учёт ГСМ → Техобслуживание (по пробегу/дате)
                                                                ↓
                                                    GPS-трекинг → Рейтинг водителей
```

**Описание:** Техника закреплена за проектами. На каждый выезд оформляется путевой лист (маршрут, водитель, время, ГСМ). По путевым листам ведётся учёт расхода топлива. ТО проводится по расписанию (по пробегу или дате). GPS отслеживает местонахождение техники.

**Модули:** Автопарк (транспорт, путевые листы, ГСМ, ТО, GPS)

---

### 13. Интеграционный поток

```
ПРИВОД ↔ 1С Бухгалтерия (контрагенты, счета, платежи, номенклатура)
ПРИВОД ↔ СБИС / Диадок (ЭДО — акты, счета-фактуры)
ПРИВОД → ИСУП Минстроя (отчётность по гос. стройкам)
ПРИВОД ↔ Банк-клиент (выписки, платёжки)
ПРИВОД ↔ Telegram (уведомления)
ПРИВОД ↔ ГРАНД-Смета (импорт ЛСР)
ПРИВОД ↔ BIM (модели IFC, BCF замечания)
```

**Модули:** Интеграции, 1C, ИСУП, ЭДО, Обмен данными

---

### 14. Поток портала заказчика

```
Заказчик входит в портал → Видит: проекты, документы, расписание, счета, КС-2, задачи, дефекты, фотоотчёты, RFI
                                                ↓
                            Согласование КП → Подписание документов → Оплата счетов
```

**Описание:** Заказчик (клиент) имеет отдельный портал с ограниченным доступом. Он видит свои проекты, может скачать документы, согласовать КП, посмотреть расписание работ, задать вопросы (RFI), видеть дефекты и фотоотчёты. Подписывает документы через КЭП.

**Модули:** Портал (дашборд, проекты, документы, задачи, сообщения, счета, КС-2, расписание, дефекты, фотоотчёты, RFI, подписи)

## Визуальная схема ключевых потоков

```
                    ┌─────────────────────────────────────────────────────────┐
                    │                   ЗАКАЗЧИК (Портал)                     │
                    │   Проекты │ Документы │ Счета │ КС-2 │ Расписание      │
                    └────────────────────────┬────────────────────────────────┘
                                             │ согласование, оплата
                    ┌────────────────────────▼────────────────────────────────┐
                    │                     ДОГОВОР                             │
                    └───────┬──────────────────────────────────┬──────────────┘
                            │                                  │
              ┌─────────────▼──────────┐         ┌─────────────▼──────────────┐
              │        ПРОЕКТ          │         │      ФИНАНСЫ               │
              │ Задачи → Наряды        │         │ Бюджет/ФМ → Счета →       │
              │ → Журнал → КС-2 → КС-3│────────▶│ Платежи → Cash Flow       │
              └───────────┬────────────┘         └────────────────────────────┘
                          │
        ┌─────────────────┼──────────────────────┐
        │                 │                      │
  ┌─────▼─────┐   ┌──────▼──────┐   ┌───────────▼──────────┐
  │ СМЕТЫ/ФМ  │   │   СКЛАД     │   │  БЕЗОПАСНОСТЬ/       │
  │ ЛСР→ФМ→КП │   │ Приход→     │   │  КАЧЕСТВО            │
  │ Спец→КЛ→ФМ│   │ Расход→М-29 │   │ Инструктаж→Инспекция │
  └────────────┘   └─────────────┘   │ →Дефект→PunchList    │
                                     └──────────────────────┘
```
