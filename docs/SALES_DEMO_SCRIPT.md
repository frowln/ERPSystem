# Сценарий демонстрации ПРИВОД ERP (15 минут)

**Целевая аудитория:** строительная компания 20-50 человек (генподрядчик или субподрядчик).
**Демо-аккаунт:** demo@privod.ru / Demo123!
**URL:** https://app.privod.ru (или локальная демо-среда http://localhost:4000)

---

## Перед демо: подготовка (1 минута до начала)

1. Открыть браузер, войти под **demo@privod.ru** (роль Админ).
2. Убедиться, что в системе есть демо-проект с данными (задачи, сметы, бюджет, КС-2, склад).
3. Открыть вкладку **Дашборд** — она будет стартовой точкой.
4. Выключить уведомления браузера, чтобы не мешали.
5. Держать открытыми: Дашборд, Проекты, Сметы — для быстрого переключения.

> **Ключевой посыл при знакомстве:** «ПРИВОД — это единая платформа для строительной компании: от сметы до сдачи объекта. Не Excel, не 1С, не 5 отдельных программ — одна система.»

## Блок 1: Дашборд — здоровье компании за 30 секунд (2 мин)

**Экран:** `/` (главная)

**Что показываем:**
- **KPI-виджеты** вверху: количество проектов, задачи к сроку, бюджет, просроченные платежи.
- **Портфель объектов** — RAG-матрица (красный/жёлтый/зелёный) по 7 показателям: бюджет, сроки, качество, безопасность, документы, закупки, риски.
- **Графики**: распределение по статусам, кэш-флоу, ход выполнения.

**Говорим:**
> «Каждое утро руководитель открывает одну страницу и видит: какие объекты горят, где просрочка по платежам, какой CPI/SPI. Не нужно обзванивать прорабов — всё здесь.»

**Конкурентное преимущество:**
> «У Procore похожий дашборд, но он стоит от $375/мес за проект. У нас — за всю компанию.»

## Блок 2: Проекты и задачи — канбан (2 мин)

**Экран:** `/projects` → выбрать проект → `/tasks`

**Что показываем:**
1. **Список проектов** — карточки с прогрессом, менеджером, сроками.
2. Заходим в проект → **Канбан-доска задач**: колонки «Новые → В работе → На проверке → Готово».
3. **Перетаскиваем карточку** (работает и на мобильном — touch DnD).
4. Открываем задачу: чек-листы, вложения, комментарии, трекинг времени, зависимости.
5. Показываем **Ганта** (диаграмма): зависимости, критический путь.

**Говорим:**
> «Прораб создаёт задачу с телефона, прикладывает фото, инженер видит её на канбане. Не WhatsApp-группа на 200 сообщений, а структурированная работа.»

**Живое действие:** создаём задачу «Залить фундамент секция Б» → назначаем ответственного → ставим срок → перетаскиваем в «В работе».

## Блок 3: Сметы → ФМ → КП — уникальная цепочка (3 мин)

**Это наш главный козырь. На него тратим больше всего времени.**

**Экран:** `/estimates`

### Шаг 1: Импорт ЛСР из ГРАНД-Сметы
1. Открываем **Сметы** → **Импорт ЛСР**.
2. Загружаем xlsx-файл из ГРАНД-Сметы (Методика 2020 РИМ).
3. Система парсит: разделы, позиции, ресурсы, оборудование, итоги.
4. Показываем дерево сметы: секции → позиции → ресурсы.

**Говорим:**
> «Вы экспортировали смету из ГРАНД-Сметы — и за 5 секунд она в системе. Со всеми разделами, кодами ГЭСН/ФЕР, ресурсами. Не нужно руками перебивать 800 позиций.»

### Шаг 2: Автоматическое создание Финансовой модели (ФМ)
1. При создании проекта автоматически создаётся ФМ.
2. Позиции ЛСР транслируются в строки ФМ: наименование, сметная стоимость (revenue), НДС.
3. Показываем **ФМ**: таблица с KPI — итого доходы, расходы, маржа, НДС.

**Говорим:**
> «Смета загрузилась — и ФМ уже считает вашу маржу. Видите: выручка 75 млн, НДС 12.5 млн, маржа 18%. Если маржа ниже 15% — красный флаг, есть риск убытка.»

### Шаг 3: Спецификация → Конкурентный лист → КП
1. **Спецификация**: загружаем xlsx с позициями (наименование, тип, код, завод, кол-во, вес).
2. **Конкурентный лист**: минимум 3 поставщика, сравнение цен, выбор победителя.
3. **Коммерческое предложение (КП)**: формируем из ФМ + КЛ, печатная форма.

**Говорим:**
> «Вся цепочка: Спецификация → Конкурентный лист → ФМ → КП. Ни в 1С:УСО, ни в PlanRadar этого нет. Обычно это 3 файла Excel, которые никак не связаны. У нас — одна цепочка с автоматическим расчётом.»

## Блок 4: КС-2 / КС-3 — исполнительная документация (2 мин)

**Экран:** `/closing/ks2`

**Что показываем:**
1. **Журнал форм КС-2**: список актов, привязка к договору, период.
2. Открываем КС-2: позиции из журнала работ, объёмы, цены, итоги.
3. **Печатная форма**: стандартный бланк КС-2 (Постановление Госкомстата №100).
4. **КС-3**: автоматическая генерация из КС-2, нарастающий итог.

**Говорим:**
> «Прораб отмечает выполненные работы в журнале — КС-2 формируется автоматически. Не нужно сидеть вечерами в Excel. Печатная форма по ГОСТ — сразу в PDF.»

**Живое действие:** кликаем «Печать» → показываем PDF с правильными шапками, ИНН, подписями.

## Блок 5: Склад + М-29 (1 мин)

**Экран:** `/warehouse`

**Что показываем:**
1. **Остатки** — текущие запасы по складам.
2. **Движения** — приход, расход, перемещения между объектами.
3. **М-29** (лимитно-заборные карты): привязка к объекту, контроль лимитов.

**Говорим:**
> «Вы всегда знаете, сколько арматуры на объекте и сколько ещё нужно. М-29 формируется автоматически — бухгалтерия получает готовый документ, а не стопку бумаг.»

## Ответы на типичные возражения

### «У нас уже есть 1С»
> «Отлично! ПРИВОД не заменяет 1С:Бухгалтерия — он интегрируется с ней. 1С — это бухучёт. А где у вас канбан задач, конкурентные листы, дашборд проектов, М-29? В Excel? ПРИВОД закрывает то, что 1С не покрывает — управление стройкой.
> У нас есть модуль интеграции с 1С: синхронизация контрагентов, экспорт КС-2/КС-3, импорт платежей.»

### «Дорого»
> «4 990 руб./мес — это стоимость 3 часов работы одного инженера. Система экономит 15+ часов в неделю. Посчитайте: если инженер стоит 150 000 руб./мес, экономия на его времени — 60 000 руб./мес. ROI = 12x.
> Плюс бесплатный план — попробуйте на одном объекте, убедитесь сами.»

### «Сложно внедрять / люди не будут пользоваться»
> «Интерфейс сделан для прорабов, а не для айтишников. Канбан — как доска на стене. Задачи — как список дел в телефоне.
> Средний срок внедрения — 1 неделя. Мы помогаем: загружаем ваши данные, обучаем команду.
> PWA — ставится на телефон как приложение, работает даже без интернета.»

### «Мы маленькие, нам не нужна ERP»
> «Как раз маленьким компаниям критично не терять деньги. Когда 5 прорабов ведут 3 объекта — без системы неизбежны потери на материалах, просрочки по актам, пропущенные инструктажи.
> Бесплатный план — 3 пользователя, 2 проекта. Начните без риска.»

### «А если вы закроетесь?»
> «Все ваши данные доступны через API и экспорт в Excel/PDF в любой момент. Плюс платформа развёртывается на вашем сервере (self-hosted) — полная независимость.»
