# Золотой стандарт: идеальная SaaS ERP для строительства (Россия)

> **Что это**: полный эталон того, что должна иметь идеальная облачная ERP-система
> для строительных компаний, готовая к продакшену и продажам на российском рынке.
>
> **Зачем**: чтобы понимать полную картину — что должно быть, чтобы система была
> конкурентоспособной, безопасной, законной, удобной и продаваемой.
>
> **Как читать**: документ разбит на направления. Каждое направление содержит описание
> с точки зрения конкретных ролей — кому это нужно и почему.

---

## ЧАСТЬ 1. ПРОДУКТОВЫЕ ФИЧИ — ЧТО ДОЛЖНА УМЕТЬ СИСТЕМА

### 1.1. Управление проектами

**Кому нужно**: директор, руководитель проекта, ПТО

- Создание проекта с полной карточкой: название, адрес, тип (жильё, коммерция, инфраструктура, промышленность), заказчик, генподрядчик, сроки, бюджет, ответственные
- Жизненный цикл проекта с фазами: инициация → проектирование → тендер → строительство → пусконаладка → сдача → гарантийный период. Автоматические переходы между фазами при выполнении условий
- Шаблоны проектов — создал шаблон один раз, дальше новые проекты по нему: задачи, роли, документы, чек-листы копируются автоматически
- Портфель проектов — сводный дашборд руководителя: все проекты, их статус, маржинальность, отклонения по срокам и деньгам на одном экране
- Мастер-проект с суб-проектами (для крупных застройщиков с несколькими очередями)
- Вехи (milestones) с привязкой к контрактным обязательствам
- Карточка проекта для клиента (заказчика) — ограниченный вид, без внутренней кухни
- Монте-Карло или иной вероятностный анализ рисков по срокам и бюджету
- Реестр рисков: описание, вероятность, воздействие, план митигации, ответственный

### 1.2. Планирование и календарное графикование

**Кому нужно**: ПТО, руководитель проекта, прораб

- Календарный график работ — визуально (диаграмма Ганта), с зависимостями (FS, FF, SS, SF), лагами и ограничениями
- WBS (иерархическая структура работ) — от объекта до элементарной работы
- Критический путь (CPM) — визуальное выделение работ, задержка которых сдвигает весь проект
- Базовые линии (baselines) — зафиксировать план, потом сравнивать план vs факт
- Еженедельные графики (lookahead schedules) на 2-4 недели — что конкретно делать прямо сейчас
- Ресурсное планирование — кто где работает, какая техника где задействована, нет ли конфликтов
- EVM (Earned Value Management) — объективная оценка прогресса через CPI, SPI и прогнозы EAC, ETC
- Импорт/экспорт из MS Project (.mpp) и Primavera P6 (.xer) — потому что многие уже пользуются ими
- Автоматическое обновление графика на основе ежедневных отчётов с площадки
- Учёт погодных дней — интеграция с API погоды, автоматическая пометка "непогода" в журнале работ
- 4D визуализация (связь графика с BIM-моделью) — видеть прогресс на 3D модели по времени

### 1.3. Финансы, сметы и бюджетирование

**Кому нужно**: сметчик, экономист, финансовый директор, бухгалтер, руководитель проекта

**Сметная часть:**
- Локальные сметы, объектные сметы, сводный сметный расчёт — полная иерархия
- Ресурсный метод расценки и базисно-индексный метод
- Справочники ГЭСН, ФЕР, ТЕР (актуальные редакции) — встроенные или импортируемые
- Индексы пересчёта в текущие цены (Минстрой, региональные центры)
- Коэффициенты: стеснённость, зимнее удорожание, высотность и т.д.
- Накладные расходы и сметная прибыль по видам работ (МДС 81-33.2004 / МДС 81-25.2001)
- НДС — правильный расчёт, включая случаи с УСН
- Экспорт сметы в форматы ГРАНД-Смета, Гектор, XML-обмен Минстроя

**Бюджетирование:**
- Бюджет проекта с кост-кодами (WBS-привязка)
- План-факт анализ в реальном времени: бюджет vs обязательства vs фактические затраты vs прогноз
- Денежный поток (cash flow) — прогноз поступлений и выплат по неделям/месяцам
- Commit (обязательства) — сумма подписанных контрактов и заказов
- Change orders — дополнительные работы, их влияние на бюджет
- Прогнозирование — автоматический расчёт "до конца проекта" (EAC) на основе текущих трендов
- Маржинальность проекта — live-показатель, видимый руководству

**Взаиморасчёты:**
- КС-2 (акт выполненных работ) и КС-3 (справка о стоимости) — генерация по данным системы, по ГОСТ
- Счета на оплату, счета-фактуры, акты сверки
- Авансы и их зачёт
- Удержания (гарантийные, штрафные)
- Взаимозачёты между контрагентами
- Автоматическое сопоставление: смета → КС-2 → оплата → факт — полная цепочка
- Признание выручки по проценту завершения (МСФО 15 / ФСБУ)

**Интеграция с бухгалтерией:**
- Синхронизация с 1С:Бухгалтерия — проводки, платежи, акты
- Импорт банковских выписок (через 1С или напрямую)
- Генерация закрывающих документов для клиентов SaaS (акт, счёт-фактура)

### 1.4. Контракты и договорная работа

**Кому нужно**: юрист, руководитель проекта, коммерческий директор

- Реестр контрактов: все договоры компании в одном месте, статус, суммы, сроки
- Типы: генподрядный, субподрядный, на поставку, на проектирование, на услуги
- Шаблоны типовых договоров с автозаполнением реквизитов из карточки контрагента
- Версионирование — полная история изменений каждого договора
- Доп.соглашения с автопересчётом общей суммы контракта
- Привязка контракта к бюджету проекта — автоматическое отражение в финансовой модели
- Контроль сроков: уведомления о приближающемся окончании, пролонгации, дедлайнах
- Претензионная работа: фиксация нарушений, формирование претензий, отслеживание ответов
- Гарантийные обязательства — сроки, суммы удержаний, чек-листы гарантийного периода
- Электронная подпись (КЭП) — подписание документов прямо в системе
- Согласование договоров — цепочка: инициатор → юрист → экономист → директор (с комментариями и историей)

### 1.5. Закупки и снабжение

**Кому нужно**: снабженец, начальник ОМТС, финансовый директор

- Заявки на материалы — создание прямо из сметы или вручную, с привязкой к проекту и работе
- Конкурентные листы — минимум 3 коммерческих предложения от поставщиков, сравнение
- Тендерные процедуры — создание тендера, рассылка приглашений, приём предложений, скоринг
- Преквалификация поставщиков — анкета, проверка, рейтинг, чёрный список
- Заказы поставщикам — формирование, согласование, отправка
- Отслеживание поставок — статус: заказано → отгружено → в пути → доставлено → принято
- Приходование на склад с проверкой количества и качества
- Автоматические заявки при достижении минимального остатка на складе
- Учёт материалов по проектам — кто сколько получил, сколько использовал, сколько вернул
- Форма М-29 (отчёт о расходе материалов) — генерация по данным системы
- QR/штрих-коды для складского учёта — мобильный сканер для прихода/расхода
- Рейтинг поставщиков — по срокам, качеству, ценам, количеству рекламаций

### 1.6. Складской учёт

**Кому нужно**: кладовщик, снабженец, прораб

- Приходы, расходы, перемещения, списания, возвраты
- Учёт по партиям и срокам годности (для цемента, добавок и т.д.)
- Инвентаризация — генерация ведомостей, фиксация факта, акт расхождений
- Минимальные и максимальные остатки — алерты
- Несколько складов (центральный, приобъектные, передвижные)
- Резервирование — забронировать материал под конкретный проект/работу
- Учёт спецодежды и СИЗ (средств индивидуальной защиты) по сотрудникам

### 1.7. Документооборот

**Кому нужно**: делопроизводитель, ПТО, юрист, все

- Единое хранилище документов (CDE — Common Data Environment) — вся документация проекта в одном месте
- Структура папок по ГОСТ/стандарту компании
- Версионирование — каждое изменение сохраняет предыдущую версию
- Реестры документов: входящие, исходящие, внутренние, проектная документация
- Автоматическая нумерация документов по шаблону
- Полнотекстовый поиск по содержимому документов (включая сканы через OCR)
- Шаблоны документов с автозаполнением — письма, акты, справки, протоколы
- Watermark / защита документов от несанкционированного распространения
- Электронный архив — долгосрочное хранение с быстрым доступом

**ЭДО (электронный документооборот):**
- Интеграция с Диадок, СБИС, Контур — обмен юридически значимыми документами
- КЭП (квалифицированная электронная подпись) — подписание в системе
- Маршруты согласования — настраиваемые цепочки (параллельные, последовательные)
- Журнал согласований — кто когда посмотрел, прокомментировал, утвердил/отклонил

**Исполнительная документация:**
- Акты скрытых работ (АОСР)
- Акты освидетельствования ответственных конструкций
- Общий и специальные журналы работ
- Исполнительные схемы
- Паспорта и сертификаты на материалы
- Комплектование — автоматическая проверка полноты комплекта ИД перед сдачей

### 1.8. Цепочки согласований (Workflow Engine)

**Кому нужно**: все роли — это сквозной механизм

- Визуальный конструктор маршрутов согласования (без кода)
- Шаблоны маршрутов: для договоров, счетов, КС-2, заявок на материалы, приказов и т.д.
- Параллельные и последовательные этапы
- Условная маршрутизация — если сумма > 1 млн, добавить согласование директора
- Делегирование — замещающий может согласовать за отсутствующего
- Эскалация — если не согласовано за N часов, уведомить руководителя
- Комментарии на каждом этапе — не просто "утвердил/отклонил", а с обоснованием
- Полная история — кто, когда, какое решение принял, с какими комментариями
- Мобильное согласование — утвердить/отклонить прямо со смартфона

### 1.9. Полевые операции (Field Management)

**Кому нужно**: прораб, мастер, начальник участка

- Ежедневный журнал работ — что делали, сколько людей, какая техника, погода, простои
- Наряд-задания — выдача, контроль выполнения, закрытие
- Фиксация дефектов — фото + описание + привязка к чертежу + ответственный
- Punch-list (ведомость замечаний) — перед приёмкой, каждое замечание с фото и дедлайном
- Диспетчеризация — распределение бригад и техники по объектам/участкам
- Табель учёта рабочего времени — с GPS-привязкой "пришёл/ушёл" на объект
- Ежедневные/еженедельные отчёты — автогенерация на основе внесённых данных
- Фотофиксация прогресса — привязка к дате, месту, элементу конструкции
- 360° фото для документирования текущего состояния
- Привязка замечаний к чертежам — пометка прямо на плане "тут проблема"
- Сравнение версий чертежей — наложение, подсветка изменений
- Оффлайн-режим — работа без интернета с последующей синхронизацией (стройка = плохой сигнал)

### 1.10. Качество (QA/QC)

**Кому нужно**: инженер по качеству, ПТО, заказчик

- Планы качества по проекту — какие проверки, на каких этапах, кем выполняются
- Чек-листы проверок (инспекций) — настраиваемые шаблоны по видам работ
- Входной контроль материалов — проверка при приёмке, фиксация несоответствий
- Предписания и замечания — от технадзора, авторского надзора, заказчика
- Регистр несоответствий — дефекты, корректирующие действия, проверка устранения
- Отслеживание сертификатов и паспортов качества на материалы
- Привязка проверок к BIM-элементам или чертежам
- Статистика по качеству — тренды, повторяющиеся дефекты, проблемные подрядчики

### 1.11. Охрана труда и безопасность

**Кому нужно**: инженер по охране труда, прораб, директор

- Инструктажи — вводный, первичный, повторный, целевой, внеплановый — журнал с подписями (электронными)
- Обучение и проверка знаний — протоколы, удостоверения, контроль сроков действия
- Медосмотры — реестр, даты прохождения, контроль сроков
- Выдача СИЗ — по нормам, учёт по сотрудникам
- Происшествия и инциденты — регистрация, расследование, корректирующие мероприятия
- Нарядо-допуски на опасные работы (высота, ограниченные пространства, огневые работы)
- Проверки безопасности (Safety walks) — мобильный чек-лист с фото
- Штрафные санкции за нарушения — предупреждение → штраф → отстранение
- Дашборд безопасности — KPI: количество инцидентов, дни без происшествий, выявленные нарушения
- Сертификация допусков персонала — СРО, электробезопасность, высотные работы, сварка

### 1.12. PM Workflows: RFI, Submittals, Issues

**Кому нужно**: ПТО, руководитель проекта, проектировщик, субподрядчик

- RFI (запросы на разъяснение) — от подрядчика к проектировщику/заказчику, с трекингом ответа
- Submittals (согласования) — материалы, оборудование, технические решения — отправка → рецензия → утверждение/отклонение
- Issues (проблемы/вопросы) — с приоритетами, ответственными, дедлайнами
- Change Orders (распоряжения об изменениях) — инициация → оценка стоимости → согласование → включение в бюджет
- SLA на каждый тип — например, ответ на RFI за 48 часов, иначе эскалация
- Привязка к спецификации — RFI на конкретный раздел / лист чертежа
- Связь с бюджетом — change order автоматически отражается в финансах
- Настраиваемый workflow — кто согласует, в каком порядке, при каких условиях

### 1.13. HR и кадры

**Кому нужно**: HR, кадровик, бухгалтер, директор

- Штатное расписание по объектам и подразделениям
- Карточки сотрудников — ФИО, должность, допуски, квалификация, документы
- Приём, перевод, увольнение — с генерацией приказов
- Табель учёта рабочего времени (Т-12/Т-13)
- Отпуска и больничные — заявки, согласование, график отпусков
- Расчёт зарплаты с учётом сдельной/повременной оплаты, надбавок, ночных, вахтовых
- Самозанятые — учёт, акты, отчётность
- Рекрутинг — заявки на подбор, воронка кандидатов
- Обучение — планы повышения квалификации, учёт пройденных курсов
- Командировки — оформление, суточные, авансовые отчёты
- Расчёт себестоимости человеко-часа по проектам — сколько реально стоит каждый рабочий

### 1.14. Техника и оборудование

**Кому нужно**: механик, диспетчер, прораб, финансист

- Реестр техники — собственная и арендованная
- Учёт наработки (моточасы, пробег)
- ТО и ремонты — графики, история, напоминания
- Распределение техники по объектам — план, факт, конфликты
- Заправка и ГСМ — учёт расхода топлива по единицам техники
- GPS-трекинг (при наличии оборудования) — местоположение в реальном времени
- Амортизация — автоматический расчёт
- Сменные рапорты механизаторов
- Учёт мелкого инструмента — выдача, возврат, списание
- IoT-интеграция — датчики техники, автоматический сбор данных

### 1.15. CRM и коммерция

**Кому нужно**: коммерческий директор, менеджер по продажам, маркетолог

- Реестр контрагентов — заказчики, подрядчики, поставщики — с полной карточкой (ИНН, реквизиты, контакты, история)
- Проверка контрагентов — по ИНН через ФНС / DaData / Контур.Фокус
- Воронка продаж — от лида до подписания контракта
- Коммерческие предложения — генерация из шаблона, версионирование, трекинг
- Тендерный отдел — мониторинг тендеров (госзакупки, коммерческие), календарь дедлайнов
- История взаимодействий — звонки, письма, встречи по каждому контрагенту
- Интеграция с email — переписка привязывается к карточке контрагента/проекта
- Сегментация клиентов — по типу, региону, объёму, статусу
- Автоматические задачи — "позвонить через 3 дня", "отправить КП"
- Партнёрские отношения — кто привёл клиента, партнёрская программа

### 1.16. Аналитика и отчётность

**Кому нужно**: директор, финансист, ПТО, все руководители

- Дашборды — настраиваемые, с виджетами (график, таблица, KPI, chart)
- Портфельная аналитика — все проекты на одном экране: статус, маржа, отклонения
- Детальные отчёты — по проекту, по подрядчику, по статье затрат, по сотруднику
- Конструктор отчётов — построить отчёт самому, без разработчика (drag-and-drop)
- Экспорт в Excel, PDF, Word
- Автоматические отчёты по расписанию — "каждый понедельник утром отправлять сводку"
- KPI дашборд руководителя — маржинальность, выручка, дебиторка, загрузка техники, безопасность
- Сравнительный анализ — проект vs проект, подрядчик vs подрядчик, план vs факт
- Предиктивная аналитика — прогноз сроков и бюджета на основе текущих трендов

### 1.17. BIM

**Кому нужно**: проектировщик, ПТО, заказчик

- Просмотр IFC-моделей прямо в браузере (3D viewer)
- Привязка задач, дефектов, RFI к элементам модели
- Поиск коллизий (clash detection) — пересечения инженерных сетей
- 4D (время) — визуализация хода строительства на модели
- 5D (стоимость) — привязка стоимостей к элементам модели
- Импорт из Revit, ArchiCAD, Navisworks (через IFC)
- Управление версиями моделей

### 1.18. Коммуникации

**Кому нужно**: все

- Внутренний мессенджер — личные сообщения и групповые чаты по проектам
- Комментарии к любой сущности — к задаче, документу, КС-2, дефекту и т.д.
- @упоминания с уведомлением
- Уведомления — in-app, email, push (мобильные), Telegram-бот
- Календарь — личный и проектный, синхронизация с Google Calendar / Outlook
- Видеозвонки — встроенные или через интеграцию (Jitsi / Zoom)
- Лента активности — что происходит по проекту в реальном времени
- Email-интеграция — письма привязываются к проекту/контрагенту

### 1.19. Закрытие проекта (Closeout)

**Кому нужно**: ПТО, руководитель проекта, заказчик

- Чек-лист приёмки — все пункты, которые надо закрыть перед сдачей
- Punch-list — ведомость замечаний → устранение → повторная проверка
- Сборка исполнительной документации — автоматическая проверка полноты комплекта
- Акт приёмки объекта — генерация по форме
- Handoff (передача) — передача объекта заказчику / эксплуатирующей организации
- Гарантийный период — сроки, учёт гарантийных обращений, ответственные
- Архивация проекта — все документы, история, финансы — в архив с возможностью доступа
- Lessons Learned — фиксация выводов: что пошло хорошо, что плохо, что улучшить

## ЧАСТЬ 2. АРХИТЕКТУРА И ТЕХНОЛОГИИ

**Кому нужно**: CTO, архитектор, разработчик

### 2.1. Серверная архитектура
- Модульный монолит или микросервисы — зависит от масштаба. Для старта модульный монолит правильный выбор
- REST API с полной OpenAPI-документацией (Swagger)
- WebSocket для real-time обновлений (уведомления, чат, live-данные)
- Message Queue (RabbitMQ или Kafka) — для асинхронных операций (генерация отчётов, email, интеграции)
- CQRS для аналитики — отдельные модели для чтения тяжёлых отчётов
- Кэширование — Redis для сессий, справочников, часто запрашиваемых данных
- Файловое хранилище — S3-совместимое (MinIO / Yandex Object Storage)
- Полнотекстовый поиск — Elasticsearch или PostgreSQL FTS
- Фоновые задачи — планировщик (cron / Quartz) для отчётов, уведомлений, backup

### 2.2. База данных
- PostgreSQL — основная СУБД
- Flyway / Liquibase — миграции, версионирование схемы
- Партиционирование больших таблиц (audit_log, notifications, activity_feed)
- Индексы — правильные, проверенные explain analyze
- Connection pooling (HikariCP + PgBouncer для прод)
- Read replicas — для отчётности, чтобы не грузить основную базу
- Backup — автоматический, ежедневный, с проверкой восстановления
- Row Level Security (RLS) — для изоляции данных тенантов

### 2.3. Фронтенд
- SPA (React / Vue) с TypeScript
- Роутинг, code splitting, lazy loading по модулям
- Design System — собственная библиотека компонентов
- Responsive — работает на десктопе, планшете, мобильном
- Dark mode
- i18n — русский как минимум, английский желательно
- Оффлайн-режим (Service Worker + IndexedDB) для полевых операций
- Storybook — каталог компонентов для разработки и QA
- Accessibility (WCAG 2.1 AA) — для госзаказчиков это может быть требованием
- Performance — bundle budgets, Lighthouse score > 80

### 2.4. Качество кода
- Единый стиль — linter (ESLint, Checkstyle), formatter (Prettier, google-java-format)
- Статический анализ — SonarQube или аналог
- Архитектурные тесты (ArchUnit) — модули не лезут друг в друга
- Unit-тесты — покрытие критических бизнес-расчётов (сметы, финансы, КС-2)
- Integration-тесты — API + база данных
- E2E тесты — ключевые сценарии (Playwright / Cypress)
- Нагрузочные тесты — k6 / Gatling / JMeter
- Единый GlobalExceptionHandler с кодами ошибок
- API rate limiting — глобальный + per-endpoint

## ЧАСТЬ 3. МУЛЬТИТЕНАНТНОСТЬ (SaaS)

**Кому нужно**: архитектор, безопасник, бизнес

### 3.1. Изоляция данных
- Каждая компания-клиент (тенант) видит ТОЛЬКО свои данные — это не опция, это обязательство
- Row Level Security (RLS) в PostgreSQL — на уровне БД, а не только в коде
- Автоматическая фильтрация на уровне ORM (Hibernate filter по tenant_id)
- Изолированные файлы — каждый тенант имеет свой префикс/бакет в S3
- Redis — кэш namespace per tenant
- Тесты на утечку данных — автоматические, в CI

### 3.2. Подписки и биллинг
- Тарифные планы: Free (ознакомительный) → Starter → Professional → Enterprise
- Ограничения по плану: количество проектов, пользователей, объём хранилища, модули
- Trial период (14-30 дней) с полным функционалом
- Оплата: помесячная и годовая (скидка за год)
- Интеграция с платёжной системой (YooKassa для РФ)
- Автоматическое выставление счетов и генерация закрывающих документов
- Dunning — обработка неудачных платежей (retry, уведомления, блокировка)
- Prorated billing — пересчёт при смене тарифа
- Usage-based элемент — доплата за доп.пользователей / проекты / хранилище

### 3.3. Tenant self-service
- Регистрация компании — через сайт, без участия вендора
- Настройка: логотип, реквизиты, роли, структура компании
- Управление подпиской — смена тарифа, добавление пользователей, история платежей
- Экспорт всех данных — по требованию (ФЗ-152 требует)
- Удаление аккаунта — с полной очисткой данных

## ЧАСТЬ 4. БЕЗОПАСНОСТЬ

**Кому нужно**: безопасник, CTO, юрист, enterprise-клиенты

### 4.1. Аутентификация и авторизация
- JWT или session-based аутентификация
- MFA (TOTP — Google Authenticator, SMS)
- SSO — SAML 2.0 и OIDC (для enterprise-клиентов с Active Directory / Keycloak)
- SCIM — автоматическое создание/удаление пользователей из AD/LDAP
- RBAC — роли с гранулярными правами (не просто 5 ролей, а настраиваемая матрица)
- IP whitelist — ограничение доступа по IP (для enterprise)
- Настраиваемая политика паролей — длина, сложность, срок действия
- Защита от брутфорса — progressive delay, account lockout, CAPTCHA
- Сессии — лимит одновременных сессий, принудительный logout
- API Keys для интеграций — с ограничением по scope

### 4.2. Защита данных
- HTTPS everywhere (TLS 1.2+)
- Шифрование данных at rest (PostgreSQL TDE или column-level для PII)
- Шифрование файлов в S3 (SSE)
- Аудит-лог — кто что когда сделал, неизменяемый (immutable)
- Автоматическая ротация секретов и JWT-ключей
- Маскирование PII в логах
- DLP (Data Loss Prevention) — предупреждение при скачивании большого объёма данных
- Vulnerability scanning — Dependabot / Snyk / Trivy в CI
- SAST/DAST — статический и динамический анализ безопасности в CI pipeline
- Penetration testing — минимум раз в год (для enterprise-клиентов это требование)

### 4.3. Compliance (соответствие закону)
- 152-ФЗ (персональные данные):
  - Регистрация в реестре операторов ПДн (Роскомнадзор)
  - Политика обработки персональных данных
  - Согласие на обработку ПДн
  - Модель угроз
  - Серверы в РФ (для хранения ПДн граждан РФ)
  - Назначение ответственного за обработку ПДн
- 149-ФЗ (информация, информационные технологии) — если есть публичная часть
- ФСТЭК — если работа с госзаказчиками, может потребоваться аттестация
- SOC 2 — если планируется работа с крупными корпоративными клиентами

## ЧАСТЬ 5. ИНФРАСТРУКТУРА И DevOps

**Кому нужно**: DevOps, CTO

### 5.1. CI/CD
- Автоматический pipeline: commit → тесты → сборка → деплой на staging → smoke-тесты → деплой на прод (с ручным approval)
- Docker-контейнеризация всех сервисов
- Docker Compose для dev, Kubernetes или Docker Swarm для прод
- Blue-green или canary deployments — для zero-downtime
- Database migration dry-run перед деплоем
- Автоматический rollback при ошибках
- Feature flags — включение/выключение фич без деплоя (для A/B тестов, постепенного раскатывания)
- Infrastructure as Code (Terraform) — инфраструктура описана в коде, воспроизводима

### 5.2. Мониторинг и алертинг
- Метрики сервисов — Prometheus + Grafana
- Логи — Loki / ELK
- Трейсы — Jaeger / OpenTelemetry (APM)
- Алерты — в Telegram / Email / PagerDuty при критических событиях
- Uptime monitoring — внешний (UptimeRobot / Better Uptime) — проверка "жив ли сервис" извне
- Status page — публичная страница status.tvойдомен.ru — показывает клиентам состояние сервиса
- RUM (Real User Monitoring) — скорость загрузки у реальных пользователей
- Error tracking — Sentry для бэкенда и фронтенда
- SLO/SLI — определённые цели: uptime 99.9%, response time < 500ms

### 5.3. Масштабирование
- Horizontal scaling — бэкенд должен быть stateless
- CDN для статики (Cloudflare / Yandex CDN)
- Database connection pooling (PgBouncer)
- Redis Sentinel или Cluster
- Auto-scaling — по CPU/memory
- Нагрузочные тесты прогоняются регулярно, не один раз

### 5.4. Backup и DR (Disaster Recovery)
- Автоматический backup: daily incremental + weekly full
- Бэкапы в отдельный регион/датацентр
- Тестирование восстановления — ежемесячно, документировано
- RTO (Recovery Time Objective) < 1 час
- RPO (Recovery Point Objective) < 15 минут
- DR Runbook — пошаговая инструкция "что делать если всё упало"
- Backup для файлов (S3) — не только БД

## ЧАСТЬ 6. ИНТЕГРАЦИИ (РОССИЯ)

**Кому нужно**: все — интеграции определяют, вписывается ли система в экосистему клиента

### 6.1. 1С (must have #1)
- Двусторонняя синхронизация с 1С:Бухгалтерия / 1С:ERP:
  - Справочники: контрагенты, номенклатура, счета
  - Документы: счета, акты, накладные, платёжки
  - Оплаты и банковские выписки
- Формат обмена: XML (CommerceML / EnterpriseData) или REST API
- Настраиваемый маппинг полей — у каждого клиента 1С настроена по-своему
- Журнал синхронизации — видеть что ушло, что пришло, где ошибки

### 6.2. ЭДО
- Диадок (Контур) — обмен актами, счетами-фактурами, УПД
- СБИС (Тензор) — альтернативный провайдер ЭДО
- Подписание КЭП прямо из системы

### 6.3. Государственные системы
- ИСУП (информационная система управления проектами) — для госконтрактов
- Единая информационная система (ЕИС) — мониторинг госзакупок
- ФНС — проверка контрагентов (ИНН → выписка ЕГРЮЛ, проверка благонадёжности)
- Росреестр — проверка земельных участков, объектов недвижимости
- ГИС ЖКХ — если объект жилой
- ФГИС ЦС (ценообразование в строительстве) — актуальные индексы Минстроя

### 6.4. Прочие интеграции
- Банк-клиент — импорт выписок, формирование платёжек
- Сметные программы — ГРАНД-Смета, Гектор, РИК — импорт/экспорт смет
- CAD/BIM — импорт IFC, DWG (через конвертацию)
- Метеосервис — автоматическая подгрузка погоды для журнала работ
- Телефония — запись звонков, привязка к CRM (Mango Office, Sipuni)
- Email — IMAP/SMTP для привязки переписки к проектам
- Telegram Bot — уведомления, быстрое согласование
- Календарь — Google Calendar / Outlook sync

### 6.5. API для разработчиков
- REST API с OAuth 2.0
- Webhooks — уведомления о событиях (создан проект, подписан КС-2 и т.д.)
- API-документация (OpenAPI / Swagger) — публичная
- Rate limiting для API
- Sandbox (тестовая среда) для разработчиков клиента
- SDK — хотя бы для Python и JavaScript

## ЧАСТЬ 7. МОБИЛЬНАЯ ВЕРСИЯ

**Кому нужно**: прораб, мастер, инженер, снабженец — все, кто на объекте

Для строительной отрасли мобильное приложение — это не "nice to have", а MUST HAVE. Прорабы, мастера и инженеры на площадке не сидят за компьютером.

- PWA или нативное приложение (React Native / Flutter)
- Полноценная работа в оффлайн-режиме — на стройке часто нет интернета. Данные сохраняются локально и синхронизируются при появлении связи
- Конфликт-резолюция — если двое редактировали одно и то же оффлайн
- Камера — фото/видео прямо в задачу, дефект, журнал
- GPS — автоматическая отметка присутствия на объекте
- QR-сканер — приёмка материалов, выдача инструмента
- Push-уведомления
- Биометрическая аутентификация (Touch ID / Face ID)
- Упрощённый UI — на стройке нужны большие кнопки, минимум текста, скорость
- Отдельная мобильная навигация — не уменьшенная десктопная версия

## ЧАСТЬ 8. UX / UI И ДИЗАЙН

**Кому нужно**: UX-дизайнер, продакт, все пользователи

### 8.1. Общий UX
- Onboarding — пошаговый мастер при первом входе: "создай компанию → добавь проект → пригласи коллег"
- Контекстные подсказки — "?" рядом с каждым непонятным полем
- Global search (Cmd+K / Ctrl+K) — поиск по всем сущностям: проекты, контрагенты, документы, сотрудники
- Быстрые действия — недавние, избранное, часто используемое
- Breadcrumbs — всегда видно, где ты находишься
- Empty states — не пустые страницы, а "создай первый проект" с CTA
- Skeleton loaders — не белый экран при загрузке
- Error boundaries — если сломался один модуль, остальное работает
- Keyboard shortcuts — для опытных пользователей
- Drag-and-drop — где уместно (Kanban, порядок задач, файлы)
- Bulk actions — выбрать несколько элементов → массовое действие
- Saved views / filters — "мои задачи", "просроченные КС-2", "проекты в моём городе"
- Настраиваемые дашборды — виджеты, которые пользователь расставляет сам
- Animations и transitions — система должна ощущаться "живой", не дёрганой

### 8.2. Доступность (Accessibility)
- WCAG 2.1 AA — контрастность, навигация клавиатурой, screen reader
- Масштабируемость шрифтов
- Для госзаказчиков это может быть требованием тендера

### 8.3. Локализация
- Русский — основной
- Форматы: даты (дд.мм.гггг), числа (1 234 567,89), валюта (₽)
- Часовые пояса — Россия огромная, проекты в разных регионах
- Печатные формы — по ГОСТ (шрифты, размеры, поля)

## ЧАСТЬ 10. ПОДДЕРЖКА КЛИЕНТОВ

**Кому нужно**: support-менеджер, customer success, пользователи

### 10.1. Техподдержка
- Тикетная система — приоритеты (P1-P4), SLA на время ответа/решения
- Живой чат внутри системы — быстрая помощь без выхода из приложения
- Телефонная линия — для Enterprise
- Эскалация — если тикет не решён за N часов, переходит на следующий уровень
- CSAT / NPS опросы — после каждого тикета

### 10.2. База знаний
- Статьи с пошаговыми инструкциями — "Как создать проект", "Как сформировать КС-2"
- Видео-туториалы — встроены в статьи
- Поиск по базе знаний
- Контекстные ссылки из интерфейса — нажал "?" рядом с полем → открылась нужная статья
- FAQ — часто задаваемые вопросы
- Changelog / What's New — что нового в каждом обновлении
- API-документация — для технических клиентов

### 10.3. Customer Success
- Onboarding-менеджер — для крупных клиентов: помогает настроить систему, обучить команду
- Регулярные check-in — "всё ли хорошо? чего не хватает?"
- Health score — метрика здоровья клиента: как часто заходят, сколько модулей используют
- Churn prediction — предсказание оттока на основе падения активности
- Feature request board — клиенты голосуют за фичи
- Сообщество — Telegram-чат пользователей, обмен опытом

## ЧАСТЬ 11. ЮРИДИЧЕСКАЯ ЧАСТЬ

**Кому нужно**: юрист, основатель, enterprise-клиенты

### 11.1. Обязательные документы
- Пользовательское соглашение (Terms of Service) — правила использования
- Политика конфиденциальности (Privacy Policy) — что с данными
- Оферта — для юрлиц (публичная или индивидуальная)
- Лицензионный договор — если продаёшь право использования ПО
- SLA (Service Level Agreement) — гарантии uptime, время ответа поддержки
- DPA (Data Processing Agreement) — соглашение об обработке данных
- NDA — шаблон для работы с enterprise-клиентами
- Согласие на обработку персональных данных

### 11.2. Compliance 152-ФЗ
- Регистрация в реестре операторов ПДн (Роскомнадзор) — заявление через gosuslugi
- Политика обработки персональных данных — опубликована на сайте
- Модель угроз безопасности ИСПДн — документ, описывающий риски
- Назначение ответственного за организацию обработки ПДн
- Приказы и положения — об обработке, хранении, уничтожении ПДн
- Серверы в РФ — персональные данные граждан РФ хранятся на территории РФ
- Уведомление об инцидентах — в случае утечки → уведомить Роскомнадзор в 24 часа

### 11.3. Юридическое лицо
- ООО или ИП — зарегистрировано, с расчётным счётом
- ОКВЭД — 62.01 (разработка ПО), 63.11 (обработка данных)
- Бухгалтерия — для генерации закрывающих документов (акт, счёт-фактура)
- Торговая марка — зарегистрировать название и логотип в Роспатенте
- Свидетельство о регистрации ПО — в Реестре российского ПО (Минцифры) — преимущество при госзакупках

## ЧАСТЬ 12. AI И ИННОВАЦИИ

**Кому нужно**: продакт, CTO, конечные пользователи

### 12.1. Практичный AI (не хайп, а польза)
- AI-ассистент в системе — отвечает на вопросы по проекту ("какой бюджет остался?", "кто ответственный за это RFI?")
- Автогенерация ежедневных отчётов — на основе внесённых данных
- Автоклассификация документов — загрузил файл → система определила тип
- Умный RFI — предложение ответов на основе похожих RFI из прошлых проектов
- Предиктивный анализ — "на основе текущих трендов проект завершится на 2 месяца позже"
- Извлечение данных из документов (OCR + NLP) — загрузил скан накладной → данные в системе
- Автозаполнение форм — на основе контекста и истории
- Анализ фото со стройки — определение процента готовности
- Суммаризация переписки — "о чём договорились в тредке из 50 сообщений"

### 12.2. На перспективу
- No-code Agent Builder — пользователь создаёт своих AI-агентов (как Procore Agent Builder)
- Голосовой ввод — "создай задачу: залить фундамент блока 3, дедлайн пятница"
- Предиктивное ТО техники на основе IoT-данных
- Оптимизация логистики поставок (ИИ-маршрутизация)

## ЧАСТЬ 13. ПОРТАЛ КЛИЕНТА / ЗАКАЗЧИКА

**Кому нужно**: заказчик (застройщик, инвестор), технадзор

Отдельный интерфейс для внешних участников — заказчик не должен видеть вашу внутреннюю кухню.

- Дашборд проекта — прогресс, сроки, финансы (только то, что положено видеть)
- Документы — доступ к согласованным документам, акты, КС-2
- Согласования — утверждение/отклонение документов прямо в портале
- Замечания — фиксация замечаний от технадзора / заказчика
- Фотоотчёты — прогресс стройки с фото
- Ежедневные/еженедельные отчёты — автоматически формируемые
- Личный кабинет заказчика — без регистрации в основной системе
- Брендирование — логотип и цвета подрядчика на портале

## ЧАСТЬ 14. ПОРТАЛ СУБПОДРЯДЧИКА / ПОСТАВЩИКА

**Кому нужно**: субподрядчик, поставщик

- Бесплатный аккаунт для субподрядчика (как у PlanRadar — клиент платит за себя, а субчики бесплатно)
- Получение нарядов / заданий
- Загрузка отчётов о выполненных работах
- Просмотр чертежей и спецификаций (только своя часть)
- Участие в тендерах — получение приглашений, подача предложений
- КС-2 — подписание со стороны субподрядчика
- Переписка с генподрядчиком по проекту
- Минимальный порог входа — регистрация за 2 минуты, интуитивный UI

## ЧАСТЬ 15. АДМИН-ПАНЕЛЬ (PANEL ВЕНДОРА)

**Кому нужно**: ты как вендор (владелец платформы)

Это не admin для клиентов, а super-admin для тебя — управление всей платформой.

- Список всех тенантов — кто зарегистрирован, какой тариф, когда последний вход
- Метрики — MAU, DAU, retention, churn, MRR, ARR
- Здоровье системы — загрузка сервера, БД, ошибки, алерты
- Управление подписками — ручное продление, блокировка, смена тарифа
- Управление пользователями — заблокировать, сбросить пароль
- Feature flags — включить/выключить фичу для конкретного тенанта
- Аудит-лог — все действия всех пользователей
- Массовые уведомления — "завтра плановое обслуживание"
- Аналитика по использованию — какие модули популярны, где люди застревают
- Управление базой знаний — CRUD статей
- Обратная связь — сводка запросов фич, жалоб, оценок

## ЧАСТЬ 16. ПЕЧАТНЫЕ ФОРМЫ И ОТЧЁТЫ (РОССИЙСКАЯ СПЕЦИФИКА)

**Кому нужно**: бухгалтер, ПТО, сметчик, юрист

Для российского рынка правильные печатные формы — это не "nice to have", а обязательное требование.

- КС-2 (акт о приёмке выполненных работ) — по Постановлению Госкомстата №1
- КС-3 (справка о стоимости выполненных работ и затрат)
- КС-6а (журнал учёта выполненных работ)
- М-29 (отчёт о расходе материалов)
- Форма С-3 (акт инвентаризации незавершённого строительства)
- Акт скрытых работ (АОСР)
- Общий журнал работ
- Специальные журналы (бетонных работ, сварочных работ и т.д.)
- Счёт на оплату, счёт-фактура, УПД
- Акт сверки взаиморасчётов
- Товарная накладная (ТОРГ-12)
- Доверенность (М-2)
- Приказы (по кадрам)
- Табель учёта рабочего времени (Т-12, Т-13)
- Все формы — с возможностью кастомизации (свой логотип, реквизиты, допполя)
- Экспорт — PDF, Word, Excel
