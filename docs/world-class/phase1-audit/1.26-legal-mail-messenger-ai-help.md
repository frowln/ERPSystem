## 3. Модуль MAIL (почта)

### Оценка: **7.5/10** — production-ready с оговорками

Полноценный email-клиент (Yandex IMAP/SMTP) + система транзакционных уведомлений с 17 HTML-шаблонами.

### Архитектура (АРХИТЕКТОР ✅)
- 4 сервиса: Sync (IMAP), Compose (SMTP), Mailbox (CRUD), Notification (system)
- Scheduled sync каждые 120с с AtomicBoolean guard (дедупликация)
- Batch IMAP fetch (200 msg/batch), MIME multipart parsing, attachment extraction
- Thymeleaf HTML templates с base-layout наследованием
- Email logging в email_logs с status tracking (QUEUED/SENT/FAILED)
- @Async для уведомлений — не блокирует бизнес-процессы
- 14+ scheduled jobs используют EmailService для напоминаний

### Безопасность (БЕЗОПАСНИК ⚠️)
- **EmailMessage НЕТ organizationId** — все письма глобальны, нет tenant isolation
  - Это единый корпоративный ящик (office@privod-electro.ru), но в multi-tenant SaaS это архитектурная проблема
  - Любой ADMIN/MANAGER/ENGINEER/ACCOUNTANT видит ВСЕ письма ящика
- RBAC на endpoints: правильная ролевая модель (Send: ADMIN/MANAGER/ACCOUNTANT, Delete: ADMIN/MANAGER only)
- Yandex password через env variable — не hardcoded ✅
- Attachment storage: filesystem `data/email-attachments/` — нет S3, нет virus scanning
- `EmailNotificationService.buildHtml()` — HTML строится конкатенацией, но данные из БД (не user input) → low XSS risk

### UX (UX ДИЗАЙНЕР ✅)
- 3-pane layout (sidebar + list + detail) — Gmail-style
- Folders: Inbox, Sent, Drafts, Trash
- Search, unread badge, star, pagination
- Reply/ReplyAll/Forward с правильной логикой CC
- Attachment download, project linking
- Dark mode, 20+ i18n ключей

### Чего не хватает (ПРОДАКТ МЕНЕДЖЕР)
- Нет rich-text editor в compose (только plain text → `<br>` conversion)
- Нет upload attachments в compose (только forward существующих)
- Нет inline images
- Нет body search (индексирован только subject/from)
- Нет calendar/iCal invite parsing
- Нет per-tenant email accounts (один ящик на всех)

### Баги

|---|---|----------|------|
| B4 | P1 | **EmailMessage НЕТ organizationId** — единый ящик, но нет tenant isolation. В multi-tenant SaaS все тенанты видят ВСЮ переписку | `EmailMessage.java` |

## 5. Модуль AI (AI-ассистент)

### Оценка: **6.0/10** — functional prototype, needs activation

Архитектурно сильный (multi-provider, SSE streaming, usage analytics), но 60% функций работают через mock/fallback.

### Архитектура (АРХИТЕКТОР ✅)
- Multi-provider design: OpenAI, GigaChat (Sber), Yandex GPT, Self-hosted
- LlmProvider interface + LlmProviderFactory — правильный Strategy pattern
- Per-organization model config (ai_model_configs table)
- SSE streaming через SseEmitter + HttpURLConnection
- Usage tracking: tokens input/output, cost (RUB), response time

## 8. Оценка по ролям

### СМЕТЧИК 🔧
- Не применимо к данной группе модулей (нет финансовых расчётов)

### ПРОРАБ 👷
- Мессенджер — **отлично** на стройке: @mentions, voice messages, DM, project channels
- Нет offline mode для чата — на стройке плохой интернет, сообщения теряются
- Help drawer с контекстной справкой — полезно для обучения новых сотрудников

### БУХГАЛТЕР 💼
- Mail module: проект-линкинг писем — удобно привязать переписку с контрагентом к объекту
- Нет автоматической привязки писем к контрагентам по email-адресу

### БЕЗОПАСНИК 🔐
- **2 P1**: EmailMessage без tenant isolation, AI conversation IDOR
- **25+ GET endpoints без @PreAuthorize** (legal 7 + messenger 10 + CallSession ~4)
- OpenAI data localization — 152-ФЗ нарушение
- Email attachment storage на filesystem без virus scanning

### АРХИТЕКТОР 🏗️
- Legal: **эталонная архитектура** — чистые слои, правильный tenant isolation, compact service
- Messenger: впечатляющий feature set (WebRTC, STOMP, reactions, threads) при clean code
- AI: правильный Strategy pattern (multi-provider), но IDOR в ядре
- Help: **гибридная архитектура — антипаттерн** (static JSON + backend API не связаны)
- Mail: single-mailbox дизайн не масштабируется для multi-tenant SaaS

### UX ДИЗАЙНЕР 🎨
- Все модули имеют dark mode ✅
- i18n полный (ru + en) ✅
- Messenger UI — уровень Telegram/Slack
- AI assistant — ChatGPT-like UX
- KB: premium CSS styling, context-sensitive help

### ПРОДАКТ МЕНЕДЖЕР 📊
- **Legal**: лучше Procore (у них нет модуля legal). Конкурент: PlanRadar (ведомости замечаний, но не юридические дела)
- **Mail**: уровень Procore (project-linked emails), но нет attachment upload в compose
- **Messenger**: лучше Procore (у них basic commenting), на уровне PlanRadar Chat
- **AI**: на уровне Procore AI (у них Procore Agent Builder, у нас multi-turn chat). Photo analysis = наше преимущество (когда заработает)
- **Help**: 92 статьи — больше чем у большинства конкурентов. Context-sensitive help = уникально

### QA ИНЖЕНЕР 🧪
- **0 тестов** во всех 5 модулях — критический gap
- Нет unit тестов для LegalService, MessagingService, AiAssistantService
- Нет integration тестов для IMAP sync, WebSocket messaging
- Нет E2E тестов для messenger, AI chat, KB navigation
