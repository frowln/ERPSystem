## 2. Что работает хорошо

### Projects
1. **Финансовый движок проекта** — `ProjectFinancialService` агрегирует данные из 6 таблиц: invoices, payments, commitments, costCodes, estimates, budgets. Формулы расчёта CPI/SPI и ComputedFinancials правильные
2. **Tenant isolation на Project entity** — `@Filter(name = "tenantFilter")` на Project, service-level `getProjectOrThrow()` проверяет organizationId
3. **Полноценный статусный автомат** — `ProjectStatus` с allowedTransitions: DRAFT→PLANNING→IN_PROGRESS→ON_HOLD→COMPLETED/CANCELLED. Прямые переходы валидируются
4. **Portfolio Health Page** — 7-метрический RAG (лучше Procore 5-6), сортировка, фильтры, CSV export, dual views (таблица + карточки)
5. **Pre-Construction Dashboard** — единый экран для ПТО: спецификации, разрешения, риски, обследования, конструктивность
6. **Monte Carlo Analysis** — 10,000 итераций, box plot, P10/P50/P90 — конкурентно с Oracle Primavera
7. **Risk Register** — 5×5 матрица вероятность×воздействие, категоризация, статусы OPEN/MITIGATING/CLOSED
8. **Project Templates** — JSONB для stages, budgetItems, taskTemplates — гибкая структура

### Site Assessments
1. **12-пунктный чек-лист** — подъездные пути, инженерные сети, рельеф, грунтовые воды, экология, соседние строения, шум/вибрация, электроснабжение, водоснабжение, газ, связь, канализация
2. **GO/CONDITIONAL/NO_GO scoring** — автоматический расчёт рекомендации по сумме баллов
3. **Хорошие dark mode классы** на всех трёх страницах
4. **Latitude/longitude** — привязка к координатам площадки

## 8. Проверка расчётов (взгляд Сметчика/Бухгалтера)

### 8.1 ProjectFinancialService формулы

```
estimateTotal = sum(estimates.totalAmount) WHERE projectId            ✅ корректно
budgetTotal = sum(costCodes.budgetAmount) WHERE projectId             ✅ корректно
contractRevenue = sum(invoices.total) WHERE type IN ('GENERAL')       ⚠️ hardcoded type code
contractCosts = sum(invoices.total) WHERE type IN ('SUBCONTRACT')     ⚠️ hardcoded type code
commitments = sum(commitments.revisedAmount) WHERE projectId          ✅ корректно
actualCosts = sum(payments.amount) WHERE projectId                    ✅ корректно

margin = contractRevenue > 0 ? ((contractRevenue - actualCosts) / contractRevenue) * 100 : 0
         ✅ формула корректна, проверка деления на ноль есть

cashPosition = totalPaid - totalCosts
         ✅ корректно

profitability = totalPaid > 0 ? ((totalPaid - totalCosts) / totalPaid) * 100 : 0
         ✅ корректно
```

**Замечание**: нет учёта НДС. Financial summary показывает суммы без разделения с/без НДС. Для российского бухучёта это критичный gap — бухгалтер не может выделить НДС из данных проекта.

### 8.2 Site Assessment scoring

```
Backend:  score >= 10 → GO, score >= 6 → CONDITIONAL, else → NO_GO
Frontend: score >= 10 → GO, score >= 7 → CONDITIONAL, else → NO_GO
```

**Баг**: пороги не совпадают. Score=6: backend=CONDITIONAL, frontend=NO_GO.

### 8.3 Portfolio Health RAG

```
schedule: computed from project progress vs timeline    ✅ реальный расчёт
budget:   computed from actual/budget ratio             ✅ реальный расчёт
risk:     computed from risk count                      ✅ реальный расчёт
safety:   hashCode(id + 'safety') % 5                  ❌ СИМУЛЯЦИЯ
quality:  hashCode(id + 'quality') % 5                 ❌ СИМУЛЯЦИЯ
procurement: hashCode(id + 'procurement') % 5           ❌ СИМУЛЯЦИЯ
itd:      hashCode(id + 'itd') % 5                     ❌ СИМУЛЯЦИЯ
```

4 из 7 RAG метрик — pseudo-random, не отражают реальное состояние проекта.
