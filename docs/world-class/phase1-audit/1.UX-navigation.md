# 1.UX — Аудит навигации глазами прораба/директора

> **Дата**: 2026-03-18
> **Методика**: Прохожу каждый пункт сайдбара как руководитель строительной компании на 50 человек, 5 объектов.
> **Роли**: прораб, сметчик, снабженец, директор — в одном лице (типичная малая стройкомпания).

---

## 0. ОБЩАЯ СТАТИСТИКА НАВИГАЦИИ

| Метрика | Значение | Проблема? |
|---------|----------|-----------|
| Групп в сайдбаре | **24** | ДА — у конкурентов 5-12 |
| Пунктов меню всего | **~185** | КРИТИЧНО — у Procore ~12, у PlanRadar ~7 |
| Макс. пунктов в группе | **29** (Финансы) | КРИТИЧНО — это не меню, это справочник |
| Групп с 1 пунктом | **5** (задачи, календарь, мессенджер, почта, знания) | Зачем группа ради одного пункта? |
| Групп с 20+ пунктов | **5** (финансы 29, кадры 28, качество 27, снабжение 27, документы 20) | Не ищется, не запоминается |
| Дублирующих пунктов | **6+** | Один функционал в 2-3 местах |
| Англицизмов/непонятных терминов | **15+** | Строитель не знает CDE, RFI, Quality Gates |

**ВЕРДИКТ: Система выглядит как каталог функций, а не как рабочий инструмент.**

## 2. ПРОБЛЕМЫ НАВИГАЦИИ

### 2.1 Критический перегруз

**5 групп с 20+ пунктами:**
- Финансы: **29** пунктов (человек откроет, увидит стену текста, закроет)
- Кадры: **28** пунктов
- Качество и регуляторика: **27** пунктов
- Снабжение и склад: **27** пунктов
- Документы: **20** пунктов

**Правило**: Одна группа сайдбара — не более **7-9 пунктов** (Miller's Law). Мы превышаем в 3-4 раза.

- **Журналы работ** — в группе «Стройплощадка» (группа 18 из 24!). Прораб использует КАЖДЫЙ ДЕНЬ, но надо скроллить мимо 17 групп

- **Быстрая приёмка** — в группе «Снабжение» (27 пунктов). Полевая функция для мобильного

### 2.5 Что торчит на виду, но нужно редко

- **IoT устройства/датчики/оповещения** — у 50-человечной компании нет IoT
- **AI-анализ фото, AI-риски** — mock, не работает
- **Калькулятор факторинга** — экзотическая функция
- **Рейтинг водителей** — mock, не работает
- **Базы расценок, Нормативные расценки, Калькулятор расценок** — настройки, не рабочие инструменты
- **Настройка 1С, Журнал обмена 1С** — административные функции

### 4.2 Детальный разбор каждого конкурента

#### Procore — золотой стандарт (50+ tools)

**Архитектура навигации:**
- **Нет сайдбара с группами!** Нет секций «Финансы», «Качество», «HR». Tools — плоский список, упорядоченный по важности
- Верхняя панель: переключатель компания/проект, **Favorites bar** (до 25 личных закреплённых tools), **Toolbox** (mega-menu со ВСЕМИ доступными tools)
- Два уровня: Company-level (~20 tools: Analytics, Portfolio, Prequalifications) и Project-level (~40 tools: Budget, Daily Log, RFIs, Punch List...)

**4 механизма борьбы с перегрузом:**

| Механизм | Как работает | Аналог в PRIVOD |
|----------|-------------|-----------------|
| **Permission-based hiding** | Tool с «None» permission = невидим. PM видит ~33 tools, Foreman видит ~20 (все финансовые скрыты) | Есть RBAC, но 185 пунктов видны всем |
| **Per-user Favorites** (до 25) | Каждый user закрепляет свои tools в хедере. Drag-and-drop. Admin НЕ может ставить favorites за других | Нет |
| **Per-project tool toggle** | Admin включает/выключает tools per-project. Жилой объект без BIM — отключить. Маленький без Estimating — отключить | Есть admin module toggle (недавно добавлен) |
| **Reorderable tool order** | Admin drag-and-drop переупорядочивает Project Tools Menu. Важные наверх | Нет |

**Что видит Foreman (прораб) в Procore:**
Daily Log, Drawings, Documents, Photos, Inspections, Observations, Punch List, RFIs, Schedule, Timesheets, Directory, Home. **~12 tools.** Бюджет, Bidding, Change Orders, Commitments, Prime Contract — **полностью скрыты**.

**Вывод для PRIVOD**: Скопировать 4 механизма: (1) скрытие по ролям, (2) пользовательские favorites, (3) toggle per-project, (4) reorder. Убрать секционные заголовки, сделать плоский tool list.

### 5.4 СТРАТЕГИЧЕСКИ (архитектура навигации)

| # | Действие | Почему |
|---|----------|--------|
| 12 | **Проектно-центричная навигация** (как Procore) | Объект → инструменты этого объекта. Прораб работает в контексте своего объекта. У Procore PM видит ~33 tools, Foreman ~12 — внутри проекта |
| 13 | **Permission-based tool hiding** (как Procore) | Tool с «None» permission = полностью невидим. НЕ серый, а ОТСУТСТВУЕТ. Foreman не знает что Budget существует |
| 14 | **Per-user Favorites bar** (как Procore) | До 25 закреплённых tools в хедере, drag-and-drop. Admin НЕ ставит favorites за пользователя — это личное |
| 15 | **Один раздел «Аналитика»** с табами | Заменяет 12 разбросанных дашбордов |
| 16 | **Группировка по бизнес-процессам** (как 1С:УСО) | «Казначейство» вместо 7 пунктов (ДДС, платежи, банк, факторинг...). «Регламентированный учёт» вместо КС-2+КС-3+М-29+ЭДО |
| 17 | **Cmd+K / Ctrl+K поиск** | Поиск по всем разделам — альтернатива скроллингу 185 пунктов |
| 18 | **Мобильная навигация** | Отдельная, 5 пунктов bottom tab: Объекты, Журнал, Задачи, Фото, Ещё. Как у PlanRadar: Tickets, Reports, Documents, Schedule |
| 19 | **Pipeline-навигация в снабжении** (как pusk.app) | Провести пользователя по цепочке: Потребность→Заявка→КП→Счёт→Оплата→Приёмка. Вместо 27 пунктов |
| 20 | **Subscription gating** (как Buildertrend) | Basic: 5 core tools. Pro: +BIM, Scheduling, AI. Enterprise: +Portal, IoT, Estimating. Прорабу на Basic не показывать IoT |

### 5.5 ЧЕГО НЕ ХВАТАЕТ (что ожидает прораб/директор)

| Ожидание | Статус | Комментарий |
|----------|--------|-------------|
| Погода на объекте | Нет | Встроить виджет в Журнал работ |
| «Мой день» — что сегодня | Нет | Персональный дашборд: задачи на сегодня, согласования, просрочки |
| Быстрое фото → привязка к объекту | Частично | Есть Фото, но нет quick-capture с GPS-тегом |
| Уведомления в реальном времени | Частично | WebSocket есть, но нет push на мобиле |
| Сравнение план/факт на одном экране | Нет | Базовая функция для прораба — план на неделю vs что сделано |
| Отчёт для заказчика «в 1 клик» | Нет | КС-2 + КС-3 + фото + журнал → PDF пакет |
| Печатные формы (М-29, Т-12, Т-13, АОСР, С-3) | Нет | Юридическая обязанность |
