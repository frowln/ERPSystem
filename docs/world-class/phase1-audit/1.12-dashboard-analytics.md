## 2. Что работает хорошо

### 2.1 Архитектура (8/10)
- **Многоуровневая система**: Dashboard CRUD + Widget system + KPI engine + Report builder + Predictive analytics + ABC/XYZ + Bonus calculations — полнота на уровне Procore/Oracle Primavera
- **Раздельные сервисы**: 11 сервисов с чётким разделением ответственности (кроме AnalyticsDataService god-class)
- **Widget registry**: DashboardWidgetSystem.tsx — элегантная система метаданных для 16 типов виджетов с lazy-loading
- **KPI health status**: Поддержка inverted logic (higher_better vs lower_better) — продуманная KPI-логика
- **Bonus state machine**: DRAFT→CALCULATED→APPROVED→PAID с валидацией переходов
- **@Scheduled predictions**: PredictiveAnalyticsService ежедневно в 3:00 обновляет прогнозы для активных проектов

### 2.2 Frontend (7/10)
- **DashboardPage** — полностью на реальных API, graceful degradation через Promise.allSettled(), skeleton loaders
- **ExecutiveDashboardPage** — RAG-матрица с 7 измерениями (budget, schedule, docs, safety, quality, procurement, ITD) на реальных данных, лучше Procore (5-6 измерений)
- **Dark mode** — полная поддержка, Recharts цвета адаптируются через useThemeStore
- **i18n** — все строки через t(), 100+ ключей analytics.*
- **Recharts** — стабильная библиотека, 8 файлов с ResponsiveContainer для адаптивности
- **Lazy loading** — React.lazy() для chart components с Suspense fallback

### 2.3 Backend AnalyticsController (7/10)
- Class-level @PreAuthorize (хотя с проблемами, см. ниже)
- AnalyticsDataService — tenant isolation через SecurityUtils.requireCurrentOrganizationId() во всех методах
- Export endpoint — 16 типов отчётов (project-summary, financial-report, safety-report, budget-variance, progress-report, daily-log-report, ks2-report, ks3-report, evm-report, change-order-report, quality-report, labor-report, subcontractor-report, rfi-submittal-report, material-consumption, fleet-utilization) с PDF/Excel/CSV

### 2.4 ReportBuilder — лучший в классе для строительных ERP
- 12 data sources (Projects, Contracts, Invoices, Payments, Employees, Materials, DailyLogs, QualityChecks, SafetyIncidents, KS2, Tasks, PurchaseRequests)
- Tenant filter через @Filter на ReportTemplate
- Настраиваемые columns, filters, groupBy, sortBy, chartType — конструктор без кода
- 6 типов диаграмм: BAR, LINE, PIE, AREA, STACKED_BAR, TABLE
- Execution history с row_count, execution_time_ms

## 3. Найденные баги

### P1 — Critical

#### BUG-DA1: Dashboard entity — НЕТ tenant isolation
**Файл**: `modules/analytics/domain/Dashboard.java`
**Проблема**: Нет organization_id, нет @Filter(name="tenantFilter"). Дашборды глобальные — тенант A видит дашборды тенанта B при прямом доступе по UUID.
**Влияние**: Cross-tenant data leakage. При 100+ организациях — массовая утечка.
**Fix**: ADD organization_id + @Filter + Flyway migration + service-level predicate

#### BUG-DA2: KpiDefinition entity — НЕТ tenant isolation
**Файл**: `modules/analytics/domain/KpiDefinition.java`
**Проблема**: Нет organization_id, нет @Filter. KPI-определения глобальные. Один тенант создаёт KPI — все тенанты его видят.
**Влияние**: Information leakage + data pollution (все видят чужие KPI targets/thresholds)
**Fix**: ADD organization_id + @Filter + Flyway migration

#### BUG-DA3: SavedReport entity — НЕТ tenant isolation
**Файл**: `modules/analytics/domain/SavedReport.java`
**Проблема**: Нет organization_id, нет @Filter. Сохранённые отчёты глобальные.
**Влияние**: Cross-tenant access. Пользователь может запустить чужой отчёт и увидеть queryConfig (JSONB с фильтрами).
**Fix**: ADD organization_id + @Filter + Flyway migration

#### BUG-DA4: AnalyticsDataService — hardcoded cost ratio 0.85
**Файл**: `modules/analytics/service/AnalyticsDataService.java:101`
**Проблема**: `totalCosts = totalBudget * 0.85` — финансовая сводка ВСЕГДА показывает затраты = 85% бюджета, а не реальные данные из budget_items.
**Влияние**: **СМЕТЧИК/БУХГАЛТЕР**: Руководитель видит фальшивую маржу 15% на каждом проекте. Budget utilization KPI (getKpiItems) тоже использует это значение → max 85%.
**Fix**: Заменить на реальный SUM(actual_cost) из budget_items

#### BUG-DA5: PredictiveAnalyticsService — DEFECT_WEIGHT copy-paste bug
**Файл**: `modules/analytics/service/PredictiveAnalyticsService.java:241`
**Проблема**: В calculateDelayProbability(), change order impact умножается на DEFECT_WEIGHT (0.15) вместо правильного веса. Copy-paste ошибка.
**Влияние**: Delay prediction занижает/завышает вклад change orders — неточные прогнозы задержек.
**Fix**: Создать CHANGE_ORDER_WEIGHT = 0.15 (или другое значение) и использовать его

### P2 — High

#### BUG-DA6: 8 analytics GET endpoints с ослабленным RBAC
**Файл**: `modules/analytics/web/AnalyticsController.java:99,107,115,123` и далее
**Проблема**: Class-level @PreAuthorize = `hasAnyRole('ADMIN','PROJECT_MANAGER','FINANCE_MANAGER')`, но 13+ endpoints переопределяют на `isAuthenticated()`. VIEWER видит:
- `/api/analytics/dashboard` (org summary)
- `/api/analytics/dashboard/financial` (финансовые данные!)
- `/api/analytics/dashboard/tasks`
- `/api/analytics/dashboard/safety`
- `/api/analytics/financial` (monthly bars)
- `/api/analytics/kpis` (KPI values)
- `/api/analytics/project-budgets` (бюджеты проектов!)
- `/api/analytics/export/{type}` (export любого отчёта!)
**Влияние**: VIEWER/ENGINEER видят полные финансовые данные организации, включая бюджеты, маржу, KPI.
**Fix**: Убрать `@PreAuthorize("isAuthenticated()")` overrides, оставить class-level RBAC

#### BUG-DA7: AnalyticsDataService — fabricated monthly financial bars
**Файл**: `modules/analytics/service/AnalyticsDataService.java:263-277`
**Проблема**: getFinancialBars() делит total budget на 6 месяцев с variance factors (0.7→1.0) — синтетические данные вместо реальных месячных сумм.
**Влияние**: **БУХГАЛТЕР**: Графики финансовой аналитики показывают фейковое распределение по месяцам. Директор принимает решения на ложных данных.
**Fix**: GROUP BY EXTRACT(MONTH FROM ...) на реальных invoices/payments

#### BUG-DA8: AnalyticsDataService — fabricated safety metrics distribution
**Файл**: `modules/analytics/service/AnalyticsDataService.java:300-303`
**Проблема**: getSafetyMetricsList() распределяет total incidents поровну на 6 месяцев. Если за полгода было 12 инцидентов — каждый месяц показывает 2.
**Влияние**: **БЕЗОПАСНИК**: Тренд безопасности ВСЕГДА flat — невозможно увидеть пик инцидентов в декабре или снижение после обучения.

#### BUG-DA9: Report execution = mock

## 4. Анализ по ролям

### СМЕТЧИК
- **Хорошо**: KPI budget utilization, cost breakdown by category, EVM metrics (CPI/SPI)
- **Плохо**: CPI формула семантически неверная (planned/actual вместо earned/actual). Все финансовые графики показывают fabricated monthly data (total/6 с variance factors). Budget utilization capped at 85% из-за hardcoded ratio.
- **Не хватает**: Plan-fact сравнение по сметным позициям, индексация по Минстрою, trend analysis по статьям затрат

### ПРОРАБ
- **Хорошо**: DashboardPage с метриками проекта, milestones, recent activity. KPI page с фильтрами по проекту.
- **Плохо**: PredictiveAnalyticsPage — красивая, но mock. What-if scenarios не подключены к реальным данным.
- **Не хватает**: Мобильный дашборд (есть route site-mobile-dash, но не связан с analytics). Offline-кэш метрик для стройки.

### БУХГАЛТЕР
- **Хорошо**: ReportsPage — 16 типов отчётов включая КС-2, КС-3, budget-variance, financial-report
- **Плохо**: Report execution = mock. Нельзя реально скачать PDF/Excel. Financial summary = budget*0.85.
- **Не хватает**: Отчёт по дебиторке/кредиторке aging, реконcиляция invoices↔payments, отчёт по НДС

### БЕЗОПАСНИК
- **Хорошо**: Safety metrics endpoint, safety incidents chart, RAG safety dimension в executive dashboard
- **Плохо**: getSafetyMetricsList() распределяет инциденты поровну по месяцам — нет реального тренда. SafetyDashboard metrics endpoint = zeros (stub).
- **Не хватает**: LTIFR/TRIR тренд на dashboard, comparison with industry benchmarks

### АРХИТЕКТОР
- **Хорошо**: Модульная структура (11 сервисов), widget registry pattern, @Scheduled predictions
- **Плохо**: AnalyticsDataService = god-class (1136 LOC). 3 entity без tenant isolation. 4 N+1 patterns. getDistinctOrganizationIds() in-memory.
- **Критично**: Dashboard/KpiDefinition/SavedReport = potential cross-tenant в multi-tenant SaaS

### UX ДИЗАЙНЕР
- **Хорошо**: Dark mode полный, responsive grids, Recharts ResponsiveContainer, skeleton loaders
- **Плохо**: No error boundaries. Export = toast. PredictiveAnalytics project selector не работает. ReportBuilder без editing/scheduling UI. Audit log без date filter.
- **Не хватает**: Dashboard customization (drag-drop widgets), saved views, period comparison, drill-down navigation

### ПРОДАКТ МЕНЕДЖЕР
- **Хорошо**: Scope амбициозный — dashboard, KPI, reports, report builder, predictive, ABC/XYZ, bonuses, audit — полный набор на уровне Procore Analytics
- **Плохо**: 30-40% функционала = заглушки. PredictiveAnalytics = демка. Reports = не генерируются. Dashboard widgets = не настраиваются. Bonuses = localStorage seed.
- **Не хватает по сравнению с конкурентами**:
  - **Procore**: Configurable dashboards (drag-drop), auto-email reports, portfolio benchmark
  - **Oracle Primavera**: Real EVM dashboard, what-if scheduling, Monte Carlo
  - **PlanRadar**: Photo-based progress tracking on dashboard, defect heat maps
  - **Buildertrend**: Client-facing dashboard, daily log summary auto-generation

### QA ИНЖЕНЕР
- **Тесты**: 9 API-level тестов в analytics.test.ts. 0 компонентных тестов. 0 E2E тестов.
- **Не покрыты**: KPI health calculation, RAG matrix logic, report generation, widget rendering, chart data transformation
- **Edge cases**: NaN data в Recharts, empty project list, date range validation, concurrent report execution
