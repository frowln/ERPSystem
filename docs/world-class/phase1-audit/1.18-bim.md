# 1.18 Глубокий аудит: BIM (модели, clash, overlay, pins, прогресс, heatmap)

> **Дата**: 2026-03-16
> **Оценка**: 5.0/10
> **Статус**: needs-work (значительная часть — прототип / mock)
> **Масштаб**: ~120 backend файлов (17 entities, 14 enums, 17 repos, 11 services, 10 controllers, 40+ DTOs, 3 migrations) + 13 frontend pages + 1 API layer + 120+ i18n keys + 3 backend tests, 0 frontend tests

---

## 1. Общая картина

BIM-модуль — один из самых амбициозных в системе: 3D просмотр IFC, clash detection, drawing overlay/pins, defect heatmap, 4D construction progress, BCF issues, property sets. Архитектурно модуль разделён на две "волны":

- **Волна 1 (V32 migration)**: BimModel, BimElement, BimClash, BimVersion, BimViewer, DesignPackage, DesignDrawing, DrawingAnnotation, PhotoProgress, PhotoComparison, PhotoAlbum — **11 entities БЕЗ organizationId, БЕЗ tenant filter**
- **Волна 2 (V225-V226)**: BimClashTest, BimClashResult, BimViewerSession, BimElementMetadata, DefectBimLink, BimDefectView — **6 entities С organizationId и @Filter**

Результат: половина модуля — enterprise-level security, половина — полностью открыта для cross-tenant атак.

Frontend: 7 из 13 страниц подключены к реальным API-эндпоинтам, 2 страницы — 100% mock, 3 страницы вызывают несуществующие бэкенд-эндпоинты, **10+ API path/method mismatches**.

---

## 2. Архитектура (оценка: 6/10)

### Что хорошо
- Чёткое разделение domain/repository/service/web/dto
- State machines для каждой entity (BimModel: DRAFT→IMPORTED→PROCESSED, DesignPackage: DRAFT→IN_REVIEW→APPROVED, и т.д.)
- JSONB fields с `@JdbcTypeCode(SqlTypes.JSON)` для BimElement properties/geometry, viewer positions
- Soft-delete паттерн во всех entities
- AuditService.logCreate/logUpdate/logStatusChange во всех сервисах
- BimClashDetectionService и BimDefectLinkService — образцовая tenant isolation с SecurityUtils
- BimViewer.tsx — реальный IFC.js 3D viewer (web-ifc-viewer), не заглушка

### Что плохо
- **Двойная архитектура clash**: BimClash (старый, без tenant) + BimClashTest/BimClashResult (новый, с tenant) — дублирование и путаница
- **simulateClashDetection() — FAKE**: генерирует Random clashes между случайными элементами, пишет в production DB синтетические данные
- API path mismatches: frontend и backend используют разные URL (10+ несовпадений)
- 3 backend endpoints не существуют: `/bim/construction-progress`, `/bim/property-sets`, `/bim/bcf-topics`
- Нет cross-module linking backend для clashes→Issues/RFI/ChangeOrders (frontend вызывает `/bim/clashes/linked` — 404)

## 4. Frontend↔Backend интеграция (оценка: 3/10) — CRITICAL MISMATCHES

### 4.1 API path mismatches (фронтенд → бэкенд → результат)

| Frontend path | Backend path | Проблема |
|--------------|-------------|----------|
| `POST /bim/clashes/detect` | `POST /bim/clash-detection/tests/{id}/run` | Разные URL + семантика |
| `GET /bim/clash-results` | `GET /bim/clash-detection/tests/{testId}/results` | Frontend не передаёт testId |
| `PATCH /bim/clash-results/{id}/status` | `PATCH /bim/clash-detection/results/{id}/resolve` / `ignore` | Структура ≠ |
| `GET /bim/defect-heatmap` | `GET /bim/defect-links/heatmap/{projectId}` | Разные URL |
| `POST /bim/design-packages/{id}/approve` | `PATCH /bim/design-packages/{id}/approve` | POST ≠ PATCH |
| `POST /bim/design-packages/{id}/reject` | — | **Бэкенд не имеет reject** |
| `POST /bim/clashes/{id}/resolve` | `PATCH /bim/clashes/{id}/resolve` | POST ≠ PATCH |
| `POST /bim/models` (multipart/form-data) | `POST /bim/models` (JSON body) | Content-Type ≠ |
| `GET /bim/clashes/linked` | — | **Нет бэкенда** |
| `GET /bim/clashes/{id}/linked-items` | — | **Нет бэкенда** |
| `POST /bim/clashes/{id}/linked-items` | — | **Нет бэкенда** |

## 6. UX анализ (оценка: 6/10)

### Что хорошо
- Loading states (isLoading prop) в 9 из 11 API-connected pages
- Empty states (emptyTitle, emptyDescription) в большинстве страниц
- Dark mode: все страницы поддерживают `dark:` classes
- i18n: 120+ ключей, все строки через `t()`
- DataTable с фильтрами, поиском, пагинацией, CSV export
- DefectHeatmapPage — наглядная цветовая карта дефектов по зонам с drill-down модалкой
- ConstructionProgressPage — план/факт визуализация с прогресс-барами и deviation calculation
- ClashBoardPage — Kanban с @dnd-kit (мобильные поддерживаются)

### Что плохо
- **BimViewer** — 3D viewer есть, но **нет интеграции с clash detection** — нельзя кликнуть clash и увидеть его в модели
- **No breadcrumbs** — navigation context потерян при drill-down (модель→элементы→clashes)
- **DrawingOverlayComparisonPage** — красивый canvas comparison, но **данные фейковые** — пользователь не может загрузить свои чертежи
- **DrawingPinsPage** — SVG floor plan hardcoded — нельзя подгрузить реальный план этажа
- **No bulk actions** — нельзя массово resolve/ignore clashes
- **No file upload UI** для BIM models — только JSON create, а multipart в API не совпадает с бэкендом
- **BimModelDetailPage** — clash summary всегда 0, linked packages всегда пустые → пользователь думает данных нет

## 7. Сметчик / Бухгалтер / Прораб

### Сметчик
- **5D (стоимость→BIM)**: НЕТ привязки стоимостей к BIM-элементам. Gold Standard требует. Нет ни backend entity, ни frontend UI.
- **IFC Property Sets**: Backend entity (BimElement.properties JSONB) есть, но frontend PropertySetsPage вызывает несуществующий эндпоинт.

### Прораб
- **4D прогресс**: Фронтенд ConstructionProgressPage есть и красив (план/факт bars), но **бэкенд не реализован** — данные никогда не придут.
- **Photo Progress**: Бэкенд полный (CRUD + albums + comparisons), фронтенд не подключён к photo comparison/album endpoints.
- **Drawing Pins**: Мощный UI для аннотаций на чертежах, но **100% mock** — на стройке бесполезен.

### Бухгалтер
- BIM модуль не имеет финансовой привязки — нет 5D, нет связи с бюджетом, нет расчётов.

## 9. Сравнение с конкурентами

### Autodesk Construction Cloud (ACC)
- Real IFC/Revit viewer (Forge Viewer) с навигацией по модели
- Clash detection через Navisworks integration — реальный IFC анализ
- 4D scheduling linked to BIM elements
- **PRIVOD gap**: нет реального парсера IFC, нет связи графика с BIM элементами

### Procore
- 2D/3D viewer через Procore Documents
- Basic clash workflow (manual, not automated)
- BIM→Issues linking, BIM→RFI linking
- **PRIVOD advantage**: DefectBimLink heatmap + saved views — уникальная фича
- **PRIVOD gap**: нет реального viewer integration, cross-linking не работает

### PlanRadar
- 2D plan annotation + pin-based defects на чертежах
- 360° photo integration
- **PRIVOD advantage**: Canvas drawing comparison с overlay/swipe modes — лучше чем PlanRadar
- **PRIVOD gap**: DrawingPinsPage — mock, не подключена к API

### Trimble Connect
- Full IFC parsing + clash detection engine
- BCF import/export (openBIM standard)
- 4D/5D integration
- **PRIVOD gap**: BCF, 4D, 5D — всё frontend-only или отсутствует

## 10. Лучшие стороны модуля

1. **DefectBimLink heatmap** — уникальная фича: агрегация дефектов по этажам/системам/типам элементов, цветовая карта плотности. Ни у Procore, ни у PlanRadar такого нет.
2. **BimViewer.tsx (IFC.js)** — настоящий 3D viewer в браузере с zoom/pan/orbit, highlighting clashes.
3. **DrawingOverlayComparisonPage** — 3 режима сравнения чертежей (overlay/side-by-side/swipe) с zoom/pan — лучше чем у PlanRadar. Нужно только подключить к API.
4. **BimClashDetectionService** — правильная архитектура (test→run→results workflow), tenant isolation, audit logging.
5. **Design Package workflow** — DRAFT→IN_REVIEW→APPROVED с auto-generated codes и sequence numbers.
