## 2. Анализ по ролям

### АРХИТЕКТОР: Паттерны и масштабируемость

**Сильные стороны:**
- Богатая доменная модель: 18 entities покрывают tasks, stages, comments, participants, dependencies, approvals, activities, time entries, labels, checklists, templates, milestones, recurrences — на уровне YouGile/Planfix
- Чёткая state machine TaskStatus с `canTransitionTo()` — предотвращает невалидные переходы (BACKLOG→TODO→IN_PROGRESS→IN_REVIEW→DONE)
- Критический путь (CPM) реализован алгоритмом Кана + longest-path — production-grade расчёт зависимостей
- Detect circular dependencies: `wouldCreateCycle()` — правильная рекурсивная проверка
- 4 типа зависимостей (FS, SS, FF, SF) + lag days — уровень MS Project
- Participant model (RESPONSIBLE/CO_EXECUTOR/OBSERVER) — гибкие роли на задачах
- Visibility levels (PARTICIPANTS_ONLY/PROJECT/ORGANIZATION) — гранулярный контроль доступа
- Zustand store с persist для UI state (viewMode, filters, columns) — сохраняется между сессиями
- TaskBoardStore.stagesToColumns() корректно маппит кастомные стадии проекта на Kanban-колонки

**Проблемы:**
- **CalendarEvent entity — НЕТ @Filter tenant isolation** (критично!)
- **ConstructionSchedule entity — НЕТ @Filter tenant isolation** (критично!)
- TaskParticipantService.hasAccess() — PROJECT visibility STUB (`return true` с комментарием "for now allow")
- tasks.ts = 764 LOC — граничный god-file (можно разбить на tasks-core.ts + tasks-labels.ts + tasks-time.ts + tasks-deps.ts)
- CalendarPage.tsx = 797 LOC — god-component (month view + week view + event detail + legend + filters в одном файле)
- Favorites хранятся в localStorage — нет серверной персистенции, теряются при смене устройства

### ПРОРАБ: Удобство на стройке

**Хорошо:**
- Kanban-доска: drag-and-drop через @dnd-kit (мобильные работают)
- MyTasks: 3 вкладки (назначенные, делегированные, избранные) — прораб видит свои задачи
- Делегирование задач: модалка с комментарием — удобно передать задачу мастеру
- Таймер: start/stop для учёта времени — полезно для нарядов
- Множественные виды: Board / List / Gantt / Calendar / MyTasks — каждый выберет удобный
- Зависимости между задачами: FS/SS/FF/SF — для планирования работ

**Не хватает:**
- Оффлайн-режим — на стройке плохой сигнал, задачи нужно создавать оффлайн
- Голосовой ввод — "создай задачу: залить фундамент блока 3, дедлайн пятница"
- Привязка задачи к чертежу/BIM-элементу — "вот тут проблема на плане"
- Фото-вложения прямо с камеры — без выхода из приложения
- Push-уведомления — "тебе назначена задача" / "deadline через час"
- Шаблоны повторяющихся задач — "каждый понедельник: осмотр опалубки"
- Быстрое создание (mobile-first) — 2 тапа вместо полной формы

## 6. Положительные находки

1. **CPM (Critical Path Method)** — алгоритм Кана + longest-path, production-grade. Лучше чем у Procore (у них нет CPM)
2. **4 типа зависимостей** + lag days — уровень MS Project / Primavera P6
3. **Кастомные стадии per project** с drag-and-drop reorder — гибкие workflow
4. **State machine TaskStatus** с `canTransitionTo()` — предотвращает невалидные переходы
5. **Multi-view** (5 видов) — каждый пользователь выбирает удобное представление
6. **Approval chains** — для согласования задач (backend полностью готов)
7. **Time tracking** с live-таймером — для учёта рабочего времени
8. **Task delegation** — передача задач с аудит-трейлом (кто, когда, комментарий)
9. **Visibility levels** — гранулярный контроль видимости (архитектурно правильно, хотя PROJECT не реализован)
10. **Production calendar** (V1176) — российские праздники/рабочие субботы
11. **Task deadline overlay** в календаре — уникальная фича, показывает дедлайны задач поверх событий
12. **Circular dependency detection** — wouldCreateCycle() предотвращает зацикливание графа
13. **Soft-delete** на всех entities — данные не теряются
14. **Dark mode** — полная поддержка во всех 15 компонентах
