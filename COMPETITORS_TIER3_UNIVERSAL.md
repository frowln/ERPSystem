## 1. Monday.com

### 1.1 Общая информация о компании

**Строительная вертикаль:** Monday.com не раскрывает долю строительных клиентов в выручке. Строительство — одна из ~15 отраслевых вертикалей, представленных на сайте. Компания предлагает набор строительных шаблонов, но не имеет выделенной строительной BU (бизнес-единицы) или специализированной команды разработки для строительства.

### 1.2 Строительные шаблоны и решения

Monday.com предлагает следующие строительные шаблоны в Template Center:

- **Construction Management** — основная доска для управления строительным проектом
- **Construction Schedule** — календарное планирование с зависимостями на диаграмме Ганта
- **Construction Roadmap** — визуализация этапов проекта
- **Construction Project Tracking** — отслеживание прогресса от начала до завершения
- **General Contractor Solution** — управление генподрядной деятельностью
- **Daily Report Template** — шаблон ежедневной отчётности

Все шаблоны построены на стандартной архитектуре Monday.com (boards, items, columns, views) и не содержат строительно-специфической бизнес-логики — это лишь преднастроенные структуры данных.

### 1.3 Строительные кейсы

- **Falkbuilt** — канадская компания по производству модульных конструкций использует Monday.com для координации проектов. Кейс опубликован на сайте Monday.com
- **Малые и средние строительные компании** — на форуме Monday.com Community активно обсуждается использование платформы подрядчиками для управления множественными объектами
- Крупные строительные корпорации (типа Bechtel, Skanska, Turner) **не упоминаются** среди клиентов Monday.com

### 1.4 Строительные интеграции

### 1.5 Как строительные компании реально используют Monday.com

На основании анализа форумов, YouTube-туториалов и кейсов:

1. **Управление списком объектов** — доска со статусами проектов (Тендер → В работе → Сдан)
2. **Простое календарное планирование** — диаграмма Ганта с основными этапами (без CPM)
3. **Отслеживание бюджета** — числовые колонки с суммами, формулы для отклонений
4. **Управление закупками** — доски для отслеживания заказов материалов
5. **Распределение ресурсов** — Workload View для визуализации загрузки команды
6. **Ежедневная отчётность** — формы для заполнения прорабами

### 1.6 Сильные стороны для строительства

- **Интуитивный интерфейс** — низкий порог входа, прорабы и бригадиры могут освоить за 1–2 дня
- **Мобильное приложение** — работает офлайн, удобно для полевой работы
- **Гибкие визуализации** — Ганта, Канбан, Таймлайн, Карта, Календарь
- **Автоматизации без кода** — уведомления при изменении статуса, дедлайна
- **Совместная работа** — комментарии, @упоминания, обновления в реальном времени
- **Dashboard** — сводные панели по нескольким проектам одновременно
- **Цена** — от $9/пользователь/мес (Basic) до $19/пользователь/мес (Pro)

### 1.7 Фундаментальные слабости для строительства

| **RFI/Submittals** | КРИТИЧНАЯ | Нет встроенного процесса запросов информации и передачи документации |
| **Управление чертежами** | КРИТИЧНАЯ | Нет версионирования чертежей, маркапов, сравнения ревизий |

### 1.8 Почему строительные компании перерастают Monday.com

1. **Масштаб проекта** — при >50 задачах на одном объекте и >3 объектах одновременно система становится громоздкой
2. **Отсутствие CPM** — невозможно профессионально планировать расписание с тысячами зависимостей
3. **Нет полевого управления** — ежедневные журналы, фотоотчёты, замечания по качеству требуют отдельного ПО
4. **Нет интеграции с проектной документацией** — BIM, чертежи, спецификации живут отдельно
5. **Отчётность** — ограниченные возможности сложных аналитических отчётов
6. **Регуляторные требования** — нет поддержки строительных стандартов, ГОСТ, СНиП

3. **Нет полевого управления** — daily logs, фотоотчёты, чек-листы качества живут отдельно
4. **Нет BIM** — координация моделей невозможна
5. **Кривая обучения** — несмотря на "простоту", продвинутые функции (динамические дашборды, автоматизации) сложны
6. **Стоимость масштабирования** — недавнее изменение политики "member" привело к резкому росту затрат

**Строительная вертикаль:** Microsoft Project — один из старейших инструментов на рынке PM, широко используется в строительстве, особенно на небольших и средних проектах. Microsoft не выделяет строительство как отдельную вертикаль для MS Project — это горизонтальный инструмент.

### 3.3 Строительные шаблоны и решения

Microsoft Project **не имеет строительных шаблонов из коробки**. Однако существуют:

- **Construction Project (CP) — сторонний add-in** — добавляет строительно-специфическую среду:
  - C-Tasks — назначение физических объёмов к задачам MS Project
  - Расчёт множественных стоимостей на основе назначенных ресурсов
  - F-Tasks — управление платежами и графиками оплат по проценту выполнения
  - Назначение подрядчиков
  - Расчёт длительности через производительность

- **Многочисленные YouTube-туториалы** по использованию MS Project для строительных расписаний

### 3.4 Строительные функции MS Project

**Сильные стороны:**

- **CPM (метод критического пути)** — полноценная реализация:
  - Расчёт раннего/позднего старта и финиша
  - Total Float и Free Float
  - Определение критического пути
  - Прямой и обратный проход
- **Ресурсное выравнивание (Resource Leveling)** — автоматическое разрешение конфликтов ресурсов
- **Базовые планы (Baselines)** — до 11 базовых планов для сравнения
- **Earned Value** — базовые метрики EVM (BCWS, BCWP, ACWP, CPI, SPI)
- **Интеграция с Microsoft 365** — Excel, Teams, SharePoint, Outlook, Power BI

**Ограничения CPM:**

Важное ограничение: после ресурсного выравнивания значения Total Float становятся некорректными. Логический анализ ресурсно-выровненных расписаний несовместим с алгоритмом rule-based leveling в MS Project. Это приводит к искажённому представлению критического пути — серьёзная проблема для профессиональных планировщиков в строительстве.

2. **Расчёт критического пути** — определение задач, влияющих на сроки проекта
3. **Ресурсное планирование** — назначение рабочей силы, техники, материалов

### 3.9 Почему строительные компании перерастают MS Project

1. **Десктопная парадигма** — MS Project Professional работает на одном компьютере, нет реального совместного доступа к расписанию (кроме дорогого Project Online)
2. **Нет полевого управления** — невозможно вести строительный процесс с площадки
3. **Отсутствие строительных процессов** — RFI, submittals, change orders, punch lists живут отдельно
4. **Крутая кривая обучения** — интерфейс сложный и неинтуитивный для полевых сотрудников
5. **Нет интеграции с проектной документацией** — чертежи, BIM, спецификации вне системы
6. **Рассинхронизация данных** — расписания, бюджеты, документы в разных системах (MS Project + Excel + Email)
7. **Лимит масштабирования** — для мега-проектов с тысячами activity недостаточно мощности

### 4.3 Глубокий разбор функционала Primavera P6

#### 4.3.1 Календарное планирование (Scheduling)

- **CPM (Critical Path Method)** — промышленный стандарт:
  - Прямой проход (Forward Pass) — расчёт раннего старта/финиша
  - Обратный проход (Backward Pass) — расчёт позднего старта/финиша
  - Total Float / Free Float — определение резервов времени
  - Определение критического пути проекта
- **Activity Codes** — многомерная классификация работ
- **WBS** — иерархическая структура работ с неограниченной глубиной
- **Relationships** — все 4 типа зависимостей (FS, FF, SS, SF) с лагами
- **Constraints** — ограничения на даты (As Soon As Possible, Must Start On и т.д.)
- **Multiple Calendars** — разные календари для разных ресурсов и работ
- **Scheduling Options** — Retained Logic, Progress Override, Actual Dates
- **До 100,000 activities на проект** (P6 Professional)
- **Миллионы activities** в P6 EPPM (корпоративная версия)

#### 4.3.2 Ресурсное управление (Resource Management)

- **Неограниченное количество ресурсов** (labor, nonlabor, material)
- **Resource Leveling** — автоматическое выравнивание перегрузки
- **Resource Curves** — профили загрузки (S-кривые, Bell-кривые)
- **Workload Analysis** — анализ загрузки по ресурсам и временным периодам
- **Role-based Planning** — планирование по ролям с последующим назначением
- **Timesheets** — учёт рабочего времени с утверждением
- **Resource Assignments** — детальные назначения с единицами и стоимостью

#### 4.3.3 Управление рисками (Risk Analysis)

- **Risk Register** — реестр рисков с вероятностью и воздействием
- **Monte Carlo Simulation** — вероятностный анализ сроков (через Primavera Risk Analysis)
- **Sensitivity Analysis** — определение наиболее критичных рисков
- **Risk Mitigation Strategies** — стратегии снижения рисков, привязанные к работам
- **Probabilistic Scheduling** — P50, P80, P90 даты завершения

#### 4.3.4 Earned Value Management (EVM)

Primavera P6 — один из наиболее полных инструментов EVM на рынке:

| Метрика | Описание |
|---|---|
| **PV (Planned Value)** | Плановая стоимость запланированных работ (BCWS) |
| **EV (Earned Value)** | Освоенный объём выполненных работ (BCWP) |
| **AC (Actual Cost)** | Фактическая стоимость выполненных работ (ACWP) |
| **CPI** | Индекс стоимости выполнения (EV/AC) |
| **SPI** | Индекс выполнения расписания (EV/PV) |
| **CV / SV** | Отклонения стоимости и расписания |
| **EAC** | Оценка при завершении |
| **ETC** | Оценка до завершения |
| **BAC** | Бюджет при завершении |
| **VAC** | Отклонение при завершении |
| **TCPI** | Индекс необходимой производительности |

Встроенные дашборды и отчёты EVM доступны непосредственно из Ганта. Отдельный модуль **Primavera Earned Value Management** расширяет возможности: управление бюджетом, фактическими, прогнозными и освоенными затратами.

- **What-If Scenarios** — моделирование сценариев

### 4.10 Будущее Primavera P6 vs облачные альтернативы

**Стратегия Oracle:** Oracle активно продвигает миграцию на Primavera Cloud (OPC) — облачную SaaS-платформу.

**Проблемы перехода на OPC:**
1. **Потеря функционала** — OPC не может обрабатывать мега-проекты так же, как P6 EPPM на реляционной базе Oracle Database
2. **Нет Portfolio capabilities** — отсутствуют portfolio analysis, capacity planning, performance status
3. **Нет стандартных интеграций** — потеря ERP-интеграций (SAP, Oracle EBS, JDE, Maximo)
4. **Разные среды** — P6 EPPM и OPC не синхронизируют данные автоматически

**Облачные альтернативы:**
- **Procore** — лидирует в полевом управлении, но слаб в CPM
- **Asta Powerproject** — британский CPM-инструмент, набирает популярность
- **InEight** — комплексная платформа (бывший Hard Dollar + BIM)
- **Oracle Primavera Cloud** — собственная облачная замена P6

**Прогноз:** P6 Professional останется стандартом для мега-проектов ещё 5–10 лет. Но для проектов среднего масштаба облачные альтернативы (включая PRIVOD) имеют окно возможностей за счёт лучшего UX, мобильности и интеграции полевого + офисного управления.

**Типичные результаты внедрения:**
- Снижение финансовых издержек и непредвиденных расходов минимум в 2 раза
- Ликвидация хаоса в процессах: потерянных заявок, некоординированных поставок
- Повышение прозрачности: все участники видят статус задач и сроки

### 5.5 Интеграция с 1С

Интеграция Битрикс24 с 1С — **ключевое преимущество** на российском рынке:

**Возможности:**
- Синхронизация контрагентов (1С → Б24 и Б24 → 1С)
- Выставление счетов из CRM Битрикс24 с передачей в 1С:Бухгалтерию
- Синхронизация статусов оплат
- Обмен номенклатурой (товары, услуги, работы)
- Формирование актов, счетов-фактур

**Ограничения интеграции 1С:**
- REST API Битрикс24 принимает только 50 элементов за запрос — медленно при больших объёмах
- Для файловых баз 1С: только одна фоновая задача — синхронизация Б24→1С не работает в фоне
- Нет просмотра остатков на складах, нет деления по ценам, нет доступа к характеристикам товаров
- Товар может принадлежать только одной категории
- 1С:Фреш (облачная 1С): закрытый конфигуратор — интеграция работает только без сложной кастомизации
- Высокая вероятность нарушения синхронизации при обновлениях программ

### 5.6 Почему строительные компании выбирают Битрикс24

1. **Импортозамещение** — после ухода западных вендоров (Microsoft, Salesforce) из России
2. **Бесплатный тариф** — можно начать без вложений
3. **Цена за компанию** — не per-seat, выгодно для больших команд
4. **Экосистема 1С** — нативная интеграция с доминирующей учётной системой России
5. **Русский интерфейс** — полная локализация, русскоязычная поддержка
6. **Мобильное приложение** — доступно на Android и iOS
7. **Знакомость** — многие менеджеры уже имеют опыт работы с Битрикс24

- Шаблоны интернет-магазинов строительных материалов (17 готовых решений)

### 5.9 Где Битрикс24 терпит неудачу в строительстве

1. **Модуль "Задачи и проекты" недостаточен** — пользователи признают, что он "не может перекрыть все потребности в управлении строительством"
2. **Требуется дорогая кастомизация** — стандартный функционал не соответствует строительной специфике, каждая компания нуждается в индивидуальных доработках
3. **Плохая адаптация под специфику работ** — уникальные процессы согласования, учёта материалов, управления проектами приводят к "дефициту необходимых инструментов и повышенной затратности на доработку функционала"
4. **Не заменяет сметное ПО** — ГРАНД-Смета, 1С:Смета, Smeta.RU остаются отдельными системами
5. **Не заменяет MS Project / Primavera** — для серьёзного календарного планирования нужно отдельное ПО
6. **Ограничения интеграции с 1С** — технические лимиты API, проблемы с синхронизацией
7. **Нет отраслевого понимания** — платформа создана для generic-бизнеса, строительная терминология и процессы чужды

## 7. Выводы для PRIVOD

### 7.1 Ключевые возможности позиционирования

**Против Monday.com / Smartsheet:**
- PRIVOD — это **специализированное строительное решение**, а не универсальная платформа, адаптированная под строительство
- Monday.com и Smartsheet заставляют строительные компании "строить" свои процессы вручную из универсальных компонентов
- PRIVOD предлагает **готовые строительные workflow** из коробки: КС-2/КС-3, AOSR, инспекции, punch-листы, наряд-допуски
- При этом цена PRIVOD может быть сопоставима ($10–30/пользователь/мес)

**Против MS Project:**
- PRIVOD объединяет **планирование + полевое управление + финансы** в одной системе, устраняя рассинхронизацию данных
- PRIVOD имеет полноценное мобильное приложение для полевой работы
- PRIVOD изначально облачный с совместным доступом
- PRIVOD покрывает российскую специфику (КС-2, КС-3, 1С)

**Против Oracle Primavera P6:**
- PRIVOD предлагает **современный UX** vs устаревший интерфейс P6
- PRIVOD объединяет планирование + полевое управление (P6 не имеет field management)
- PRIVOD доступен по подписке от ~$10/чел/мес vs $3,500+ perpetual у P6
- PRIVOD имеет кривую обучения в часы, а не в недели
- **Но:** для мега-проектов с >100K activities P6 остаётся непревзойдённым; PRIVOD должен позиционироваться для проектов малого и среднего масштаба

**Против Битрикс24:**
- PRIVOD — **строительная платформа с CRM**, а не CRM-платформа, адаптированная для строительства
- Битрикс24 признаёт, что его модуль "Задачи и проекты" не покрывает строительные потребности
- PRIVOD имеет все строительные модули (КС-2/КС-3, AOSR, качество, ОТ, склад), которых нет в Б24
- PRIVOD интегрируется с 1С на том же уровне + добавляет интеграции со строительным ПО
- PRIVOD имеет CPM-планирование, EVM, ресурсное управление — то, чего нет в Б24 вообще

### 7.2 Стратегические рекомендации

1. **Нишевое позиционирование** — "Не заставляйте строительный бизнес работать в CRM-системе. Используйте платформу, созданную для строительства."

2. **Migration path от универсальных инструментов:**
   - Предложить импорт данных из Monday.com, Smartsheet (CSV/API)
   - Предложить импорт расписаний из MS Project (.mpp) и Primavera P6 (.xer)
   - Предложить миграцию CRM-данных из Битрикс24

3. **Ценовое позиционирование:**
   - Monday.com: $9–19/чел/мес → PRIVOD сопоставим, но 10x больше строительного функционала
   - Smartsheet: $9–19/чел/мес → аналогично
   - MS Project: $10–55/чел/мес → PRIVOD в том же диапазоне, но с полевым управлением
   - Primavera P6: $3,500+ за лицензию → PRIVOD в 10–50x дешевле для СМП
   - Битрикс24: 0–168K₽/год → PRIVOD дороже, но покрывает строительную специфику

4. **Конкурентные killer features PRIVOD:**
   - ✅ Российская специфика из коробки (КС-2, КС-3, AOSR, 1С)
   - ✅ Единая платформа (планирование + поле + финансы + CRM + склад)
   - ✅ Современный UX + мобильное приложение
   - ✅ Доступная цена per-seat
   - ✅ AI-ассистент для строительства (уникальное преимущество)

### 7.3 Риски и ограничения

| Риск | Описание | Митигация |
|---|---|---|
| **Monday/Smartsheet добавят строительные модули** | Крупные платформы могут создать строительные решения | Глубина специализации — их модули будут поверхностными |
| **Primavera Cloud улучшит UX** | Oracle инвестирует в модернизацию | У Oracle это займёт 3–5 лет, фокус на мега-проекты |
| **Битрикс24 создаст строительный модуль** | 1С-Битрикс может выпустить отраслевое решение | Маловероятно — нет экспертизы в строительстве |
| **Клиенты не хотят менять инструменты** | Инерция перехода высока | Предложить пилотный проект, параллельную работу |
