## 2. Табель Т-13 формируется?

### Privod
- **Страница**: `/hr/timesheet-t13` — **РЕАЛИЗОВАНА**
- **Сетка**: сотрудники × дни месяца, коды посещаемости (Я/Б/О/К/НН/В/Н)
- **Редактирование**: клик по ячейке → модал с выбором кода и часов
- **Итоги**: дни, часы, ночные часы по сотруднику

### Что нужно для бухгалтерии
| Требование | Статус | Комментарий |
|-----------|--------|-------------|
| Форма по Госкомстату | ⚠️ | Интерактивная сетка есть, но нет экспорта в утверждённую форму Т-13 |
| Экспорт в Excel/PDF | ❌ | Бухгалтерия ожидает скачать готовую форму |
| Подписи (ответственный, руководитель) | ❌ | Т-13 требует 3 подписи |
| Автозаполнение выходных | ⚠️ | Нужен производственный календарь РФ |

**Вердикт**: Т-13 **частично готов** — интерактивная сетка работает, но бухгалтерия не сможет отправить его в ФНС без экспорта в утверждённую форму.

## 5. Блокеры для кадрового учёта РФ

| Документ/Форма | Описание | Статус | Критичность |
|---------------|---------|--------|-------------|
| Т-2 (личная карточка) | Полная форма с 11 разделами | ❌ Нет | HIGH — нужно для кадрового делопроизводства |
| Т-1 (приказ о приёме) | Приказ о приёме на работу | ❌ Нет | HIGH — обязательный кадровый документ |
| Т-5 (приказ о переводе) | Приказ о переводе | ❌ Нет | MEDIUM |
| Т-6 (приказ об отпуске) | Приказ о предоставлении отпуска | ❌ Нет | HIGH — обязательный |
| Т-8 (приказ об увольнении) | Приказ о прекращении ТД | ❌ Нет | HIGH — обязательный |
| Т-13 (табель) | Табель учёта рабочего времени | ⚠️ Частично | Сетка есть, экспорт нет |
| Т-49/51 (ведомости) | Расчётно-платёжные ведомости | ❌ Нет | LOW — обычно в 1С |
| СЗВ-ТД / ЕФС-1 | Отчёт в ПФР (электронная трудовая) | ❌ Нет | HIGH — штраф за непредставление |
| Трудовой договор | Печатная форма по шаблону | ⚠️ Частично | Запись есть, шаблон/печать нет |
| График отпусков (Т-7) | Годовой график отпусков | ❌ Нет | MEDIUM — утверждается ежегодно |

**Вердикт**: Для полного кадрового учёта РФ не хватает **6 обязательных форм**. Это не блокер для запуска (все стройкомпании ведут кадры в 1С), но блокер для позиционирования как "полный ERP".

## 6. Самозанятые: учёт ведётся?

**Статус**: ❌ **НЕТ**

На стройке ~30% рабочей силы — самозанятые (штукатуры, плиточники, отделочники, специалисты ИП). Для них:

| Требование | Обычный сотрудник | Самозанятый/ИП |
|-----------|-------------------|----------------|
| Договор | Трудовой (ТД) | ГПХ / договор подряда |
| Налоги | НДФЛ 13% + взносы 30% | НПД 6% (сам платит) |
| Табель | Обязателен (Т-13) | Акт выполненных работ |
| Отпуск | Обязателен (28 дней) | Нет |
| Больничный | ФСС оплачивает | Нет |
| Увольнение | По ТК РФ (14 дней) | Расторжение договора |
| СИЗ | Работодатель выдаёт | Свои |

**Рекомендация**: Добавить в систему тип "Подрядчик/Самозанятый" с упрощённой карточкой (ИНН, договор ГПХ, акты, оплата). Это покроет 30% рабочей силы стройки.

## 7. Допуски: система блокирует работу без действующего допуска?

**Статус**: ⚠️ **ЧАСТИЧНО**

| Проверка | Реализовано | Комментарий |
|---------|-------------|-------------|
| Матрица сертификатов | ✅ Да | valid/expiring/expired статусы |
| Предупреждение за 30 дней до истечения | ✅ Да | expiringCount в метриках |
| Блокировка назначения на наряд без допуска | ❌ Нет | Система не проверяет при создании наряда |
| Блокировка допуска на площадку без инструктажа | ❌ Нет | Нет сквозной проверки briefing→work_order |
| Автоуведомление ответственного | ❌ Нет | Нет email/push при истечении |

**Рекомендация (P0)**:
1. При создании наряда проверять: у сотрудника есть действующий допуск для данного вида работ?
2. При истечении сертификата: автоматически блокировать назначение + уведомить ответственного по ОТ.
3. Вводный инструктаж → проставить флаг `briefingPassed` на employee → проверять при назначении в бригаду.
