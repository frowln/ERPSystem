## 2. RFI workflow: создание → ответ → закрытие

### Privod
**Статус**: Реализовано
- RFI: list + board + create + detail + edit ✅
- Статусы: OPEN → ANSWERED → CLOSED ✅
- Поля: title, description, project, priority ✅

### Лучший конкурент
**Кто**: Procore (мировой лидер в RFI)
**Как у них**:
- Поле «Кому» (ответственный за ответ) — обязательное
- Срок ответа с автоматическими напоминаниями
- Эскалация при просрочке (уведомление руководителю)
- Связь RFI → Change Event автоматическая
- Ball-in-court tracking (кто сейчас должен действовать)

### GAP анализ
- **Нет эскалации при просрочке** — если ответа нет >5 дней, стройка простаивает
- **Нет ball-in-court** — неясно кто сейчас должен действовать
- **Нет автоматических напоминаний** по deadline
- Приоритет: **HIGH**
- Сложность: 3-5 дней (notifications + escalation rules)
- Почему важно: RFI без ответа = простой стройки = потеря денег

## 3. Change Management: event → assessment → order → budget

### Privod
**Статус**: Реализовано (полный цикл)
- Change Events: list + detail + create (modal) + status workflow ✅
- Impact Assessment: cost impact + schedule impact на detail page ✅
- Change Orders: list + board + form + detail + line items ✅
- Status flow: IDENTIFIED → UNDER_REVIEW → APPROVED_FOR_PRICING → PRICED → APPROVED ✅
- CO Status: DRAFT → PENDING_APPROVAL → APPROVED → EXECUTED ✅
- Budget impact analytics ✅
- Schedule impact analytics ✅
- Trend analysis ✅
- Sources: RFI, Design Change, Owner Request, Field Condition, Regulatory, Value Engineering ✅
- CO Types: Addition, Deduction, Substitution, Time Extension, Mixed ✅

### Лучший конкурент
**Кто**: Procore
**Как у них**:
- Change Events linked to RFIs automatically
- CO → автоматическое обновление бюджета проекта
- Multi-tier approval (порог суммы → разные уровни утверждающих)
- Commitment change orders (субподрядчики) vs prime change orders (заказчик)

### GAP анализ
- **CO не обновляет бюджет автоматически** — утверждённый CO должен создавать строку в ФМ
- **Нет порогов утверждения** — CO на 100K и на 10M проходят одинаково
- **Нет разделения prime/commitment CO** — разные документы для заказчика и подрядчика
- Приоритет: **HIGH** (budget sync), **MEDIUM** (thresholds)
- Сложность: 5-7 дней

- Приоритет: **LOW** (ISO 19650 stages), **MEDIUM** (diff)
- Сложность: 1-2 недели

## 5. ЭДО: интеграция

### Privod
**Статус**: Страницы существуют (/russian-docs/edo, /russian-docs/sbis), но без реальной интеграции

### Лучший конкурент
**Кто**: СБИС (Тензор)
**Как у них**: Нативная интеграция с ФНС, роуминг с контрагентами, КЭП.
**Кто**: 1С:УСО — встроенный ЭДО через Тензор.

### GAP анализ
- **Нет реальной интеграции с оператором ЭДО** (СБИС, Диадок, Контур)
- **Блокер для крупных заказчиков** — по 63-ФЗ многие требуют электронные первичные документы
- КС-2/КС-3 через ЭДО = экономия 3-5 дней на каждом акте
- Приоритет: **HIGH** (конкурентный разрыв)
- Сложность: 2-4 недели (API интеграция)
- Почему важно: без ЭДО крупные заказчики (госсектор, крупный бизнес) не будут работать
