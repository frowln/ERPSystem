## 1. Полная цепочка: Заявка → КЛ → PO → Доставка → Склад → Выдача

### Реализованная архитектура

```
Прораб → Заявка (PurchaseRequest) → Согласование → КЛ (CompetitiveList)
                                                     ↓
                                              PO (PurchaseOrder)
                                                     ↓
                                         Dispatch (DispatchOrder)
                                                     ↓
                                          Приход (StockMovement RECEIPT)
                                                     ↓
                                         Выдача (StockMovement ISSUE)
                                                     ↓
                                            Остаток (Stock Balance)
```

### Оценка цепочки

| Этап | Модуль | Статус | Примечание |
|------|--------|--------|------------|
| Заявка на закупку | PurchaseRequest (10 статусов) | ✅ Полный | DRAFT→SUBMITTED→APPROVED→ORDERED→DELIVERED→CLOSED |
| Согласование | Approval workflow | ⚠️ Базовый | Нет многоуровневого (директор для >500K ₽) |
| Конкурентный лист | CompetitiveList | ✅ Есть | ≥3 поставщика, скоринг по совокупности |
| Заказ поставщику | PurchaseOrder (8 статусов) | ✅ Полный | Привязка к проекту, заявке, материалам |
| Отправка PO | Send action | ⚠️ Базовый | Нет интеграции с email/ЭДО |
| Доставка | DispatchOrder (7 статусов) | ✅ Есть | Транспорт, водитель, маршрут, календарь |
| Приход на склад | StockMovement RECEIPT | ✅ Есть | Quick receipt + подтверждение |
| Выдача на объект | StockMovement ISSUE | ✅ Есть | С проектом и ответственным |
| Остатки | Stock balance | ✅ Есть | По локациям, с резервом |

**Вердикт**: 8 из 8 этапов реализованы. Цепочка замкнута. Обрывов нет.

6. **Адресное хранение** — зона/стеллаж/ячейка

8. **КЛ с ≥3 поставщиками** — ни у кого нет (1С:УСО, HubEx, PlanRadar — нет)

### ⚠️ Что нужно улучшить
1. **Нет сводного dashboard снабженца** — Морозова открывает 5+ страниц
2. **Нет порога для упрощённой закупки** — КЛ для изоленты на 2 250 ₽ = бюрократия
3. **Нет многоуровневого согласования** — заявка >500K без директора
4. **Нет интеграции PO → email/ЭДО** — отправка поставщику вручную
5. **Потребность в материалах** — нет "spec→ordered→received→issued→balance" на одном экране

### ❌ Что отсутствует
1. **Procurement dashboard** с KPI (активные заявки, в пути, критический запас)
2. **Quick PO from Purchase Request** — автоматическое создание PO из согласованной заявки
3. **Мобильное приложение снабженца** (сравни: HubEx с GPS-заявками)

## 5. Негативные остатки

**Тест**: попытка списать 2000 м кабеля при остатке ~1000 м.

- Если API отклоняет (400/422) → ✅ Правильно
- Если API создаёт движение → 🔴 [CRITICAL] — negative stock, data integrity at risk

**Рекомендация**: backend MUST enforce `available_qty >= issue_qty` на уровне SQL constraint или service validation.
