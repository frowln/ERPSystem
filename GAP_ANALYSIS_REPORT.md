## МЕТОДОЛОГИЯ

Каждая capability оценивается по шкале:

| Уровень | Описание |
|---------|----------|
| **5 — Production** | Полнофункциональный, протестированный, готов к production |
| **4 — Advanced** | Работающий функционал с minor gaps |
| **3 — Intermediate** | Базовый CRUD + бизнес-логика, но неполный |
| **2 — Basic/Skeleton** | Scaffolding, минимальная логика |
| **1 — Planned** | Есть в UI/API но не реализовано |
| **0 — Absent** | Отсутствует полностью |

**Benchmark**: средний уровень Procore + CMiC + Fieldwire для соответствующей функции.
