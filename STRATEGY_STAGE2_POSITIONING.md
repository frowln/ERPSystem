#### ПОЧЕМУ именно этот сегмент (обоснование данными)

**1. Самый крупный неокученный рынок.** Анализ 58+ конкурентов показал фрагментацию:
- Enterprise (>500 чел.) уже внедрили 1С:УСО (684 000 руб. коробка + 2-20 млн внедрение) или Pragmacore (30 клиентов, enterprise-only).
- SMB (<50 чел.) используют Gectaro (бесплатно / 1 600 руб./user/мес.), Smetter (480 руб./мес.) или просто Excel — не готовы платить за профессиональные инструменты.
- **Mid-market (50-500 чел.) — "мёртвая зона"**: слишком крупные для Gectaro/Smetter, но слишком маленькие для 1С:УСО с внедрением за 5-20 млн руб. 59% клиентов Viewpoint Vista (мировой аналог) — компании именно этого размера (101-1 000 сотрудников).

**2. Рынок растёт в 4 раза к 2028 году.** По прогнозу Strategy Partners, объём рынка цифровизации строительства вырастет с $70 млн (2025) до $280+ млн (2028). 70%+ средних и крупных компаний увеличили IT-бюджеты в 2024, при этом 45% направили рост на внедрение отечественных решений.

**3. Максимальная боль — фрагментация данных.** Карта 85 строительных процессов показала: Pain Point #1 (10/10 по severity) — средняя строительная компания работает с 4-7 несвязанными системами (сметы в ГРАНД-Смета, графики в MS Project, бухгалтерия в 1С, документы в файловых папках, табели на бумаге, контроль качества в Excel, коммуникации в WhatsApp). Ручной перенос данных между системами создаёт ошибки, задержки и отсутствие единой картины.

**4. Импортозамещение создаёт окно возможностей.** Procore, Autodesk BIM 360, Trimble — все ушли с рынка РФ. Доля отечественных BIM-решений выросла с 15% (2020) до 65% (2025). Обязательный BIM по ПП 331 с июля 2024 усиливает спрос.

**5. Ни один конкурент не закрывает потребности mid-market полностью.** Сводная таблица покрытия процессов из анализа 41 российского конкурента:

#### KILLER FEATURE #1: Автоматическая генерация КС-2 из полевых данных (One-Click Billing)

**Описание:** Сквозной конвейер данных: ежедневный журнал работ (#24) -> учёт объёмов (#25) -> КС-6а -> КС-2 с автоматическим применением актуальных индексов Минстроя (#16) -> цифровая подпись КЭП -> КС-3 -> выгрузка в 1С. Один клик — и месячная КС-2 сформирована.

**ПОЧЕМУ это killer:**
- **Pain Point #3 (Severity 9/10):** Формирование КС-2 — самая трудоёмкая задача в строительстве. Средний цикл выставления актов: 30-60 дней. Ошибки в объёмах или индексах приводят к отклонению и обнулению платёжного цикла.
- **UX-аудит подтверждает:** FormKs2Page оценён на 5.5/10. НДС захардкожен как `totalAmount * 0.2`. Нет построчного НДС. Нет print stylesheet. 35-66 минут на один документ вручную.
- **Конкурентный разрыв:** ГРАНД-Смета и Smeta.RU работают со сметами, 1С — с бухгалтерией, но мост между ними — ручной перенос данных через 3-4 инструмента. **Ни один российский ERP не делает это сквозь.**

**Кто из конкурентов делает и почему мы сделаем лучше:**
- **Gectaro:** Обещает "КС-2 за 3 минуты", но без ГЭСН/ФЕР — не подходит для госзаказа и крупных контрактов.
- **1С:УСО:** Формирует КС-2, но только из данных, уже введённых в 1С. Нет полевого ввода, нет мобильного, нет связи с ежедневными журналами.
- **Адепт:** Формирует КС-2 из сметных данных, но нет финансового модуля для отслеживания оплат.
- **ПРИВОД сделает лучше:** Данные вводятся прорабом с телефона (ежедневный журнал -> объёмы), автоматически агрегируются, оцениваются по сметным нормам с актуальными индексами, формируются в КС-2, подписываются КЭП, и передаются в 1С через API. **Цикл: 30 дней -> 5 дней.**

**Подход к реализации:**
- *Что уже есть:* DailyLog entity, KS-2/KS-3 pages (FormKs2Page, FormKs3Page), EVM module, backend entities для исполнительной документации, интеграция СБИС (SbisService.java), KepSignWizard.tsx
- *Что доработать на frontend:* Импорт спецификации в FormKs2Page (из EstimateImportWizard), построчный НДС, print stylesheet (CSS @media print), автоподсчёт итогов, валидация (qty > 0, workName required)
- *Что доработать на backend:* Сервис автогенерации КС-2 из отслеженных объёмов + применение индексов, двусторонний обмен с 1С (OneCDataExchangeService.java уже существует, требует доработки), связка с PricingController (backend есть, frontend — 0)
- *Что нового:* Модуль сметных баз ГЭСН/ФЕР/ТЕР (импорт справочников ФСНБ-2022), компонент PricingIntegration UI (24 часа по UX-аудиту)

**Трудозатраты:** 8-10 недель (2 backend + 1 frontend инженера)

**Ожидаемый эффект на продажи:**
- Открывает 100% российского рынка подрядных организаций (каждый рубль проходит через КС-2)
- Снижение биллингового цикла с 30 до 5 дней = прямая экономия на cash flow
- Конверсия demo -> покупка: ожидаемая +40% (клиент видит немедленный эффект)

#### KILLER FEATURE #2: Единая платформа "ERP + Стройконтроль + BIM" с мобильным offline

**Описание:** Прораб с телефона (offline): создаёт ежедневный журнал, фотофиксирует работы с GPS-привязкой, фиксирует дефекты на чертежах/BIM-модели, заполняет акты скрытых работ, отмечает табели. Данные синхронизируются при появлении связи. Все данные автоматически попадают в ERP: бюджет, план-факт, дашборды руководства.

**ПОЧЕМУ это killer:**
- **Pain Point #1 (Severity 10/10):** Фрагментация данных между 4-7 системами. Прораб ведёт журнал на бумаге, фото скидывает в WhatsApp, дефекты записывает в Excel, табели заполняет в бумажную форму. Данные вводятся повторно 3-4 раза.
- **Pain Point #2 (Severity 9/10):** Бумажная регуляторная документация — журналы, акты скрытых работ, инструктажи.
- **UX-аудит (Sergey/Foreman): 3.2/10** — "CRITICAL: System unusable in field". Нет touch targets 44x44px, нет barcode сканера, нет Copy from yesterday, нет фото geotagging, DataTable чекбоксы 20x20px (невозможно в перчатках).

**Кто из конкурентов делает и почему мы сделаем лучше:**
- **PlanRadar (3.5/10 Capterra):** Лучший UX стройконтроля, offline-режим, BIM. НО: иностранный (риск ухода), нет ERP, нет КС-2/КС-3, нет интеграции с 1С. Цена: $35-179/user/мес.
- **Fieldwire (4.7 App Store):** Лучший offline и OCR чертежей. НО: нет финансов, нет HR, нет российских документов.
- **Адепт:Стройконтроль:** Мобильное приложение (iOS/Android), инспекции, фото. НО: нет ERP, нет финансов, нет HR, нет интеграции с 1С.
- **1С:УСО:** Нет мобильного приложения вообще. Desktop-only.
- **ПРИВОД сделает лучше:** Единственная платформа, где прораб вводит данные с телефона, и они **автоматически** попадают в ERP (финансы, HR, бюджеты, аналитика). Нет ручного переноса. Один ввод данных -> все модули.

**Подход к реализации:**
- *Что уже есть:* PWA с offline-поддержкой (offlineQueue.ts), MobileReportNewPage.tsx (оценён 7.1/10 — "excellent offline support"), BimViewer.tsx, DailyLogCreatePage.tsx, QualityCheckCreateModal.tsx, PunchItemCreateModal.tsx, BottomNav для мобильного
- *Что доработать:* Touch targets увеличить до 44x44px (8h по UX-аудиту), DataTable mobile card view (16h), DailyLogCreatePage: "Copy from yesterday" + inline фото (12h), barcode scanner для MovementFormPage (16h), мобильный quick-confirm для склада (8h), BIM viewer dark mode (0xf0f4f8 -> theme-aware)
- *Что нового:* React Native / Capacitor-обёртка для нативного мобильного с GPS, камерой, NFC; offline-first архитектура для АОСР, инспекций, табелей

**Трудозатраты:** 12-16 недель (1 mobile + 1 frontend + 1 backend инженера)

**Ожидаемый эффект на продажи:**
- +80% DAU (по аналогии с Fieldwire, где 80%+ пользователей приходят через мобильное)
- Снижение времени ввода данных на 70% (один ввод вместо 3-4)
- Конкурентное преимущество #1 в демо: "Прораб вводит с телефона — директор видит на дашборде через 30 секунд"

#### KILLER FEATURE #3: Бесшовная двусторонняя интеграция с 1С + ЭДО

**Описание:** Двусторонний обмен данными с 1С:Бухгалтерия / 1С:УСО в реальном времени: контрагенты, номенклатура, счета-фактуры, КС-2/КС-3, табели, зарплаты, банковские выписки. Плюс интеграция с СБИС/Диадок для электронного документооборота: отправка КС-2 с КЭП через ЭДО напрямую из ПРИВОД.

**ПОЧЕМУ это killer:**
- **1С занимает 55-60% рынка строительного ERP** в России. Даже компании, использующие другое ПО для управления строительством, ведут бухгалтерию в 1С. Вопрос "А как с 1С?" — первый на каждой демонстрации.
- **Pain Point #4 (Severity 8/10):** Нет видимости затрат в реальном времени. Менеджер узнаёт о перерасходе только при закрытии месячной бухгалтерии в 1С, через 3-4 недели после фактических затрат.
- **Карта процессов:** Процесс #58 (Real-Time Plan/Fact Analysis) требует данных из учёта (1С), закупок (committed costs) и полевых данных (progress tracking). Без интеграции с 1С — данные всегда запаздывают.

**Кто из конкурентов делает и почему мы сделаем лучше:**
- **Адепт:** Нет интеграции с 1С (только через Excel). Это главная слабость.
- **Pragmacore:** Нет 1С. Enterprise-only.
- **ЦУС:** Нет 1С. Молодая платформа.
- **PlanRadar:** Иностранный, нет 1С.
- **Gectaro:** Импорт из 1С через Albato (400+ сервисов). Не двусторонний.
- **1С:УСО:** Нативная 1С, но нет стройконтроля, BIM, мобильного.
- **ПРИВОД сделает лучше:** Не заменяем 1С, а становимся **"полевым дополнением к 1С"**. Бухгалтер продолжает работать в 1С, прораб — в ПРИВОД. Данные синхронизируются автоматически. Позиция: "ПРИВОД + 1С = полное покрытие" вместо "ПРИВОД ИЛИ 1С".

**Подход к реализации:**
- *Что уже есть:* OneCIntegrationService.java (335 строк), OneCDataExchangeService.java (fullSync в одной транзакции — нужен chunking), OneCConfig entity, SbisService.java (296 строк), EdoIntegrationService.java, KepSignWizard.tsx, EdoSettingsPage.tsx
- *Что доработать:* OneCDataExchangeService — разбить fullSync на chunks (P1-8 batch processing issue из аудита), двустороннее API вместо только экспорта, SbisService — добавить SSRF validation (C9 из security audit)
- *Что нового:* 1С REST API connector (веб-сервис для 1С:Бухгалтерия 3.0), маппинг справочников (контрагенты, номенклатура, статьи ДДС), автоматическая выгрузка КС-2/КС-3 в ЭДО

**Трудозатраты:** 6-8 недель (2 backend инженера + 1С-разработчик)

**Ожидаемый эффект на продажи:**
- **Снимает главный барьер** на российском рынке: "А как с 1С?"
- Открывает партнёрство с 1С-франчайзи (рекомендуют ПРИВОД как полевое дополнение к 1С)
- Снижение конкурентного давления от 1С:УСО: позиционируем не как замену, а как дополнение

#### KILLER FEATURE #4: AI-ассистент для строительства на русском языке

**Описание:** Conversational AI для работы с данными проекта на русском языке: "Покажи все проекты с перерасходом > 10%", "Сформируй КС-2 по объекту Жилой Дом 5 за январь", "Какие допуски у крановщика Иванова истекают в этом месяце?". Плюс: AI-предупреждения о рисках (задержки, перерасход, нехватка материалов), автоматическая проверка объёмов КС-2 по сметным данным.

**ПОЧЕМУ это killer:**
- **60% ConTech-инвестиций 2025 ($2,22 млрд) направлены в AI.** Procore запустил Agent Builder и приобрёл Datagrid (январь 2026). Autodesk запустил Assistant. Bluebeam Max интегрировал Anthropic Claude. **НИ ОДИН конкурент не имеет AI на русском языке для строительства.**
- **Pain Point #5 (Severity 8/10):** Субподрядная непрозрачность. Генподрядчики не видят реальную картину по субподрядчикам. AI может агрегировать данные и давать рекомендации.
- **Pain Point #4 (Severity 8/10):** Нет видимости затрат в реальном времени. AI-дашборд с natural language запросами позволяет руководителю получать ответы без навигации по 173+ пунктам меню.

**Кто из конкурентов делает и почему мы сделаем лучше:**
- **Procore Assist:** Conversational AI, но только на английском. Не доступен в РФ.
- **Pragmacore:** Предиктивная аналитика на базе 300+ проектов. Но дорого (enterprise-only, ~30 клиентов).

### 2.3 Ценовая стратегия

#### Модель: Модульная подписка per-user/month

**Обоснование выбора модели per-user:**
1. Procore перешёл на per-project модель — и это стало его главной слабостью для SMB ("дорого для мелких проектов"). Мы не повторяем эту ошибку.

3. Per-user позволяет клиенту начать с 5 пользователей и масштабироваться до 100+.

- Импорт из 1С (через OneCDataExchangeService), Excel, ГРАНД-Смета.

**5. "Старт" как воронка для роста (land-and-expand)**
- Клиент начинает со "Старт" (1 200 руб./user x 5 user = 6 000 руб./мес.) — минимальный порог входа.

- Модель Atlassian/Slack: дешевый вход, расширение по мере вовлечения.
